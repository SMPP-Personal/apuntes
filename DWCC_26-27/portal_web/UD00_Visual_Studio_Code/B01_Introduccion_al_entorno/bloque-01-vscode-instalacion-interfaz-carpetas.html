Situación inicial

El problema habitual

Al empezar, muchos fallos no son de programación.
Son de entorno, carpetas, nombres, rutas o cambios sin guardar.

Lo que suele pasar

  • Abro un archivo suelto.
  • Creo index.html donde no toca.
  • No encuentro el archivo recién creado.

Lo que aprenderemos

  • Abrir la carpeta correcta.
  • Organizar archivos.
  • Comprobar rutas y guardado.

Idea clave

VS Code no ejecuta la web.
VS Code sirve para escribir, organizar y revisar el código.
La web se prueba en Chrome. Los errores se revisan en Chrome, VS Code o la terminal, según el caso.

Nombres correctos e incorrectos

Recomendados

  • index.html
  • estilos.css
  • script.js
  • css
  • js
  • img

Evitar

  • Index.html
  • index.HTML
  • estilos css.css
  • mi pagina.html
  • script final bueno.js
  • Nueva carpeta
En programación, los nombres importan. Un espacio, una tilde o una mayúscula pueden complicar el trabajo.

Mini código de prueba

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi primer proyecto</title>
  <link rel="stylesheet" href="css/estilos.css">
</head>
<body>
  <h1>Mi primer proyecto</h1>
  <p>Estoy trabajando con VS Code.</p>

  <script src="js/script.js"></script>
</body>
</html>
No es aprender HTML todavía. Es comprobar que los archivos están bien colocados.
