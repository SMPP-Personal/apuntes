Qué es una extensión

Una extensión añade funciones a VS Code.

  • Abrir una página con servidor local.
  • Formatear código.
  • Detectar errores visualmente.
  • Cambiar idioma.
  • Mejorar la escritura de código.

Advertencia

Más extensiones no significa mejor entorno. Demasiadas extensiones pueden confundir, ralentizar o esconder problemas.

Para qué sirve Live Server

Live Server abre tu proyecto web usando un servidor local.

  • Prueba la página en Chrome.
  • Recarga al guardar cambios.
  • Evita abrir archivos manualmente todo el tiempo.
  • Se parece más a un entorno real.
Live Server no cambia tu código.

Solo cambia la forma de servir la página al navegador.

Error preparado · Go Live no aparece

Posibles causas

  • Live Server no está instalado.
  • La extensión está deshabilitada.
  • No has abierto una carpeta de proyecto.
  • VS Code necesita recargarse.
  • Estás mirando otra ventana de VS Code.

Soluciones razonables

  1. Revisar extensiones instaladas.
  2. Abrir una carpeta, no solo un archivo.
  3. Cerrar y abrir VS Code.
  4. Botón derecho → Open with Live Server.

Qué significa formatear código

Formatear código significa ordenarlo visualmente:

  • sangrías correctas;
  • saltos de línea claros;
  • espacios consistentes;
  • etiquetas bien alineadas;
  • bloques fáciles de leer.

Idea clave

Formatear no significa corregir la lógica. Un código puede estar muy bonito y estar mal.

Bloque 5 · Extensiones, Live Server y formateo20/34

Código sin formato

Puede funcionar, pero se lee mal:

<!DOCTYPE html><html lang="es"><head><meta charset="UTF-8"><title>Mal formato</title><link rel="stylesheet" href="css/estilos.css"></head><body><h1>Texto</h1><p>Un párrafo</p><script src="js/script.js"></script></body></html>
Pregunta: ¿funciona? Puede que sí. ¿Se lee bien? No.
Bloque 5 · Extensiones, Live Server y formateo21/34

Código formateado

El mismo código ordenado:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Buen formato</title>
  <link rel="stylesheet" href="css/estilos.css">
</head>
<body>
  <h1>Texto</h1>
  <p>Un párrafo</p>

  <script src="js/script.js"></script>
</body>
</html>
El formato ayuda a leer, detectar errores y trabajar en equipo.

Formatear al guardar: sí o no

Ventaja

Mantiene el código limpio automáticamente.

Riesgo

Si no entiendes qué cambia, puede confundirte al principio.

Primero aprender a formatear manualmente. Después, si se trabaja con seguridad, activar formato al guardar.

const numero2 = 5;

Problemas frecuentes

No aparece Go Live

Revisar extensión instalada, carpeta abierta y VS Code reiniciado.

La página no cambia

Revisar guardar, URL correcta y archivo correcto.

El CSS no carga

Revisar ruta, nombre del archivo y Network.

Código formateado pero falla

Revisar Console, nombres de variables, rutas y sintaxis.
