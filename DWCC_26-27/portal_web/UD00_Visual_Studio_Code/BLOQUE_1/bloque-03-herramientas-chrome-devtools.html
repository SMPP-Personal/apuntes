El problema habitual

“He escrito el código, pero no funciona.”

Esa frase no basta. A partir de hoy necesitamos responder con datos:

Preguntas útiles

  • ¿Qué no funciona?
  • ¿Dónde falla?
  • ¿Qué dice Chrome?
  • ¿Qué archivo no carga?
  • ¿Qué línea provoca el error?

Respuesta mejor

“El CSS no carga: en Network aparece estilos.css con estado 404.”

Bloque 3 · Chrome DevTools06

Demostración: comprobar JavaScript

Pasos

  1. Abrir script.js.
  2. Escribir un console.log().
  3. Guardar.
  4. Recargar Chrome.
  5. Abrir Console.
  6. Comprobar el mensaje.
console.log("Estoy comprobando la consola");
Si no aparece: guardar, recargar, revisar enlace del JS y comprobar que abrimos el HTML correcto.

Trabajo guiado

Vas a recibir un script.js con errores.

  • Abrir Console.
  • Localizar el primer error.
  • Anotar archivo y línea.

Elements: para qué sirve

Elements permite ver:

  • el HTML que Chrome interpreta;
  • qué elemento ocupa cada zona;
  • qué estilos CSS se aplican;
  • qué estilos están tachados;
  • cambios visuales temporales.
Elements no modifica tus archivos. Solo permite probar cambios en el navegador.

Bloque 3 · Chrome DevTools18

Network: para qué sirve

Network permite ver qué archivos carga Chrome:

  • index.html
  • estilos.css
  • script.js
  • imágenes, fuentes y otros recursos.
Si un archivo no carga, la página puede verse mal o JavaScript puede no ejecutarse.

Bloque 3 · Chrome DevTools20

Error preparado · CSS con ruta incorrecta

HTML incorrecto
<link rel="stylesheet" href="estilos.css">
Corrección
<link rel="stylesheet" href="css/estilos.css">
Actividad: recarga, abre Network, busca estilos.css, comprueba si aparece 404, corrige la ruta y recarga de nuevo.
Bloque 3 · Chrome DevTools21

Error preparado · JavaScript con ruta incorrecta

HTML incorrecto
<script src="script.js"></script>
Corrección
<script src="js/script.js"></script>
Console ayuda a ver si JavaScript se ejecuta. Network ayuda a ver si el archivo JavaScript se ha cargado.
Bloque 3 · Chrome DevTools22

Console, Elements y Network: diferencias

Console

¿Hay mensajes o errores de JavaScript?

Network

¿Se han cargado los archivos necesarios?

Elements

¿Qué HTML y qué CSS está usando Chrome?

Ejemplo: si no se aplica el CSS, Console puede no mostrar nada, Network puede mostrar un 404 y Elements puede mostrar que no hay estilos aplicados.

Bloque 3 · Chrome DevTools30

Cierre del bloque

Chrome no solo sirve para ver la página.

También sirve para preguntar: ¿qué ha cargado?, ¿qué no ha cargado?, ¿qué error aparece?, ¿dónde está el fallo?
Frase final:
Programar no es escribir código y esperar que funcione. Programar también es comprobar, leer errores y corregir.
