Qué es la terminal

La terminal es una forma de dar instrucciones al ordenador escribiendo comandos.

La usaremos para

  • Movernos por carpetas.
  • Comprobar herramientas instaladas.
  • Ejecutar archivos JavaScript.
  • Arrancar servidores más adelante.
  • Usar Git más adelante.
No necesitamos cientos de comandos.
Solo los necesarios para trabajar con proyectos.

Qué es Node.js

Node.js permite ejecutar JavaScript desde la terminal.
Pero Node.js no es Chrome:
No muestra HTML, no aplica CSS, no tiene document ni window como el navegador.

Bloque 4 · Terminal integrada y Node.js22

Chrome y Node.js no son lo mismo

Chrome

  • Ejecuta JS dentro de una página web.
  • Tiene HTML y CSS.
  • Tiene document y window.
  • Los errores se ven en Console.

Node.js

  • Ejecuta JS desde terminal.
  • No carga una página HTML.
  • No tiene document ni window como Chrome.
  • Los errores se ven en la terminal.

JavaScript en Chrome

En navegador, JavaScript puede trabajar con la página

console.log("JavaScript en Chrome");

Chrome tiene document, window, HTML cargado, CSS aplicado y eventos de la página.

Leer errores de terminal

Cuando aparece un error, busca:

  1. Qué comando he escrito.
  2. Qué archivo intenta abrir.
  3. Qué mensaje aparece.
  4. En qué línea falla si el archivo se ejecutó.
  5. Si estoy en la carpeta correcta.
Error típico
Cannot find module
Node.js no encuentra el archivo o módulo que intentas ejecutar.

Detener procesos: Ctrl + C

Ctrl + C sirve para detener un proceso que está ejecutándose en la terminal.

Lo usaremos para detener servidores, cancelar comandos, salir de procesos abiertos y parar ejecuciones cuando sea necesario.
