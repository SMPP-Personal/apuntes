Tres palabras que conviene separar

Error

Algo está mal en el código o en el entorno.

Fallo

Lo que vemos cuando el programa no se comporta como esperamos.

Depuración

Investigar paso a paso hasta encontrar la causa.

Ejemplo: si al pulsar el botón no pasa nada, puede fallar el selector, el evento, la ruta del JS o un error anterior.

Bloque 6 · Detección de errores y depuración básica8/42

Funciona, pero calcula mal

Código válido, resultado incorrecto

const precio = 100;
const descuento = 20;

const total = precio + descuento;
console.log(total);

Corrección lógica

const precio = 100;
const descuento = 20;

const total = precio - descuento;
console.log(total);
Console no marca esto como error porque sumar es una operación válida. El problema está en la lógica.

¿Qué tipo de error es?

A. Falta un paréntesis.

B. Una variable no existe.

C. El cálculo usa + en vez de -.

D. El archivo JS no carga.

E. La página muestra 30 cuando debería mostrar 20.

F. class="destcado", pero en CSS pone .destacado.

A → sintaxis

B → ejecución

C → lógico

D → entorno / ruta

E → lógico

F → nombre / conexión HTML-CSS

Una pausa en el código

Breakpoint

Detiene la ejecución en una línea concreta para observar qué está pasando.

Sirve para

  • pausar antes de un cálculo;
  • ver valores de variables;
  • avanzar paso a paso;
  • comprobar si una función se ejecuta.

Actividad 6 · Por parejas

  1. Una persona provoca un error preparado.
  2. La otra persona debe detectarlo.
  3. Debe explicar qué falla, dónde lo ha visto, qué herramienta ha usado y cómo lo corregiría.
  4. Después se intercambian los roles.
El objetivo no es corregir por intuición. Es verbalizar el diagnóstico.
