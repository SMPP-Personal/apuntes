Bloque 7 · Git y GitHub básico4/46

Git y GitHub no son lo mismo

Git

Herramienta de control de versiones. Trabaja dentro de tu carpeta de proyecto.

  • registra cambios;
  • crea commits;
  • consulta historial;
  • permite volver atrás.

GitHub

Plataforma en la nube para alojar repositorios Git.

  • guarda copia remota;
  • permite compartir;
  • permite clonar;
  • facilita colaboración posterior.
Bloque 7 · Git y GitHub básico5/46

Guardar, commit y push

Guardar

Guarda cambios en el archivo abierto.

Ctrl + S

Commit

Guarda una versión del proyecto en el historial local de Git.

Push

Sube tus commits desde el ordenador a GitHub.

Guardar no es hacer commit. Hacer commit no es subir a GitHub.

Comprobar instalación de Git

Terminal
git --version

Resultado esperado

Aparece algo como git version 2.x.x.

Si falla

Git no está instalado o la terminal no lo encuentra.

Bloque 7 · Git y GitHub básico8/46

Error preparado: Git no reconocido

Terminal
"git" no se reconoce como un comando interno o externo

Causas probables

  • Git no está instalado.
  • VS Code estaba abierto durante la instalación.
  • El PATH no se actualizó.

Primeras soluciones

  1. Cerrar VS Code.
  2. Abrir VS Code de nuevo.
  3. Abrir terminal nueva.
  4. Probar git --version.

Configurar identidad

Git necesita saber quién hace los commits.

Inicializar repositorio

Terminal · dentro del proyecto

Convierte la carpeta actual en un repositorio Git.

Hazlo solo en la carpeta del proyecto. No inicialices Git en una carpeta enorme.

Actividad 1 · Primer commit

Bloque 7 · Git y GitHub básico35/46

Cuándo hacer commit

Buen momento

  • crear estructura inicial;
  • añadir estilos;
  • conectar JavaScript;
  • corregir error de ruta;
  • terminar mini práctica.

Mal hábito

  • un único commit al final;
  • commits cada 10 segundos sin sentido;
  • proyecto roto sin indicarlo;
  • mensajes tipo final-final.
Bloque 7 · Git y GitHub básico36/46

Cuándo hacer push

Haz push cuando quieras subir a GitHub los commits locales.

□ al terminar una sesión
□ antes de cambiar de ordenador
□ antes de entregar
□ para guardar copia remota
Si haces commit pero no haces push, GitHub no recibe esos cambios.
Bloque 7 · Git y GitHub básico37/46

Errores preparados

Commit sin add

no changes added to commit

Hay cambios, pero no están preparados.

Push sin commit

Push solo sube commits. Si no hay commit, no hay nueva versión que subir.

git add .
git commit -m "Mensaje claro"
git push
Bloque 7 · Git y GitHub básico38/46

Error: repositorio en carpeta equivocada

Creo Git en una carpeta, pero edito archivos en otra.

Síntomas

  • Git no ve cambios.
  • VS Code muestra otro proyecto.
  • GitHub no recibe los archivos esperados.

Revisión

  • Nombre de carpeta abierta.
  • Ruta de la terminal.
  • Archivos visibles.
  • git status.

Cierre del bloque

Hoy no hemos aprendido Git avanzado. Hemos aprendido el flujo mínimo para no perder el control de un proyecto.
guardar→commit→historial→push→clone
Git no es solo para entregar trabajos. Git sirve para no perder el control de tu propio proyecto.
Bloque 7 · Git y GitHub básico45/46

Idea final

Git

Controla versiones.

Permite saber qué cambió, cuándo y con qué mensaje.

GitHub

Guarda una copia remota.

Permite compartir, clonar y recuperar proyectos.

Git no es una herramienta para “subir cosas”. Es una herramienta para controlar versiones.
