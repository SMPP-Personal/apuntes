UD2_B3c. jQuery básico: DOM y eventos

1. Contexto y finalidad

1.1. ¿Por qué incluimos jQuery en esta unidad?

jQuery es una biblioteca de JavaScript que simplifica tareas frecuentes en el navegador:

  • seleccionar elementos del DOM;
  • modificar texto, HTML, atributos, propiedades, estilos y clases;
  • gestionar eventos;
  • crear, insertar y eliminar elementos;
  • aplicar efectos visuales sencillos;
  • comprender y mantener código existente que ya utiliza jQuery.

Actualmente, JavaScript estándar moderno permite realizar estas tareas con herramientas como:

  • querySelector() y querySelectorAll();
  • textContent e innerHTML;
  • classList;
  • addEventListener();
  • createElement() y append();
  • fetch().

Aun así, jQuery sigue siendo útil para:

  • entender proyectos antiguos o mantenidos con jQuery;
  • leer código heredado;
  • trabajar con plantillas o plugins que dependen de esta biblioteca;
  • comparar la sintaxis de JavaScript nativo con la de una biblioteca;
  • comprender ejemplos y documentación creados durante muchos años.

En este módulo utilizaremos jQuery como complemento didáctico, no como sustituto de JavaScript moderno ni del aprendizaje del DOM nativo.

1.2. ¿Qué es una biblioteca JavaScript?

Una biblioteca es un conjunto de código ya preparado que podemos incorporar a un proyecto para facilitar tareas habituales.

jQuery no sustituye ni modifica el lenguaje JavaScript. Proporciona funciones y métodos que simplifican operaciones frecuentes sobre el DOM, los eventos y otros elementos del navegador.

Ejemplo comparativo:

// JavaScript estándar
const titulo = document.querySelector("#titulo");
titulo.textContent = "Nuevo título";

// jQuery
$("#titulo").text("Nuevo título");

En ambos casos se selecciona el mismo elemento y se modifica su texto.

1.3. Situación actual de jQuery

En los ejemplos nuevos de este documento utilizaremos jQuery 4.0.0.

Esto no significa que todos los proyectos existentes utilicen esa versión. Es frecuente encontrar aplicaciones con jQuery 1.x, 2.x o 3.x.

En un proyecto heredado no debemos cambiar directamente la versión existente de jQuery. Antes hay que revisar la compatibilidad del código, los plugins utilizados y la guía de migración correspondiente.

2. Carga y preparación de jQuery

2.1. Añadir jQuery desde un CDN

Podemos cargar jQuery mediante una etiqueta <script>:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  >

  <title>Ejemplo con jQuery</title>
</head>
<body>
  <h1 id="titulo">Título inicial</h1>

  <script src="https://code.jquery.com/jquery-4.0.0.min.js"></script>
  <script src="script.js"></script>
</body>
</html>

El orden es importante:

  1. primero se carga jQuery;
  2. después se carga nuestro archivo script.js.

Si nuestro script intenta utilizar $ antes de que jQuery esté disponible, se producirá un error.

2.2. Cargar una copia local

También podemos descargar jQuery y guardarlo dentro del proyecto:

proyecto/
├── index.html
├── js/
│   ├── jquery-4.0.0.min.js
│   └── script.js
└── css/
    └── estilos.css

En el HTML:

<script src="js/jquery-4.0.0.min.js"></script>
<script src="js/script.js"></script>

La copia local permite trabajar sin conexión a Internet.

2.3. Versión completa y versión slim

jQuery ofrece una compilación completa y una compilación reducida denominada slim.

La versión slim no incluye:

  • el módulo AJAX;
  • los efectos y animaciones de jQuery.

En este documento se muestran métodos como:

$.get();
$("#panel").fadeIn();
$("#panel").slideToggle();

Por tanto, para probar todos los ejemplos necesitamos la versión completa, no la versión slim.

2.4. Esperar a que el documento esté preparado

En código jQuery tradicional es frecuente utilizar:

$(document).ready(function () {
  console.log("Documento preparado");
});

La forma abreviada es:

$(function () {
  console.log("Documento preparado");
});

Estas funciones ejecutan el código cuando el DOM está disponible.

No siempre son obligatorias:

  • si el script está al final de <body>, los elementos anteriores ya han sido procesados;
  • si el script utiliza defer, se ejecuta después de analizar el HTML;
  • en código heredado o cuando no controlamos la posición del script, $(function () { ... }) sigue siendo un patrón habitual.

3. Selectores y objetos jQuery

3.1. El símbolo $

En jQuery se utiliza frecuentemente el símbolo $:

$("#titulo")

Este código:

  1. busca los elementos que coinciden con el selector;
  2. devuelve un objeto jQuery;
  3. permite utilizar métodos como .text(), .html(), .addClass() o .on().

De forma aproximada:

document.querySelector("#titulo");

y:

$("#titulo");

realizan una selección similar, pero no devuelven el mismo tipo de objeto.

3.2. Seleccionar por identificador

HTML:

<h1 id="titulo">Apuntes de DWCC</h1>

jQuery:

$("#titulo").text("UD2: JavaScript avanzado");

3.3. Seleccionar por clase

HTML:

<p class="aviso">Primer aviso</p>
<p class="aviso">Segundo aviso</p>

jQuery:

$(".aviso").text("Aviso modificado");

Se modifican todos los elementos que tienen la clase aviso.

3.4. Seleccionar por etiqueta

$("p").addClass("parrafo-destacado");

Selecciona todos los párrafos.

3.5. Selectores combinados

jQuery utiliza selectores CSS:

$("main .tarjeta h2").addClass("titulo-tarjeta");

Podemos combinar:

  • etiquetas;
  • identificadores;
  • clases;
  • atributos;
  • descendientes;
  • pseudoclases admitidas por el selector.

3.6. Objeto jQuery frente a elemento DOM

Esta selección devuelve un objeto jQuery:

const tituloJQuery = $("#titulo");

Para obtener el primer elemento DOM de la selección:

const tituloDOM1 = tituloJQuery[0];
const tituloDOM2 = tituloJQuery.get(0);

Los métodos dependen del tipo de objeto:

// Método jQuery
$("#titulo").text("Nuevo título");

// Propiedad del elemento DOM
$("#titulo")[0].textContent = "Nuevo título";

No debemos mezclar directamente ambas interfaces:

// Incorrecto: textContent no es un método del objeto jQuery
$("#titulo").textContent = "Nuevo título";

3.7. Selecciones sin resultados

Si querySelector() no encuentra un elemento, devuelve null:

const elemento =
  document.querySelector("#noExiste");

console.log(elemento); // null

Una selección jQuery vacía sigue siendo un objeto jQuery:

const resultado =
  $("#noExiste");

console.log(resultado.length); // 0

Esta operación no produce necesariamente un error:

$("#noExiste").text("Texto");

Simplemente no modifica ningún elemento.

Para comprobar si existe alguna coincidencia:

if ($("#titulo").length > 0) {
  console.log("El elemento existe");
}

4. Texto, HTML y valores de formularios

4.1. Leer y modificar texto con .text()

HTML:

<h1 id="titulo">JavaScript</h1>

Leer el texto:

const texto = $("#titulo").text();

console.log(texto);

Modificarlo:

$("#titulo").text("JavaScript avanzado");

.text() equivale aproximadamente a trabajar con textContent en JavaScript estándar.

4.2. Leer y modificar HTML con .html()

HTML:

<section id="contenedor"></section>

jQuery:

$("#contenedor").html(
  "<p>Texto creado desde jQuery</p>"
);

.html() interpreta la cadena como contenido HTML y equivale aproximadamente a innerHTML.

4.3. Seguridad al utilizar .html()

No debemos insertar con .html() contenido:

  • escrito por personas usuarias;
  • recibido de una fuente no fiable;
  • procedente de datos externos no controlados.

La validación no convierte automáticamente un texto en HTML seguro.

Ejemplo inseguro:

const nombre = $("#nombre").val();

$("#salida").html(nombre);

Alternativa segura:

const nombre = $("#nombre").val();

$("#salida").text(nombre);

Cuando solo queremos mostrar texto, debemos utilizar .text().

Las cadenas HTML literales son adecuadas cuando el contenido está completamente controlado por el programa:

$("#salida").html(
  "<strong>Operación completada</strong>"
);

4.4. Leer y modificar valores con .val()

HTML:

<input id="nombre" type="text">

Leer el valor:

const nombre = $("#nombre").val();

console.log(nombre);

Cambiarlo:

$("#nombre").val("Ana");

.val() equivale aproximadamente a la propiedad .value de JavaScript estándar.

4.5. Formularios y métodos adecuados

Elemento JavaScript estándar jQuery
Texto de un h1 o p textContent .text()
HTML interno innerHTML .html()
Valor de input, select o textarea .value .val()
Estado de un checkbox .checked .prop("checked")

5. Atributos, propiedades, clases y estilos

5.1. Atributos con .attr()

HTML:

<img
  id="foto"
  src="img/default.png"
  alt="Imagen por defecto"
>

Leer un atributo:

const ruta = $("#foto").attr("src");

console.log(ruta);

Modificar atributos:

$("#foto").attr("src", "img/nueva.png");
$("#foto").attr("alt", "Nueva imagen");

.attr() es apropiado para atributos como:

  • src;
  • href;
  • alt;
  • title;
  • data-*.

5.2. Propiedades con .prop()

Algunos estados de los controles se representan mediante propiedades del DOM.

Para consultar un checkbox:

const aceptado =
  $("#aceptaCondiciones").prop("checked");

Para desactivar un botón:

$("#botonEnviar").prop("disabled", true);

Para activarlo:

$("#botonEnviar").prop("disabled", false);

Para estados booleanos como checked, disabled o selected, debemos utilizar .prop().

Comparación:

Necesidad Método recomendado
Leer src, href o alt .attr()
Saber si un checkbox está marcado .prop("checked")
Activar o desactivar un botón .prop("disabled", valor)
Saber si una opción está seleccionada .prop("selected")

5.3. Añadir, quitar y alternar clases

Añadir una clase:

$("#mensaje").addClass("correcto");

Quitar una clase:

$("#mensaje").removeClass("error");

Alternar una clase:

$("#panel").toggleClass("oculto");

Comprobar si existe una clase con .hasClass():

if ($("#panel").hasClass("oculto")) {
  console.log("El panel está oculto");
}

Comparación:

document
  .querySelector("#mensaje")
  .classList
  .add("correcto");

$("#mensaje").addClass("correcto");

5.4. Estilos con .css()

Modificar una propiedad:

$("#titulo").css("color", "blue");

Modificar varias:

$("#titulo").css({
  color: "blue",
  backgroundColor: "#eef",
  padding: "1rem"
});

En proyectos reales suele ser preferible añadir o quitar clases:

$("#titulo").addClass("titulo-destacado");

Esto permite mantener la presentación en el CSS.

6. Creación y manipulación del DOM

6.1. Crear elementos

Podemos crear un elemento con $():

const parrafo = $("<p></p>");

parrafo.text(
  "Párrafo creado con jQuery"
);

También podemos usar una cadena HTML controlada:

const parrafo =
  $("<p>Párrafo creado con jQuery</p>");

Cuando el contenido procede del usuario, es más seguro separar la creación y el texto:

const textoUsuario =
  $("#entrada").val();

const parrafo =
  $("<p></p>").text(textoUsuario);

6.2. Insertar al final con .append()

<section id="lista"></section>

const elemento =
  $("<p></p>").text("Nuevo elemento");

$("#lista").append(elemento);

6.3. Insertar al principio con .prepend()

const elemento =
  $("<p></p>").text("Elemento inicial");

$("#lista").prepend(elemento);

6.4. Insertar antes o después

const antes =
  $("<p></p>").text("Texto antes");

const despues =
  $("<p></p>").text("Texto después");

$("#titulo").before(antes);
$("#titulo").after(despues);

6.5. Eliminar elementos

$("#aviso").remove();

Si la selección contiene varios elementos:

$(".temporal").remove();

se eliminan todos.

6.6. Recorrer elementos con .each()

HTML:

<ul>
  <li>Ana</li>
  <li>Luis</li>
  <li>María</li>
</ul>

jQuery:

$("li").each(function (indice, elemento) {
  console.log(
    indice,
    $(elemento).text()
  );
});

Dentro de .each():

  • indice es la posición;
  • elemento es un elemento DOM;
  • $(elemento) lo envuelve en un objeto jQuery.
$(elemento).addClass("item-lista");

7. Eventos directos y delegados

7.1. Gestionar eventos con .on()

HTML:

<button id="boton">Pulsar</button>
<p id="mensaje"></p>

jQuery:

$("#boton").on("click", function () {
  $("#mensaje").text("Botón pulsado");
});

Equivalente aproximado:

document
  .querySelector("#boton")
  .addEventListener("click", function () {
    document
      .querySelector("#mensaje")
      .textContent = "Botón pulsado";
  });

7.2. Evento input

$("#nombre").on("input", function () {
  const valor = $(this).val();

  $("#salida").text(valor);
});

7.3. Evento change

$("#modalidad").on("change", function () {
  const modalidad = $(this).val();

  $("#salida").text(modalidad);
});

7.4. Evento submit

$("#formulario").on(
  "submit",
  function (event) {
    event.preventDefault();

    const nombre =
      $("#nombre").val().trim();

    if (nombre === "") {
      $("#error").text(
        "El nombre es obligatorio"
      );

      return;
    }

    $("#error").text("");
    $("#resultado").text(
      "Formulario correcto"
    );
  }
);

7.5. this, event.target y event.currentTarget

En una función tradicional:

$(".boton-accion").on(
  "click",
  function (event) {
    console.log(this);
    console.log(event.currentTarget);
    console.log(event.target);
  }
);
  • this representa el elemento para el que jQuery está ejecutando el manejador;
  • normalmente coincide con event.currentTarget;
  • event.target es el elemento concreto en el que se originó el evento;
  • event.target puede ser un descendiente del elemento que tiene el manejador.

Ejemplo:

<button class="boton-accion">
  <span>Guardar</span>
</button>

Si se pulsa el <span>:

  • event.target puede ser el span;
  • this representa el botón.

7.6. Funciones flecha y this

Las funciones flecha no crean su propio this.

Evitar cuando necesitamos el elemento del evento:

$(".boton-accion").on(
  "click",
  () => {
    $(this).addClass("activo");
  }
);

Utilizar una función tradicional:

$(".boton-accion").on(
  "click",
  function () {
    $(this).addClass("activo");
  }
);

7.7. Eventos directos

$(".boton").on(
  "click",
  function () {
    console.log("Pulsado");
  }
);

Este listener se asocia a los elementos que existen en el momento de ejecutar la selección.

7.8. Eventos delegados

Si los botones se crean después, podemos delegar el evento:

$("#lista").on(
  "click",
  ".boton-eliminar",
  function () {
    $(this).closest("li").remove();
  }
);

En este caso:

  • el listener se asocia a #lista;
  • jQuery comprueba si el evento procede de .boton-eliminar;
  • funciona también con botones creados dinámicamente.

La delegación es habitual en aplicaciones que generan contenido desde JavaScript.

8. Visibilidad y efectos

8.1. Mostrar, ocultar y alternar

$("#panel").hide();
$("#panel").show();
$("#panel").toggle();

8.2. Efectos sencillos

$("#panel").fadeIn();
$("#panel").fadeOut();
$("#panel").slideToggle();

Estos efectos:

  • son una mejora visual;
  • no deben sustituir una estructura correcta de HTML, CSS y JavaScript;
  • no deben impedir el acceso al contenido;
  • no están disponibles en la compilación slim.

Conviene evitar animaciones excesivas o innecesarias.

9. Ejemplo guiado: lista de tareas con jQuery

9.1. HTML

<h1>Lista de tareas</h1>

<form id="formulario-tarea">
  <label for="tarea">Nueva tarea:</label>

  <input
    id="tarea"
    name="tarea"
    type="text"
  >

  <button type="submit">
    Añadir
  </button>
</form>

<p
  id="mensaje"
  aria-live="polite"
></p>

<ul id="lista-tareas"></ul>

9.2. JavaScript con jQuery

$(function () {
  $("#formulario-tarea").on(
    "submit",
    function (event) {
      event.preventDefault();

      const texto =
        $("#tarea").val().trim();

      if (texto === "") {
        $("#mensaje")
          .text("Debes escribir una tarea")
          .removeClass("correcto")
          .addClass("error");

        return;
      }

      const item =
        $("<li></li>")
          .addClass("tarea");

      const contenido =
        $("<span></span>")
          .text(texto);

      const botonEliminar =
        $("<button></button>")
          .attr("type", "button")
          .addClass("boton-eliminar")
          .text("Eliminar");

      item.append(
        contenido,
        botonEliminar
      );

      $("#lista-tareas").append(item);

      $("#mensaje")
        .text("Tarea añadida correctamente")
        .removeClass("error")
        .addClass("correcto");

      $("#tarea")
        .val("")
        .trigger("focus");
    }
  );

  $("#lista-tareas").on(
    "click",
    ".boton-eliminar",
    function () {
      $(this).closest("li").remove();

      $("#mensaje")
        .text("Tarea eliminada")
        .removeClass("error")
        .addClass("correcto");
    }
  );
});

10. AJAX como contexto y código heredado

10.1. AJAX en jQuery

jQuery incluye métodos para realizar peticiones asíncronas:

  • $.ajax();
  • $.get();
  • $.post().

Durante muchos años se utilizaron ampliamente.

Ejemplo:

$.get(
  "datos/productos.json",
  function (datos) {
    console.log(datos);
  }
);

Estos métodos requieren la versión completa de jQuery.

10.2. Enfoque actual del curso

En este curso priorizaremos la API estándar fetch().

Ejemplo contextual:

async function cargarProductos() {
  const respuesta =
    await fetch(
      "datos/productos.json"
    );

  const datos =
    await respuesta.json();

  console.log(datos);
}

cargarProductos();

Este ejemplo solo muestra la estructura básica.

En UD2_B4 se trabajarán:

  • comprobación de response.ok;
  • try...catch;
  • estados de carga;
  • tratamiento de errores;
  • consumo de APIs;
  • promesas;
  • async/await.

jQuery AJAX se estudia aquí únicamente para reconocer y mantener código heredado.

11.2. ¿Cuándo tiene sentido utilizar jQuery?

Puede tener sentido:

  • en un proyecto existente que ya lo utiliza;
  • cuando una plantilla o un plugin depende de jQuery;
  • para comprender código heredado;
  • para mantener aplicaciones antiguas;
  • como introducción a la idea de biblioteca JavaScript.

No suele ser imprescindible:

  • en proyectos nuevos pequeños;
  • cuando JavaScript estándar resuelve claramente el problema;
  • cuando se utilizan frameworks modernos;
  • para sustituir el aprendizaje del DOM y los eventos nativos.

12. Errores, preguntas, resumen y contenidos relacionados

12.1. Errores frecuentes

Error 1. Cargar el script propio antes de jQuery

Incorrecto:

<script src="script.js"></script>
<script src="https://code.jquery.com/jquery-4.0.0.min.js"></script>

Correcto:

<script src="https://code.jquery.com/jquery-4.0.0.min.js"></script>
<script src="script.js"></script>

Error 2. Olvidar # o . en los selectores

$("titulo");  // Busca una etiqueta <titulo>
$("#titulo"); // Busca id="titulo"
$(".titulo"); // Busca class="titulo"

Error 3. Confundir .text() y .html()

$("#salida").text(
  "<strong>Hola</strong>"
);

Muestra las etiquetas como texto.

$("#salida").html(
  "<strong>Hola</strong>"
);

Interpreta las etiquetas como HTML.

Error 4. Utilizar .val() en un elemento que no es un control

Incorrecto:

$("#titulo").val();

Correcto:

$("#titulo").text();

Error 5. Confundir objeto jQuery y elemento DOM

Incorrecto:

$("#titulo").textContent =
  "Nuevo título";

Correcto con jQuery:

$("#titulo").text(
  "Nuevo título"
);

Correcto con DOM:

$("#titulo")[0].textContent =
  "Nuevo título";

Error 6. Utilizar .attr() para estados booleanos

Problemático:

$("#acepta").attr("checked");

Correcto:

$("#acepta").prop("checked");

Error 7. Introducir datos del usuario con .html()

Incorrecto:

$("#salida").html(
  $("#nombre").val()
);

Correcto:

$("#salida").text(
  $("#nombre").val()
);

Error 8. Utilizar una función flecha cuando se necesita this

Problemático:

$(".item").on(
  "click",
  () => {
    $(this).addClass("activo");
  }
);

Correcto:

$(".item").on(
  "click",
  function () {
    $(this).addClass("activo");
  }
);

Error 9. Asociar el evento antes de crear los elementos

Problemático:

$(".boton-eliminar").on(
  "click",
  function () {
    // Los botones se crean después
  }
);

Solución delegada:

$("#lista").on(
  "click",
  ".boton-eliminar",
  function () {
    $(this).closest("li").remove();
  }
);

Error 10. Pensar que jQuery sustituye JavaScript

jQuery está escrito en JavaScript y trabaja sobre el DOM.

Para utilizarlo correctamente es necesario comprender:

  • variables;
  • funciones;
  • arrays;
  • objetos;
  • DOM;
  • eventos;
  • callbacks.

12.3. Resumen final

  • jQuery es una biblioteca JavaScript para trabajar con el DOM, eventos, efectos y AJAX.
  • Se utiliza como complemento para comprender y mantener código existente.
  • En los ejemplos nuevos se emplea jQuery 4.0.0.
  • $() devuelve un objeto jQuery, no directamente un elemento DOM.
  • Los selectores jQuery utilizan sintaxis CSS.
  • .text() trabaja con texto.
  • .html() interpreta contenido HTML.
  • .val() trabaja con valores de formularios.
  • .attr() gestiona atributos.
  • .prop() gestiona propiedades como checked o disabled.
  • .addClass(), .removeClass() y .toggleClass() modifican clases.
  • .on() gestiona eventos directos y delegados.
  • Dentro de una función tradicional, this representa el elemento para el que se ejecuta el manejador.
  • .append(), .prepend(), .before() y .after() insertan contenido.
  • .remove() elimina elementos.
  • La versión slim no incluye AJAX ni efectos.
  • En comunicación asíncrona se priorizará fetch().
