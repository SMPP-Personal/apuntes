UD2_B3. Práctica global: interacción con el usuario

1. Finalidad, resultado esperado y archivos

1.1. Finalidad de la práctica

Esta práctica global cierra el bloque UD2_B3 — Eventos, interacción con el usuario y formularios.

El objetivo es construir una pequeña aplicación web interactiva que permita al usuario:

  • consultar una lista de actividades;
  • filtrar actividades por categoría;
  • seleccionar una actividad;
  • completar un formulario de inscripción;
  • validar los datos introducidos;
  • mostrar mensajes de error o confirmación;
  • actualizar la interfaz según las acciones realizadas.

La práctica integra contenidos de bloques anteriores:

  • objetos y arrays de objetos;
  • métodos de array;
  • manipulación del DOM;
  • creación dinámica de elementos;
  • eventos;
  • formularios;
  • validación básica;
  • expresiones regulares sencillas.

En esta práctica no se utilizarán fetch, promesas, async/await, APIs externas ni almacenamiento persistente. Esos contenidos se trabajarán en UD2_B4.

1.2. Resultado final esperado

La aplicación se llamará:

Gestor interactivo de actividades

Deberá permitir:

  1. mostrar una lista de actividades disponibles;
  2. filtrar las actividades por categoría;
  3. indicar cuántas actividades están visibles;
  4. seleccionar una actividad mediante un botón;
  5. mostrar los datos de la actividad seleccionada;
  6. completar un formulario de inscripción;
  7. validar el formulario antes de enviarlo;
  8. mostrar mensajes de error junto a los controles;
  9. usar clases CSS para diferenciar estados correctos e incorrectos;
  10. activar o desactivar el botón de envío según el estado del formulario;
  11. mostrar un resumen final seguro;
  12. mantener correctamente el estado de la actividad seleccionada aunque cambie el filtro.

1.3. Nivel de autonomía

Esta es una práctica global, no una práctica completamente guiada.

Se proporcionan:

  • estructura general;
  • datos iniciales;
  • firmas de funciones;
  • reglas obligatorias;
  • algunas pistas y ejemplos parciales;
  • pruebas de comprobación.

No se proporciona la solución completa. Debes decidir cómo organizar parte del código y completar las funciones indicadas.

2.2. Uso de novalidate

El formulario contiene:

<form id="formularioInscripcion" novalidate>

Se utiliza para que la validación y los mensajes se gestionen mediante JavaScript.

Los atributos required, min, max, step y minlength se mantienen porque:

  • documentan las reglas;
  • mejoran la semántica;
  • permiten comparar posteriormente validación HTML y JavaScript.

function validarNombre() {
  const valor = nombre.value.trim();

  if (valor === "") {
    mostrarError(
      nombre,
      errorNombre,
      "El nombre es obligatorio."

      nombre,
      errorNombre,
      "El nombre debe tener al menos 3 caracteres."

    return false;
  }

  mostrarCorrecto(nombre, errorNombre);
  return true;
}

function validarFormulario() {
  const actividadValida =
    validarActividadSeleccionada();

  const nombreValido =
    validarNombre();

  const emailValido =
    validarEmail();

  const edadValida =
    validarEdad();

  const nivelValido =
    validarNivel();

  const condicionesValidas =
    validarCondiciones();

  return (
    actividadValida
    && nombreValido
    && emailValido
    && edadValida
    && nivelValido
    && condicionesValidas
  );
}

Las validaciones se ejecutan por separado antes del return para que aparezcan todos los errores y no se produzca un cortocircuito que impida validar los campos posteriores.

function crearDatoResumen(etiqueta, valor) {

  parrafo.append(strong, String(valor));

  return parrafo;
}

10.2. Elige una ampliación adicional

Opción A. Actividades sin plazas

  • conserva al menos una actividad con plazas: 0;
  • aplica una clase especial;
  • desactiva su botón;

Opción B. Reiniciar la aplicación

Añade un botón que:

  • limpie el formulario;
  • borre mensajes y clases;
  • oculte o reinicie el resumen;
  • elimine la actividad seleccionada;
  • restablezca el filtro;
  • vuelva a renderizar todas las actividades;
  • desactive el botón de envío.

No modifiques directamente el array original si puedes evitarlo. Puedes utilizar una copia:

const copia = [...actividades];

11.2. Errores frecuentes

Error 1. No usar preventDefault()

formulario.addEventListener("submit", () => {
  mostrarResumenInscripcion();
});

El formulario puede recargar la página.

Error 2. Usar = en lugar de ===

if (categoria = "todas") {
  // Error
}

Debe utilizarse:

if (categoria === "todas") {
  // Correcto
}

Error 3. No convertir dataset.id

const id = event.target.dataset.id;

dataset.id devuelve texto. Convierte con:

const id = Number(botonSeleccionar.dataset.id);

Error 4. Acceder a una actividad inexistente

Antes de usar:

actividadElegida.titulo

comprueba que actividadElegida no sea null.

Error 5. Cortocircuitar las validaciones visibles

Evita:

return (
  validarActividadSeleccionada()
  && validarNombre()
  && validarEmail()
);

Si una validación falla, las siguientes pueden no ejecutarse.

Error 6. Confiar solo en el botón desactivado

El formulario debe volver a validarse dentro de submit.

Error 7. Usar innerHTML con datos del usuario

Usa nodos DOM y textContent.

Error 8. Detectar el botón únicamente con tagName

Es más robusto usar:

event.target.closest("button[data-id]")

Error 9. Aceptar una edad decimal

Comprueba:

Number.isInteger(valor)
