UD2_B2. Práctica guiada: Galería generada con DOM

1. Objetivo de la práctica

En esta práctica vas a crear una pequeña galería de tarjetas generada desde JavaScript. La página HTML tendrá una estructura mínima y será el script quien cree dinámicamente los elementos de la galería mediante el DOM.

La práctica sirve para consolidar los contenidos del bloque UD2_B2: DOM, estructura, acceso y manipulación, sin entrar todavía en eventos ni formularios.

Al terminar, tendrás una página que muestra una colección de lugares a partir de un array de objetos.

2. Qué vas a practicar

En esta práctica trabajarás:

  • selección de elementos con querySelector;
  • escritura y modificación de contenido mediante textContent;
  • creación de elementos con createElement;
  • modificación de atributos con setAttribute;
  • trabajo con clases CSS mediante classList.add();
  • inserción de elementos con append();
  • eliminación de elementos con remove();
  • renderizado de contenido a partir de un array de objetos;
  • separación entre HTML, CSS y JavaScript.

En este bloque no usaremos todavía eventos como click, submit o input. Estos contenidos se trabajarán en el bloque UD2_B3.

Copia este código en index.html:

  <meta name="viewport" content="width=device-width, initial-scale=1.0">

    </p>
  </header>

  <main class="contenedor">
    <section class="panel-info">
      <h2>Resumen</h2>
      <p id="total-elementos">Pendiente de calcular...</p>
    </section>

    <section>
      <h2>Lugares destacados</h2>
      <p id="mensaje-inicial">La galería todavía no se ha generado.</p>
      <div id="galeria" class="galeria"></div>
    </section>
  </main>

  <script src="js/app.js"></script>

</html>

Observa que el HTML incluye un contenedor vacío:

<div id="galeria" class="galeria"></div>

Ese contenedor se rellenará desde JavaScript.

El elemento:

<p id="mensaje-inicial">La galería todavía no se ha generado.</p>

se eliminará cuando se renderice la galería por primera vez.

Pregunta de reflexión

¿Por qué usamos textContent para escribir el título, la descripción o la localización?

Comprobar la respuesta

Porque se trata de texto normal y no necesitamos que el navegador interprete etiquetas HTML. textContent evita insertar HTML accidentalmente y es la opción adecuada cuando solo queremos mostrar texto.

11. Paso 6: convertir la creación de tarjetas en una función

Ahora vamos a organizar mejor el código. Crea una función llamada crearTarjetaLugar.

function crearTarjetaLugar(lugar) {
  const tarjeta = document.createElement("article");
  tarjeta.classList.add("tarjeta");

  if (lugar.destacado) {
    tarjeta.classList.add("destacado");
  }

  const imagen = document.createElement("img");
  imagen.setAttribute("src", lugar.imagen);
  imagen.setAttribute("alt", lugar.textoAlternativo);

  const contenido = document.createElement("div");
  contenido.classList.add("tarjeta-contenido");

  const titulo = document.createElement("h3");
  titulo.textContent = lugar.nombre;

  const descripcion = document.createElement("p");
  descripcion.textContent = lugar.descripcion;

  const localizacion = document.createElement("p");
  localizacion.classList.add("localizacion");
  localizacion.textContent = `Localización: ${lugar.localizacion}`;

  const etiqueta = document.createElement("span");
  etiqueta.classList.add("etiqueta");
  etiqueta.textContent = lugar.categoria;

  contenido.append(titulo, descripcion, localizacion, etiqueta);
  tarjeta.append(imagen, contenido);

  return tarjeta;
}

Esta función:

  1. recibe un objeto lugar;
  2. crea una tarjeta completa;
  3. devuelve el elemento creado.

Sustitución obligatoria

Elimina ahora el código temporal de los pasos 4 y 5, desde:

const primerLugar = lugares[0];

hasta:

galeria.append(tarjeta);

La función crearTarjetaLugar() sustituye todo ese bloque.

No debes conservar simultáneamente la creación manual y la función, porque estarías repitiendo el mismo proceso dos veces.

12. Paso 7: limpiar la galería antes de renderizar

Antes de pintar todos los lugares, limpia el contenido de la galería:

galeria.textContent = "";

En este caso usamos textContent = "" para vaciar el contenedor.

13. Paso 8: generar toda la galería con forEach

Usa forEach para crear una tarjeta por cada lugar:

lugares.forEach((lugar) => {
  const tarjetaLugar = crearTarjetaLugar(lugar);
  galeria.append(tarjetaLugar);
});

Ahora la página debe mostrar todas las tarjetas del array.

Resultado esperado

La página debe mostrar:

  • cuatro tarjetas;
  • dos tarjetas con la clase destacado;
  • una imagen, título, descripción, localización y categoría en cada tarjeta;
  • el texto La galería contiene 4 lugares.;
  • ningún error en la consola.

Código temporal: este recorrido se integrará en una función en el siguiente paso.

14. Paso 9: renderizar desde una función

Crea una función llamada renderizarGaleria para agrupar el proceso completo:

function renderizarGaleria(listaLugares) {
  galeria.textContent = "";

  if (listaLugares.length === 0) {
    const mensajeVacio = document.createElement("p");
    mensajeVacio.classList.add("mensaje-vacio");
    mensajeVacio.textContent = "No hay lugares para mostrar.";
    galeria.append(mensajeVacio);
  } else {
    listaLugares.forEach((lugar) => {
      const tarjetaLugar = crearTarjetaLugar(lugar);
      galeria.append(tarjetaLugar);
    });
  }

  totalElementos.textContent =
    `La galería contiene ${listaLugares.length} lugares.`;
}

Al final del archivo, elimina el recorrido temporal del paso anterior y llama a la función:

renderizarGaleria(lugares);

Sustitución obligatoria

No debes conservar a la vez:

lugares.forEach(...)

fuera de la función y:

renderizarGaleria(lugares);

La función renderizarGaleria() sustituye el recorrido anterior.

15. Paso 10: eliminar el mensaje inicial

Una vez que la galería se ha renderizado, elimina el mensaje inicial:

mensajeInicial.remove();

Coloca esta instrucción después de:

renderizarGaleria(lugares);

El final del archivo debe quedar así:

renderizarGaleria(lugares);
mensajeInicial.remove();

Este paso practica el método remove().

En una aplicación real, los mensajes de estado suelen mostrarse u ocultarse según haya o no resultados. En esta práctica eliminamos el mensaje inicial porque solo informa de que la galería todavía no se ha generado. El caso de una lista vacía ya se gestiona dentro de renderizarGaleria().

16. Paso 11: probar una galería filtrada sin eventos

En este bloque todavía no vamos a usar botones ni eventos. Sin embargo, podemos practicar el renderizado con una lista filtrada.

Crea una constante con los lugares destacados:

const lugaresDestacados = lugares.filter(
  (lugar) => lugar.destacado
);

filter() no modifica el array original. Devuelve un nuevo array que contiene únicamente los elementos que cumplen la condición.

Prueba temporalmente esta llamada:

renderizarGaleria(lugaresDestacados);

La página debe mostrar solo dos tarjetas.

Después vuelve a dejar:

renderizarGaleria(lugares);

Pregunta de reflexión

¿Por qué en este bloque no usamos todavía un botón para cambiar entre todos los lugares y los destacados?

Comprobar la respuesta

Porque un botón necesitaría detectar una interacción del usuario mediante un evento. Los eventos con addEventListener() se trabajarán en UD2_B3. En esta práctica llamamos directamente a la función desde el código.

19. Ampliación controlada

Estas mejoras no introducen eventos y siguen perteneciendo a UD2_B2.

Ampliación 1: mostrar el número de destacados

Crea una constante:

const numeroDestacados = lugares.filter(
  (lugar) => lugar.destacado
).length;

Modifica el resumen para que muestre:

La galería contiene 4 lugares. Hay 2 destacados.

Para que el resumen funcione también con listas filtradas, puedes calcular los destacados dentro de renderizarGaleria():

const numeroDestacados = listaLugares.filter(
  (lugar) => lugar.destacado
).length;

totalElementos.textContent =
  `La galería contiene ${listaLugares.length} lugares. ` +
  `Hay ${numeroDestacados} destacados.`;

Ampliación 2: añadir una propiedad nueva

Añade a cada objeto una propiedad duracionRecomendada:

duracionRecomendada: "1 día"

Después muestra ese dato dentro de cada tarjeta.

Esta propiedad evita simplificar información geográfica y permite seguir practicando la lectura de propiedades de un objeto.

Ampliación 3: generar una lista de categorías

Crea un array con las categorías de los lugares mediante map():

const categorias = lugares.map((lugar) => lugar.categoria);
console.log(categorias);

map() devuelve un nuevo array con el resultado de transformar cada elemento.

En esta ampliación no es obligatorio mostrar las categorías en el DOM. Puedes dejarlas visibles en la consola.

20. Errores frecuentes

Error 1: seleccionar mal el contenedor

const galeria = document.querySelector("galeria");

Falta # porque galeria es un id.

Correcto:

const galeria = document.querySelector("#galeria");

Error 2: usar append() sobre un elemento inexistente

const galeria = document.querySelector("#contenedor-galeria");
galeria.append(tarjeta);

Si no existe ningún elemento con ese id, galeria será null y el código fallará.

Error 3: olvidar devolver la tarjeta en la función

function crearTarjetaLugar(lugar) {
  const tarjeta = document.createElement("article");
  // Falta return
}

La función debe devolver la tarjeta creada:

return tarjeta;

Error 4: escribir HTML donde basta texto

Para textos normales es preferible:

titulo.textContent = lugar.nombre;

No es necesario usar innerHTML para escribir un título simple.

Error 5: conservar código temporal

Si mantienes la creación manual de la primera tarjeta, el recorrido con forEach() y la función renderizarGaleria(), estarás repitiendo el mismo proceso varias veces.

Debes sustituir cada bloque temporal cuando el documento lo indique.

Error 6: ruta incorrecta de una imagen

imagen: "imagenes/santiago.jpg"

Si la carpeta se llama img, la ruta será incorrecta.

Correcto:

imagen: "img/santiago.jpg"

Error 7: texto alternativo que no describe la imagen

textoAlternativo: "Santiago"

Es preferible una descripción más precisa:

textoAlternativo:
  "Fachada de la catedral de Santiago de Compostela"

  • usa remove() para eliminar el mensaje inicial;
  • renderiza todos los objetos del array;
  • gestiona correctamente una lista vacía;
  • mantiene separados HTML, CSS y JavaScript;
  • utiliza imágenes cuyas rutas y textos alternativos sean coherentes.
