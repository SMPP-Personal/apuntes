UD2_B2b. Apuntes: BOM básico en JavaScript

1. Qué vamos a aprender en este documento

En los bloques anteriores hemos trabajado JavaScript como lenguaje y hemos empezado a manipular la página con el DOM.

En este documento vamos a estudiar el BOM (Browser Object Model), nombre habitual que se utiliza para agrupar los objetos y las APIs que el navegador ofrece para interactuar con la ventana, la URL, el historial, la información del navegador, la pantalla y algunos mecanismos de temporización.

Al finalizar este documento deberías ser capaz de:

  • diferenciar DOM y BOM;
  • usar el objeto window;
  • consultar el tamaño visible de la ventana;
  • mostrar diálogos básicos con alert, confirm y prompt;
  • consultar información de la URL con location;
  • moverte por el historial con history;
  • consultar información básica del navegador con navigator;
  • consultar información básica de la pantalla con screen;
  • usar temporizadores con setTimeout y setInterval;
  • cancelar temporizadores con clearTimeout y clearInterval;
  • aplicar buenas prácticas para no abusar de ventanas emergentes, redirecciones o mensajes intrusivos.

2. DOM y BOM: diferencia básica

El DOM representa el documento HTML cargado en el navegador.

Por ejemplo:

const titulo = document.querySelector("h1");
titulo.textContent = "Nuevo título";

El BOM es un término habitual para agrupar elementos y APIs del navegador que rodean al documento: la ventana, la barra de direcciones, el historial, la información del navegador, la pantalla, los temporizadores, etc.

Por ejemplo:

console.log(window.innerWidth);
console.log(location.href);
console.log(navigator.language);

Podemos resumirlo así:

Concepto Qué representa Ejemplos
DOM El documento HTML document, querySelector, textContent, createElement
BOM El navegador y el entorno de la ventana window, location, history, navigator, screen

DOM y BOM no son mundos separados. El objeto document, que representa el DOM, también es accesible desde window:

console.log(document);
console.log(window.document);

3. El objeto window

window representa la ventana o pestaña del navegador donde se está ejecutando el código JavaScript.

En el navegador, muchas funciones y objetos globales pertenecen realmente a window.

Por ejemplo, estas dos instrucciones son equivalentes:

alert("Hola");
window.alert("Hola");

También ocurre con document:

console.log(document);
console.log(window.document);

Normalmente escribimos document, alert o setTimeout directamente, sin poner window., porque JavaScript los busca en el objeto global de la ventana.

Ejemplos equivalentes:

setTimeout(() => {
  console.log("Mensaje");
}, 1000);

window.setTimeout(() => {
  console.log("Mensaje");
}, 1000);

4. Tamaño de la ventana y de la pantalla

4.1. Tamaño visible de la ventana

Podemos consultar el tamaño visible de la ventana del navegador:

console.log(window.innerWidth);
console.log(window.innerHeight);

Ejemplo:

const ancho = window.innerWidth;
const alto = window.innerHeight;

console.log("Ancho de la ventana:", ancho);
console.log("Alto de la ventana:", alto);

window.innerWidth y window.innerHeight indican las dimensiones aproximadas del área visible o viewport de la página, expresadas en píxeles CSS.

Esto puede ser útil para adaptar ciertos comportamientos, aunque el diseño adaptable debe resolverse principalmente con CSS mediante técnicas como media queries, Flexbox o Grid.

4.2. El objeto screen

screen contiene información sobre la pantalla del dispositivo que el navegador decide exponer.

console.log(screen.width);
console.log(screen.height);

Ejemplo:

console.log(
  "Dimensiones de pantalla comunicadas por el navegador:",
  screen.width + "x" + screen.height
);

No debemos confundir:

  • screen.width: ancho lógico de la pantalla comunicado por el navegador;
  • screen.height: alto lógico de la pantalla comunicado por el navegador;
  • window.innerWidth: ancho visible de la ventana o viewport;
  • window.innerHeight: alto visible de la ventana o viewport.

Estos valores se expresan normalmente en píxeles CSS y no tienen por qué coincidir con la resolución física exacta del panel.

5. Diálogos básicos del navegador

JavaScript ofrece tres diálogos básicos:

  • alert();
  • confirm();
  • prompt().

Son fáciles de usar, pero deben emplearse con moderación porque interrumpen la navegación y la interacción del usuario mientras están abiertos.

Los navegadores también pueden limitar o bloquear estos diálogos en determinadas situaciones.

5.1. alert()

alert() muestra un mensaje informativo.

alert("Bienvenido/a a la aplicación");

alert() devuelve undefined, aunque normalmente ese valor no se utiliza. Su finalidad es mostrar un mensaje y esperar a que el usuario pulse aceptar.

Uso razonable:

alert("Datos guardados correctamente");

Uso poco recomendable:

alert("Hola");
alert("Sigue leyendo");
alert("Otro mensaje más");

En aplicaciones reales suele ser preferible mostrar mensajes dentro de la propia página mediante el DOM.

5.2. confirm()

confirm() muestra una pregunta con dos opciones: aceptar o cancelar.

Devuelve:

  • true si el usuario acepta;
  • false si cancela.

Ejemplo:

const borrar = confirm("¿Seguro que quieres borrar este elemento?");

if (borrar) {
  console.log("Elemento borrado");
} else {
  console.log("Operación cancelada");
}

Es útil para pedir confirmación antes de acciones importantes.

5.3. prompt()

prompt() permite pedir un dato al usuario.

Devuelve:

  • un string con el texto escrito;
  • null si el usuario cancela.

Ejemplo:

const nombre = prompt("Escribe tu nombre:");

if (nombre !== null) {
  console.log("Hola, " + nombre);
} else {
  console.log("El usuario canceló la entrada");
}

Hay que recordar que prompt() siempre devuelve texto cuando el usuario acepta. Si queremos trabajar con números, debemos validar primero la entrada y después convertir el valor.

const edadTexto = prompt("Escribe tu edad:");

if (edadTexto === null || edadTexto.trim() === "") {
  console.log("No se ha introducido una edad");
} else {
  const edad = Number(edadTexto);

  if (Number.isNaN(edad)) {
    console.log("No has escrito un número válido");
  } else {
    console.log("Edad:", edad);
  }
}

No es correcto convertir directamente sin comprobar antes la cancelación o una entrada vacía, porque:

Number(null); // 0
Number("");   // 0

6. El objeto location

location contiene información sobre la URL actual de la página y permite realizar algunas operaciones de navegación.

6.1. Consultar la URL actual

Algunas propiedades útiles son:

Propiedad Qué indica
location.href URL completa
location.protocol Protocolo o esquema, por ejemplo https:, http: o file:
location.hostname Nombre del servidor
location.pathname Ruta dentro del sitio
location.search Parámetros de consulta de la URL
location.hash Fragmento de la URL que comienza por #

Ejemplo:

console.log(location.href);
console.log(location.protocol);
console.log(location.hostname);
console.log(location.pathname);
console.log(location.search);
console.log(location.hash);

Ejemplo de URL:

https://www.ejemplo.com/cursos/javascript.html?unidad=2#temporizadores

En ese caso:

console.log(location.protocol); // "https:"
console.log(location.hostname); // "www.ejemplo.com"
console.log(location.pathname); // "/cursos/javascript.html"
console.log(location.search);   // "?unidad=2"
console.log(location.hash);     // "#temporizadores"

location.hash resulta especialmente útil para identificar enlaces a secciones concretas de una misma página.

6.2. Cambiar de página o recargarla

También se puede modificar la URL desde JavaScript.

location.href = "https://www.example.com";

Otra posibilidad es:

location.assign("https://www.example.com");

También podemos recargar la página:

location.reload();

Hay que usar estas instrucciones con cuidado. Redirigir al usuario sin motivo puede ser molesto y confuso.

Antes de probar una redirección, conviene guardar el trabajo, porque la página actual será sustituida por la nueva dirección.

7. El objeto history

history permite interactuar con el historial de navegación de la pestaña.

Métodos básicos:

history.back();
history.forward();
  • history.back() intenta volver a la página anterior.
  • history.forward() intenta avanzar a la página siguiente.

Ejemplo:

function volverAtras() {
  history.back();
}

En prácticas de aula hay que tener cuidado, porque si no hay una entrada anterior o posterior en el historial puede parecer que el método “no hace nada”.

También podemos consultar cuántas entradas contiene aproximadamente el historial de la pestaña:

console.log(history.length);

Este valor indica el número de entradas disponibles, pero no permite leer las direcciones visitadas por motivos de privacidad.

8. El objeto navigator

navigator ofrece información sobre el navegador y el entorno del usuario.

Ejemplos útiles:

console.log(navigator.language);
console.log(navigator.onLine);
console.log(navigator.userAgent);
Propiedad Qué indica
navigator.language Idioma preferido del navegador
navigator.onLine Si el navegador considera que existe conexión de red
navigator.userAgent Cadena con información declarada por el navegador

Ejemplo:

if (navigator.onLine) {
  console.log("El navegador podría tener conexión de red");
} else {
  console.log("El navegador indica que está sin conexión");
}

navigator.onLine no garantiza que exista acceso real a Internet ni que un servidor concreto responda. Un dispositivo puede estar conectado a una red local y, aun así, no tener salida a Internet.

navigator.userAgent no debe usarse como base principal para decidir si una función está disponible. Es mejor comprobar directamente si la funcionalidad existe.

Ejemplo de detección de funcionalidad:

if ("geolocation" in navigator) {
  console.log("La API de geolocalización está disponible");
} else {
  console.log("La API de geolocalización no está disponible");
}

En este documento no se desarrollará la geolocalización; el ejemplo solo muestra cómo comprobar si una API existe.

9. Temporizadores

Los temporizadores permiten programar la ejecución de funciones.

Los dos mecanismos principales son:

  • setTimeout(): programa una ejecución única;
  • setInterval(): programa ejecuciones repetidas.

El tiempo se expresa en milisegundos:

  • 1000 ms = 1 segundo;
  • 2000 ms = 2 segundos;
  • 5000 ms = 5 segundos.

Los tiempos indicados son aproximados. El navegador intenta ejecutar la función cuando ha transcurrido, como mínimo, el retraso señalado, pero puede hacerlo algo más tarde si está ocupado con otras tareas.

9.1. setTimeout()

setTimeout() permite programar una función para que se ejecute una sola vez después de que haya transcurrido, como mínimo, el tiempo indicado.

Sintaxis básica:

setTimeout(() => {
  console.log("Han pasado al menos 2 segundos");
}, 2000);

Ejemplo con función declarada:

function mostrarMensaje() {
  console.log("Mensaje retrasado");
}

setTimeout(mostrarMensaje, 3000);

Fíjate en que escribimos mostrarMensaje, no mostrarMensaje().

  • mostrarMensaje pasa la función para que se ejecute más tarde.
  • mostrarMensaje() ejecutaría la función inmediatamente.

9.2. Cancelar un setTimeout()

setTimeout() devuelve un identificador que podemos guardar.

const temporizador = setTimeout(() => {
  console.log("Esto se ejecutará si no se cancela");
}, 5000);

clearTimeout(temporizador);

clearTimeout() cancela el temporizador antes de que se ejecute.

Ejemplo con una condición:

const temporizador = setTimeout(() => {
  console.log("Operación realizada");
}, 5000);

const cancelar = confirm("¿Quieres cancelar la operación programada?");

if (cancelar) {
  clearTimeout(temporizador);
  console.log("Temporizador cancelado");
}

9.3. setInterval()

setInterval() ejecuta una función repetidamente dejando, como mínimo, el intervalo indicado entre una ejecución y la siguiente.

setInterval(() => {
  console.log("Este mensaje se repite aproximadamente cada segundo");
}, 1000);

Ejemplo con contador:

let contador = 0;

const intervalo = setInterval(() => {
  contador++;
  console.log("Contador:", contador);
}, 1000);

9.4. Detener un setInterval()

Para detener un intervalo usamos clearInterval().

let contador = 0;

const intervalo = setInterval(() => {
  contador++;
  console.log("Contador:", contador);

  if (contador === 5) {
    clearInterval(intervalo);
  }
}, 1000);

Este ejemplo muestra los números del 1 al 5 y después detiene el intervalo.

Conviene detener los intervalos cuando ya no son necesarios para evitar ejecuciones inútiles y comportamientos difíciles de controlar.

10. Ejemplo guiado: cuenta atrás

El siguiente ejemplo muestra una cuenta atrás de cinco segundos:

let segundos = 5;

console.log("Quedan", segundos, "segundos");

const cuentaAtras = setInterval(() => {
  segundos--;

  if (segundos > 0) {
    console.log("Quedan", segundos, "segundos");
  } else {
    clearInterval(cuentaAtras);
    console.log("Tiempo terminado");
  }
}, 1000);

Secuencia aproximada mostrada en consola:

Quedan 5 segundos
Quedan 4 segundos
Quedan 3 segundos
Quedan 2 segundos
Quedan 1 segundos
Tiempo terminado

Podemos mejorar el mensaje para usar singular cuando queda un segundo:

let segundos = 5;

function mostrarTiempo() {
  if (segundos === 1) {
    console.log("Queda 1 segundo");
  } else {
    console.log("Quedan", segundos, "segundos");
  }
}

mostrarTiempo();

const cuentaAtras = setInterval(() => {
  segundos--;

  if (segundos > 0) {
    mostrarTiempo();
  } else {
    clearInterval(cuentaAtras);
    console.log("Tiempo terminado");
  }
}, 1000);

Este ejemplo puede servir como base para temporizadores, mensajes retardados o pequeños juegos.

11. Ejemplo integrador: mostrar información del navegador en el DOM

Como ya conocemos el DOM, podemos combinarlo con información del BOM.

HTML:

<section id="informacion-navegador">
  <h2>Información del entorno</h2>
  <p id="url-actual"></p>
  <p id="idioma-navegador"></p>
  <p id="tamano-ventana"></p>
  <p id="estado-conexion"></p>
</section>

JavaScript:

const parrafoUrl = document.querySelector("#url-actual");
const parrafoIdioma = document.querySelector("#idioma-navegador");
const parrafoTamano = document.querySelector("#tamano-ventana");
const parrafoConexion = document.querySelector("#estado-conexion");

parrafoUrl.textContent = "URL actual: " + location.href;
parrafoIdioma.textContent = "Idioma: " + navigator.language;
parrafoTamano.textContent =
  `Ventana: ${window.innerWidth} × ${window.innerHeight}`;

if (navigator.onLine) {
  parrafoConexion.textContent =
    "El navegador podría tener conexión de red.";
} else {
  parrafoConexion.textContent =
    "El navegador indica que está sin conexión.";
}

En este ejemplo:

  • el BOM proporciona la URL, el idioma, el tamaño de la ventana y el estado de conexión;
  • el DOM permite mostrar esa información dentro de la página;
  • textContent evita interpretar los datos como HTML.

12. Buenas prácticas con BOM

Conviene recordar:

  • no abusar de alert, confirm y prompt;
  • preferir mensajes integrados en el DOM cuando sea posible;
  • no redirigir al usuario sin avisar;
  • no recargar una página automáticamente sin una razón clara;
  • no depender de navigator.userAgent si se puede comprobar una funcionalidad directamente;
  • no asumir que navigator.onLine === true garantiza acceso real a Internet;
  • detener los intervalos cuando ya no sean necesarios;
  • cancelar los temporizadores pendientes si dejan de tener sentido;
  • no usar temporizadores para ocultar errores de lógica;
  • no esperar que los temporizadores sean exactos;
  • comprobar los valores devueltos por prompt() antes de convertirlos;
  • tener en cuenta que el comportamiento puede variar ligeramente entre navegadores.

13. Errores frecuentes

Error 1. Ejecutar la función inmediatamente en setTimeout()

Incorrecto:

setTimeout(mostrarMensaje(), 2000);

En este caso, mostrarMensaje() se ejecuta inmediatamente y su valor de retorno se pasa a setTimeout().

Correcto:

setTimeout(mostrarMensaje, 2000);

También es correcto utilizar una función flecha:

setTimeout(() => {
  mostrarMensaje();
}, 2000);

Error 2. No detener un intervalo

setInterval(() => {
  console.log("Esto no se detiene nunca");
}, 1000);

Si el intervalo ya no hace falta, debemos guardar su identificador y usar clearInterval().

const intervalo = setInterval(() => {
  console.log("Mensaje repetido");
}, 1000);

clearInterval(intervalo);

Error 3. Olvidar que prompt() devuelve texto

const numero = prompt("Escribe un número:");
console.log(numero + 10);

Si el usuario escribe 5, el resultado será "510", porque se concatena texto.

Tampoco es suficiente hacer directamente:

const numero = Number(prompt("Escribe un número:"));
console.log(numero + 10);

Si el usuario cancela o deja la entrada vacía, el resultado puede convertirse en 0.

Correcto:

const numeroTexto = prompt("Escribe un número:");

if (numeroTexto === null || numeroTexto.trim() === "") {
  console.log("No se ha introducido ningún número");
} else {
  const numero = Number(numeroTexto);

  if (Number.isNaN(numero)) {
    console.log("El valor introducido no es un número válido");
  } else {
    console.log(numero + 10);
  }
}

Error 4. Redirigir accidentalmente

location.href = "https://www.example.com";

Esta instrucción cambia la página actual. No debe usarse en pruebas si no se quiere abandonar la página.

Antes de ejecutar una redirección, conviene comprobar que la URL es correcta y que el usuario espera ese cambio.

Error 5. Creer que un temporizador es exacto

setTimeout(() => {
  console.log("Exactamente un segundo");
}, 1000);

El mensaje puede ejecutarse algo después de un segundo.

Mejor descripción:

setTimeout(() => {
  console.log("Ha transcurrido al menos aproximadamente un segundo");
}, 1000);

Error 6. Confiar completamente en navigator.onLine

if (navigator.onLine) {
  console.log("Internet funciona correctamente");
}

Este mensaje es demasiado rotundo. navigator.onLine solo refleja el estado de conexión que el navegador cree detectar.

Mejor:

if (navigator.onLine) {
  console.log("El navegador podría tener conexión de red");
}

16. Resumen final

El BOM permite interactuar con el navegador más allá del documento HTML.

Los objetos y funciones más importantes de este bloque son:

  • window: representa la ventana o pestaña del navegador;
  • location: permite consultar la URL y realizar operaciones de navegación;
  • history: permite retroceder y avanzar por el historial;
  • navigator: ofrece información básica sobre el navegador y su entorno;
  • screen: comunica información básica sobre la pantalla;
  • alert, confirm y prompt: muestran diálogos modales;
  • setTimeout: programa una ejecución única;
  • clearTimeout: cancela un setTimeout;
  • setInterval: programa ejecuciones repetidas;
  • clearInterval: detiene un setInterval.

Ideas esenciales:

console.log(window.innerWidth);
console.log(location.href);
console.log(history.length);
console.log(navigator.language);
console.log(screen.width);

Temporizador de una sola ejecución:

const temporizador = setTimeout(() => {
  console.log("Mensaje retrasado");
}, 2000);

clearTimeout(temporizador);

Temporizador repetido:

const intervalo = setInterval(() => {
  console.log("Mensaje repetido");
}, 1000);

clearInterval(intervalo);

El BOM es útil, pero debe usarse con cuidado para no interrumpir, confundir o desplazar al usuario de forma inesperada. También debemos recordar que los datos proporcionados por el navegador pueden ser aproximados y que los temporizadores no son mecanismos de precisión.
