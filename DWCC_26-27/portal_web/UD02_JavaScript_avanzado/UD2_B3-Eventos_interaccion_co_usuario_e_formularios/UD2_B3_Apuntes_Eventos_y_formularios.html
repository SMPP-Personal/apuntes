UD2_B3. Eventos, interacción con el usuario y formularios

1. Introducción y objetivos del bloque

1.1. Qué vamos a trabajar

En los bloques anteriores ya hemos trabajado la base de JavaScript, los objetos, los arrays de objetos, el BOM y la manipulación básica del DOM.

En este bloque vamos a dar un paso más: la página dejará de ser solo un documento que modificamos desde JavaScript y empezará a responder a las acciones del usuario.

Trabajaremos:

  • qué es un evento;
  • cómo asociar código JavaScript a un evento;
  • cómo usar addEventListener();
  • qué información proporciona el objeto event;
  • las diferencias entre event.target y event.currentTarget;
  • eventos habituales como click, input, change y submit;
  • lectura de datos introducidos en formularios;
  • diferencia entre id, name y la asociación con <label>;
  • validación básica mediante HTML;
  • validación básica mediante JavaScript;
  • mensajes de error y confirmación;
  • modificación de clases CSS en función de la interacción del usuario;
  • prevención del envío automático de formularios con preventDefault();
  • creación de contenido dinámico como respuesta a un formulario;
  • organización del código en funciones;
  • accesibilidad básica en mensajes dinámicos.

Importante: en este bloque no vamos a trabajar todavía fetch, promesas, async/await ni datos externos. Eso queda para el bloque de comunicación asíncrona.

1.2. Relación con DWCC

Este bloque se relaciona principalmente con:

  • la interacción del usuario con la página web;
  • la captura y gestión de eventos;
  • el uso de formularios desde código JavaScript;
  • la validación de datos introducidos por el usuario;
  • la modificación dinámica del contenido y la apariencia de la página.

Después de este bloque, el alumnado debería ser capaz de crear pequeñas páginas interactivas donde JavaScript responda a acciones como pulsar un botón, escribir en un campo, cambiar una selección o enviar un formulario.

1.3. Cómo utilizar los ejemplos

Los ejemplos de este documento son independientes.

Si se reúnen varios ejemplos en una misma página:

  • cada elemento debe tener un id único;
  • no deben repetirse nombres de constantes en el mismo ámbito;
  • conviene organizar cada ejemplo en una función;
  • debe comprobarse que cada selector coincide con un elemento real del HTML.

2. Fundamentos de los eventos

2.1. Qué es un evento

Un evento es algo que ocurre en la página y que JavaScript puede detectar.

Algunos ejemplos de eventos son:

Evento Cuándo ocurre
click Cuando un elemento se activa, normalmente con ratón, panel táctil o teclado.
input Cada vez que cambia el contenido o valor de un campo mientras el usuario interactúa con él.
change Cuando el usuario confirma o consolida un cambio en un control.
submit Cuando se intenta enviar un formulario.
keydown Cuando se pulsa una tecla.
mouseover Cuando el puntero pasa por encima de un elemento.

Ejemplo cotidiano:

  • El usuario pulsa un botón.
  • Se produce un evento click.
  • JavaScript ejecuta una función.
  • La página cambia algo: un texto, una clase, un mensaje, una lista, etc.

2.2. Preparar el HTML para trabajar con eventos

Para trabajar con eventos necesitamos tener elementos HTML que podamos seleccionar desde JavaScript.

Ejemplo de HTML:

<h1 id="titulo">Eventos en JavaScript</h1>
<button id="botonSaludo" type="button">Mostrar saludo</button>
<p id="mensaje" aria-live="polite"></p>

Y un fichero JavaScript enlazado desde el HTML:

<script src="script.js"></script>

En el fichero script.js seleccionaremos los elementos y les asociaremos eventos.

2.3. Seleccionar elementos antes de usar eventos

Antes de asociar un evento a un elemento, debemos seleccionarlo.

const boton = document.querySelector("#botonSaludo");
const mensaje = document.querySelector("#mensaje");

A partir de ahí, ya podemos indicar qué debe ocurrir cuando el usuario interactúe con el botón.

Conviene comprobar que el selector y el HTML coinciden. Si el elemento no existe, querySelector() devuelve null.

2.4. addEventListener(): asociar un evento a un elemento

La forma recomendada de asociar eventos en JavaScript moderno es usar addEventListener().

Sintaxis básica:

elemento.addEventListener("evento", funcion);

Ejemplo:

const boton = document.querySelector("#botonSaludo");
const mensaje = document.querySelector("#mensaje");

boton.addEventListener("click", function () {
  mensaje.textContent = "Hola, estás usando eventos en JavaScript.";
});

En este ejemplo:

  • boton es el elemento que escucha el evento;
  • "click" es el tipo de evento;
  • la función se ejecuta cuando ocurre el evento;
  • se modifica el contenido del párrafo mensaje.

2.5. Funciones callback en eventos

La función que se ejecuta cuando ocurre un evento es una función callback.

Podemos escribirla directamente dentro de addEventListener():

boton.addEventListener("click", function () {
  mensaje.textContent = "Has pulsado el botón.";
});

También podemos definirla aparte:

function mostrarMensaje() {
  mensaje.textContent = "Has pulsado el botón.";
}

boton.addEventListener("click", mostrarMensaje);

Atención: al pasar la función a addEventListener(), no se escriben paréntesis.

Correcto:

boton.addEventListener("click", mostrarMensaje);

Incorrecto:

boton.addEventListener("click", mostrarMensaje());

En el segundo caso, la función se ejecutaría inmediatamente al cargar la página, no cuando se active el botón.

2.6. Funciones flecha en eventos

También podemos usar funciones flecha como callbacks:

boton.addEventListener("click", () => {
  mensaje.textContent = "Evento gestionado con una función flecha.";
});

Son muy habituales en JavaScript moderno.

Sin embargo, si dentro del evento necesitamos usar this, conviene tener cuidado, porque las funciones flecha no tienen su propio this.

En este bloque trabajaremos principalmente con:

  • el elemento seleccionado;
  • event.target;
  • event.currentTarget.

Este enfoque suele ser más claro para este nivel.

2.7. El objeto event

Cuando ocurre un evento, JavaScript proporciona información sobre él mediante el objeto event.

Ejemplo:

const boton = document.querySelector("#botonSaludo");

boton.addEventListener("click", function (event) {
  console.log(event);
});

El objeto event contiene información como:

  • qué tipo de evento ocurrió;
  • sobre qué elemento se originó;
  • qué elemento está ejecutando el manejador;
  • valores asociados al evento;
  • métodos útiles como preventDefault().

Una propiedad muy usada es event.target, que indica el elemento concreto sobre el que se originó el evento.

boton.addEventListener("click", function (event) {
  console.log(event.target);
});

También es importante event.currentTarget, que indica el elemento al que se asoció el listener que se está ejecutando.

Ejemplo:

<button id="botonGuardar" type="button">
  <span>Guardar</span>
</button>

const botonGuardar = document.querySelector("#botonGuardar");

botonGuardar.addEventListener("click", (event) => {
  console.log("Origen:", event.target);
  console.log("Listener asociado a:", event.currentTarget);
});

Si se pulsa sobre el <span>:

  • event.target puede ser el <span>;
  • event.currentTarget será el botón.

En este bloque no desarrollaremos todavía la propagación, la captura ni la delegación de eventos, pero conviene conocer esta diferencia.

3. Eventos de interacción habituales

3.1. Evento click

El evento click ocurre cuando un elemento se activa, normalmente mediante ratón, panel táctil o teclado.

Ejemplo:

<button id="botonColor" type="button">Cambiar color</button>
<p id="texto">Texto de ejemplo</p>

const botonColor = document.querySelector("#botonColor");
const texto = document.querySelector("#texto");

botonColor.addEventListener("click", () => {
  texto.classList.add("destacado");
});

CSS de ejemplo:

.destacado {
  color: darkblue;
  font-weight: bold;
}

Este ejemplo no cambia directamente el estilo desde JavaScript, sino que añade una clase CSS. Esta suele ser una práctica más limpia.

3.2. Añadir, quitar y alternar clases con eventos

Podemos combinar eventos con classList.

elemento.classList.add("clase");      // añade una clase
elemento.classList.remove("clase");   // quita una clase
elemento.classList.toggle("clase");   // añade o quita según esté o no esté

Ejemplo:

<button id="botonAlternar" type="button">Mostrar / ocultar aviso</button>
<p id="aviso" class="oculto">Este es un aviso importante.</p>

.oculto {
  display: none;
}

const botonAlternar = document.querySelector("#botonAlternar");
const aviso = document.querySelector("#aviso");

botonAlternar.addEventListener("click", () => {
  aviso.classList.toggle("oculto");
});

3.3. Evento input

El evento input ocurre cada vez que cambia el contenido de un campo mientras el usuario escribe o modifica su valor.

Ejemplo:

<label for="nombre">Nombre:</label>
<input id="nombre" name="nombre" type="text">
<p id="salida" aria-live="polite"></p>

const nombre = document.querySelector("#nombre");
const salida = document.querySelector("#salida");

nombre.addEventListener("input", () => {
  salida.textContent = "Estás escribiendo: " + nombre.value;
});

La propiedad .value permite leer el valor actual de un campo de formulario.

3.4. Evento change

El evento change ocurre cuando el usuario confirma o consolida el cambio de un control.

Se usa mucho con:

  • listas desplegables (select);
  • casillas de verificación (checkbox);
  • botones de opción (radio);
  • algunos campos de formulario.

Ejemplo con select:

<label for="curso">Curso:</label>
<select id="curso" name="curso">
  <option value="">Elige una opción</option>
  <option value="DWCC">DWCC</option>
  <option value="DWES">DWES</option>
  <option value="DIW">DIW</option>
</select>

<p id="cursoElegido" aria-live="polite"></p>

const curso = document.querySelector("#curso");
const cursoElegido = document.querySelector("#cursoElegido");

curso.addEventListener("change", () => {
  cursoElegido.textContent = "Has elegido: " + curso.value;
});

3.5. Diferencia entre input y change

Evento Comportamiento habitual
input Se produce cada vez que el usuario modifica el valor.
change Se produce cuando el usuario confirma o consolida el cambio.

En un campo de texto:

  • input se ejecuta mientras se escribe;
  • change suele producirse cuando el campo pierde el foco después de haber cambiado.

Ejemplo comparativo:

<label for="textoComparacion">Texto:</label>
<input id="textoComparacion" name="textoComparacion" type="text">

<p id="salidaInput"></p>
<p id="salidaChange"></p>

const textoComparacion = document.querySelector("#textoComparacion");
const salidaInput = document.querySelector("#salidaInput");
const salidaChange = document.querySelector("#salidaChange");

textoComparacion.addEventListener("input", () => {
  salidaInput.textContent = "input: " + textoComparacion.value;
});

textoComparacion.addEventListener("change", () => {
  salidaChange.textContent = "change: " + textoComparacion.value;
});

4. Formularios HTML y lectura de datos

4.1. Formularios en HTML

Un formulario permite recoger datos del usuario.

Ejemplo básico:

<form id="formularioRegistro">
  <label for="nombreRegistro">Nombre:</label>
  <input id="nombreRegistro" name="nombre" type="text">

  <label for="edadRegistro">Edad:</label>
  <input id="edadRegistro" name="edad" type="number">

  <button type="submit">Enviar</button>
</form>

<p id="mensajeFormulario" aria-live="polite"></p>

Elementos habituales:

Elemento Uso
<form> Agrupa los campos del formulario.
<input> Permite introducir datos.
<label> Texto descriptivo asociado a un campo.
<select> Lista desplegable.
<textarea> Campo de texto largo.
<button> Botón para enviar o ejecutar una acción.

4.2. Diferencia entre id, name y <label>

Ejemplo:

<label for="email">Correo:</label>
<input id="email" name="email" type="email">
  • id identifica el elemento dentro del documento y permite seleccionarlo desde JavaScript.
  • name identifica el dato cuando el formulario se envía o cuando se construye un FormData.
  • for="email" asocia el <label> con el elemento cuyo id es email.

Usar correctamente <label> mejora la accesibilidad y permite activar el campo al pulsar sobre el texto de la etiqueta.

4.3. Leer valores de un formulario

Para leer el valor de un campo usamos .value.

const nombre = document.querySelector("#nombreRegistro");
const edad = document.querySelector("#edadRegistro");

console.log(nombre.value);
console.log(edad.value);

Importante: aunque un campo sea type="number", el valor que leemos con .value llega como texto. Si queremos operar numéricamente, debemos convertirlo.

const edadNumero = Number(edad.value);

Antes de convertir, conviene comprobar que el campo no esté vacío:

const edadTexto = edad.value.trim();

if (edadTexto === "") {
  console.log("Debes introducir una edad.");
} else {
  const edadNumero = Number(edadTexto);
  console.log(edadNumero);
}

4.4. valueAsNumber como alternativa

En un <input type="number"> también se puede usar:

const edadNumero = edad.valueAsNumber;

Si el campo está vacío o no contiene un número válido, valueAsNumber devuelve NaN.

Ejemplo:

if (Number.isNaN(edad.valueAsNumber)) {
  console.log("La edad no es válida.");
}

En este bloque seguiremos usando principalmente .value, trim() y Number() porque permiten practicar de forma explícita la validación y la conversión.

5. Envío de formularios

5.1. Evento submit

El evento submit ocurre cuando el usuario intenta enviar un formulario.

Ejemplo:

const formulario = document.querySelector("#formularioRegistro");

formulario.addEventListener("submit", function (event) {
  console.log("Formulario enviado");
});

Por defecto, al enviar un formulario el navegador puede navegar o recargar la página. Para evitarlo mientras validamos o procesamos datos con JavaScript, usamos preventDefault().

5.2. event.preventDefault()

preventDefault() evita el comportamiento predeterminado asociado al evento.

En formularios, impide que se realice el envío tradicional mientras procesamos los datos con JavaScript.

const formulario = document.querySelector("#formularioRegistro");
const mensaje = document.querySelector("#mensajeFormulario");

formulario.addEventListener("submit", function (event) {
  event.preventDefault();

  mensaje.textContent = "Formulario procesado con JavaScript.";
});

Esto nos permite:

  • leer los datos;
  • validarlos;
  • mostrar errores;
  • mostrar mensajes de confirmación;
  • preparar el envío real más adelante, cuando trabajemos comunicación asíncrona.

preventDefault() no detiene automáticamente la propagación del evento. Evitar la acción predeterminada y detener la propagación son operaciones diferentes. En este bloque no trabajaremos stopPropagation() como contenido principal.

6. Validación de formularios

6.1. Qué significa validar

Validar significa comprobar que los datos introducidos por el usuario son adecuados antes de aceptarlos.

La validación puede realizarse en varios niveles:

  • validación básica mediante atributos HTML;
  • validación adicional mediante JavaScript;
  • validación en el servidor, cuando exista un backend.

La validación JavaScript mejora la experiencia del usuario, pero no sustituye la validación del servidor en una aplicación real.

6.2. Validación básica mediante HTML

HTML ofrece atributos de validación integrados:

<form id="formularioRegistro">
  <label for="nombreRegistro">Nombre:</label>
  <input
    id="nombreRegistro"
    name="nombre"
    type="text"
    required
    minlength="2"
  >

  <label for="edadRegistro">Edad:</label>
  <input
    id="edadRegistro"
    name="edad"
    type="number"
    required
    min="0"
  >

  <label for="emailRegistro">Correo:</label>
  <input
    id="emailRegistro"
    name="email"
    type="email"
    required
  >

  <button type="submit">Enviar</button>
</form>

Atributos habituales:

Atributo Función
required El campo no puede quedar vacío.
minlength Longitud mínima de texto.
maxlength Longitud máxima de texto.
min Valor numérico mínimo.
max Valor numérico máximo.
type="email" Comprueba un formato básico de correo.
pattern Permite indicar un patrón.

La validación HTML y la validación JavaScript pueden combinarse.

Cuando un formulario no supera la validación HTML integrada, el navegador puede impedir el envío antes de que se ejecute el código asociado al evento submit. Por eso conviene probar y entender por separado la validación HTML y la validación personalizada con JavaScript.

6.3. Validación básica de formularios mediante JavaScript

Ejemplo corregido:

<form id="formularioRegistro">
  <label for="nombreRegistro">Nombre:</label>
  <input id="nombreRegistro" name="nombre" type="text">

  <label for="edadRegistro">Edad:</label>
  <input id="edadRegistro" name="edad" type="number">

  <button type="submit">Enviar</button>
</form>

<p id="mensajeFormulario" aria-live="polite"></p>

const formulario = document.querySelector("#formularioRegistro");
const nombre = document.querySelector("#nombreRegistro");
const edad = document.querySelector("#edadRegistro");
const mensajeFormulario = document.querySelector("#mensajeFormulario");

formulario.addEventListener("submit", function (event) {
  event.preventDefault();

  const nombreLimpio = nombre.value.trim();
  const edadTexto = edad.value.trim();

  mensajeFormulario.classList.remove("error", "correcto");

  if (nombreLimpio === "") {
    mensajeFormulario.textContent = "El nombre no puede estar vacío.";
    mensajeFormulario.classList.add("error");
    nombre.focus();
    return;
  }

  if (edadTexto === "") {
    mensajeFormulario.textContent = "Debes introducir una edad.";
    mensajeFormulario.classList.add("error");
    edad.focus();
    return;
  }

  const edadNumero = Number(edadTexto);

  if (
    Number.isNaN(edadNumero)
    || !Number.isInteger(edadNumero)
    || edadNumero < 0
  ) {
    mensajeFormulario.textContent =
      "La edad debe ser un número entero válido.";
    mensajeFormulario.classList.add("error");
    edad.focus();
    return;
  }

  mensajeFormulario.textContent =
    "Datos correctos. Bienvenido/a, " + nombreLimpio + ".";
  mensajeFormulario.classList.add("correcto");
});

CSS de ejemplo:

.error {
  color: darkred;
  font-weight: bold;
}

.correcto {
  color: darkgreen;
  font-weight: bold;
}

Se utiliza classList en lugar de asignar directamente className, porque className = "error" reemplazaría todas las clases existentes del elemento.

6.4. Validar mientras el usuario escribe

A veces interesa validar en tiempo real, mientras el usuario escribe.

Ejemplo: comprobar la longitud de una contraseña.

<label for="password">Contraseña:</label>
<input id="password" name="password" type="password">
<p id="ayudaPassword" aria-live="polite"></p>

const password = document.querySelector("#password");
const ayudaPassword = document.querySelector("#ayudaPassword");

password.addEventListener("input", () => {
  ayudaPassword.classList.remove("error", "correcto");

  if (password.value.length < 8) {
    ayudaPassword.textContent =
      "La contraseña debe tener al menos 8 caracteres.";
    ayudaPassword.classList.add("error");
  } else {
    ayudaPassword.textContent = "Longitud correcta.";
    ayudaPassword.classList.add("correcto");
  }
});

La validación en tiempo real debe ayudar al usuario, no interrumpirlo constantemente ni mostrar errores agresivos antes de que haya podido terminar de escribir.

6.5. Activar o desactivar un botón

Podemos usar la propiedad disabled para activar o desactivar botones.

<label>
  <input id="acepta" name="acepta" type="checkbox">
  Acepto las condiciones
</label>

<button id="botonEnviar" type="button" disabled>Enviar</button>

const acepta = document.querySelector("#acepta");
const botonEnviar = document.querySelector("#botonEnviar");

acepta.addEventListener("change", () => {
  botonEnviar.disabled = !acepta.checked;
});

En este ejemplo:

  • acepta.checked vale true si la casilla está marcada;
  • el botón solo se activa cuando el usuario acepta las condiciones.

6.6. Validación sencilla con expresiones regulares

Una expresión regular permite comprobar si un texto cumple un patrón.

En este bloque solo las usaremos de forma básica.

Ejemplo: comprobar si un texto parece un correo electrónico sencillo.

const patronEmail = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

console.log(patronEmail.test("ana@example.com")); // true
console.log(patronEmail.test("ana.example.com")); // false

Aplicado a un formulario:

<label for="email">Correo:</label>
<input id="email" name="email" type="email">
<p id="mensajeEmail" aria-live="polite"></p>

const email = document.querySelector("#email");
const mensajeEmail = document.querySelector("#mensajeEmail");
const patronEmail = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

email.addEventListener("input", () => {
  mensajeEmail.classList.remove("error", "correcto");

  if (patronEmail.test(email.value)) {
    mensajeEmail.textContent = "Correo válido.";
    mensajeEmail.classList.add("correcto");
  } else {
    mensajeEmail.textContent = "Correo no válido.";
    mensajeEmail.classList.add("error");
  }
});

Esta expresión regular es una aproximación didáctica sencilla. No valida todas las direcciones de correo posibles.

No es necesario memorizar expresiones regulares complejas en este punto. Lo importante es entender que permiten validar patrones de texto.

Antes de recurrir a una expresión regular, conviene aprovechar también:

<input id="email" name="email" type="email" required>

7. Formularios y contenido dinámico

7.1. Generar contenido en respuesta a un formulario

Podemos usar un formulario para recoger datos y después crear contenido en el DOM.

Ejemplo:

<form id="formularioTarea">
  <label for="tarea">Nueva tarea:</label>
  <input id="tarea" name="tarea" type="text">
  <button type="submit">Añadir</button>
</form>

<ul id="listaTareas"></ul>
<p id="mensajeTarea" aria-live="polite"></p>

const formularioTarea = document.querySelector("#formularioTarea");
const tarea = document.querySelector("#tarea");
const listaTareas = document.querySelector("#listaTareas");
const mensajeTarea = document.querySelector("#mensajeTarea");

formularioTarea.addEventListener("submit", (event) => {
  event.preventDefault();

  const textoTarea = tarea.value.trim();

  mensajeTarea.classList.remove("error", "correcto");

  if (textoTarea === "") {
    mensajeTarea.textContent = "Debes escribir una tarea.";
    mensajeTarea.classList.add("error");
    tarea.focus();
    return;
  }

  const li = document.createElement("li");
  li.textContent = textoTarea;
  listaTareas.append(li);

  formularioTarea.reset();

  mensajeTarea.textContent = "Tarea añadida correctamente.";
  mensajeTarea.classList.add("correcto");
  tarea.focus();
});

Para limpiar únicamente el campo de la tarea también podríamos escribir:

tarea.value = "";

formularioTarea.reset() restablece todos los controles del formulario, mientras que tarea.value = "" limpia solo ese campo.

Este ejemplo combina:

  • evento submit;
  • preventDefault();
  • lectura con .value;
  • validación con trim();
  • creación de elementos con createElement();
  • inserción en el DOM con append();
  • limpieza del formulario con reset();
  • recuperación del foco con focus().

7.2. textContent frente a innerHTML

Para mostrar texto simple, es preferible usar:

mensaje.textContent = "Datos correctos";

textContent trata el contenido como texto.

innerHTML interpreta la cadena como marcado HTML:

mensaje.innerHTML = "<strong>Datos correctos</strong>";

No se debe insertar directamente con innerHTML contenido introducido por el usuario o recibido de una fuente no confiable.

Ejemplo inseguro:

salida.innerHTML = inputUsuario.value;

Alternativa segura para mostrar texto:

salida.textContent = inputUsuario.value;

Reservamos innerHTML para casos donde realmente necesitemos insertar HTML controlado y conozcamos el origen del contenido.

8. Organización del código y separación de responsabilidades

8.1. Separar responsabilidades

Cuando el código empieza a crecer, conviene dividirlo en funciones.

Ejemplo:

function obtenerTextoLimpio(input) {
  return input.value.trim();
}

function limpiarEstado(elemento) {
  elemento.classList.remove("error", "correcto");
}

function mostrarError(elemento, mensaje) {
  limpiarEstado(elemento);
  elemento.textContent = mensaje;
  elemento.classList.add("error");
}

function mostrarCorrecto(elemento, mensaje) {
  limpiarEstado(elemento);
  elemento.textContent = mensaje;
  elemento.classList.add("correcto");
}

Después podemos usarlas dentro del evento:

formulario.addEventListener("submit", (event) => {
  event.preventDefault();

  const nombreLimpio = obtenerTextoLimpio(nombre);

  if (nombreLimpio === "") {
    mostrarError(
      mensajeFormulario,
      "El nombre no puede estar vacío."
    );
    nombre.focus();
    return;
  }

  mostrarCorrecto(
    mensajeFormulario,
    "Formulario correcto."
  );
});

Esto hace que el código sea más claro, reutilizable y fácil de depurar.

8.2. Seleccionar elementos al principio

Una estructura habitual es:

const formulario = document.querySelector("#formulario");
const nombre = document.querySelector("#nombre");
const mensaje = document.querySelector("#mensaje");

function validarNombre() {
  // lógica
}

formulario.addEventListener("submit", (event) => {
  event.preventDefault();
  validarNombre();
});

Así se separan:

  • selección de elementos;
  • funciones auxiliares;
  • asociación de eventos.

9.2. JavaScript

const formularioPerfil =
  document.querySelector("#formularioPerfil");

const nombrePerfil =
  document.querySelector("#nombrePerfil");

const emailPerfil =
  document.querySelector("#emailPerfil");

const cursoPerfil =
  document.querySelector("#cursoPerfil");

const aceptaPerfil =
  document.querySelector("#aceptaPerfil");

const enviarPerfil =
  document.querySelector("#enviarPerfil");

const mensajePerfil =
  document.querySelector("#mensajePerfil");

const listaPerfiles =
  document.querySelector("#listaPerfiles");

const patronEmail = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

function mostrarEstado(elemento, mensaje, clase) {
  elemento.classList.remove("error", "correcto");
  elemento.textContent = mensaje;
  elemento.classList.add(clase);
}

aceptaPerfil.addEventListener("change", () => {
  enviarPerfil.disabled = !aceptaPerfil.checked;
});

emailPerfil.addEventListener("input", () => {
  if (emailPerfil.value === "") {
    mensajePerfil.textContent = "";
    mensajePerfil.classList.remove("error", "correcto");
    return;
  }

  if (patronEmail.test(emailPerfil.value)) {
    mostrarEstado(
      mensajePerfil,
      "El correo tiene un formato válido.",
      "correcto"
    );
  } else {
    mostrarEstado(
      mensajePerfil,
      "El correo no tiene un formato válido.",
      "error"
    );
  }
});

formularioPerfil.addEventListener("submit", (event) => {
  event.preventDefault();

  const nombreLimpio = nombrePerfil.value.trim();
  const emailLimpio = emailPerfil.value.trim();
  const cursoSeleccionado = cursoPerfil.value;

  if (nombreLimpio.length < 2) {
    mostrarEstado(
      mensajePerfil,
      "El nombre debe tener al menos dos caracteres.",
      "error"
    );
    nombrePerfil.focus();
    return;
  }

  if (!patronEmail.test(emailLimpio)) {
    mostrarEstado(
      mensajePerfil,
      "Debes introducir un correo válido.",
      "error"
    );
    emailPerfil.focus();
    return;
  }

  if (cursoSeleccionado === "") {
    mostrarEstado(
      mensajePerfil,
      "Debes seleccionar un curso.",
      "error"
    );
    cursoPerfil.focus();
    return;
  }

  if (!aceptaPerfil.checked) {
    mostrarEstado(
      mensajePerfil,
      "Debes aceptar las condiciones.",
      "error"
    );
    aceptaPerfil.focus();
    return;
  }

  const li = document.createElement("li");
  li.textContent =
    `${nombreLimpio} · ${emailLimpio} · ${cursoSeleccionado}`;

  listaPerfiles.append(li);

  mostrarEstado(
    mensajePerfil,
    "Perfil creado correctamente.",
    "correcto"
  );

  formularioPerfil.reset();
  enviarPerfil.disabled = true;
  nombrePerfil.focus();
});

Este ejemplo integra:

  • input;
  • change;
  • submit;
  • preventDefault();
  • validación HTML;
  • validación JavaScript;
  • expresiones regulares sencillas;
  • checked;
  • disabled;
  • classList;
  • creación de contenido;
  • reset();
  • focus();
  • mensajes dinámicos accesibles.

10. Buenas prácticas, seguridad y accesibilidad

10.1. Buenas prácticas generales

  • Seleccionar los elementos al principio del script.
  • Usar nombres descriptivos para variables y funciones.
  • Asociar eventos con addEventListener().
  • Evitar mezclar demasiado HTML dentro de JavaScript si no es necesario.
  • Usar textContent para textos simples.
  • Usar innerHTML solo cuando sea necesario y con contenido controlado.
  • Validar los datos antes de aceptarlos.
  • Combinar validación HTML y JavaScript cuando sea apropiado.
  • Mostrar mensajes claros al usuario.
  • Evitar que el formulario se envíe automáticamente si lo estamos procesando con JavaScript.
  • Dividir el código en funciones cuando empiece a repetirse.
  • Usar classList en lugar de sobrescribir todas las clases con className.
  • Utilizar type="button" en botones que no envían formularios.
  • Utilizar type="submit" en los botones que sí envían el formulario.
  • Mantener identificadores únicos.
  • Devolver el foco al campo que contiene el primer error.
  • Limpiar los formularios con reset() cuando proceda.

10.2. Accesibilidad básica de mensajes dinámicos

Cuando un mensaje cambia mediante JavaScript, puede añadirse:

<p id="mensaje" aria-live="polite"></p>

aria-live="polite" permite que las tecnologías de asistencia anuncien el cambio sin interrumpir de forma brusca.

Para errores importantes puede usarse:

<p id="mensajeError" role="alert"></p>

No conviene usar role="alert" para todos los mensajes, porque puede resultar intrusivo.

10.3. Seguridad al mostrar datos

No introduzcas directamente datos no confiables con innerHTML.

Incorrecto:

resultado.innerHTML = campo.value;

Correcto para texto:

resultado.textContent = campo.value;

11. Errores frecuentes

11.1. Escribir mal el selector

const boton = document.querySelector("botonEnviar");

Si queremos seleccionar por id, falta #:

const boton = document.querySelector("#botonEnviar");

11.2. Ejecutar la función en lugar de pasarla

Incorrecto:

boton.addEventListener("click", mostrarMensaje());

Correcto:

boton.addEventListener("click", mostrarMensaje);

11.3. Olvidar preventDefault() en formularios

Si no usamos event.preventDefault(), la página puede navegar o recargarse antes de que veamos el resultado.

formulario.addEventListener("submit", (event) => {
  event.preventDefault();
  // código de validación
});

11.4. Leer un número como texto

const edad = edadInput.value;

if (edad >= 18) {
  console.log("Mayor de edad");
}

Aunque a veces parezca funcionar por conversión implícita, es mejor convertir explícitamente:

const edad = Number(edadInput.value);

Antes hay que comprobar si el campo está vacío:

const edadTexto = edadInput.value.trim();

if (edadTexto === "") {
  console.log("Debes introducir una edad.");
} else {
  const edad = Number(edadTexto);
}

11.5. No limpiar espacios

if (nombre.value === "") {
  console.log("Nombre vacío");
}

Si el usuario escribe espacios, esa condición no detecta correctamente el problema. Mejor:

if (nombre.value.trim() === "") {
  console.log("Nombre vacío");
}

11.6. Usar innerHTML sin necesidad

Para mostrar texto simple, es mejor usar textContent:

mensaje.textContent = "Datos correctos";

Reservamos innerHTML para casos donde realmente necesitemos insertar etiquetas HTML controladas.

11.7. No comprobar si un elemento existe

Si el selector no encuentra nada, devuelve null.

const boton = document.querySelector("#boton");

boton.addEventListener("click", () => {
  console.log("clic");
});

Si no existe un elemento con id="boton", se producirá un error.

Podemos comprobarlo:

const boton = document.querySelector("#boton");

if (boton !== null) {
  boton.addEventListener("click", () => {
    console.log("clic");
  });
}

La solución principal sigue siendo revisar que el HTML y el selector coincidan.

11.8. Sobrescribir todas las clases con className

mensaje.className = "error";

Esto reemplaza todas las clases existentes.

Mejor:

mensaje.classList.remove("correcto");
mensaje.classList.add("error");

11.9. Confundir target y currentTarget

boton.addEventListener("click", (event) => {
  console.log(event.target);
});

Si se pulsa un elemento hijo del botón, target puede ser ese elemento hijo.

Cuando se necesita el elemento que tiene asociado el listener:

console.log(event.currentTarget);

11.10. Confiar solo en la validación JavaScript

La validación JavaScript puede manipularse o desactivarse.

En una aplicación real:

  • se aprovecha la validación HTML;
  • se mejora la experiencia con JavaScript;
  • se validan de nuevo los datos en el servidor.

12.2. Resumen final

En este bloque hemos visto que JavaScript puede responder a acciones del usuario mediante eventos.

Ideas clave:

  • Un evento es algo que ocurre en la página.
  • addEventListener() permite asociar una función a un evento.
  • La función asociada al evento es un callback.
  • El objeto event aporta información sobre lo ocurrido.
  • event.target indica dónde se originó el evento.
  • event.currentTarget indica qué elemento está ejecutando el listener.
  • click, input, change y submit son eventos muy habituales.
  • En formularios, preventDefault() evita la acción predeterminada de envío.
  • Los valores de formulario se leen con .value.
  • Los valores leídos suelen llegar como texto.
  • La validación HTML y la validación JavaScript pueden combinarse.

Cuando un formulario no supera la validación HTML integrada, el navegador puede impedir el envío antes de que se ejecute el código asociado al evento submit. Por eso conviene probar y entender por separado la validación HTML y la validación personalizada con JavaScript.

  • La validación permite comprobar datos antes de aceptarlos.
  • classList ayuda a modificar la apariencia mediante clases CSS.
  • Las expresiones regulares permiten validar patrones sencillos de texto.
  • textContent es la opción adecuada para mostrar texto no confiable.
  • reset() permite limpiar un formulario.
  • focus() permite dirigir al usuario al campo que necesita corregir.
  • aria-live mejora la accesibilidad de los mensajes dinámicos.

Este bloque prepara el terreno para crear páginas realmente interactivas y para, más adelante, conectar esas interacciones con datos externos mediante comunicación asíncrona.
