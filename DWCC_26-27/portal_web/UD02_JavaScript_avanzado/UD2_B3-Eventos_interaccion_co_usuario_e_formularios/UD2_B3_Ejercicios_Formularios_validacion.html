UD2_B3. Ejercicios: formularios y validación

1. Objetivo de la hoja

En esta hoja vas a practicar la interacción con formularios HTML desde JavaScript.

El objetivo principal es que seas capaz de:

  • capturar el envío de un formulario;
  • evitar el comportamiento automático del navegador con preventDefault();
  • leer valores escritos por el usuario;
  • validar datos antes de aceptarlos;
  • mostrar mensajes de error y confirmación;
  • modificar clases CSS desde JavaScript;
  • activar o desactivar botones según el estado del formulario;
  • usar expresiones regulares sencillas en validaciones básicas;
  • organizar la validación en funciones reutilizables;
  • mostrar datos en el DOM de forma segura;
  • mantener una aplicación sin errores ni interferencias entre ejercicios.

En esta hoja no trabajaremos todavía fetch, APIs externas, promesas ni async/await. Eso quedará para el bloque de asincronía.

2. Cómo se desarrollará la hoja

2.1. Carácter acumulativo

Los ejercicios 1 a 15 se realizan de forma acumulativa sobre el mismo formulario y el mismo proyecto.

Esto significa que:

  • no debes borrar el código de los ejercicios anteriores;
  • cada ejercicio amplía o mejora el anterior;
  • las funciones de validación deben reutilizarse;
  • el formulario debe seguir funcionando al llegar a los ejercicios posteriores;
  • no debes comentar código válido para poder probar otro ejercicio.

Los ejercicios 16 a 19 son fragmentos independientes para corregir errores.

El ejercicio 20 es una mini-entrega final. Puedes reutilizar y reorganizar el código desarrollado en los ejercicios anteriores.

2.2. Campos obligatorios por fases

Durante los ejercicios 1 a 6 se trabajará con:

  • nombre;
  • correo;
  • edad;
  • modalidad;
  • aceptación de condiciones.

En el ejercicio 7 se añade:

  • precio de matrícula.

En el ejercicio 10 se añade:

  • teléfono.

A partir de ese momento, precio y teléfono también deben considerarse al validar el formulario completo, salvo que la profesora indique expresamente que son opcionales.

3. Instrucciones generales

  1. Crea una carpeta llamada UD2_B3_formularios_validacion.
  2. Dentro de ella crea, como mínimo, estos archivos:
UD2_B3_formularios_validacion/
├── index.html
├── css/
│   └── estilos.css
└── js/
    └── app.js
  1. Enlaza correctamente el archivo JavaScript desde el HTML:
<script src="js/app.js" defer></script>
  1. Usa addEventListener(), no atributos HTML como onclick, onsubmit, etc.
  2. No uses alert() como forma principal de mostrar resultados. Usa elementos del DOM.
  3. Comenta las partes importantes del código.
  4. Comprueba la consola del navegador para detectar errores.
  5. Usa identificadores id únicos.
  6. Utiliza classList para añadir y quitar clases.
  7. Usa textContent, createElement() y append() para mostrar datos introducidos por el usuario.
  8. No uses innerHTML con datos procedentes de campos de formulario.
  9. Mantén separados HTML, CSS y JavaScript.
  10. No dependas únicamente de que el botón esté desactivado: valida de nuevo dentro del evento submit.

4.2. Por qué se usa novalidate

El formulario inicial contiene:

<form id="formularioRegistro" novalidate>

novalidate desactiva la validación automática del navegador durante el envío.

En estos primeros ejercicios se utiliza para que:

  • todos los mensajes sean gestionados por JavaScript;
  • puedas practicar la validación personalizada;
  • se vea con claridad qué comprueba cada función.

Más adelante puedes retirar novalidate y añadir atributos como:

required
minlength
min
max

para comparar la validación HTML integrada con la validación JavaScript.

5. Parte 1. Capturar el envío y leer valores

Ejercicio 1. Evitar el envío automático

  1. Selecciona el formulario con querySelector().
  2. Añade un listener al evento submit.
  3. Recibe el objeto event como parámetro.
  4. Dentro del callback, usa event.preventDefault().
  5. Muestra en consola el mensaje:
Formulario capturado correctamente

Punto de partida:

const formulario = document.querySelector("#formularioRegistro");

formulario.addEventListener("submit", (event) => {
  event.preventDefault();

  console.log("Formulario capturado correctamente");
});

Ejercicio 2. Leer los valores del formulario

A partir del formulario anterior:

  1. Selecciona los campos nombre, correo y edad.
  2. Al enviar el formulario, muestra en consola:
    • el nombre escrito;
    • el correo escrito;
    • la edad escrita.

Ejemplo de salida esperada:

Nombre: Ana
Correo: ana@correo.com
Edad: 20

Recuerda que los valores de los campos se leen con .value.

Ejemplo:

const inputNombre = document.querySelector("#nombre");
const inputCorreo = document.querySelector("#correo");
const inputEdad = document.querySelector("#edad");

console.log("Nombre:", inputNombre.value);
console.log("Correo:", inputCorreo.value);
console.log("Edad:", inputEdad.value);

Aunque el campo de edad sea type="number", .value devuelve texto.

6. Parte 2. Validar campos obligatorios

Ejercicio 3. Validar que el nombre no esté vacío

  1. Lee el valor del campo nombre.
  2. Elimina espacios al principio y al final usando .trim().
  3. Si el nombre queda vacío, muestra el mensaje:
El nombre es obligatorio.
  1. El mensaje debe aparecer en el elemento #errorNombre.
  2. Si el nombre es correcto, limpia el mensaje de error.
  3. Coloca el foco en el campo cuando haya error.

Pista:

const nombre = inputNombre.value.trim();

if (nombre === "") {
  errorNombre.textContent = "El nombre es obligatorio.";
  inputNombre.focus();
} else {
  errorNombre.textContent = "";
}

Ejercicio 4. Añadir y quitar clases según la validación

Modifica el ejercicio anterior:

  1. Si el nombre está vacío:
    • añade la clase incorrecto al input;
    • elimina la clase correcto.
  2. Si el nombre es válido:
    • añade la clase correcto;
    • elimina la clase incorrecto.

Usa classList.add() y classList.remove().

Ejemplo:

inputNombre.classList.remove("correcto");
inputNombre.classList.add("incorrecto");

No permitas que un mismo campo tenga simultáneamente las clases correcto e incorrecto.

Ejercicio 5. Validar varios campos obligatorios

Valida también:

  • edad: no puede estar vacía;

Para cada campo, muestra el error en su elemento correspondiente:

Campo Elemento de error

7. Parte 3. Validar números

Ejercicio 6. Convertir y validar la edad

  1. Lee la edad como texto.
  2. Aplica trim().
  3. Comprueba primero si está vacía.
  4. Conviértela a número usando Number().
  5. Comprueba si la conversión ha fallado usando Number.isNaN().
  6. Comprueba que sea un número entero mediante Number.isInteger().
  7. Valida que la edad:
    • sea un número;
    • sea entera;
    • sea mayor o igual que 16;
    • sea menor o igual que 99.

Mensajes posibles:

La edad es obligatoria.
La edad debe ser un número.
La edad debe ser un número entero.
La edad debe estar entre 16 y 99 años.

Orden recomendado:

const edadTexto = inputEdad.value.trim();

if (edadTexto === "") {
  // La edad es obligatoria
} else {
  const edadNumero = Number(edadTexto);

  if (Number.isNaN(edadNumero)) {
    // No es un número
  } else if (!Number.isInteger(edadNumero)) {
    // No es entero
  } else if (edadNumero < 16 || edadNumero > 99) {
    // Fuera del rango
  }
}

No conviertas antes de comprobar el vacío, porque:

Number(""); // 0

Ejercicio 7. Precio de matrícula con descuento

Añade al formulario un campo nuevo:

<div class="campo">
  <label for="precio">Precio de matrícula</label>
  <input
    type="number"
    id="precio"
    name="precio"
    min="0"
    step="0.01"
  >
  <p
    class="error"
    id="errorPrecio"
    aria-live="polite"
  ></p>
</div>

Después:

  1. Lee el precio como texto.
  2. Comprueba que no esté vacío.
  3. Convierte el valor con Number().
  4. Valida que sea un número positivo.
  5. Si es válido, calcula un descuento del 10 %.
  6. Muestra el precio final con dos decimales usando .toFixed(2).

Ejemplo:

Precio final con descuento: 90.00 €

Recuerda:

toFixed(2)

devuelve una cadena de texto.

También puedes comparar el comportamiento de:

parseFloat("100abc"); // 100
Number("100abc");     // NaN

parseFloat() puede aceptar una parte inicial numérica aunque el resto del texto sea incorrecto. Para validar todo el contenido del campo, en este ejercicio se recomienda Number().

8. Parte 4. Validar texto, correo y teléfono

Ejercicio 8. Validar longitud mínima del nombre

Amplía la validación del nombre:

  1. El nombre no puede estar vacío.
  2. El nombre debe tener al menos 3 caracteres.

Mensajes posibles:

El nombre es obligatorio.
El nombre debe tener al menos 3 caracteres.

La longitud debe comprobarse sobre el texto limpio:

const nombre = inputNombre.value.trim();

Ejercicio 9. Validar correo con una expresión regular sencilla

Valida el correo electrónico con una expresión regular básica:

const patronCorreo = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  1. Si el campo está vacío, muestra:
El correo es obligatorio.
  1. Si no cumple el patrón, muestra:
El correo no tiene un formato válido.
  1. Si es correcto, limpia el mensaje de error.

Pista:

patronCorreo.test(correo)

Esta expresión es una aproximación didáctica sencilla. No representa una validación completa de todas las direcciones de correo posibles.

Ejercicio 10. Validar teléfono español básico

Añade un campo de teléfono:

<div class="campo">
  <label for="telefono">Teléfono</label>
  <input
    type="tel"
    id="telefono"
    name="telefono"
    autocomplete="tel"
  >
  <p
    class="error"
    id="errorTelefono"
    aria-live="polite"
  ></p>
</div>

Valida que:

  • no esté vacío;
  • tenga exactamente 9 cifras;
  • empiece por 6, 7, 8 o 9.

Puedes usar este patrón:

const patronTelefono = /^[6789]\d{8}$/;

Este patrón es una simplificación didáctica para números nacionales de nueve cifras:

  • sin espacios;
  • sin guiones;
  • sin prefijo internacional.

9. Parte 5. Validación en tiempo real y estado del botón

Ejercicio 11. Validar mientras se escribe

Añade un evento input al campo nombre.

  1. Cada vez que el usuario escriba, valida si el nombre tiene al menos 3 caracteres.
  2. Muestra o elimina el mensaje de error según corresponda.
  3. Añade o quita las clases correcto e incorrecto.
  4. Reutiliza la misma función validarNombre() que se usará al enviar.

Este ejercicio debe funcionar sin esperar al envío del formulario.

La validación en tiempo real debe ayudar al usuario, no mostrar mensajes agresivos antes de que haya tenido tiempo de escribir.

Ejercicio 12. Activar o desactivar el botón de envío

Modifica el formulario para que el botón de envío empiece desactivado:

<button type="submit" id="botonEnviar" disabled>
  Enviar
</button>

Después:

  1. Añade eventos input o change a los campos principales.
  2. Comprueba si:
    • el nombre tiene al menos 3 caracteres;
    • el correo tiene formato válido;
    • la edad es un entero entre 16 y 99;
    • se ha seleccionado una modalidad;
    • se han aceptado las condiciones;
    • el precio es válido y positivo;
    • el teléfono cumple el patrón básico.
  3. Si todo es correcto, activa el botón.
  4. Si falta algo, mantenlo desactivado.

Pista:

botonEnviar.disabled = !formularioValido;

Crea una función:

function actualizarEstadoBoton() {
  const formularioValido =
    validarNombre()
    && validarCorreo()
    && validarEdad()
    && validarModalidad()
    && validarCondiciones()
    && validarPrecio()
    && validarTelefono();

  botonEnviar.disabled = !formularioValido;
}

Aunque el botón esté desactivado, debes volver a ejecutar validarFormulario() dentro del evento submit.

10. Parte 6. Mostrar un resumen y limpiar el formulario

Ejercicio 13. Mostrar confirmación en el DOM

Cuando el formulario sea válido:

  1. Limpia los mensajes de error.
  2. Muestra en #resultado un resumen con los datos introducidos:
Registro realizado correctamente.

Nombre: Ana
Correo: ana@correo.com
Edad: 20
Modalidad: Presencial
  1. No uses alert().
  2. No recargues la página.
  3. No uses innerHTML con los datos introducidos por el usuario.
  4. Crea los elementos con createElement(), asigna el texto con textContent e insértalos con append() o replaceChildren().

Ejemplo seguro:

const titulo = document.createElement("h2");
titulo.textContent = "Registro realizado correctamente";

const parrafoNombre = document.createElement("p");
parrafoNombre.textContent = "Nombre: " + nombre;

const parrafoCorreo = document.createElement("p");
parrafoCorreo.textContent = "Correo: " + correo;

resultado.replaceChildren(
  titulo,
  parrafoNombre,
  parrafoCorreo
);

Validar un dato no significa que sea seguro interpretarlo como HTML.

Ejercicio 14. Limpiar el formulario tras enviar

Después de mostrar el resumen:

  1. Limpia los campos con formulario.reset().
  2. Elimina las clases correcto e incorrecto.
  3. Limpia los mensajes de error.
  4. Desactiva de nuevo el botón de envío.
  5. Restablece las variables internas necesarias.
  6. Devuelve el foco al primer campo.
  7. Mantén visible el resumen final.

Recuerda que:

formulario.reset();

solo restablece los controles del formulario. No elimina automáticamente:

  • mensajes;
  • clases CSS;
  • contenido de #resultado;
  • variables JavaScript;
  • estado calculado del botón.

Crea una función:

function limpiarFormulario() {
  formulario.reset();
  limpiarErrores();
  limpiarClasesValidacion();
  botonEnviar.disabled = true;
  inputNombre.focus();
}

11. Parte 7. Organizar el código en funciones

Ejercicio 15. Crear funciones de validación

Reorganiza el código creando funciones como estas:

function validarNombre() {
  // Actualiza mensaje y clases.
  // Devuelve true si el nombre es válido.
}

function validarCorreo() {
  // Actualiza mensaje y clases.
  // Devuelve true si el correo es válido.
}

function validarEdad() {
  // Actualiza mensaje y clases.
  // Devuelve true si la edad es válida.
}

function validarFormulario() {
  const nombreValido = validarNombre();
  const correoValido = validarCorreo();
  const edadValida = validarEdad();

  return nombreValido && correoValido && edadValida;
}

Las funciones deben reutilizarse:

  • durante la validación en tiempo real;
  • al actualizar el estado del botón;
  • dentro del evento submit.

No escribas reglas diferentes para el mismo campo en distintos lugares.

Una estructura posible:

formulario.addEventListener("submit", (event) => {
  event.preventDefault();

  if (!validarFormulario()) {
    return;
  }

  mostrarResumen();
  limpiarFormulario();
});

El objetivo es evitar que todo el código esté metido directamente dentro del evento submit.

12. Parte 8. Corrige los errores

Ejercicio 16. Error con preventDefault()

El siguiente código no evita correctamente que el formulario se envíe:

const formulario = document.querySelector("#formularioRegistro");

formulario.addEventListener("submit", () => {
  event.preventDefault();
  console.log("Enviado");
});
  1. Localiza el error.
  2. Corrige el código.
  3. Explica por qué fallaba.
  4. No confíes en una posible variable global llamada event.

Corrección esperada:

formulario.addEventListener("submit", (event) => {
  event.preventDefault();
  console.log("Enviado");
});

Ejercicio 17. Error al leer el valor de un input

const nombre = document.querySelector("#nombre").textContent;
console.log(nombre);
  1. Explica por qué no es correcto para un <input>.
  2. Corrige el código usando la propiedad adecuada.

Corrección:

const nombre = document.querySelector("#nombre").value;
console.log(nombre);

Ejercicio 18. Error al comprobar un checkbox

const acepta = document.querySelector("#aceptaCondiciones").value;

if (acepta) {
  console.log("Aceptado");
}
  1. Explica por qué .value no sirve para saber si el checkbox está marcado.
  2. Corrige el código usando la propiedad adecuada.

Corrección:

const acepta =
  document.querySelector("#aceptaCondiciones").checked;

if (acepta) {
  console.log("Aceptado");
}

La propiedad .value suele devolver el valor configurado del checkbox, frecuentemente "on", aunque no esté marcado.

Ejercicio 19. Error con clases CSS

inputNombre.classList.add("incorrecto");
inputNombre.classList.add("correcto");
  1. ¿Qué problema puede provocar este código?
  2. Corrígelo para que no tenga las dos clases a la vez.

Ejemplo:

inputNombre.classList.remove("correcto");
inputNombre.classList.add("incorrecto");

Y para el estado correcto:

inputNombre.classList.remove("incorrecto");
inputNombre.classList.add("correcto");

13. Parte 9. Mini-entrega final

Ejercicio 20. Formulario de inscripción a un curso

Debe incluir:

  • nombre;
  • apellidos;
  • correo electrónico;
  • teléfono;

Correo Obligatorio y con formato válido
Teléfono 9 cifras y comienza por 6, 7, 8 o 9
Edad Número entero entre 16 y 99
Modalidad Debe seleccionarse una opción
Nivel Debe seleccionarse una opción

Para el nivel inicial utiliza un grupo de botones de opción:

<fieldset>

Requisitos funcionales

El formulario debe:

  2. Usar preventDefault().
