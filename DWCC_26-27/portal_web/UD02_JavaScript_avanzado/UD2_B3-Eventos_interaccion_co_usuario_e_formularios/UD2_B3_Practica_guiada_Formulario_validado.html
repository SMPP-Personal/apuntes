UD2_B3. Práctica guiada: formulario validado con JavaScript

1. Objetivo, escenario y archivos de trabajo

1.1. Objetivo de la práctica

En esta práctica vas a construir un formulario de inscripción y vas a validarlo con JavaScript antes de aceptar el envío.

La práctica se centra en los contenidos del bloque UD2_B3: Eventos, interacción con el usuario y formularios.

Vas a trabajar:

  • selección de elementos del DOM;
  • eventos input, change y submit;
  • addEventListener();
  • objeto event;
  • event.preventDefault();
  • lectura de valores con .value;
  • validación de campos obligatorios;
  • validación de números;
  • validación básica con expresiones regulares;
  • mensajes de error y confirmación;
  • modificación de clases CSS con classList;
  • organización del código en funciones;
  • separación entre reglas de validación y presentación de mensajes;
  • creación segura de contenido en el DOM;
  • accesibilidad básica de formularios.

En esta práctica no se envían datos a ningún servidor. Todo se comprueba en el navegador.

1.2. Escenario de la práctica

Vas a crear un formulario para simular la inscripción de una persona en una actividad formativa.

El formulario tendrá estos campos:

Campo Tipo Validación requerida
Nombre completo Texto Obligatorio. Mínimo 3 caracteres.
Correo electrónico Email Obligatorio. Formato básico válido.
Edad Número Obligatoria. Número entero entre 16 y 99.
Curso Lista desplegable Debe seleccionarse una opción válida.
Modalidad Botones de opción Debe seleccionarse una modalidad.
Aceptación de normas Casilla de verificación Debe estar marcada.

Cuando el formulario sea correcto, se mostrará un resumen seguro de los datos introducidos.

2.2. Por qué usamos novalidate

El formulario contiene:

<form id="formularioInscripcion" novalidate>

novalidate evita que el navegador bloquee el envío mediante su validación automática.

En esta práctica se utiliza para:

  • gestionar todos los mensajes desde JavaScript;
  • comprender qué regla comprueba cada función;
  • controlar cómo y dónde aparecen los errores;
  • comparar más adelante la validación HTML con la validación JavaScript.

Aunque el formulario use novalidate, conservamos atributos como required, min, max, step y minlength porque documentan las reglas y mejoran la semántica del formulario.

3.3. Separar reglas y presentación

La práctica utilizará dos tipos de funciones:

  1. Funciones puras de comprobación

    • reciben valores;
    • no modifican el DOM;
    • no muestran mensajes;
    • devuelven true o false.

  2. Funciones de interfaz

    • muestran mensajes;
    • añaden o quitan clases;
    • actualizan atributos de accesibilidad;
    • colocan el foco cuando procede.

Esta separación permite:

  • reutilizar las reglas sin mostrar errores;
  • activar o desactivar el botón sin llenar la página de mensajes;
  • probar las reglas de forma independiente;
  • evitar duplicar condiciones.

4. Funciones puras de comprobación

4.1. Patrón de correo

Declara el patrón una sola vez:

const PATRON_EMAIL =
  /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

No es necesario memorizar expresiones regulares complejas. En este bloque basta con comprender que .test() comprueba si un texto cumple un patrón.

4.2. Comprobar el nombre

function comprobarNombre(valor) {
  return valor.trim().length >= 3;
}

4.3. Comprobar el correo

function comprobarEmail(valor) {
  return PATRON_EMAIL.test(valor.trim());
}

4.4. Comprobar la edad

function comprobarEdad(valor) {
  const texto = valor.trim();

  if (texto === "") {
    return false;
  }

  const edad = Number(texto);

  return (
    Number.isInteger(edad)
    && edad >= 16
    && edad <= 99
  );
}

4.5. Comprobar el curso

function comprobarCurso(valor) {
  return valor !== "";
}

4.6. Comprobar la modalidad

function obtenerModalidadSeleccionada() {
  return document.querySelector(
    "input[name='modalidad']:checked"
  );
}

function comprobarSeleccion(valorSeleccionado) {
  return valorSeleccionado !== null;
}

obtenerModalidadSeleccionada() consulta el DOM. La regla comprobarSeleccion() recibe el resultado y realiza la comprobación sin modificar la página.

4.7. Comprobar la aceptación de normas

function comprobarNormas(estaMarcado) {
  return estaMarcado;
}

4.8. Comprobar el formulario sin mostrar mensajes

function comprobarFormularioCompleto() {
  const modalidadSeleccionada =
    obtenerModalidadSeleccionada();

  return (
    comprobarNombre(inputNombre.value)
    && comprobarEmail(inputEmail.value)
    && comprobarEdad(inputEdad.value)
    && comprobarCurso(selectCurso.value)
    && comprobarSeleccion(modalidadSeleccionada)
    && comprobarNormas(inputNormas.checked)
  );
}

Esta función será útil para activar o desactivar el botón sin mostrar todos los errores.

5. Funciones de interfaz y mensajes

5.1. Mostrar un error

function mostrarError(campo, elementoError, mensaje) {
  campo.classList.remove("correcto");
  campo.classList.add("incorrecto");
  campo.setAttribute("aria-invalid", "true");

  elementoError.textContent = mensaje;
}

5.2. Marcar un campo como correcto

function marcarCorrecto(campo, elementoError) {
  campo.classList.remove("incorrecto");
  campo.classList.add("correcto");
  campo.setAttribute("aria-invalid", "false");

  elementoError.textContent = "";
}

5.3. Limpiar el estado de un campo

function limpiarEstado(campo, elementoError) {
  campo.classList.remove("correcto", "incorrecto");
  campo.removeAttribute("aria-invalid");

  elementoError.textContent = "";
}

5.4. Por qué separamos estas funciones

Las funciones puras solo responden:

¿El dato cumple la regla?

Las funciones de interfaz responden:

¿Cómo mostramos el resultado al usuario?

Así evitamos que una comprobación utilizada para activar el botón muestre errores prematuramente.

6. Validación visible de los campos

6.1. Validar el nombre

function validarNombre() {
  const nombre = inputNombre.value.trim();

  if (nombre === "") {
    mostrarError(
      inputNombre,
      errorNombre,
      "El nombre es obligatorio."
    );

    return false;
  }

  if (!comprobarNombre(nombre)) {
    mostrarError(
      inputNombre,
      errorNombre,
      "El nombre debe tener al menos 3 caracteres."
    );

    return false;
  }

  marcarCorrecto(inputNombre, errorNombre);
  return true;
}

6.2. Validar el correo electrónico

function validarEmail() {
  const email = inputEmail.value.trim();

  if (email === "") {
    mostrarError(
      inputEmail,
      errorEmail,
      "El correo electrónico es obligatorio."
    );

    return false;
  }

  if (!comprobarEmail(email)) {
    mostrarError(
      inputEmail,
      errorEmail,
      "Introduce un correo electrónico válido."
    );

    return false;
  }

  marcarCorrecto(inputEmail, errorEmail);
  return true;
}

6.3. Validar la edad

function validarEdad() {
  const edadTexto = inputEdad.value.trim();

  if (edadTexto === "") {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad es obligatoria."
    );

    return false;
  }

  const edad = Number(edadTexto);

  if (Number.isNaN(edad)) {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad debe ser un número."
    );

    return false;
  }

  if (!Number.isInteger(edad)) {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad debe ser un número entero."
    );

    return false;
  }

  if (!comprobarEdad(edadTexto)) {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad debe estar entre 16 y 99 años."
    );

    return false;
  }

  marcarCorrecto(inputEdad, errorEdad);
  return true;
}

6.4. Validar el curso

function validarCurso() {
  if (!comprobarCurso(selectCurso.value)) {
    mostrarError(
      selectCurso,
      errorCurso,
      "Debes seleccionar un curso."
    );

    return false;
  }

  marcarCorrecto(selectCurso, errorCurso);
  return true;
}

6.5. Validar la modalidad

function validarModalidad() {
  const modalidadSeleccionada =
    obtenerModalidadSeleccionada();

  if (!comprobarSeleccion(modalidadSeleccionada)) {
    mostrarError(
      grupoModalidad,
      errorModalidad,
      "Debes seleccionar una modalidad."
    );

    return false;
  }

  marcarCorrecto(
    grupoModalidad,
    errorModalidad
  );

  return true;
}

6.6. Validar la aceptación de normas

function validarNormas() {
  if (!comprobarNormas(inputNormas.checked)) {
    mostrarError(
      grupoNormas,
      errorNormas,
      "Debes aceptar las normas para continuar."
    );

    return false;
  }

  marcarCorrecto(
    grupoNormas,
    errorNormas
  );

  return true;
}

7. Validación completa y envío

7.1. Validar el formulario mostrando errores

function validarFormulario() {
  const nombreValido = validarNombre();
  const emailValido = validarEmail();
  const edadValida = validarEdad();
  const cursoValido = validarCurso();
  const modalidadValida = validarModalidad();
  const normasValidas = validarNormas();

  return (
    nombreValido
    && emailValido
    && edadValida
    && cursoValido
    && modalidadValida
    && normasValidas
  );
}

Cada función devuelve true o false.

Se ejecutan todas las funciones para que el usuario pueda ver todos los errores del formulario, no solo el primero.

7.2. Capturar el evento submit

formulario.addEventListener("submit", (event) => {
  event.preventDefault();

  const formularioValido =
    validarFormulario();

  if (formularioValido) {
    mostrarResumen();
  } else {
    resultado.classList.add("oculto");
  }
});

event.preventDefault() evita el envío tradicional y la recarga de la página.

Esto permite:

  • validar los datos;
  • mostrar errores;
  • decidir cuándo se acepta la inscripción;
  • generar el resumen en el DOM.

8. Creación segura del resumen

8.1. Agrupar los datos en un objeto

function obtenerDatosInscripcion() {
  const modalidadSeleccionada =
    obtenerModalidadSeleccionada();

  return {
    nombre: inputNombre.value.trim(),
    email: inputEmail.value.trim(),
    edad: Number(inputEdad.value),
    curso: selectCurso.value,
    modalidad: modalidadSeleccionada.value
  };
}

Esto conecta con lo trabajado en el bloque anterior sobre objetos.

Ejemplo de objeto:

const datosInscripcion = {
  nombre: "Ana",
  email: "ana@correo.com",
  edad: 22,
  curso: "DOM y eventos",
  modalidad: "Presencial"
};

8.2. Crear una fila del resumen

function crearDatoResumen(etiqueta, valor) {
  const parrafo = document.createElement("p");
  const strong = document.createElement("strong");

  strong.textContent = etiqueta + ": ";

  parrafo.append(
    strong,
    String(valor)
  );

  return parrafo;
}

8.3. Mostrar el resumen

function mostrarResumen() {
  const datosInscripcion =
    obtenerDatosInscripcion();

  const nombre = crearDatoResumen(
    "Nombre",
    datosInscripcion.nombre
  );

  const email = crearDatoResumen(
    "Email",
    datosInscripcion.email
  );

  const edad = crearDatoResumen(
    "Edad",
    datosInscripcion.edad
  );

  const curso = crearDatoResumen(
    "Curso",
    datosInscripcion.curso
  );

  const modalidad = crearDatoResumen(
    "Modalidad",
    datosInscripcion.modalidad
  );

  resumen.replaceChildren(
    nombre,
    email,
    edad,
    curso,
    modalidad
  );

  resultado.classList.remove("oculto");
}

8.4. Por qué no usamos innerHTML

No usamos:

resumen.innerHTML = inputNombre.value;

porque los datos escritos por el usuario no deben interpretarse como HTML.

Usamos:

  • createElement();
  • textContent;
  • append();
  • replaceChildren().

Así, cualquier etiqueta escrita por el usuario se muestra como texto y no se ejecuta como código.

9.2. Activar el botón sin mostrar errores prematuros

En el HTML, modifica el botón:

<button
  type="submit"
  id="botonEnviar"
  disabled
>
  Enviar inscripción
</button>

Crea:

function actualizarBoton() {
  botonEnviar.disabled =
    !comprobarFormularioCompleto();
}

Esta función usa las reglas de comprobación, pero no las funciones que muestran mensajes.

Así evitamos que, al escribir la primera letra del nombre, aparezcan errores en todos los campos restantes.

9.3. Combinar validación visible y actualización del botón

inputNombre.addEventListener("input", () => {
  validarNombre();
  actualizarBoton();
});

inputEmail.addEventListener("input", () => {
  validarEmail();
  actualizarBoton();
});

inputEdad.addEventListener("input", () => {
  validarEdad();
  actualizarBoton();
});

selectCurso.addEventListener("change", () => {
  validarCurso();
  actualizarBoton();
});

inputNormas.addEventListener("change", () => {
  validarNormas();
  actualizarBoton();
});

radiosModalidad.forEach((radio) => {
  radio.addEventListener("change", () => {
    validarModalidad();
    actualizarBoton();
  });
});

Aunque el botón esté desactivado mientras el formulario sea incompleto, el evento submit vuelve a ejecutar validarFormulario().

El estado del botón mejora la interfaz, pero no sustituye la validación final.

const inputNormas =
  document.querySelector("#aceptaNormas");

const grupoModalidad =
  document.querySelector("#grupoModalidad");

const grupoNormas =
  document.querySelector("#grupoNormas");

const errorNombre =
  document.querySelector("#errorNombre");

const errorEmail =
  document.querySelector("#errorEmail");

const errorEdad =
  document.querySelector("#errorEdad");

const errorCurso =
  document.querySelector("#errorCurso");

const errorModalidad =
  document.querySelector("#errorModalidad");

const errorNormas =
  document.querySelector("#errorNormas");

const resultado =
  document.querySelector("#resultado");

const resumen =
  document.querySelector("#resumen");

const botonEnviar =
  document.querySelector("#botonEnviar");

const radiosModalidad =
  document.querySelectorAll(
    "input[name='modalidad']"
  );

const PATRON_EMAIL =
  /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

function comprobarNombre(valor) {
  return valor.trim().length >= 3;
}

function comprobarEmail(valor) {
  return PATRON_EMAIL.test(valor.trim());
}

function comprobarEdad(valor) {
  const texto = valor.trim();

  if (texto === "") {
    return false;
  }

  const edad = Number(texto);

  return (
    Number.isInteger(edad)
    && edad >= 16
    && edad <= 99
  );
}

function comprobarCurso(valor) {
  return valor !== "";
}

function comprobarSeleccion(valorSeleccionado) {
  return valorSeleccionado !== null;
}

function comprobarNormas(estaMarcado) {
  return estaMarcado;
}

function obtenerModalidadSeleccionada() {
  return document.querySelector(
    "input[name='modalidad']:checked"
  );
}

function comprobarFormularioCompleto() {
  return (
    comprobarNombre(inputNombre.value)
    && comprobarEmail(inputEmail.value)
    && comprobarEdad(inputEdad.value)
    && comprobarCurso(selectCurso.value)
    && comprobarSeleccion(
      obtenerModalidadSeleccionada()
    )
    && comprobarNormas(inputNormas.checked)
  );
}

function mostrarError(campo, elementoError, mensaje) {
  campo.classList.remove("correcto");
  campo.classList.add("incorrecto");
  campo.setAttribute("aria-invalid", "true");

  elementoError.textContent = mensaje;
}

function marcarCorrecto(campo, elementoError) {
  campo.classList.remove("incorrecto");
  campo.classList.add("correcto");
  campo.setAttribute("aria-invalid", "false");

  elementoError.textContent = "";
}

function limpiarEstado(campo, elementoError) {
  campo.classList.remove("correcto", "incorrecto");
  campo.removeAttribute("aria-invalid");

  elementoError.textContent = "";
}

function validarNombre() {
  const nombre = inputNombre.value.trim();

  if (nombre === "") {
    mostrarError(
      inputNombre,
      errorNombre,
      "El nombre es obligatorio."
    );

    return false;
  }

  if (!comprobarNombre(nombre)) {
    mostrarError(
      inputNombre,
      errorNombre,
      "El nombre debe tener al menos 3 caracteres."
    );

    return false;
  }

  marcarCorrecto(inputNombre, errorNombre);
  return true;
}

function validarEmail() {
  const email = inputEmail.value.trim();

  if (email === "") {
    mostrarError(
      inputEmail,
      errorEmail,
      "El correo electrónico es obligatorio."
    );

    return false;
  }

  if (!comprobarEmail(email)) {
    mostrarError(
      inputEmail,
      errorEmail,
      "Introduce un correo electrónico válido."
    );

    return false;
  }

  marcarCorrecto(inputEmail, errorEmail);
  return true;
}

function validarEdad() {
  const edadTexto = inputEdad.value.trim();

  if (edadTexto === "") {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad es obligatoria."
    );

    return false;
  }

  const edad = Number(edadTexto);

  if (Number.isNaN(edad)) {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad debe ser un número."
    );

    return false;
  }

  if (!Number.isInteger(edad)) {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad debe ser un número entero."
    );

    return false;
  }

  if (!comprobarEdad(edadTexto)) {
    mostrarError(
      inputEdad,
      errorEdad,
      "La edad debe estar entre 16 y 99 años."
    );

    return false;
  }

  marcarCorrecto(inputEdad, errorEdad);
  return true;
}

function validarCurso() {
  if (!comprobarCurso(selectCurso.value)) {
    mostrarError(
      selectCurso,
      errorCurso,
      "Debes seleccionar un curso."
    );

    return false;
  }

  marcarCorrecto(selectCurso, errorCurso);
  return true;
}

function validarModalidad() {
  const modalidadSeleccionada =
    obtenerModalidadSeleccionada();

  if (!comprobarSeleccion(modalidadSeleccionada)) {
    mostrarError(
      grupoModalidad,
      errorModalidad,
      "Debes seleccionar una modalidad."
    );

    return false;
  }

  marcarCorrecto(
    grupoModalidad,
    errorModalidad
  );

  return true;
}

function validarNormas() {
  if (!comprobarNormas(inputNormas.checked)) {
    mostrarError(
      grupoNormas,
      errorNormas,
      "Debes aceptar las normas para continuar."
    );

    return false;
  }

  marcarCorrecto(
    grupoNormas,
    errorNormas
  );

  return true;
}

function validarFormulario() {
  const nombreValido = validarNombre();
  const emailValido = validarEmail();
  const edadValida = validarEdad();
  const cursoValido = validarCurso();
  const modalidadValida = validarModalidad();
  const normasValidas = validarNormas();

  return (
    nombreValido
    && emailValido
    && edadValida
    && cursoValido
    && modalidadValida
    && normasValidas
  );
}

function obtenerDatosInscripcion() {
  const modalidadSeleccionada =
    obtenerModalidadSeleccionada();

  return {
    nombre: inputNombre.value.trim(),
    email: inputEmail.value.trim(),
    edad: Number(inputEdad.value),
    curso: selectCurso.value,
    modalidad: modalidadSeleccionada.value
  };
}

function crearDatoResumen(etiqueta, valor) {
  const parrafo = document.createElement("p");
  const strong = document.createElement("strong");

  strong.textContent = etiqueta + ": ";

  parrafo.append(
    strong,
    String(valor)
  );

  return parrafo;
}

function mostrarResumen() {
  const datosInscripcion =
    obtenerDatosInscripcion();

  resumen.replaceChildren(
    crearDatoResumen(
      "Nombre",
      datosInscripcion.nombre
    ),
    crearDatoResumen(
      "Email",
      datosInscripcion.email
    ),
    crearDatoResumen(
      "Edad",
      datosInscripcion.edad
    ),
    crearDatoResumen(
      "Curso",
      datosInscripcion.curso
    ),
    crearDatoResumen(
      "Modalidad",
      datosInscripcion.modalidad
    )
  );

  resultado.classList.remove("oculto");
}

function actualizarBoton() {
  botonEnviar.disabled =
    !comprobarFormularioCompleto();
}

formulario.addEventListener("submit", (event) => {
  event.preventDefault();

  const formularioValido =
    validarFormulario();

  if (formularioValido) {
    mostrarResumen();
  } else {
    resultado.classList.add("oculto");
  }
});

inputNombre.addEventListener("input", () => {
  validarNombre();
  actualizarBoton();
});

inputEmail.addEventListener("input", () => {
  validarEmail();
  actualizarBoton();
});

inputEdad.addEventListener("input", () => {
  validarEdad();
  actualizarBoton();
});

selectCurso.addEventListener("change", () => {
  validarCurso();
  actualizarBoton();
});

inputNormas.addEventListener("change", () => {
  validarNormas();
  actualizarBoton();
});

radiosModalidad.forEach((radio) => {
  radio.addEventListener("change", () => {
    validarModalidad();
    actualizarBoton();
  });
});

actualizarBoton();

11. Ampliación opcional: limpiar el formulario

Después de mostrar el resumen puedes limpiar el formulario, manteniendo visible el resultado.

function limpiarFormulario() {
  formulario.reset();

  limpiarEstado(
    inputNombre,
    errorNombre
  );

  limpiarEstado(
    inputEmail,
    errorEmail
  );

  limpiarEstado(
    inputEdad,
    errorEdad
  );

  limpiarEstado(
    selectCurso,
    errorCurso
  );

  limpiarEstado(
    grupoModalidad,
    errorModalidad
  );

  limpiarEstado(
    grupoNormas,
    errorNormas
  );

  actualizarBoton();
  inputNombre.focus();
}

Para usarla:

if (formularioValido) {
  mostrarResumen();
  limpiarFormulario();
}

formulario.reset() solo restablece los controles. No elimina por sí mismo:

  • clases;
  • mensajes;
  • atributos aria-invalid;
  • estado calculado del botón;
  • variables JavaScript.

12. Pruebas, errores frecuentes, entrega y evaluación

12.1. Pruebas obligatorias

Prueba Resultado esperado
Enviar el formulario vacío Aparecen mensajes de error.
Escribir solo espacios en el nombre Error de nombre obligatorio.
Escribir un nombre de 2 caracteres Error de longitud mínima.
Escribir un email sin @ Error de email no válido.
Escribir edad 15 Error de edad fuera de rango.
Escribir edad 16 Edad válida.
Escribir edad 99 Edad válida.
Escribir edad 100 Error de edad fuera de rango.
Escribir una edad decimal Error: la edad debe ser entera.
No seleccionar curso Error de curso obligatorio.
No seleccionar modalidad Error de modalidad obligatoria.
No aceptar normas Error de aceptación de normas.
Corregir un campo incorrecto Desaparece el error y cambia la clase.
Completar todo correctamente Aparece el resumen de inscripción.
Introducir etiquetas en el nombre Se muestran como texto, no como HTML.
Enviar dos veces con datos diferentes El resumen anterior se sustituye.
Formulario incompleto El botón permanece desactivado.
Todos los datos válidos El botón se activa.
Invalidar un campo válido El botón vuelve a desactivarse.

12.2. Errores frecuentes

Error 1. Olvidar preventDefault()

Incorrecto:

formulario.addEventListener("submit", (event) => {
  validarFormulario();
});

Correcto:

formulario.addEventListener("submit", (event) => {
  event.preventDefault();
  validarFormulario();
});

Error 2. Leer el elemento en lugar de su valor

Incorrecto:

const nombre = inputNombre;

Correcto:

const nombre = inputNombre.value;

Error 3. No convertir la edad

Incorrecto:

const edad = inputEdad.value;

Correcto:

const edad = Number(inputEdad.value);

Error 4. Acceder a un radio sin comprobarlo

Incorrecto:

const modalidad =
  document.querySelector(
    "input[name='modalidad']:checked"
  ).value;

Correcto:

const modalidadSeleccionada =
  document.querySelector(
    "input[name='modalidad']:checked"
  );

if (modalidadSeleccionada !== null) {
  console.log(
    modalidadSeleccionada.value
  );
}

Error 5. Mantener clases incompatibles

Incorrecto:

inputNombre.classList.add("incorrecto");
inputNombre.classList.add("correcto");

Correcto:

inputNombre.classList.remove("correcto");
inputNombre.classList.add("incorrecto");

Error 6. Mostrar datos del usuario con innerHTML

Incorrecto:

resumen.innerHTML = inputNombre.value;

Correcto:

resumen.textContent = inputNombre.value;

Error 7. Aceptar edades decimales

Incorrecto:

if (edad >= 16 && edad <= 99) {
  console.log("Edad válida");
}

Correcto:

if (
  Number.isInteger(edad)
  && edad >= 16
  && edad <= 99
) {
  console.log("Edad válida");
}

12.5. Lista de comprobación final

Antes de entregar, revisa:

  • El HTML carga correctamente el CSS.
  • El HTML carga correctamente el JavaScript.
  • No aparecen errores en la consola.
  • El formulario vacío no se acepta.
  • Los errores aparecen junto a cada campo.
  • Los errores desaparecen cuando el campo es correcto.
  • El correo se valida con una expresión regular sencilla.
  • La edad se convierte a número.
  • La edad debe ser entera.
  • El formulario no recarga la página.
  • El resumen solo aparece con todos los datos válidos.
  • El resumen no usa innerHTML con datos del usuario.
  • Las reglas de validación están separadas de los mensajes.
  • El botón se activa y desactiva correctamente.
  • El formulario vuelve a validarse en submit.
  • Los estados visuales no dependen solo del color.
  • Los mensajes dinámicos incluyen accesibilidad básica.

12.6. Contenidos aplazados

No forman parte de esta práctica:

  • almacenamiento en localStorage;
  • envío real de datos a un servidor;
  • fetch;
  • promesas;
  • async/await;
  • consumo de APIs;
  • validaciones asíncronas;
  • frameworks o librerías externas;
  • componentes reutilizables avanzados;
  • propagación avanzada y delegación de eventos.
