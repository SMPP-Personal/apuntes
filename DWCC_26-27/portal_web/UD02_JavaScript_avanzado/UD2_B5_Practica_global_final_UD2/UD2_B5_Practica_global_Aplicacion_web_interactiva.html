UD2_B5. Práctica global final: aplicación web interactiva con seguimiento de recursos

Práctica global final de cierre de la UD2: integración, persistencia local y seguimiento personal de recursos.

1. Finalidad de la práctica

Esta práctica sirve como cierre de la UD2: JavaScript avanzado en entorno cliente .

El objetivo es construir una aplicación web completa que integre los contenidos trabajados en la unidad y que no se limite a cargar datos y pintarlos en pantalla. La aplicación deberá permitir al usuario interactuar con esos datos, modificar su estado, conservar parte de esa información en el navegador y explicar técnicamente las decisiones tomadas.

En esta práctica vas a demostrar que sabes:

  • trabajar con objetos y arrays de objetos;
  • cargar datos externos en formato JSON usando fetch ;
  • usar async / await y try...catch ;
  • comprobar response.ok antes de leer la respuesta;
  • convertir una respuesta HTTP en datos JavaScript con await response.json() ;
  • renderizar contenido dinámico en el DOM;
  • gestionar eventos en elementos estáticos y generados dinámicamente;
  • validar formularios;
  • filtrar, ordenar, buscar y calcular estadísticas;
  • usar localStorage para conservar información del usuario;
  • separar el código en funciones con responsabilidades claras;
  • probar, depurar y justificar el funcionamiento de la aplicación.

No se trata de copiar código de apuntes anteriores. La práctica debe funcionar como una aplicación propia y debe poder explicarse.

2. Diferencia con la práctica global de UD2_B4

La práctica global de UD2_B4 se centraba principalmente en cargar datos con fetch , mostrarlos en tarjetas, filtrarlos y añadir un recurso visualmente.

Esta práctica final sube un nivel:

  • el fichero JSON será el catálogo base de recursos;
  • el usuario podrá marcar recursos como favoritos;
  • el usuario podrá cambiar el estado de cada recurso: pendiente, en progreso o completado;
  • el usuario podrá guardar una nota personal asociada a un recurso;
  • esa información personal se guardará en localStorage ;
  • se calcularán estadísticas combinando datos del JSON y datos guardados por el usuario;
  • se añadirá una pequeña zona de exportación o depuración con JSON.stringify() y JSON.parse() ;
  • se entregará una explicación técnica final.

Por tanto, esta práctica no consiste solo en consultar y mostrar datos , sino en crear una pequeña aplicación interactiva de seguimiento.

3. Aplicación que se va a construir

Vas a crear una aplicación llamada:

Panel personal de recursos de aprendizaje

La aplicación cargará una lista de recursos desde un archivo JSON y permitirá al usuario gestionar su propio seguimiento.

La aplicación deberá permitir:

  1. Cargar recursos desde datos/recursos.json usando fetch .
  2. Mostrar los recursos en tarjetas creadas desde JavaScript.
  3. Buscar recursos por texto.
  4. Filtrar por tipo.
  5. Filtrar por dificultad.
  6. Filtrar por estado personal: pendiente, en progreso o completado.
  7. Mostrar solo favoritos.
  8. Ordenar por título, duración o dificultad.
  9. Cambiar el estado personal de un recurso.
  10. Marcar y desmarcar recursos como favoritos.
  11. Escribir una nota personal breve en cada recurso.
  12. Guardar progreso, favoritos y notas en localStorage .
  13. Restaurar automáticamente esa información al recargar la página.
  14. Mostrar estadísticas generales y de progreso.
  15. Exportar en consola el estado actual de la aplicación en formato JSON.
  16. Validar un formulario para añadir un recurso personalizado durante la sesión.
  17. Mostrar mensajes de carga, error, confirmación y resultado vacío.

Los recursos personalizados añadidos desde el formulario sí se conservarán en localStorage , pero no se modificarán físicamente en el archivo recursos.json .

11. Carga de datos con fetch

En script.js , crea al menos estas variables globales controladas: 

const CLAVE_SEGUIMIENTO = "ud2_b5_seguimiento_recursos";
const CLAVE_RECURSOS_PERSONALES = "ud2_b5_recursos_personales";

let recursosBase = [];
let recursosPersonales = [];
let seguimiento = {};

Crea una función asíncrona para cargar el catálogo base: 

async function cargarRecursos() {
  try {
    mostrarMensaje("Cargando recursos...", "cargando");

    const respuesta = await fetch("datos/recursos.json");

    if (!respuesta.ok) {
      throw new Error(`Error HTTP: ${respuesta.status}`);
    }

    recursosBase = await respuesta.json();
    recursosPersonales = leerRecursosPersonales();
    seguimiento = leerSeguimiento();

    rellenarFiltros();
    actualizarVista();
    mostrarMensaje("Recursos cargados correctamente.", "ok");
  } catch (error) {
    mostrarMensaje("No se pudieron cargar los recursos: " + error.message, "error");
    console.error("Error al cargar recursos:", error);
  }
}

Recuerda:

  • fetch() inicia la petición;
  • response.ok permite comprobar si la respuesta HTTP es correcta;
  • response.json() devuelve una promesa;
  • por eso se usa await response.json() ;
  • try...catch captura errores de red y errores lanzados manualmente.

14. Seguimiento personal de cada recurso

Crea una función para obtener los datos personales de un recurso. Si todavía no existen, debe devolver valores por defecto. 

function obtenerSeguimientoRecurso(id) {
  if (!seguimiento[id]) {
    seguimiento[id] = {
      estado: "pendiente",
      favorito: false,
      nota: ""
    };
  }

  return seguimiento[id];
}

Cuando el usuario cambie el estado, marque favorito o escriba una nota, debes actualizar seguimiento y guardar los cambios en localStorage .

Ejemplo orientativo: 

function actualizarSeguimientoRecurso(id, cambios) {
  const datosActuales = obtenerSeguimientoRecurso(id);

  seguimiento[id] = {
    ...datosActuales,
    ...cambios
  };

  guardarSeguimiento();
  actualizarVista();
}

Este patrón usa el operador spread para combinar el estado anterior con los cambios nuevos.

15. Guardar y recuperar con localStorage

Crea funciones separadas para leer y guardar. 

function leerSeguimiento() {
  const texto = localStorage.getItem(CLAVE_SEGUIMIENTO);

  if (texto === null) {
    return {};
  }

  return JSON.parse(texto);
}

function guardarSeguimiento() {
  localStorage.setItem(CLAVE_SEGUIMIENTO, JSON.stringify(seguimiento));
}

Haz lo mismo con los recursos personalizados: 

function leerRecursosPersonales() {
  const texto = localStorage.getItem(CLAVE_RECURSOS_PERSONALES);

  if (texto === null) {
    return [];
  }

  return JSON.parse(texto);
}

function guardarRecursosPersonales() {
  localStorage.setItem(CLAVE_RECURSOS_PERSONALES, JSON.stringify(recursosPersonales));
}

Para esta práctica no es necesario capturar errores de JSON.parse() con try...catch , pero puedes hacerlo como ampliación si quieres una solución más robusta.

16. Buscar, filtrar y ordenar

Crea una función: 

function obtenerRecursosVisibles() {
  // código
}

Primero debes unir los recursos del JSON con los recursos personalizados: 

const todosLosRecursos = [...recursosBase, ...recursosPersonales];

Después aplica búsqueda, filtros y ordenación.

La búsqueda debe funcionar, como mínimo, sobre:

  • título;
  • tipo;
  • dificultad;
  • etiquetas.

Ejemplo orientativo para la búsqueda: 

const textoEtiquetas = recurso.etiquetas.join(" ").toLowerCase();

const coincideTexto =
  recurso.titulo.toLowerCase().includes(texto) ||
  recurso.tipo.toLowerCase().includes(texto) ||
  recurso.dificultad.toLowerCase().includes(texto) ||
  textoEtiquetas.includes(texto);

El filtro por estado debe consultar el seguimiento personal: 

const seguimientoRecurso = obtenerSeguimientoRecurso(recurso.id);
const coincideEstado = filtroEstado === "todos" || seguimientoRecurso.estado === filtroEstado;

El filtro de favoritos debe funcionar así: 

const coincideFavorito = !soloFavoritos.checked || seguimientoRecurso.favorito;

Para ordenar, no modifiques directamente el array original. Usa una copia: 

let resultado = [...todosLosRecursos];

Para ordenar por dificultad puedes usar un objeto de pesos: 

const pesoDificultad = {
  basica: 1,
  media: 2,
  avanzada: 3
};

Y después: 

if (orden === "dificultadAsc") {
  resultado.sort((a, b) => pesoDificultad[a.dificultad] - pesoDificultad[b.dificultad]);
}

const duracionMedia = duracionTotal / listaVisible.length;

  titulo: titulo.value.trim(),
  tipo: tipo.value,
  dificultad: dificultad.value,
  duracion: Number(duracion.value),
  etiquetas: etiquetas.value
    .split(",")
    .map((etiqueta) => etiqueta.trim())
    .filter((etiqueta) => etiqueta !== ""),
  publicado: false,
  personalizado: true
};

Usa una función para generar identificadores: 

function generarNuevoId() {
  const todosLosRecursos = [...recursosBase, ...recursosPersonales];

  if (todosLosRecursos.length === 0) {
    return 1;
  }

  const ids = todosLosRecursos.map((recurso) => recurso.id);
  return Math.max(...ids) + 1;
}

Después de añadir un recurso personalizado:

  1. Añádelo a recursosPersonales .
  2. Guarda recursosPersonales en localStorage .
  3. Crea su seguimiento inicial si es necesario.
  4. Limpia el formulario.
  5. Limpia filtros para que el recurso añadido sea visible.
  6. Actualiza la vista.
  7. Muestra un mensaje de confirmación.

21. Reiniciar seguimiento

Añade un botón para borrar los datos personales guardados.

La función debe:

  1. Pedir confirmación con confirm() .
  2. Borrar las claves usadas en localStorage .
  3. Vaciar las variables seguimiento y recursosPersonales .
  4. Actualizar la vista.
  5. Mostrar un mensaje de confirmación.

Ejemplo orientativo: 

function reiniciarSeguimiento() {
  const confirmar = confirm("¿Seguro que quieres borrar tu seguimiento y recursos personalizados?");

  if (!confirmar) {
    return;
  }

  localStorage.removeItem(CLAVE_SEGUIMIENTO);
  localStorage.removeItem(CLAVE_RECURSOS_PERSONALES);

  seguimiento = {};
  recursosPersonales = [];

  actualizarVista();
  mostrarMensaje("Seguimiento reiniciado.", "ok");
}
