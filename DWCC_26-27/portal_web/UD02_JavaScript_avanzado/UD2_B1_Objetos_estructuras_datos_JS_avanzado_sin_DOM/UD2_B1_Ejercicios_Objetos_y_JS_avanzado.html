UD2_B1. Ejercicios: objetos y JavaScript avanzado sin DOM

1. Objetivo de la hoja

Esta hoja de ejercicios acompaña a los apuntes UD2_B1. Objetos y JavaScript avanzado sin DOM.

El objetivo es practicar cómo organizar información más compleja en JavaScript usando objetos, arrays de objetos y métodos avanzados de arrays, sin manipular todavía elementos HTML desde JavaScript.

Al terminar esta hoja deberías ser capaz de:

  • crear objetos literales;
  • acceder, modificar, añadir y eliminar propiedades;
  • crear métodos dentro de objetos;
  • entender el uso básico de this;
  • detectar errores al usar funciones flecha dentro de objetos;
  • trabajar con arrays de objetos;
  • aplicar map, filter, reduce, find, some, every y sort sobre arrays de objetos;
  • usar destructuring, spread y rest de forma básica;
  • convertir estructuras de datos a JSON y recuperarlas;
  • elegir la estructura adecuada para representar información.

En esta hoja no se trabaja DOM, no se usan eventos, no se trabaja asincronía y no se consume ninguna API externa. Eso se verá en bloques posteriores.

2. Instrucciones generales

  1. Crea una carpeta llamada UD2_B1_ejercicios.
  2. Dentro de ella crea un archivo llamado objetos_js_avanzado.js.
  3. Resuelve los ejercicios en orden.
  4. Separa cada ejercicio con un comentario claro:
// Ejercicio 1
  1. Comprueba el resultado en la consola del navegador o con Node.js, según indique la profesora.
  2. Usa nombres de variables descriptivos.
  3. Añade comentarios breves cuando el ejercicio lo pida.
  4. No copies soluciones completas sin entenderlas: modifica los datos y comprueba qué ocurre.

Importante sobre las variables repetidas

Todos los ejercicios se realizan en un mismo archivo. No puedes declarar varias veces una constante con el mismo nombre en el mismo ámbito.

Para evitar conflictos:

  • reutiliza el mismo array alumnos en los ejercicios que dependen de él;
  • usa nombres distintos en ejercicios independientes, como productoEj2, productoDestructuring o productoOferta;
  • en ejercicios aislados también puedes encerrar el código dentro de un bloque:
{
  // Ejercicio independiente
  const ejemplo = {
    nombre: "Ana"
  };

  console.log(ejemplo);
}

En esta hoja algunos ejemplos ya utilizan nombres diferentes para evitar redeclaraciones accidentales.

Parte 4. Funciones flecha dentro de objetos: detectar el problema

Ejercicio 9. Analizar un error con this

Lee este código:

const alumnoConFlecha = {
  nombre: "Ana",

  saludar: () => {
    console.log("Hola, soy " + this.nombre);
  }
};

alumnoConFlecha.saludar();

Responde en comentarios dentro del archivo:

  1. ¿Qué intenta hacer el código?
  2. ¿Por qué this no hace referencia al objeto alumnoConFlecha dentro de esa función flecha?
  3. Reescribe el método usando una función tradicional para que funcione correctamente.

Las funciones flecha no crean su propio this. En este ejemplo, this.nombre no apunta a la propiedad nombre del objeto alumnoConFlecha.

Solución esperada de la estructura corregida:

const alumnoCorregido = {
  nombre: "Ana",

  saludar: function () {
    console.log("Hola, soy " + this.nombre);
  }
};

Parte 10. Spread y rest

Ojo: spread y rest utilizan el mismo símbolo ..., pero no hacen lo mismo:

  • spread expande valores para copiar, combinar o crear nuevas estructuras;
  • rest recoge varios argumentos en un array.

Copias y spread Comprende cuándo conviene copiar antes de modificar u ordenar.
JSON Convierte datos a JSON y los recupera correctamente.

  • renderizado dinámico en una página web.

18. Resumen final

En esta hoja has practicado cómo JavaScript permite representar datos complejos sin necesidad de usar todavía el DOM.

Los conceptos clave son:

  • un objeto agrupa propiedades y métodos;
  • un array de objetos permite trabajar con listas de entidades complejas;
  • map transforma;
  • filter selecciona;
  • reduce acumula;
  • find busca un elemento;
  • some comprueba si alguno cumple una condición;
  • every comprueba si todos cumplen una condición;
  • sort ordena, pero modifica el array;
  • destructuring, spread y rest ayudan a escribir código más claro;
  • JSON permite convertir datos de JavaScript en texto y recuperarlos después.
