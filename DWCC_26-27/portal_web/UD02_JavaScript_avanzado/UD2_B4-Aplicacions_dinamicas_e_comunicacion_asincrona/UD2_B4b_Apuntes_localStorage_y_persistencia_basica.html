UD2_B4b. Apuntes: localStorage y persistencia básica en JavaScript

1. Objetivos y concepto de persistencia

1.1. Qué vamos a aprender

En los documentos anteriores hemos trabajado con:

  • variables;
  • arrays;
  • objetos;
  • JSON;
  • formularios;
  • eventos;
  • manipulación del DOM.

Estos datos suelen mantenerse únicamente mientras la página está abierta. Si recargamos el documento, las variables vuelven a crearse y su contenido anterior se pierde.

En este documento aprenderemos a utilizar localStorage para conservar datos sencillos en el navegador.

Al finalizar deberías ser capaz de:

  • explicar qué significa persistir datos;
  • distinguir entre estado temporal y estado persistido;
  • guardar datos con localStorage.setItem();
  • recuperarlos con localStorage.getItem();
  • eliminar claves con removeItem();
  • utilizar clear() con precaución;
  • comprender que localStorage siempre almacena texto;
  • convertir números y booleanos;
  • guardar arrays y objetos mediante JSON.stringify();
  • recuperar datos mediante JSON.parse();
  • tratar claves inexistentes, JSON incorrecto y formatos inesperados;
  • aplicar persistencia a preferencias y pequeñas listas;
  • diferenciar localStorage y sessionStorage;
  • identificar limitaciones y riesgos de seguridad.

1.2. Datos temporales y persistentes

Una variable mantiene su valor mientras se ejecuta la aplicación:

let nombre = "Ana";

console.log(nombre);

Si recargamos la página, el programa vuelve a empezar y la variable se crea de nuevo.

En cambio, un dato guardado en localStorage puede seguir disponible después de:

  • recargar la página;
  • cerrar la pestaña;
  • cerrar y volver a abrir el navegador;
  • regresar más tarde a la misma aplicación.

Ejemplos de uso:

  • recordar el tema claro u oscuro;
  • conservar una lista de tareas;
  • guardar preferencias de visualización;
  • mantener el borrador de un formulario sencillo;
  • recordar filtros o configuraciones de una práctica.

1.3. Persistir no significa guardar para siempre

localStorage no tiene una caducidad automática ordinaria, pero sus datos pueden desaparecer si:

  • el usuario borra los datos del sitio;
  • el navegador aplica una política de limpieza;
  • se utiliza navegación privada y se cierra la sesión privada;
  • se cambia de navegador, perfil o dispositivo;
  • se accede desde otro origen;
  • el almacenamiento está bloqueado.

Por tanto, localStorage permite conservar datos entre sesiones, pero no debe considerarse una copia permanente ni una base de datos segura.

2. Alcance y funcionamiento de localStorage

2.1. Qué es localStorage

localStorage es una API del navegador que permite almacenar pares clave-valor.

localStorage.setItem(
  "usuario",
  "Ana"
);

En este ejemplo:

  • "usuario" es la clave;
  • "Ana" es el valor.

Para recuperar el dato:

const usuario =
  localStorage.getItem(
    "usuario"
  );

console.log(usuario);

2.2. El almacenamiento pertenece al origen

Los datos de localStorage están asociados al origen de la página.

De forma simplificada, un origen depende de:

  • protocolo;
  • host;
  • puerto.

Estas direcciones pueden tener almacenamientos diferentes:

http://localhost:5500
http://localhost:3000
https://localhost:5500

Las páginas que comparten origen comparten también el mismo localStorage.

Por eso conviene evitar claves demasiado genéricas.

En lugar de:

const CLAVE_TEMA =
  "tema";

es preferible:

const CLAVE_TEMA =
  "dwcc.ud2.tema";

const CLAVE_TAREAS =
  "dwcc.ud2.tareas";

El prefijo ayuda a identificar qué aplicación creó cada dato y reduce el riesgo de colisiones.

2.3. localStorage es síncrono

Las operaciones de localStorage son síncronas:

localStorage.setItem(
  "clave",
  "valor"
);

localStorage.getItem(
  "clave"
);

Esto significa que se ejecutan inmediatamente y bloquean la continuación del JavaScript hasta terminar.

Comparación:

fetch()       → operación asíncrona
localStorage  → operación síncrona

Para cantidades pequeñas de datos no suele suponer un problema.

No debe utilizarse para:

  • grandes volúmenes de información;
  • escrituras continuas;
  • datos complejos de gran tamaño;
  • sustituir una base de datos.

Para almacenamientos más avanzados existe IndexedDB, pero no será contenido principal en este bloque.

2.4. Inspeccionar el almacenamiento

Puedes revisar los datos guardados desde las herramientas de desarrollo del navegador.

En Chrome suele encontrarse en:

Application
└── Storage
    └── Local Storage

El nombre exacto puede variar según el navegador.

3. Operaciones básicas

3.1. Guardar con setItem()

Sintaxis:

localStorage.setItem(
  clave,
  valor
);

Ejemplo:

localStorage.setItem(
  "dwcc.ud2.curso",
  "DWCC"
);

localStorage.setItem(
  "dwcc.ud2.idioma",
  "es"
);

Si se utiliza una clave que ya existe, su valor se sustituye:

localStorage.setItem(
  "dwcc.ud2.curso",
  "DWCC"
);

localStorage.setItem(
  "dwcc.ud2.curso",
  "DIW"
);

El valor final será:

DIW

3.2. Recuperar con getItem()

const curso =
  localStorage.getItem(
    "dwcc.ud2.curso"
  );

console.log(curso);

Si la clave no existe, getItem() devuelve:

null

Por eso debemos comprobar su existencia:

const nombre =
  localStorage.getItem(
    "dwcc.ud2.nombre"
  );

if (nombre !== null) {
  console.log(
    "Hola, " + nombre
  );
} else {
  console.log(
    "No hay nombre guardado."
  );
}

3.3. Eliminar una clave con removeItem()

localStorage.removeItem(
  "dwcc.ud2.usuario"
);

Después:

const usuario =
  localStorage.getItem(
    "dwcc.ud2.usuario"
  );

console.log(usuario);

El resultado será:

null

3.4. Eliminar todas las claves con clear()

localStorage.clear();

clear() elimina todas las claves de localStorage del origen actual, no solo las creadas por una práctica concreta.

Por eso, normalmente es preferible:

localStorage.removeItem(
  "dwcc.ud2.tareas"
);

frente a:

localStorage.clear();

En prácticas de aula, utiliza clear() únicamente cuando quieras borrar conscientemente todos los datos del origen.

4. Cadenas, números, booleanos y valores ausentes

4.1. localStorage siempre almacena texto

Aunque guardemos un número:

localStorage.setItem(
  "dwcc.ud2.edad",
  20
);

al recuperarlo obtenemos una cadena:

const edad =
  localStorage.getItem(
    "dwcc.ud2.edad"
  );

console.log(edad);
console.log(typeof edad);

Resultado:

20
string

4.2. Recuperar números de forma segura

No conviene convertir directamente sin comprobar si la clave existe:

const edad =
  Number(
    localStorage.getItem(
      "dwcc.ud2.edad"
    )
  );

Si la clave no existe:

Number(null);

produce:

0

Una forma más segura:

const edadGuardada =
  localStorage.getItem(
    "dwcc.ud2.edad"
  );

let edad = null;

if (edadGuardada !== null) {
  const numero =
    Number(edadGuardada);

  if (Number.isFinite(numero)) {
    edad = numero;
  }
}

console.log(edad);

Así diferenciamos:

  • una edad válida;
  • un valor numérico incorrecto;
  • la ausencia de la clave.

4.3. Guardar y recuperar booleanos

Al guardar un booleano:

localStorage.setItem(
  "dwcc.ud2.acepta",
  true
);

se almacena la cadena:

true

Conversión sencilla:

const acepta =
  localStorage.getItem(
    "dwcc.ud2.acepta"
  ) === "true";

Esta expresión devuelve false tanto si se guardó "false" como si la clave no existe.

Para distinguir los tres estados:

const aceptaGuardado =
  localStorage.getItem(
    "dwcc.ud2.acepta"
  );

let acepta = null;

if (aceptaGuardado !== null) {
  acepta =
    aceptaGuardado === "true";
}

Ahora acepta puede ser:

  • true;
  • false;
  • null, si todavía no se guardó ninguna decisión.

4.4. Clave inexistente y valor vacío

No es lo mismo:

localStorage.setItem(
  "dwcc.ud2.nombre",
  ""
);

que eliminar la clave:

localStorage.removeItem(
  "dwcc.ud2.nombre"
);

En el primer caso:

localStorage.getItem(
  "dwcc.ud2.nombre"
);

devuelve una cadena vacía.

En el segundo devuelve:

null

5. Objetos y arrays mediante JSON

5.1. Guardar objetos

No debemos guardar un objeto directamente:

const usuario = {
  nombre: "Ana",
  edad: 20
};

localStorage.setItem(
  "dwcc.ud2.usuario",
  usuario
);

El valor resultante sería:

[object Object]

Debemos convertir el objeto a JSON:

const usuario = {
  nombre: "Ana",
  edad: 20
};

const textoJSON =
  JSON.stringify(usuario);

localStorage.setItem(
  "dwcc.ud2.usuario",
  textoJSON
);

5.2. Recuperar objetos

const textoGuardado =
  localStorage.getItem(
    "dwcc.ud2.usuario"
  );

if (textoGuardado !== null) {
  const usuario =
    JSON.parse(textoGuardado);

  console.log(usuario.nombre);
  console.log(usuario.edad);
}

5.3. Guardar arrays

const tareas = [
  "Estudiar",
  "Practicar JavaScript",
  "Repasar DOM"
];

localStorage.setItem(
  "dwcc.ud2.tareas",
  JSON.stringify(tareas)
);

5.4. Recuperar arrays

const textoTareas =
  localStorage.getItem(
    "dwcc.ud2.tareas"
  );

let tareas = [];

if (textoTareas !== null) {
  tareas =
    JSON.parse(textoTareas);
}

Este ejemplo funciona si el texto guardado contiene JSON válido y representa realmente un array.

En la siguiente sección aprenderemos una recuperación más segura.

6. Recuperación segura y tratamiento de errores

6.1. Qué ocurre con JSON.parse(null)

JSON.parse(null);

devuelve:

null

No lanza una excepción por sí mismo.

El problema aparece después si tratamos el resultado como objeto:

const usuario =
  JSON.parse(
    localStorage.getItem(
      "dwcc.ud2.usuario"
    )
  );

console.log(usuario.nombre);

Si la clave no existe:

usuario === null

y acceder a usuario.nombre provoca un error.

6.2. JSON incorrecto o manipulado

El contenido del almacenamiento puede ser:

  • antiguo;
  • incorrecto;
  • manipulado desde DevTools;
  • incompatible con la versión actual de la aplicación.

Ejemplo de texto JSON incompleto:

[{"texto":"Estudiar"

En ese caso:

JSON.parse(texto);

lanza un SyntaxError.

La expresión:

const tareas =
  JSON.parse(
    localStorage.getItem(
      "dwcc.ud2.tareas"
    )
  ) || [];

no evita ese error, porque JSON.parse() falla antes de poder aplicar || [].

6.3. Función segura para recuperar un array

const CLAVE_TAREAS =
  "dwcc.ud2.tareas";

function recuperarTareas() {
  const texto =
    localStorage.getItem(
      CLAVE_TAREAS
    );

  if (texto === null) {
    return [];
  }

  try {
    const datos =
      JSON.parse(texto);

    if (!Array.isArray(datos)) {
      console.warn(
        "El dato guardado no es un array."
      );

      return [];
    }

    return datos;
  } catch (error) {
    console.error(
      "Las tareas guardadas no son válidas:",
      error
    );

    return [];
  }
}

Esta función trata:

  • una clave inexistente;
  • JSON incorrecto;
  • un tipo de dato inesperado;
  • un array correcto.

6.4. Guardar JSON de forma segura

setItem() puede fallar, por ejemplo:

  • por falta de espacio;
  • porque el almacenamiento esté bloqueado;
  • por una política del navegador;
  • por un contexto de seguridad incompatible.
function guardarTareas(tareas) {
  try {
    localStorage.setItem(
      CLAVE_TAREAS,
      JSON.stringify(tareas)
    );

    return true;
  } catch (error) {
    console.error(
      "No se pudieron guardar las tareas:",
      error
    );

    return false;
  }
}

La función devuelve:

  • true, si pudo guardar;
  • false, si se produjo un error.

6.5. Funciones genéricas para JSON

Podemos crear funciones reutilizables:

function guardarJSON(
  clave,
  datos
) {
  try {
    const texto =
      JSON.stringify(datos);

    localStorage.setItem(
      clave,
      texto
    );

    return true;
  } catch (error) {
    console.error(
      `No se pudo guardar ${clave}:`,
      error
    );

    return false;
  }
}

function recuperarJSON(
  clave,
  valorInicial
) {
  const texto =
    localStorage.getItem(clave);

  if (texto === null) {
    return valorInicial;
  }

  try {
    return JSON.parse(texto);
  } catch (error) {
    console.error(
      `No se pudo recuperar ${clave}:`,
      error
    );

    return valorInicial;
  }
}

Estas funciones no sustituyen la validación de tipos.

Después de recuperar debemos comprobar:

const tareasRecuperadas =
  recuperarJSON(
    CLAVE_TAREAS,
    []
  );

const tareas =
  Array.isArray(
    tareasRecuperadas
  )
    ? tareasRecuperadas
    : [];

7. Ejemplo completo: preferencia de tema

7.1. HTML

<button
  type="button"
  id="tema-claro"
>
  Tema claro
</button>

<button
  type="button"
  id="tema-oscuro"
>
  Tema oscuro
</button>

<p
  id="estado-tema"
  aria-live="polite"
></p>

7.2. CSS básico

.tema-claro {
  color: #222222;
  background-color: #ffffff;
}

.tema-oscuro {
  color: #f5f5f5;
  background-color: #1f1f1f;
}

7.3. JavaScript

const CLAVE_TEMA =
  "dwcc.ud2.tema";

const TEMAS_VALIDOS = [
  "tema-claro",
  "tema-oscuro"
];

const botonClaro =
  document.querySelector(
    "#tema-claro"
  );

const botonOscuro =
  document.querySelector(
    "#tema-oscuro"
  );

const estadoTema =
  document.querySelector(
    "#estado-tema"
  );

function aplicarTema(tema) {
  if (
    !TEMAS_VALIDOS.includes(tema)
  ) {
    return;
  }

  document.body.classList.remove(
    ...TEMAS_VALIDOS
  );

  document.body.classList.add(
    tema
  );
}

function guardarTema(tema) {
  aplicarTema(tema);

  try {
    localStorage.setItem(
      CLAVE_TEMA,
      tema
    );

    estadoTema.textContent =
      "Preferencia guardada.";
  } catch (error) {
    estadoTema.textContent =
      "El tema se aplicó, pero no pudo guardarse.";

    console.error(error);
  }
}

const temaGuardado =
  localStorage.getItem(
    CLAVE_TEMA
  );

if (
  temaGuardado !== null
  && TEMAS_VALIDOS.includes(
    temaGuardado
  )
) {
  aplicarTema(temaGuardado);
}

botonClaro.addEventListener(
  "click",
  () => {
    guardarTema("tema-claro");
  }
);

botonOscuro.addEventListener(
  "click",
  () => {
    guardarTema("tema-oscuro");
  }
);

7.4. Por qué no usamos body.className

Evita:

document.body.className =
  temaGuardado;

Esta instrucción sustituye todas las clases del elemento <body>.

Con classList podemos eliminar únicamente las clases de tema y conservar el resto.

8. Ejemplo completo: lista de tareas persistente

8.1. Flujo de una aplicación persistente

El ciclo recomendado es:

  1. recuperar el estado guardado;
  2. validarlo;
  3. renderizarlo;
  4. modificarlo como respuesta a un evento;
  5. guardar el nuevo estado;
  6. volver a renderizar.

Debemos diferenciar:

  • estado en memoria: el array tareas;
  • estado persistido: el JSON de localStorage;
  • interfaz visible: los elementos del DOM.

8.2. HTML

<form id="formulario-tarea">
  <label for="texto-tarea">
    Nueva tarea:
  </label>

  <input
    type="text"
    id="texto-tarea"
    name="texto-tarea"
  >

  <button type="submit">
    Añadir tarea
  </button>
</form>

<p
  id="estado-tareas"
  aria-live="polite"
></p>

<ul id="lista-tareas"></ul>

8.3. Estado inicial

const CLAVE_TAREAS =
  "dwcc.ud2.tareas";

let tareas =
  recuperarTareas();

No añadimos tareas automáticamente al cargar la página.

Este patrón sería incorrecto:

agregarTarea(
  "Estudiar localStorage"
);

agregarTarea(
  "Practicar JSON"
);

Si se ejecuta al cargar el script, las tareas se duplicarán después de cada recarga.

8.4. Renderizar las tareas

const listaTareas =
  document.querySelector(
    "#lista-tareas"
  );

function renderizarTareas() {
  listaTareas.replaceChildren();

  tareas.forEach((tarea) => {
    const li =
      document.createElement("li");

    li.textContent =
      tarea.texto;

    listaTareas.append(li);
  });
}

8.5. Añadir una tarea

const formulario =
  document.querySelector(
    "#formulario-tarea"
  );

const campoTarea =
  document.querySelector(
    "#texto-tarea"
  );

const estadoTareas =
  document.querySelector(
    "#estado-tareas"
  );

formulario.addEventListener(
  "submit",
  (event) => {
    event.preventDefault();

    const texto =
      campoTarea.value.trim();

    if (texto === "") {
      estadoTareas.textContent =
        "Escribe una tarea.";

      return;
    }

    const nuevaTarea = {
      id: Date.now(),
      texto
    };

    tareas.push(nuevaTarea);

    const guardado =
      guardarTareas(tareas);

    if (!guardado) {
      tareas.pop();

      estadoTareas.textContent =
        "No se pudo guardar la tarea.";

      return;
    }

    renderizarTareas();

    formulario.reset();
    campoTarea.focus();

    estadoTareas.textContent =
      "Tarea añadida.";
  }
);

8.6. Inicio de la aplicación

renderizarTareas();

Al recargar:

  1. recuperarTareas() obtiene el array;
  2. renderizarTareas() reconstruye la lista;
  3. no se añaden elementos duplicados.

9. localStorage y sessionStorage

9.1. Comparación básica

API Comportamiento
localStorage Se conserva entre sesiones hasta que se elimina o el navegador lo limpia.
sessionStorage Se conserva durante la sesión de una pestaña, incluidas sus recargas, y normalmente se elimina al cerrarla.

9.2. Ejemplo con sessionStorage

sessionStorage.setItem(
  "dwcc.ud2.pasoActual",
  "2"
);

const paso =
  sessionStorage.getItem(
    "dwcc.ud2.pasoActual"
  );

La API es similar:

sessionStorage.setItem();
sessionStorage.getItem();
sessionStorage.removeItem();
sessionStorage.clear();

9.3. Cuándo usar cada una

Usa localStorage para:

  • preferencias que deben conservarse;
  • pequeñas listas;
  • configuraciones sencillas;
  • borradores que deban seguir disponibles.

Usa sessionStorage para:

  • información temporal de una pestaña;
  • pasos de un proceso corto;
  • filtros que solo interesan durante la sesión;
  • estados que no deben conservarse al cerrar la pestaña.

10. Seguridad y datos que no deben guardarse

10.1. localStorage no es un almacén seguro

Los datos:

  • no están cifrados automáticamente;
  • pueden inspeccionarse desde DevTools;
  • pueden ser leídos por JavaScript ejecutado en el mismo origen;
  • permanecen en el dispositivo hasta que se eliminan.

No guardes:

  • contraseñas;
  • tokens de autenticación;
  • identificadores secretos de sesión;
  • datos bancarios;
  • información médica;
  • datos personales sensibles;
  • información que no debería quedar almacenada en el equipo.

10.2. Usos apropiados

Es adecuado para:

  • tema claro u oscuro;
  • preferencias de visualización;
  • filtros;
  • pequeñas listas de una práctica;
  • borradores no sensibles;
  • configuraciones locales.

10.3. Datos externos y validación

No confíes automáticamente en el contenido recuperado.

Aunque lo haya creado la propia aplicación, el dato puede:

  • estar manipulado;
  • pertenecer a una versión antigua;
  • tener un formato incorrecto;
  • contener propiedades inesperadas.

Después de JSON.parse() debemos validar:

if (!Array.isArray(datos)) {
  return [];
}

y, cuando sea necesario, comprobar las propiedades de cada elemento.

11. Errores frecuentes y buenas prácticas

11.1. Olvidar que todo se almacena como texto

Incorrecto:

localStorage.setItem(
  "dwcc.ud2.contador",
  5
);

const contador =
  localStorage.getItem(
    "dwcc.ud2.contador"
  );

console.log(contador + 1);

Resultado:

51

Corrección:

const textoContador =
  localStorage.getItem(
    "dwcc.ud2.contador"
  );

if (textoContador !== null) {
  const contador =
    Number(textoContador);

  if (
    Number.isFinite(contador)
  ) {
    console.log(
      contador + 1
    );
  }
}

11.2. Guardar objetos sin JSON.stringify()

Incorrecto:

localStorage.setItem(
  "dwcc.ud2.usuario",
  {
    nombre: "Ana"
  }
);

Correcto:

localStorage.setItem(
  "dwcc.ud2.usuario",
  JSON.stringify({
    nombre: "Ana"
  })
);

11.3. Recuperar JSON sin tratar errores

Problemático:

const tareas =
  JSON.parse(
    localStorage.getItem(
      "dwcc.ud2.tareas"
    )
  );

Mejor:

const tareas =
  recuperarTareas();

11.4. No comprobar el tipo recuperado

El JSON puede representar:

  • un array;
  • un objeto;
  • una cadena;
  • un número;
  • null.

Por eso:

const datos =
  JSON.parse(texto);

debe ir acompañado de una validación:

if (!Array.isArray(datos)) {
  return [];
}

11.5. Usar clear() sin pensar

localStorage.clear();

puede borrar claves de otras partes de la misma aplicación.

Es más seguro:

localStorage.removeItem(
  "dwcc.ud2.tareas"
);

11.6. Utilizar claves genéricas

Evita:

"tema"
"datos"
"usuario"

Prefiere:

"dwcc.ud2.tema"
"dwcc.ud2.tareas"
"dwcc.ud2.usuario"

11.7. Duplicar datos al iniciar

No añadas elementos de demostración cada vez que se carga script.js.

Incorrecto:

agregarTarea("Estudiar");

si se ejecuta siempre al iniciar.

Los datos iniciales solo deberían crearse si todavía no existe almacenamiento y la aplicación necesita realmente valores de ejemplo.

11.8. Sobrescribir todas las clases del documento

Evita:

document.body.className =
  tema;

Utiliza:

document.body.classList.remove(
  "tema-claro",
  "tema-oscuro"
);

document.body.classList.add(
  tema
);

11.9. Ignorar errores de escritura

No des por hecho que setItem() siempre funciona.

Utiliza try...catch cuando el guardado sea importante:

try {
  localStorage.setItem(
    clave,
    valor
  );
} catch (error) {
  console.error(error);
}

11.10. Guardar demasiados datos

localStorage está pensado para cantidades pequeñas.

No debe utilizarse como sustituto de:

  • una API;
  • una base de datos;
  • un sistema de archivos;
  • un almacén de grandes colecciones.

12.2. Resumen final

  • localStorage conserva datos sencillos entre recargas y sesiones.
  • El almacenamiento pertenece al origen de la página.
  • Protocolo, host y puerto influyen en el origen.
  • localStorage es una API síncrona.
  • Todas las claves y valores se almacenan como texto.
  • setItem() guarda o sustituye un valor.
  • getItem() recupera un valor y devuelve null si no existe.
  • removeItem() elimina una clave concreta.
  • clear() elimina todas las claves del origen actual.
  • Los números y booleanos necesitan conversión.
  • Los objetos y arrays requieren JSON.stringify() y JSON.parse().
  • JSON.parse() puede fallar si el texto no es JSON válido.
  • El tipo de dato recuperado debe validarse.
  • setItem() también puede producir errores.
  • Una aplicación persistente debe recuperar, renderizar, modificar, guardar y volver a renderizar.
  • sessionStorage conserva datos durante la sesión de una pestaña.
  • localStorage no debe utilizarse para secretos ni información sensible.
