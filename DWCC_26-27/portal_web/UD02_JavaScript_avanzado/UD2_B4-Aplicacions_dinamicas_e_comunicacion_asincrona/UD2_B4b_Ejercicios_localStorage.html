UD2_B4b. Ejercicios: localStorage y persistencia básica

1. Objetivos, entorno y claves

1.1. Objetivo de la hoja

En esta hoja practicarás el uso de localStorage para conservar datos sencillos en el navegador.

Trabajarás con:

  • localStorage.setItem();
  • localStorage.getItem();
  • localStorage.removeItem();
  • localStorage.clear();
  • claves con prefijo;
  • alcance por origen;
  • conversión de números y booleanos;
  • JSON.stringify();
  • JSON.parse();
  • recuperación segura;
  • validación con Array.isArray();
  • objetos y arrays persistentes;
  • integración con DOM y eventos;
  • sessionStorage;
  • seguridad y tratamiento de errores.

2. Guardar, recuperar y borrar textos

Ejercicio 2. Guardar un nombre

Al pulsar el botón Guardar:

  1. recupera el contenido del campo;
  2. elimina espacios iniciales y finales;
  3. comprueba que no esté vacío;
  4. guarda el nombre;
  5. actualiza el mensaje.

Estructura orientativa:

const campoNombre =
  document.querySelector(
    "#campo-nombre"
  );

const estadoNombre =
  document.querySelector(
    "#estado-nombre"
  );

function guardarNombre() {
  const nombre =
    campoNombre.value.trim();

  if (nombre === "") {
    estadoNombre.textContent =
      "Escribe un nombre.";

    return;
  }

  localStorage.setItem(
    CLAVE_NOMBRE,
    nombre
  );

  estadoNombre.textContent =
    `Nombre guardado: ${nombre}`;
}

Ejercicio 3. Recuperar el nombre al cargar

Crea:

function recuperarNombre() {
  const nombre =
    localStorage.getItem(
      CLAVE_NOMBRE
    );

  if (nombre === null) {
    estadoNombre.textContent =
      "Todavía no hay nombre guardado.";

    return;
  }

  campoNombre.value = nombre;

  estadoNombre.textContent =
    `Bienvenido/a de nuevo, ${nombre}.`;
}

Llama a la función al iniciar el script.

Ejercicio 4. Borrar el nombre

Al pulsar Borrar nombre:

  1. elimina la clave;
  2. limpia el campo;
  3. actualiza el mensaje.
function borrarNombre() {
  localStorage.removeItem(
    CLAVE_NOMBRE
  );

  campoNombre.value = "";

  estadoNombre.textContent =
    "El nombre se ha eliminado.";
}

3. Números y booleanos

Ejercicio 6. Contador de cargas de la página

Este ejercicio cuenta cargas, no visitantes únicos.

Proceso:

  1. recupera el valor;
  2. comprueba si existe;
  3. conviértelo a número;
  4. valida el resultado;
  5. incrementa;
  6. guarda;
  7. muestra el contador.
const textoVisitas =
  localStorage.getItem(
    CLAVE_VISITAS
  );

let visitas = 0;

if (textoVisitas !== null) {
  const valor =
    Number(textoVisitas);

  if (Number.isFinite(valor)) {
    visitas = valor;
  }
}

visitas += 1;

localStorage.setItem(
  CLAVE_VISITAS,
  visitas
);

console.log(
  `Esta página se ha cargado ${visitas} veces.`
);

Responde:

  1. ¿Qué resultado produce Number(null)?
  2. ¿Por qué se comprueba primero null?
  3. ¿Qué ocurriría si el valor guardado fuese "abc"?

Ejercicio 11. Recuperar el perfil al cargar

Debes comprobar:

  1. si existe la clave;
  2. si el JSON es válido;
  3. si el resultado es un objeto;
  4. si contiene las propiedades esperadas.

Estructura:

function recuperarPerfil() {
  const texto =
    localStorage.getItem(
      CLAVE_PERFIL
    );

  if (texto === null) {
    return null;
  }

  try {
    const perfil =
      JSON.parse(texto);

    if (
      typeof perfil !== "object"
      || perfil === null
      || Array.isArray(perfil)
    ) {
      return null;
    }

    return perfil;
  } catch (error) {
    console.error(
      "Perfil no válido:",
      error
    );

    return null;
  }
}

5. Arrays y recuperación segura

Ejercicio 13. Analizar un patrón inseguro

Explica por qué esta expresión no es completamente segura:

const tareas =
  JSON.parse(
    localStorage.getItem(
      CLAVE_TAREAS
    )
  ) || [];

Responde:

  1. ¿Qué ocurre si la clave no existe?
  2. ¿Qué ocurre si el texto no es JSON válido?
  3. ¿Qué ocurre si el JSON contiene un objeto?
  4. ¿Por qué || [] no captura un SyntaxError?

Ejercicio 14. Crear recuperarTareas()

function recuperarTareas() {
  const texto =
    localStorage.getItem(
      CLAVE_TAREAS
    );

  if (texto === null) {
    return [];
  }

  try {
    const datos =
      JSON.parse(texto);

    if (!Array.isArray(datos)) {
      console.warn(
        "El dato guardado no es un array."
      );

      return [];
    }

    return datos;
  } catch (error) {
    console.error(
      "Las tareas no son válidas:",
      error
    );

    return [];
  }
}

Prueba:

localStorage.setItem(
  CLAVE_TAREAS,
  "{texto incorrecto"
);

y después:

localStorage.setItem(
  CLAVE_TAREAS,
  JSON.stringify({
    tareas: []
  })
);

Ejercicio 15. Crear guardarJSON()

function guardarJSON(
  clave,
  datos
) {
  try {
    localStorage.setItem(
      clave,
      JSON.stringify(datos)
    );

    return true;
  } catch (error) {
    console.error(
      `No se pudo guardar ${clave}:`,
      error
    );

    return false;
  }
}

Explica:

  1. ¿Por qué devuelve un booleano?
  2. ¿Qué podría provocar un error de escritura?
  3. ¿Por qué no debe darse por hecho que el guardado siempre funciona?

11. Mini-entrega y lista de comprobación

11.1. Aplicación de preferencias y tareas

Crea una aplicación que incluya:

  • nombre persistente;
  • tema claro u oscuro;
  • tamaño de texto;
  • lista de tareas persistente;
  • eliminación individual de tareas;
  • botón para vaciar tareas;
  • botón para borrar únicamente los datos de la aplicación.

11.2. Requisitos técnicos

La aplicación debe:

  1. usar claves con prefijo;
  2. declarar constantes para las claves;
  3. validar cadenas vacías;
  4. comprobar null;
  5. convertir y validar números;
  6. diferenciar false de una clave inexistente;
  7. utilizar JSON.stringify();
  8. proteger JSON.parse() con try...catch;
  9. comprobar Array.isArray();
  10. utilizar tareas como objetos con identificador;
  11. usar createElement();
  12. usar textContent;
  13. usar append();
  14. usar replaceChildren();
  15. validar temas y tamaños;
  16. tratar errores de escritura;
  17. eliminar solo las claves propias;
  18. no guardar datos sensibles;
  19. organizar el código en funciones;
  20. no mostrar errores en consola durante el uso normal.
