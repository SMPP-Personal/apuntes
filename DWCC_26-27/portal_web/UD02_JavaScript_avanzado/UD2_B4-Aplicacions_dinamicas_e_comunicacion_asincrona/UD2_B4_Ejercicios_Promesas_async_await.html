UD2_B4. Ejercicios: promesas y async/await

1. Objetivo de la hoja

En esta hoja vas a practicar la asincronía en JavaScript antes de trabajar con fetch y datos externos.

El objetivo no es memorizar la sintaxis, sino entender qué ocurre cuando una operación no termina inmediatamente y cómo podemos gestionar su resultado de forma ordenada.

Al finalizar deberías ser capaz de:

  • distinguir código síncrono y código asíncrono;
  • crear funciones que devuelvan promesas sencillas;
  • consumir promesas con .then(), .catch() y .finally();
  • escribir funciones async;
  • usar await para esperar el resultado de una promesa;
  • capturar errores con try...catch;
  • transformar código con .then() a código con async/await;
  • razonar el orden de ejecución de un programa asíncrono.

2. Antes de empezar

Trabaja en un archivo llamado:

UD2_B4_ejercicios_promesas_async_await.js

Puedes ejecutar los ejercicios:

  • en la consola del navegador;
  • en un archivo .js enlazado desde una página HTML;
  • o con Node.js, si ya lo tienes preparado.

En esta hoja no se usa todavía fetch. Vamos a simular operaciones asíncronas con setTimeout() y promesas para entender bien la base antes de consumir datos reales.

Cómo trabajar esta hoja

La hoja contiene muchos ejercicios porque sirve como práctica completa del bloque. Si se trabaja en clase y no da tiempo a completarla entera, prioriza estos ejercicios:

  • Básicos imprescindibles: 1, 2, 3, 4, 5, 7, 8, 11, 12, 14, 15, 21, 22, 23 y 24.
  • Refuerzo recomendado: 6, 9, 10, 16, 17, 18, 19 y 20.
  • Entrega final: 25.

La idea es que primero entiendas el orden de ejecución y el funcionamiento de las promesas, y después practiques la forma moderna de escribir código asíncrono con async/await.

3. Recordatorio rápido

3.1. Promesa resuelta

const promesa = new Promise((resolve, reject) => {
  resolve("Operación completada");
});

promesa.then((resultado) => {
  console.log(resultado);
});

3.2. Promesa rechazada

const promesa = new Promise((resolve, reject) => {
  reject("Ha ocurrido un error");
});

promesa.catch((error) => {
  console.log(error);
});

3.3. Función async con await

function esperar(ms) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve("Tiempo finalizado");
    }, ms);
  });
}

async function ejecutar() {
  const resultado = await esperar(1000);
  console.log(resultado);
}

ejecutar();

Ejercicio 5. Promesa con retraso

  console.log("Han pasado 2 segundos");

function esperar(ms) {
  return new Promise((resolve) => {

      resolve();
    }, ms);

}

Ejercicio 9. Preparar, procesar y finalizar

Crea tres funciones que devuelvan promesas:

  • "Datos procesados";
  • "Proceso finalizado".

  })
  .then((mensaje) => {
    console.log(mensaje);
  })

    console.log(error);
  });

Proceso finalizado

Ejercicio 14. Capturar un error con try...catch

Usa la función:

descargarArchivo(nombreArchivo, disponible)

Crea una función async llamada probarDescarga() que:

  1. intente descargar "apuntes.pdf" con disponible = true;
  2. muestre el resultado;
  3. capture posibles errores con catch.

Después prueba también el caso de error modificando la llamada dentro de la función async. No ejecutes una promesa rechazada suelta sin capturar el error.

Ejemplo orientativo:

async function probarDescarga() {
  try {
    const mensaje = await descargarArchivo("apuntes.pdf", true);
    console.log(mensaje);

    console.log("Error capturado:", error);

5. Errores frecuentes

Error Por qué ocurre Cómo corregirlo
Usar await fuera de una función async await necesita un contexto asíncrono Crear una función async
Olvidar return al crear una promesa dentro de una función La función no devuelve la promesa Escribir return new Promise(...)
Pensar que una promesa contiene ya el resultado La promesa representa un resultado futuro Usar .then() o await
No capturar errores Si la promesa se rechaza, el error queda sin tratar Usar .catch() o try...catch
Confundir .then() con await Son dos formas distintas de consumir promesas Elegir una y usarla de forma coherente
Poner demasiada lógica dentro de .then() El código se vuelve difícil de leer Usar funciones auxiliares o async/await
Usar try...catch pero olvidarse de await El error de la promesa puede no capturarse como se espera Usar await dentro del try

7. Resumen final

En esta hoja has practicado la base de la asincronía en JavaScript:

  • una promesa representa una operación que puede completarse más adelante;
  • resolve indica que la operación ha ido bien;
  • reject indica que ha ocurrido un error;
  • .then() permite trabajar con el resultado correcto;
  • .catch() permite capturar errores;
  • .finally() se ejecuta siempre;
  • async permite declarar funciones asíncronas;
  • await permite esperar el resultado de una promesa;
  • try...catch permite capturar errores dentro de funciones async.

Estos conceptos son imprescindibles antes de trabajar con fetch y datos JSON externos.
