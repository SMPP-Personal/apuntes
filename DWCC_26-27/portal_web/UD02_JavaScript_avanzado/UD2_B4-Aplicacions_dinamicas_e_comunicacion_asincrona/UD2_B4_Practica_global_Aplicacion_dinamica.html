UD2_B4. Práctica global: aplicación dinámica con fetch, JSON y DOM

Práctica global de cierre del bloque UD2_B4.

1. Sentido de la práctica

Esta práctica cierra el bloque UD2_B4: Aplicaciones dinámicas y comunicación asíncrona.

El objetivo es construir una pequeña aplicación web que cargue datos desde un fichero JSON usando fetch, los procese con JavaScript y los muestre dinámicamente en la página mediante el DOM.

La práctica integra contenidos trabajados en la UD2:

  • objetos y arrays de objetos;
  • métodos de array (forEach, map, filter, reduce, find, some, every, sort);
  • DOM;
  • eventos;
  • formularios sencillos;
  • validación básica;
  • asincronía;
  • promesas;
  • async / await;
  • try...catch;
  • fetch;
  • lectura de JSON con .json();
  • estados de carga y error.

No se trabajará todavía almacenamiento persistente (localStorage), autenticación, cabeceras HTTP avanzadas, AbortController, módulos ni frameworks.

Importante: esta práctica carga un fichero JSON local mediante fetch. Debe ejecutarse con Live Server o con un servidor local equivalente. Si abres el HTML directamente con doble clic, la carga del JSON puede fallar por restricciones del navegador.

2. Aplicación que se va a construir

Vas a crear una aplicación llamada:

Panel dinámico de recursos de aula

La aplicación cargará una lista de recursos desde un fichero JSON y permitirá:

  1. Mostrar todos los recursos en tarjetas.
  2. Filtrar recursos por categoría.
  3. Buscar recursos por texto.
  4. Ordenar los recursos por dificultad o duración.
  5. Mostrar estadísticas generales.
  6. Ver si hay recursos pendientes.
  7. Mostrar mensajes de carga, error, éxito y resultado vacío.
  8. Recargar los datos con un botón.
  9. Añadir visualmente un nuevo recurso desde un formulario, sin guardarlo en el fichero JSON.

El recurso añadido desde el formulario se añadirá solo al array que está en memoria mientras la página está abierta.
No se escribirá realmente dentro de datos/recursos.json.

7.2. Variables globales controladas

También puedes tener referencias a elementos del DOM:

const listaRecursos = document.querySelector("#listaRecursos");

No abuses de variables globales. Usa funciones para organizar el comportamiento.

Si algo no funciona, revisa primero que los id del HTML coinciden exactamente con los usados en querySelector().

7.3. Cargar los datos con fetch

Debes crear una función asíncrona:

async function cargarRecursos() {
  // Código aquí
}

Esta función debe:

  1. Mostrar un mensaje de carga.
  2. Llamar a fetch("datos/recursos.json").
  3. Comprobar response.ok.
  4. Convertir la respuesta con await response.json().
  5. Guardar los datos en el array recursos.
  6. Rellenar el filtro de categorías.
  7. Renderizar la página.
  8. Capturar errores con try...catch.

Estructura orientativa:

async function cargarRecursos() {
  try {
    mostrarEstado("Cargando recursos...", "cargando");

    const response = await fetch("datos/recursos.json");

    if (!response.ok) {
      throw new Error("No se pudieron cargar los datos");
    }

    recursos = await response.json();

    rellenarFiltroCategorias(recursos);
    mostrarEstado("Recursos cargados correctamente.", "ok");
    actualizarInterfaz();
  } catch (error) {
    mostrarEstado("Error: " + error.message, "error");
  }
}

Explicación importante

fetch() devuelve una promesa que se resuelve con una respuesta HTTP.

Después se comprueba:

if (!response.ok) {
  throw new Error("No se pudieron cargar los datos");
}

Solo si la respuesta es correcta se leen los datos con:

recursos = await response.json();

response.json() también devuelve una promesa. Por eso se usa await.

7.5. Renderizar tarjetas

Crea una función:

function renderizarRecursos(lista) {
  // Código aquí
}

Esta función debe:

  1. Limpiar el contenedor.
  2. Comprobar si la lista está vacía.
  3. Crear una tarjeta por cada recurso.
  4. Mostrar:
    • título;
    • categoría;
    • duración;
    • dificultad;
    • si está completado o pendiente;
    • etiquetas.

Cada tarjeta debe generarse con DOM:

  • document.createElement;
  • textContent;
  • classList;
  • append.

Evita construir toda la tarjeta con una cadena enorme de innerHTML.

Estructura orientativa:

function renderizarRecursos(lista) {
  listaRecursos.innerHTML = "";

  if (lista.length === 0) {
    listaRecursos.textContent = "No hay recursos que coincidan con los filtros.";
    mostrarEstado("No hay recursos que coincidan con los filtros.", "vacio");
    return;
  }

  lista.forEach((recurso) => {
    const tarjeta = crearTarjetaRecurso(recurso);
    listaRecursos.append(tarjeta);
  });
}

La función crearTarjetaRecurso(recurso) debe devolver una tarjeta creada con DOM.

Para mostrar etiquetas, recuerda que algunos recursos nuevos pueden tener un array vacío de etiquetas:

const etiquetas = recurso.etiquetas || [];

7.6. Crear estadísticas

Crea una función:

function renderizarEstadisticas(lista) {
  // Código aquí
}

Debe calcular y mostrar:

  • número total de recursos;
  • número de recursos completados;
  • número de recursos pendientes;
  • duración total;
  • duración media;
  • si existe algún recurso pendiente usando some;
  • recurso más largo usando reduce.

Ejemplo de ideas:

const total = lista.length;

const completados = lista.filter((recurso) => recurso.completado).length;

const duracionTotal = lista.reduce((acumulador, recurso) => {
  return acumulador + recurso.duracionMinutos;
}, 0);

Control obligatorio de lista vacía

Si la lista está vacía, no debes calcular medias ni buscar el recurso más largo.

function renderizarEstadisticas(lista) {
  if (lista.length === 0) {
    estadisticas.textContent = "No hay recursos para calcular estadísticas.";
    return;
  }

  const total = lista.length;

  const completados = lista.filter((recurso) => recurso.completado).length;
  const pendientes = lista.filter((recurso) => !recurso.completado).length;

  const duracionTotal = lista.reduce((acumulador, recurso) => {
    return acumulador + recurso.duracionMinutos;
  }, 0);

  const duracionMedia = duracionTotal / total;

  const hayPendientes = lista.some((recurso) => !recurso.completado);

  const recursoMasLargo = lista.reduce((mayor, recurso) => {
    if (recurso.duracionMinutos > mayor.duracionMinutos) {
      return recurso;
    }

    return mayor;
  });

  estadisticas.innerHTML = `
    <p>Total de recursos: ${total}</p>
    <p>Completados: ${completados}</p>
    <p>Pendientes: ${pendientes}</p>
    <p>Duración total: ${duracionTotal} minutos</p>
    <p>Duración media: ${duracionMedia.toFixed(2)} minutos</p>
    <p>¿Hay recursos pendientes?: ${hayPendientes ? "Sí" : "No"}</p>
    <p>Recurso más largo: ${recursoMasLargo.titulo}</p>
  `;
}

En este caso se usa innerHTML para construir un bloque de resumen con datos controlados. Para las tarjetas de recursos se recomienda usar creación de elementos del DOM.

7.7. Crear filtro de categorías

Al cargar los datos, debes rellenar el <select id="filtroCategoria"> con las categorías existentes.

Puedes obtenerlas recorriendo el array y evitando duplicados.

Ejemplo orientativo:

function obtenerCategorias(lista) {
  const categorias = [];

  lista.forEach((recurso) => {
    if (!categorias.includes(recurso.categoria)) {
      categorias.push(recurso.categoria);
    }
  });

  return categorias;
}

Después debes crear dinámicamente las opciones del select.

Ejemplo orientativo:

function rellenarFiltroCategorias(lista) {
  const categorias = obtenerCategorias(lista);

  filtroCategoria.innerHTML = "";

  const opcionTodas = document.createElement("option");
  opcionTodas.value = "todas";
  opcionTodas.textContent = "Todas";
  filtroCategoria.append(opcionTodas);

  categorias.forEach((categoria) => {
    const option = document.createElement("option");
    option.value = categoria;
    option.textContent = categoria;
    filtroCategoria.append(option);
  });
}

Cuando añadas un recurso desde el formulario, si aparece una categoría nueva, deberás volver a llamar a rellenarFiltroCategorias(recursos).

7.8. Aplicar búsqueda, filtro y ordenación

Crea una función:

function obtenerRecursosFiltradosYOrdenados() {
  // Código aquí
}

Debe tener en cuenta:

  • texto escrito en el campo de búsqueda;
  • categoría seleccionada;
  • orden seleccionado.

Debe devolver un nuevo array con el resultado.

La búsqueda debe funcionar, como mínimo, sobre:

  • título;
  • categoría;
  • etiquetas.

Para evitar problemas con mayúsculas y espacios, usa:

texto.trim().toLowerCase()

Importante sobre sort()

sort() modifica el array sobre el que se aplica.
Por eso no debes ordenar directamente el array recursos, porque podrías perder el orden original.

Empieza creando una copia:

let lista = [...recursos];

Después filtra y ordena esa copia:

function obtenerRecursosFiltradosYOrdenados() {
  const texto = busqueda.value.trim().toLowerCase();
  const categoriaSeleccionada = filtroCategoria.value;
  const ordenSeleccionado = orden.value;

  let lista = [...recursos];

  if (texto !== "") {
    lista = lista.filter((recurso) => {
      const etiquetas = recurso.etiquetas || [];
      const textoEtiquetas = etiquetas.join(" ").toLowerCase();

      return (
        recurso.titulo.toLowerCase().includes(texto) ||
        recurso.categoria.toLowerCase().includes(texto) ||
        textoEtiquetas.includes(texto)
      );
    });
  }

  if (categoriaSeleccionada !== "todas") {
    lista = lista.filter((recurso) => recurso.categoria === categoriaSeleccionada);
  }

  if (ordenSeleccionado === "duracionAsc") {
    lista.sort((a, b) => a.duracionMinutos - b.duracionMinutos);
  }

  if (ordenSeleccionado === "duracionDesc") {
    lista.sort((a, b) => b.duracionMinutos - a.duracionMinutos);
  }

  if (ordenSeleccionado === "dificultadAsc") {
    lista.sort((a, b) => a.dificultad - b.dificultad);
  }

  if (ordenSeleccionado === "dificultadDesc") {
    lista.sort((a, b) => b.dificultad - a.dificultad);
  }

  return lista;
}

7.9. Actualizar la interfaz

Crea una función central:

function actualizarInterfaz() {
  const lista = obtenerRecursosFiltradosYOrdenados();

  renderizarEstadisticas(lista);
  renderizarRecursos(lista);
}

Cada vez que cambie un filtro, una búsqueda o el orden, se llamará a esta función.

7.11. Formulario para añadir recurso visualmente

El formulario debe permitir añadir un nuevo recurso al array recursos y actualizar la interfaz.

No hace falta guardar el recurso en el fichero JSON. Al recargar la página o pulsar Recargar datos, se perderá.

Debes validar:

  • título obligatorio;
  • categoría obligatoria;
  • duración mayor que 0;
  • dificultad entre 1 y 3.

Cuando el formulario sea correcto:

  1. Crear un objeto recurso.
  2. Añadirlo al array recursos.
  3. Actualizar filtros si aparece una categoría nueva.
  4. Limpiar búsqueda, filtro y ordenación para que el nuevo recurso se vea.
  5. Actualizar la interfaz.
  6. Limpiar el formulario.
  7. Mostrar un mensaje de confirmación.

Modelo obligatorio del nuevo recurso

El recurso añadido desde el formulario debe tener la misma estructura básica que los recursos del JSON:

const nuevoRecurso = {
  id: generarNuevoId(),
  titulo: titulo,
  categoria: categoria,
  duracionMinutos: duracion,
  dificultad: dificultad,
  completado: false,
  etiquetas: []
};

Se usa completado: false porque un recurso recién añadido se considera pendiente.

Se usa etiquetas: [] porque el formulario no pide etiquetas, pero la aplicación puede necesitar que esa propiedad exista.

Generar un nuevo id

Puedes crear una función sencilla:

function generarNuevoId() {
  if (recursos.length === 0) {
    return 1;
  }

  const ids = recursos.map((recurso) => recurso.id);
  return Math.max(...ids) + 1;
}

Ejemplo orientativo de validación y creación

function gestionarEnvioFormulario(event) {
  event.preventDefault();

  const titulo = document.querySelector("#tituloRecurso").value.trim();
  const categoria = document.querySelector("#categoriaRecurso").value.trim();
  const duracion = Number(document.querySelector("#duracionRecurso").value);
  const dificultad = Number(document.querySelector("#dificultadRecurso").value);

  if (titulo === "" || categoria === "") {
    mostrarEstado("El título y la categoría son obligatorios.", "error");
    return;
  }

  if (duracion <= 0) {
    mostrarEstado("La duración debe ser mayor que 0.", "error");
    return;
  }

  if (dificultad < 1 || dificultad > 3) {
    mostrarEstado("La dificultad debe estar entre 1 y 3.", "error");
    return;
  }

  const nuevoRecurso = {
    id: generarNuevoId(),
    titulo: titulo,
    categoria: categoria,
    duracionMinutos: duracion,
    dificultad: dificultad,
    completado: false,
    etiquetas: []
  };

  recursos.push(nuevoRecurso);

  rellenarFiltroCategorias(recursos);

  busqueda.value = "";
  filtroCategoria.value = "todas";
  orden.value = "original";

  actualizarInterfaz();

  formRecurso.reset();

  mostrarEstado("Recurso añadido visualmente. No se ha guardado en el fichero JSON.", "ok");
}

13. Errores frecuentes

Revisa especialmente estos errores:

  • escribir mal la ruta del fichero JSON;
  • abrir el HTML directamente sin servidor local;
  • olvidar await delante de fetch;
  • olvidar await delante de response.json();
  • no comprobar response.ok;
  • no usar try...catch;
  • usar innerHTML de forma descontrolada;
  • no limpiar el contenedor antes de renderizar;
  • modificar el array original al ordenar sin querer;
  • usar sort() directamente sobre recursos;
  • olvidar preventDefault() en el formulario;
  • no convertir a número los valores de los campos numéricos;
  • comparar textos sin normalizar mayúsculas/minúsculas;
  • crear un recurso nuevo sin id;
  • crear un recurso nuevo sin completado;
  • crear un recurso nuevo sin etiquetas;
  • calcular estadísticas con una lista vacía;
  • dejar errores en la consola.
