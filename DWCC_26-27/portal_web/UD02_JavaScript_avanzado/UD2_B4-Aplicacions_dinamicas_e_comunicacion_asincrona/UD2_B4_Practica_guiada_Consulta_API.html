UD2_B4. Práctica guiada: consulta de una API con fetch, JSON y renderizado en el DOM

1. Objetivo de la práctica

En esta práctica vas a construir una pequeña página web que consulta datos de una API externa, recibe información en formato JSON y la muestra dinámicamente en el navegador.

El objetivo no es memorizar una API concreta, sino entender el flujo general que se usa en muchas aplicaciones web reales:

  1. La página carga.
  2. JavaScript solicita datos a una URL externa.
  3. Se recibe una respuesta HTTP.
  4. Se comprueba si la respuesta es correcta con response.ok.
  5. Si la respuesta es correcta, se lee el cuerpo de la respuesta con response.json().
  6. Se procesan los datos recibidos.
  7. Se generan elementos HTML desde JavaScript.
  8. Se muestran estados de carga, error y resultado.

En esta práctica trabajaremos con una API pública de pruebas. No se envían datos privados ni se necesita clave de acceso.

2. Qué contenidos se trabajan

En esta práctica se integran contenidos del bloque UD2_B4:

  • código asíncrono;
  • fetch();
  • promesas;
  • async / await;
  • try...catch;
  • comprobación de response.ok;
  • lectura del cuerpo de una respuesta con .json();
  • tratamiento de errores;
  • renderizado dinámico en el DOM;
  • estados de carga, error y éxito;
  • uso de métodos de array sobre datos recibidos;
  • separación entre HTML, CSS y JavaScript.

También se repasan contenidos de bloques anteriores:

  • objetos;
  • arrays de objetos;
  • funciones;
  • forEach, map, filter y reduce;
  • DOM básico;
  • eventos básicos.

3. API que se utilizará

Usaremos JSONPlaceholder, una API pública de prueba que devuelve datos ficticios en formato JSON.

En esta práctica consultaremos principalmente estos recursos:

https://jsonplaceholder.typicode.com/posts
https://jsonplaceholder.typicode.com/users

Los datos no son reales. Se usan para practicar consultas desde JavaScript.

Esta práctica necesita conexión a Internet. Si la API no responde, revisa primero la conexión del equipo, el proxy del centro o posibles restricciones de red antes de modificar el código.

Organización recomendada

La práctica puede dividirse en dos sesiones:

  • Sesión 1: estructura del proyecto, consulta de publicaciones, renderizado y estados.
  • Sesión 2: filtro, consulta de usuarios, prueba de errores, entrega y ampliaciones.

Si no da tiempo a completar todo en clase, prioriza hasta el apartado 17. Los apartados 18 y 24 pueden quedar como mejora guiada.

8. Función para mostrar mensajes de estado

Añade esta función:

function mostrarEstado(mensaje, tipo = "normal") {
  estado.textContent = mensaje;
  estado.className = "estado";

  if (tipo === "error") {
    estado.classList.add("error");
  }

  if (tipo === "cargando") {
    estado.classList.add("cargando");
  }
}

Explicación

Esta función centraliza los mensajes que se muestran al usuario:

  • cuando la página está esperando datos;
  • cuando hay un error;
  • cuando todo ha ido bien.

Así evitamos repetir varias veces instrucciones como:

estado.textContent = "...";

10. Función para crear una tarjeta

Cada publicación se mostrará como una tarjeta. Añade esta función:

function crearTarjetaPublicacion(publicacion) {
  const tarjeta = document.createElement("article");
  tarjeta.classList.add("tarjeta");

  const titulo = document.createElement("h2");
  titulo.textContent = publicacion.title;

  const cuerpo = document.createElement("p");
  cuerpo.textContent = publicacion.body;

  const usuario = document.createElement("span");
  usuario.classList.add("etiqueta");
  usuario.textContent = `Usuario ${publicacion.userId}`;

  tarjeta.append(titulo, cuerpo, usuario);

  return tarjeta;
}

Fíjate en esto

La función recibe un objeto publicacion y devuelve un elemento HTML completo.

No añade directamente la tarjeta al DOM. Solo la crea y la devuelve. Esto hace que la función sea más reutilizable.

11. Función para renderizar publicaciones

Añade esta función:

function renderizarPublicaciones(lista) {
  resultados.innerHTML = "";

  if (lista.length === 0) {
    resultados.textContent = "No hay publicaciones que mostrar.";
    return;
  }

  lista.forEach((publicacion) => {
    const tarjeta = crearTarjetaPublicacion(publicacion);
    resultados.append(tarjeta);
  });
}

Explicación

Esta función:

  1. Limpia los resultados anteriores.
  2. Comprueba si la lista está vacía.
  3. Recorre la lista con forEach.
  4. Crea una tarjeta por cada publicación.
  5. Añade cada tarjeta al contenedor.

12. Función para mostrar un resumen

Añade esta función:

function mostrarResumen(lista) {
  const total = lista.length;

  const idsUsuarios = lista.map((publicacion) => publicacion.userId);
  const sumaUsuarios = idsUsuarios.reduce((acumulador, id) => acumulador + id, 0);

  resumen.innerHTML = `
    <strong>Resumen:</strong>
    <p>Total de publicaciones mostradas: ${total}</p>
    <p>Suma de identificadores de usuario: ${sumaUsuarios}</p>
  `;
}

Importante

Este resumen no tiene gran utilidad real, pero sirve para practicar:

  • map;
  • reduce;
  • plantillas de texto;
  • actualización del DOM.

En este caso usamos innerHTML con valores controlados por nuestro propio código. Cuando se insertan textos escritos por usuarios o datos externos sin controlar, es más seguro crear nodos con createElement() y asignar textos con textContent.

13. Función principal: consultar la API

Ahora añade la función principal de la práctica:

async function cargarPublicaciones() {
  limpiarResultados();
  mostrarEstado("Cargando publicaciones...", "cargando");

  try {
    const respuesta = await fetch("https://jsonplaceholder.typicode.com/posts");

    if (!respuesta.ok) {
      throw new Error(`Error HTTP: ${respuesta.status}`);
    }

    const datos = await respuesta.json();

    publicaciones = datos.slice(0, 12);

    renderizarPublicaciones(publicaciones);
    mostrarResumen(publicaciones);
    mostrarEstado("Publicaciones cargadas correctamente.");
  } catch (error) {
    mostrarEstado("No se pudieron cargar las publicaciones.", "error");
    console.error("Error al consultar la API:", error);
  }
}

Explicación paso a paso

const respuesta = await fetch("https://jsonplaceholder.typicode.com/posts");

Solicita datos a la URL indicada.

if (!respuesta.ok) {
  throw new Error(`Error HTTP: ${respuesta.status}`);
}

Comprueba si la respuesta HTTP es correcta. Si no lo es, genera un error.

const datos = await respuesta.json();

Lee el cuerpo de la respuesta y devuelve una promesa que se resuelve con los datos JavaScript obtenidos a partir del JSON.

Por eso se escribe await respuesta.json() y no solo respuesta.json().

publicaciones = datos.slice(0, 12);

Guarda solo las 12 primeras publicaciones para no llenar demasiado la página.

renderizarPublicaciones(publicaciones);

Muestra los datos en el DOM.

mostrarResumen(publicaciones);

Muestra un resumen básico.

catch (error) {
  ...
}

Captura errores de red, errores HTTP lanzados manualmente u otros problemas.

15. Añadir filtrado por texto

Añade este evento para filtrar las publicaciones ya cargadas:

inputBusqueda.addEventListener("input", () => {
  const texto = inputBusqueda.value.trim().toLowerCase();

  if (publicaciones.length === 0) {
    limpiarResultados();

    if (texto === "") {
      mostrarEstado("Pulsa el botón para cargar los datos.");
    } else {
      mostrarEstado("Carga primero las publicaciones para poder filtrarlas.");
    }

    return;
  }

  const filtradas = publicaciones.filter((publicacion) => {
    return (
      publicacion.title.toLowerCase().includes(texto) ||
      publicacion.body.toLowerCase().includes(texto)
    );
  });

  renderizarPublicaciones(filtradas);
  mostrarResumen(filtradas);

  if (texto === "") {
    mostrarEstado("Mostrando todas las publicaciones cargadas.");
  } else {
    mostrarEstado(`Filtro aplicado: ${texto}`);
  }
});

Qué se está practicando aquí

  • Evento input.
  • Lectura de .value.
  • Limpieza de texto con trim().
  • Conversión a minúsculas con toLowerCase().
  • Comprobación de que hay publicaciones cargadas antes de filtrar.
  • Filtrado con filter().
  • Búsqueda con includes().
  • Renderizado dinámico después de filtrar.

16. Probar un error de API

Para comprobar que el tratamiento de errores funciona, cambia temporalmente la URL:

const respuesta = await fetch("https://jsonplaceholder.typicode.com/ruta-inexistente");

Ejecuta de nuevo la página y pulsa el botón de carga.

Debe aparecer un mensaje de error.

Después, vuelve a dejar la URL correcta:

const respuesta = await fetch("https://jsonplaceholder.typicode.com/posts");

Recuerda

El objetivo de esta prueba no es dejar la aplicación rota. Es comprobar que el catch muestra un mensaje de error controlado y que no aparece una promesa rechazada sin tratar.

18. Mejora guiada: consultar usuarios

Ahora vas a ampliar la práctica para consultar también usuarios.

Añade en el HTML un nuevo botón dentro de panel-controles:

<button id="btnUsuarios">Cargar usuarios</button>

Selecciona el botón en JavaScript:

const btnUsuarios = document.querySelector("#btnUsuarios");

Crea una función para tarjetas de usuario:

function crearTarjetaUsuario(usuario) {
  const tarjeta = document.createElement("article");
  tarjeta.classList.add("tarjeta");

  const nombre = document.createElement("h2");
  nombre.textContent = usuario.name;

  const email = document.createElement("p");
  email.textContent = `Email: ${usuario.email}`;

  const ciudad = document.createElement("p");
  ciudad.textContent = `Ciudad: ${usuario.address.city}`;

  tarjeta.append(nombre, email, ciudad);

  return tarjeta;
}

Crea una función para renderizar usuarios:

function renderizarUsuarios(lista) {
  resultados.innerHTML = "";

  lista.forEach((usuario) => {
    const tarjeta = crearTarjetaUsuario(usuario);
    resultados.append(tarjeta);
  });
}

Crea la función asíncrona:

async function cargarUsuarios() {
  publicaciones = [];
  inputBusqueda.value = "";
  limpiarResultados();
  mostrarEstado("Cargando usuarios...", "cargando");

  try {
    const respuesta = await fetch("https://jsonplaceholder.typicode.com/users");

    if (!respuesta.ok) {
      throw new Error(`Error HTTP: ${respuesta.status}`);
    }

    const usuarios = await respuesta.json();

    renderizarUsuarios(usuarios);
    resumen.innerHTML = `<strong>Usuarios cargados:</strong> ${usuarios.length}`;
    mostrarEstado("Usuarios cargados correctamente.");
  } catch (error) {
    mostrarEstado("No se pudieron cargar los usuarios.", "error");
    console.error("Error al consultar usuarios:", error);
  }
}

Conecta el botón:

btnUsuarios.addEventListener("click", () => {
  cargarUsuarios();
});

Importante sobre el filtro

El campo de búsqueda de esta práctica filtra publicaciones, no usuarios. Por eso, al cargar usuarios se vacía la variable publicaciones y se limpia el campo de búsqueda. Así evitamos que el filtro intente trabajar sobre datos que no corresponden a la vista actual.

Comprobación

  • email;
  • ciudad.

20. Errores frecuentes

Error 1: olvidar await

Incorrecto:

const respuesta = fetch("https://jsonplaceholder.typicode.com/posts");
const datos = respuesta.json();

Correcto:

const respuesta = await fetch("https://jsonplaceholder.typicode.com/posts");
const datos = await respuesta.json();

respuesta.json() también es una operación asíncrona. Por eso necesita await.

Error 2: usar .json() sin comprobar la respuesta

Mejor no hacer directamente:

const datos = await respuesta.json();

Sin antes comprobar:

if (!respuesta.ok) {
  throw new Error(`Error HTTP: ${respuesta.status}`);
}

Error 3: intentar usar datos fuera de una función asíncrona antes de que existan

Incorrecto:

let publicaciones = [];

fetch("https://jsonplaceholder.typicode.com/posts");

console.log(publicaciones); // probablemente seguirá vacío

La carga de datos tarda un tiempo. Hay que trabajar con los datos cuando ya se han recibido.

Error 4: no limpiar resultados anteriores

Si no limpias el contenedor antes de volver a renderizar, las tarjetas se duplican.

resultados.innerHTML = "";

Error 5: escribir mal una propiedad del objeto recibido

Incorrecto:

titulo.textContent = publicacion.titulo;

Correcto:

titulo.textContent = publicacion.title;

Hay que fijarse en los nombres reales de las propiedades que devuelve la API.

24. Ampliaciones opcionales

Estas tareas son opcionales y solo deben hacerse si la práctica base funciona correctamente.

Ampliación 0: desactivar el botón mientras carga

Para evitar que se lancen varias peticiones si se pulsa muchas veces el botón, puedes desactivar temporalmente el botón dentro de cargarPublicaciones():

async function cargarPublicaciones() {
  limpiarResultados();
  mostrarEstado("Cargando publicaciones...", "cargando");
  btnCargar.disabled = true;

  try {
    const respuesta = await fetch("https://jsonplaceholder.typicode.com/posts");

    if (!respuesta.ok) {
      throw new Error(`Error HTTP: ${respuesta.status}`);
    }

    const datos = await respuesta.json();

    publicaciones = datos.slice(0, 12);

    renderizarPublicaciones(publicaciones);
    mostrarResumen(publicaciones);
    mostrarEstado("Publicaciones cargadas correctamente.");
  } catch (error) {
    mostrarEstado("No se pudieron cargar las publicaciones.", "error");
    console.error("Error al consultar la API:", error);
  } finally {
    btnCargar.disabled = false;
  }
}

Si haces esta ampliación, añade también un estilo para botones desactivados:

button:disabled {
  opacity: 0.6;
  cursor: not-allowed;
}

Ampliación 4: crear una función reutilizable para consultar APIs

Crea una función:

async function obtenerDatos(url) {
  const respuesta = await fetch(url);

  if (!respuesta.ok) {
    throw new Error(`Error HTTP: ${respuesta.status}`);
  }

  return await respuesta.json();
}

Después reutilízala para cargar publicaciones y usuarios.
