UD2_B4c. Ejercicios: AJAX como contexto y código heredado

1. Objetivo de la hoja

Esta hoja complementa los apuntes sobre AJAX como contexto histórico y código heredado.

El objetivo no es aprender a programar aplicaciones nuevas con XMLHttpRequest ni con $.ajax(), sino saber reconocer código antiguo, entender qué hace y compararlo con el enfoque moderno usado en el curso: fetch con async/await.

En esta unidad, para desarrollar código nuevo, se prioriza fetch, promesas y async/await.

2. Antes de empezar

Recuerda estas ideas:

  • AJAX no es una tecnología concreta, sino una forma de hacer peticiones al servidor sin recargar toda la página.
  • XMLHttpRequest fue una API clásica del navegador para hacer peticiones HTTP desde JavaScript.
  • $.ajax() es una forma de hacer AJAX usando la biblioteca jQuery.
  • fetch es la API moderna que usaremos como referencia principal.
  • JSON es uno de los formatos más habituales para enviar y recibir datos.
  • Con fetch, una respuesta HTTP como 404 o 500 no provoca automáticamente un error en el catch; por eso conviene comprobar response.ok.

Ejercicio 2. Leer código con XMLHttpRequest

Observa este código antiguo:

const xhr = new XMLHttpRequest();

xhr.open("GET", "datos/productos.json");

xhr.onload = function () {
  if (xhr.status === 200) {
    const productos = JSON.parse(xhr.responseText);
    console.log(productos);
  } else {
    console.log("Error al cargar los datos");
  }
};

xhr.onerror = function () {
  console.log("Error de conexión");
};

xhr.send();

Responde:

  1. ¿Qué archivo intenta cargar el código?
  2. ¿Qué método HTTP utiliza?
  3. ¿Qué significa comprobar xhr.status === 200 en este ejemplo?
  4. ¿Qué contiene xhr.responseText?
  5. ¿Para qué se usa JSON.parse()?
  6. ¿Qué ocurre si hay un problema de conexión?
  7. ¿Es este el estilo que vamos a usar para código nuevo en el curso?

Ejercicio 5. Comparar $.ajax() y fetch

El siguiente código usa jQuery:

$.ajax({
  url: "datos/cursos.json",
  method: "GET",
  success: function (cursos) {
    console.log(cursos);
  },
  error: function () {
    console.log("Error al cargar cursos");
  }

Completa una versión equivalente usando fetch con async/await:

async function cargarCursos() {
  try {
    const respuesta = await fetch("____________________");

    if (!respuesta.ok) {
      throw new Error("____________________");
    }

    const cursos = await respuesta.__________();
    console.log(cursos);
  } catch (error) {
    console.log("____________________");
  }
}

Ejercicio 7. Detectar errores frecuentes

Indica qué problema hay en cada fragmento y cómo se podría corregir.

Caso A

fetch("datos/productos.json")
  .then((respuesta) => respuesta)
  .then((datos) => {
    console.log(datos);
  });

Pregunta: ¿Por qué no se están obteniendo directamente los datos JSON?

Pista: piensa qué devuelve fetch() y qué método hay que llamar para transformar la respuesta en datos.

Caso B

async function cargarDatos() {
  const respuesta = await fetch("datos/productos.json");
  const datos = await respuesta.json();
  console.log(datos);
}

Pregunta: ¿Qué dos elementos faltan para tratar errores de forma más adecuada con fetch?

Pista: uno sirve para capturar errores y el otro para comprobar si la respuesta HTTP ha sido correcta.

Caso C

$.ajax({
  url: "datos/productos.json",
  success: function () {
    console.log(productos);
  }
});

Pregunta: ¿Por qué productos puede dar error?

Pista: revisa si la función success recibe algún parámetro con los datos de la respuesta.

Contenidos que no se desarrollan en esta hoja

Esta hoja es solo de reconocimiento y comparación. No se trabajarán como contenido principal:

  • cabeceras HTTP complejas;

  • CORS en profundidad;
  • AbortController;
