UD2_B4. Ejercicios: fetch y JSON

1. Objetivos y preparación del proyecto

1.1. Objetivo de la hoja

En esta hoja vas a practicar el flujo completo de trabajo con datos JSON mediante fetch():

  1. iniciar la carga desde un evento;
  2. mostrar un estado de carga;
  3. realizar la petición;
  4. comprobar response.ok;
  5. interpretar la respuesta con .json();
  6. validar la estructura recibida;
  7. procesar los datos;
  8. renderizarlos de forma segura;
  9. tratar estados sin resultados;
  10. gestionar errores;
  11. restaurar la interfaz mediante finally.

El objetivo no es memorizar una plantilla, sino comprender qué responsabilidad tiene cada parte del proceso.

En esta hoja trabajaremos principalmente con archivos JSON locales para no depender de una API externa. Aun así, aprenderemos a reconocer errores de red, HTTP, JSON, formato y CORS.

1.2. Estructura de archivos

Crea:

├── index.html

    ├── productos.json

1.3. Utilizar un servidor web

Abre el proyecto mediante Live Server o un servidor local equivalente.

Dirección correcta:

http://127.0.0.1:5500/index.html

Dirección que debes evitar:

file:///C:/proyecto/index.html

Antes de comenzar:

  1. comprueba que el servidor está activo;
  2. abre productos.json desde el navegador;
  3. verifica que la ruta respeta carpetas, mayúsculas y minúsculas;
  4. revisa que la consola no muestra errores.

Realiza una petición a:

datos/productos.json

Utiliza:

fetch("datos/productos.json")
  .then((respuesta) => {
    return respuesta.json();
  })
  .then((productos) => {
    console.log(productos);
  });

Responde:

  1. ¿Qué devuelve fetch()?

4. Response, response.ok y errores

Ejercicio 4. Comprobar response.ok

Modifica la cadena:

fetch("datos/productos.json")
  .then((respuesta) => {
    if (!respuesta.ok) {
      throw new Error(
        `Error HTTP: ${respuesta.status}`
      );
    }

    return respuesta.json();
  })
  .then((productos) => {
    console.log(productos);
  })
  .catch((error) => {
    console.error(error);
  });

Prueba una ruta incorrecta.

Responde:

  1. ¿Qué valor tiene response.ok?
  2. ¿Qué código aparece en response.status?
  3. ¿Por qué se lanza manualmente un error?

Ejercicio 18. Crear cargarProductos()

Estructura obligatoria:

async function cargarProductos() {
  mostrarCargaProductos();

  try {
    const productos =
      await obtenerProductos(
        "datos/productos.json"
      );

    if (productos.length === 0) {
      mostrarSinProductos();
      return;
    }

    const productosValidos =
      productos.filter(
        esProductoValido
      );

    renderizarProductos(
      productosValidos
    );

    const resumen =
      calcularResumen(
        productosValidos
      );

    renderizarResumen(resumen);

    estadoProductos.textContent =
      "Productos cargados correctamente.";
  } catch (error) {
    mostrarErrorProductos();
    console.error(error);
  } finally {
    finalizarCargaProductos();
  }
}
