UD2_B1. Objetos y JavaScript avanzado sin DOM

1. Qué vamos a trabajar en este bloque

En la unidad anterior se trabajó la base de JavaScript: variables, tipos, operadores, condicionales, bucles, funciones, arrays y objetos predefinidos.

En este bloque vamos a avanzar un paso más, pero sin trabajar todavía con el DOM . El objetivo es aprender a organizar mejor los datos y a escribir código más expresivo usando objetos, arrays de objetos y recursos modernos de JavaScript.

En este bloque aprenderás a:

  • crear y utilizar objetos propios;
  • distinguir entre arrays y objetos;
  • trabajar con propiedades y métodos;
  • entender el uso básico de this ;
  • evitar errores habituales con funciones flecha dentro de objetos;
  • crear arrays de objetos;
  • recorrer y transformar arrays de objetos;
  • usar métodos como find , some , every y sort ;
  • aplicar destructuring ;
  • usar spread/rest de forma básica;
  • comprender cuándo puede ser útil Map ;
  • conocer, como ampliación, arrays multidimensionales, closures y recursividad.

Importante: en este bloque todavía no manipularemos elementos HTML desde JavaScript. Eso se trabajará en el bloque de DOM.

2. Relación con DWCC

Este bloque se relaciona especialmente con:

  • el uso de objetos predefinidos y estructuras propias de JavaScript;
  • la creación y utilización de arrays;
  • las operaciones agregadas sobre colecciones;
  • la definición de objetos, propiedades y métodos;
  • la depuración y documentación del código.

El objetivo no es memorizar métodos sueltos, sino aprender a elegir la estructura adecuada para representar información y procesarla correctamente.

3. De los datos simples a los objetos

Hasta ahora hemos trabajado con datos simples: 

const nombre = "Ana";
const edad = 20;
const nota = 7.5;
const matriculado = true;

También hemos trabajado con arrays: 

const notas = [4, 7, 9, 3, 5];

Un array es útil cuando tenemos una lista ordenada de valores similares .

Pero muchas veces necesitamos representar información más compleja. Por ejemplo, un alumno no es solo un nombre o una nota. Puede tener varias propiedades:

  • nombre;
  • edad;
  • curso;
  • nota media;
  • si está aprobado o no.

Para eso usamos objetos .

4. Objetos literales: propiedades, acceso, modificación y eliminación

Un objeto permite agrupar información relacionada en una sola estructura.

Ejemplo: 

const alumno = {
  nombre: "Ana",
  edad: 20,
  curso: "DWCC",
  notaMedia: 7.5,
  aprobado: true
};

En este ejemplo:

  • alumno es un objeto;
  • nombre , edad , curso , notaMedia y aprobado son propiedades;
  • cada propiedad tiene un valor asociado.

4.1. Acceder a propiedades de un objeto

La forma más habitual es usar el punto: 

const alumno = {
  nombre: "Ana",
  edad: 20,
  curso: "DWCC"
};

console.log(alumno.nombre); // Ana
console.log(alumno.edad);   // 20
console.log(alumno.curso);  // DWCC

También se puede acceder con corchetes: 

console.log(alumno["nombre"]); // Ana

La notación con punto es la más cómoda cuando conocemos el nombre de la propiedad.

La notación con corchetes es útil cuando el nombre de la propiedad está guardado en una variable: 

const propiedad = "edad";

console.log(alumno[propiedad]); // 20

4.2. Modificar y añadir propiedades

Los objetos pueden modificarse después de crearse. 

const alumno = {
  nombre: "Ana",
  edad: 20
};

alumno.edad = 21;
alumno.curso = "DWCC";

console.log(alumno);

Resultado aproximado: 

{
  nombre: "Ana",
  edad: 21,
  curso: "DWCC"
}

Aunque el objeto esté declarado con const , podemos modificar sus propiedades. 

const alumno = {
  nombre: "Ana",
  edad: 20
};

alumno.edad = 21; // permitido

// alumno = { nombre: "Luis" }; // error: no se puede reasignar la constante

Esto es parecido a lo que ocurría con los arrays declarados con const .

4.3. Eliminar propiedades

Podemos eliminar una propiedad con delete . 

const alumno = {
  nombre: "Ana",
  edad: 20,
  telefono: "111222333"
};

delete alumno.telefono;

console.log(alumno);

Después de ejecutar el código, el objeto ya no tendrá la propiedad telefono .

En código real no conviene abusar de delete . Muchas veces es mejor diseñar bien la estructura de datos desde el principio.

5. Métodos de objetos y uso básico de this

Un objeto puede contener propiedades con datos, pero también puede contener funciones. Cuando una función está dentro de un objeto, se suele llamar método . 

const alumno = {
  nombre: "Ana",
  nota: 8,

  saludar: function () {
    console.log("Hola, soy Ana");
  }
};

alumno.saludar();

En este caso, saludar es un método del objeto alumno .

5.1. Uso básico de this

Dentro de un método, podemos usar this para referirnos al propio objeto. 

const alumno = {
  nombre: "Ana",
  nota: 8,

  saludar: function () {
    console.log("Hola, soy " + this.nombre);
  }
};

alumno.saludar(); // Hola, soy Ana

Aquí this.nombre significa “la propiedad nombre de este objeto”.

Otro ejemplo: 

const producto = {
  nombre: "Teclado",
  precio: 25,

  mostrarPrecio: function () {
    console.log(this.nombre + " cuesta " + this.precio + " euros");
  }
};

producto.mostrarPrecio();

5.2. Cuidado con funciones flecha dentro de objetos

Las funciones flecha son muy útiles, pero no se comportan igual que las funciones tradicionales con this .

Ejemplo problemático: 

const alumno = {
  nombre: "Ana",

  saludar: () => {
    console.log("Hola, soy " + this.nombre);
  }
};

alumno.saludar(); // this.nombre no apunta al objeto alumno

La función flecha no crea su propio this , por lo que this.nombre no apunta al objeto alumno .

En general, para métodos de objetos donde necesitemos usar this , es más seguro utilizar funciones tradicionales: 

const alumno = {
  nombre: "Ana",

  saludar: function () {
    console.log("Hola, soy " + this.nombre);
  }
};

alumno.saludar(); // Hola, soy Ana

Regla práctica para este nivel: si un método necesita usar this , usa function .

6. Arrays, objetos y arrays asociativos

Un array sirve para guardar una lista ordenada de elementos: 

const notas = [4, 7, 9, 3, 5];

Un objeto sirve para representar una entidad con propiedades: 

const alumno = {
  nombre: "Ana",
  edad: 20,
  nota: 8
};

Comparación:

Estructura Se usa para Acceso
Array Lista ordenada de valores notas[0]
Objeto Entidad con propiedades alumno.nombre

Ejemplo: 

const nombres = ["Ana", "Luis", "María"];

const alumno = {
  nombre: "Ana",
  edad: 20,
  nota: 8
};

6.1. Aclaración: “arrays asociativos” en JavaScript

En algunos lenguajes se habla de arrays asociativos , donde se guardan valores usando claves de texto.

En JavaScript, si queremos guardar pares clave-valor, lo normal es usar objetos: 

const persona = {
  nombre: "Ana",
  edad: 20
};

No es recomendable usar un array como si fuera un objeto: 

const datos = [];

datos["nombre"] = "Ana";
datos["edad"] = 20;

console.log(datos);

Aunque JavaScript lo permite, no es el uso adecuado de un array.

Idea clave: para listas, arrays. Para datos con nombre, objetos.

Además, aunque JavaScript permita escribir propiedades con claves de texto en un array, esas propiedades no cuentan como elementos reales del array y no afectan a length . 

const datos = [];

datos["nombre"] = "Ana";

console.log(datos.length); // 0

7. Arrays de objetos

Una estructura muy habitual en JavaScript es un array que contiene objetos.

Ejemplo: 

const alumnos = [
  {
    nombre: "Ana",
    nota: 8
  },
  {
    nombre: "Luis",
    nota: 4
  },
  {
    nombre: "María",
    nota: 9
  }
];

Aquí:

  • alumnos es un array;
  • cada elemento del array es un objeto;
  • cada objeto representa un alumno.

Podemos acceder a los datos así: 

console.log(alumnos[0].nombre); // Ana
console.log(alumnos[1].nota);   // 4

8. Recorrer, transformar, filtrar y acumular arrays de objetos

Podemos recorrer un array de objetos con un bucle for . 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

for (let i = 0; i < alumnos.length; i++) {
  console.log(alumnos[i].nombre + ": " + alumnos[i].nota);
}

Este recorrido es muy útil para entender qué está ocurriendo internamente.

8.1. Recorrer arrays de objetos con forEach

También podemos usar forEach . 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

alumnos.forEach((alumno) => {
  console.log(alumno.nombre + ": " + alumno.nota);
});

Aquí, en cada vuelta, alumno representa uno de los objetos del array.

8.2. Transformar arrays de objetos con map

map permite crear un nuevo array transformando cada elemento.

Ejemplo: obtener solo los nombres. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

const nombres = alumnos.map((alumno) => {
  return alumno.nombre;
});

console.log(nombres); // ["Ana", "Luis", "María"]

Ejemplo: generar mensajes personalizados. 

const mensajes = alumnos.map((alumno) => {
  return alumno.nombre + " tiene una nota de " + alumno.nota;
});

console.log(mensajes);

8.3. Filtrar arrays de objetos con filter

filter permite crear un nuevo array con los elementos que cumplen una condición.

Ejemplo: obtener solo los aprobados. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

const aprobados = alumnos.filter((alumno) => {
  return alumno.nota >= 5;
});

console.log(aprobados);

El array aprobados contiene los objetos de Ana y María.

8.4. Acumular información con reduce

reduce permite obtener un único resultado a partir de un array.

Ejemplo: calcular la suma de las notas. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

const sumaNotas = alumnos.reduce((acumulador, alumno) => {
  return acumulador + alumno.nota;
}, 0);

console.log(sumaNotas); // 21

Podemos calcular la media: 

const media = sumaNotas / alumnos.length;

console.log(media); // 7

Tabla mental del proceso:

Vuelta acumulador alumno.nota nuevo acumulador
1 0 8 8
2 8 4 12
3 12 9 21

9. Buscar y comprobar con find , some y every

find devuelve el primer elemento que cumple una condición. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

const alumnoBuscado = alumnos.find((alumno) => {
  return alumno.nombre === "Luis";
});

console.log(alumnoBuscado); // { nombre: "Luis", nota: 4 }

Si no encuentra nada, devuelve undefined . 

const resultado = alumnos.find((alumno) => {
  return alumno.nombre === "Pedro";
});

console.log(resultado); // undefined

9.1. Comprobar condiciones con some

some devuelve true si al menos un elemento cumple la condición. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

const haySuspensos = alumnos.some((alumno) => {
  return alumno.nota < 5;
});

console.log(haySuspensos); // true

Pregunta que responde some :

¿Hay alguno que cumpla esta condición?

9.2. Comprobar condiciones con every

every devuelve true si todos los elementos cumplen la condición. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

const todosAprobados = alumnos.every((alumno) => {
  return alumno.nota >= 5;
});

console.log(todosAprobados); // false

Pregunta que responde every :

¿Todos cumplen esta condición?

9.3. Comparativa: find , some y every

Método Pregunta que responde Resultado
find ¿Cuál es el primer elemento que cumple la condición? Un elemento o undefined
some ¿Hay al menos uno que cumple la condición? true o false
every ¿Cumplen todos la condición? true o false

Ejemplo con productos: 

const productos = [
  { nombre: "Teclado", precio: 25 },
  { nombre: "Ratón", precio: 12 },
  { nombre: "Monitor", precio: 140 }
];

const productoCaro = productos.find((producto) => producto.precio > 100);
const hayProductosBaratos = productos.some((producto) => producto.precio < 15);
const todosCuestanMasDe10 = productos.every((producto) => producto.precio > 10);

console.log(productoCaro);
console.log(hayProductosBaratos);
console.log(todosCuestanMasDe10);

10. Ordenación con sort

Importante sort() modifica el array original. Si quieres conservar el orden inicial, crea una copia antes de ordenar.

El método sort() ordena los elementos de un array.

Con textos: 

const nombres = ["María", "Ana", "Luis"];

nombres.sort();

console.log(nombres); // ["Ana", "Luis", "María"]

Con números, hay que tener cuidado: 

const numeros = [10, 2, 30, 4];

numeros.sort();

console.log(numeros); // [10, 2, 30, 4] o resultado no esperado

Por defecto, sort() ordena como texto. Para ordenar números correctamente, usamos una función de comparación: 

const numeros = [10, 2, 30, 4];

numeros.sort((a, b) => {
  return a - b;
});

console.log(numeros); // [2, 4, 10, 30]

Orden descendente: 

numeros.sort((a, b) => {
  return b - a;
});

console.log(numeros); // [30, 10, 4, 2]

10.1. Ordenar arrays de objetos

Podemos ordenar arrays de objetos usando una propiedad. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 }
];

alumnos.sort((a, b) => {
  return a.nota - b.nota;
});

console.log(alumnos);

Esto ordena de menor a mayor nota.

Para ordenar de mayor a menor: 

alumnos.sort((a, b) => {
  return b.nota - a.nota;
});

Para ordenar por nombre: 

alumnos.sort((a, b) => {
  return a.nombre.localeCompare(b.nombre);
});

sort() modifica el array original. Si queremos conservar el array inicial, podemos hacer una copia antes. 

const copiaOrdenada = alumnos.slice().sort((a, b) => {
  return b.nota - a.nota;
});

11. Destructuring

El destructuring permite extraer propiedades de un objeto y guardarlas en variables.

Sin destructuring: 

const alumno = {
  nombre: "Ana",
  nota: 8
};

const nombre = alumno.nombre;
const nota = alumno.nota;

console.log(nombre, nota);

Con destructuring: 

const alumno = {
  nombre: "Ana",
  nota: 8
};

const { nombre, nota } = alumno;

console.log(nombre, nota);

Es especialmente útil cuando trabajamos con objetos grandes.

11.1. Destructuring en parámetros de funciones

Podemos usar destructuring directamente en los parámetros. 

function mostrarAlumno({ nombre, nota }) {
  console.log(nombre + " tiene un " + nota);
}

const alumno = {
  nombre: "Ana",
  nota: 8
};

mostrarAlumno(alumno);

Esto evita escribir continuamente alumno.nombre , alumno.nota , etc.

11.2. Destructuring de arrays

También podemos extraer valores de un array. 

const coordenadas = [10, 20];

const [x, y] = coordenadas;

console.log(x); // 10
console.log(y); // 20

Otro ejemplo: 

const nombres = ["Ana", "Luis", "María"];

const [primero, segundo] = nombres;

console.log(primero); // Ana
console.log(segundo); // Luis

12. Spread y rest

Ojo Spread y rest usan el mismo símbolo ... , pero no hacen lo mismo:

  • spread expande valores;
  • rest recoge varios valores en un array.

El operador spread ... permite expandir los elementos de un array.

Crear una copia: 

const numeros = [1, 2, 3];

const copia = [...numeros];

console.log(copia);

Combinar arrays: 

const grupoA = ["Ana", "Luis"];
const grupoB = ["María", "Pedro"];

const grupoCompleto = [...grupoA, ...grupoB];

console.log(grupoCompleto);

Añadir elementos: 

const numeros = [2, 3, 4];

const nuevosNumeros = [1, ...numeros, 5];

console.log(nuevosNumeros); // [1, 2, 3, 4, 5]

12.1. Spread con objetos

También podemos usar spread con objetos. 

const alumno = {
  nombre: "Ana",
  nota: 8
};

const alumnoActualizado = {
  ...alumno,
  nota: 9
};

console.log(alumnoActualizado);

El objeto original no se modifica: 

console.log(alumno); // { nombre: "Ana", nota: 8 }

Esto es muy útil cuando queremos crear una versión modificada de un objeto sin tocar el original.

12.2. Rest en funciones

El operador rest también usa ... , pero sirve para recoger varios argumentos en un array. 

function sumar(...numeros) {
  let suma = 0;

  for (let i = 0; i < numeros.length; i++) {
    suma += numeros[i];
  }

  return suma;
}

console.log(sumar(1, 2, 3));       // 6
console.log(sumar(10, 20, 30, 40)); // 100

Aquí numeros es un array con todos los argumentos recibidos.

13. JSON, Map y estructuras clave-valor

En la unidad anterior vimos JSON.stringify() y JSON.parse() de forma básica. Ahora podemos aplicarlo a arrays de objetos. 

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 }
];

const textoJSON = JSON.stringify(alumnos);

console.log(textoJSON);

Convertir de texto JSON a estructura JavaScript: 

const datos = '[{"nombre":"Ana","nota":8},{"nombre":"Luis","nota":4}]';

const alumnos = JSON.parse(datos);

console.log(alumnos[0].nombre); // Ana

Esto será fundamental más adelante cuando trabajemos con APIs y fetch .

13.1. Map como colección clave-valor

Además de objetos y arrays, JavaScript tiene una estructura llamada Map .

Un Map permite guardar pares clave-valor. 

const telefonos = new Map();

telefonos.set("Ana", "111-111-111");
telefonos.set("Luis", "222-222-222");

console.log(telefonos.get("Ana")); // 111-111-111
console.log(telefonos.has("Luis")); // true
console.log(telefonos.size);        // 2

Recorrer un Map : 

for (const [nombre, telefono] of telefonos) {
  console.log(nombre + ": " + telefono);
}

Para este nivel:

  • si tienes una entidad con propiedades fijas, usa un objeto;
  • si tienes una lista, usa un array;
  • si necesitas una colección clave-valor que cambia mucho, Map puede ser una opción.

No será la estructura principal del curso, pero conviene conocerla.

14. Ampliaciones: arrays multidimensionales, closures y recursividad

Estas secciones son de ampliación. Conviene conocer la idea general, pero no serán el eje principal del bloque.

14.1. Ampliación: arrays multidimensionales

Un array multidimensional es un array que contiene otros arrays.

Ejemplo de tabla: 

const tablero = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
];

console.log(tablero[0][0]); // 1
console.log(tablero[1][1]); // 5
console.log(tablero[2][2]); // 9

Podemos recorrerlo con bucles anidados: 

for (let fila = 0; fila < tablero.length; fila++) {
  for (let columna = 0; columna < tablero[fila].length; columna++) {
    console.log(tablero[fila][columna]);
  }
}

Este contenido se considera ampliación porque añade complejidad al manejo de arrays.

14.2. Ampliación: closures

Un closure aparece cuando una función recuerda variables del entorno en el que fue creada.

Ejemplo sencillo: 

function crearContador() {
  let contador = 0;

  return function () {
    contador++;
    return contador;
  };
}

const contador1 = crearContador();

console.log(contador1()); // 1
console.log(contador1()); // 2
console.log(contador1()); // 3

La función interna sigue recordando la variable contador , aunque crearContador() ya haya terminado.

En este bloque solo necesitamos entender la idea general. Se profundizará si es necesario más adelante.

14.3. Ampliación: recursividad

Una función recursiva es una función que se llama a sí misma.

Ejemplo clásico: cuenta atrás. 

function cuentaAtras(numero) {
  if (numero <= 0) {
    console.log("Fin");
    return;
  }

  console.log(numero);
  cuentaAtras(numero - 1);
}

cuentaAtras(5);

Una función recursiva necesita siempre:

  • un caso base;
  • una llamada recursiva que se acerque al caso base.

Si no hay caso base, se produce una recursión infinita. 

function error() {
  error();
}

// error(); // No ejecutar: provocaría un error por recursión infinita

16. Ejemplo integrador 

const productos = [
  { nombre: "Teclado", precio: 25, stock: 10 },
  { nombre: "Ratón", precio: 12, stock: 0 },
  { nombre: "Monitor", precio: 140, stock: 4 },
  { nombre: "Cable HDMI", precio: 8, stock: 20 }
];

// Mostrar productos
productos.forEach((producto) => {
  console.log(producto.nombre + " - " + producto.precio + " €");
});

// Obtener solo los nombres
const nombres = productos.map((producto) => {
  return producto.nombre;
});

// Filtrar productos disponibles
const disponibles = productos.filter((producto) => {
  return producto.stock > 0;
});

// Calcular valor total del stock
const valorStock = productos.reduce((total, producto) => {
  return total + producto.precio * producto.stock;
}, 0);

// Buscar un producto concreto
const monitor = productos.find((producto) => {
  return producto.nombre === "Monitor";
});

// Comprobar condiciones
const haySinStock = productos.some((producto) => {
  return producto.stock === 0;
});

const todosTienenPrecio = productos.every((producto) => {
  return producto.precio > 0;
});

// Ordenar copia por precio
const ordenadosPorPrecio = [...productos].sort((a, b) => {
  return a.precio - b.precio;
});

console.log("Nombres:", nombres);
console.log("Disponibles:", disponibles);
console.log("Valor total del stock:", valorStock);
console.log("Monitor:", monitor);
console.log("Hay productos sin stock:", haySinStock);
console.log("Todos tienen precio:", todosTienenPrecio);
console.log("Ordenados por precio:", ordenadosPorPrecio);

Este ejemplo resume muchas de las operaciones más habituales con arrays de objetos.

17. Errores frecuentes

17.1. Confundir array y objeto

Incorrecto si queremos representar una entidad: 

const alumno = ["Ana", 20, 8];

Es mejor: 

const alumno = {
  nombre: "Ana",
  edad: 20,
  nota: 8
};

17.2. Usar corchetes cuando la notación con punto sería más clara

Esto no es un error técnico: 

console.log(alumno["nombre"]);

La notación con corchetes es válida y puede ser útil si el nombre de la propiedad está guardado en una variable. Pero si conocemos la propiedad, suele ser más claro escribir: 

console.log(alumno.nombre);

17.3. Olvidar que find puede devolver undefined 

const alumno = alumnos.find((a) => a.nombre === "Pedro");

console.log(alumno.nota); // error si alumno es undefined

Mejor: 

const alumno = alumnos.find((a) => a.nombre === "Pedro");

if (alumno !== undefined) {
  console.log(alumno.nota);
} else {
  console.log("Alumno no encontrado");
}

17.4. Usar map cuando se necesita filter

Si queremos quedarnos solo con aprobados, usamos filter : 

const aprobados = alumnos.filter((alumno) => alumno.nota >= 5);

No map .

17.5. Usar forEach esperando un nuevo array

Incorrecto: 

const dobles = numeros.forEach((n) => n * 2);

console.log(dobles); // undefined

Correcto: 

const dobles = numeros.map((n) => n * 2);

17.6. Olvidar que sort modifica el array original 

const ordenados = productos.sort((a, b) => a.precio - b.precio);

Si queremos conservar el array original: 

const ordenados = [...productos].sort((a, b) => a.precio - b.precio);

17.7. Usar funciones flecha como métodos cuando se necesita this

Problemático: 

const alumno = {
  nombre: "Ana",
  saludar: () => {
    console.log(this.nombre);
  }
};

Mejor: 

const alumno = {
  nombre: "Ana",
  saludar: function () {
    console.log(this.nombre);
  }
};

19. Resumen final

En este bloque hemos dado el salto desde datos simples y arrays básicos hacia estructuras de datos más reales.

Ideas clave:

  • Los objetos permiten representar entidades con propiedades.
  • Los métodos son funciones dentro de objetos.
  • this permite acceder al propio objeto desde un método.
  • Los arrays de objetos son una estructura muy habitual en JavaScript.
  • map , filter , reduce , find , some , every y sort permiten procesar colecciones de datos.
  • destructuring ayuda a extraer información de arrays y objetos.
  • spread permite copiar y combinar estructuras.
  • rest permite recoger varios argumentos en una función.
  • JSON permite convertir datos entre estructuras JavaScript y texto.
  • Map , arrays multidimensionales, closures y recursividad son recursos avanzados que conviene conocer, aunque no serán el eje principal de los siguientes bloques.
