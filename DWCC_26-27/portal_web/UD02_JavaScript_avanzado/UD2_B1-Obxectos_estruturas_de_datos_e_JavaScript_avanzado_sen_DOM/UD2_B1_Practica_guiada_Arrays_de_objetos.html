UD2_B1. Práctica guiada: arrays de objetos en JavaScript

Procesamiento de una colección de productos mediante arrays de objetos, funciones y métodos avanzados, sin utilizar todavía el DOM.

1. Objetivo de la práctica

En esta práctica vas a trabajar con una colección de datos representada mediante un array de objetos.

El objetivo no es crear una página web ni manipular el DOM todavía. En este bloque seguimos trabajando con JavaScript avanzado sin DOM, usando la consola para comprobar los resultados.

Al finalizar la práctica deberías ser capaz de:

  • crear objetos literales;
  • guardar varios objetos dentro de un array;
  • acceder a propiedades de objetos;
  • recorrer arrays de objetos;
  • usar map, filter, reduce, find, some y every sobre arrays de objetos;
  • ordenar arrays con sort() y función de comparación;
  • usar destructuring y spread en ejemplos sencillos;
  • convertir datos a JSON con JSON.stringify() y recuperarlos con JSON.parse();
  • organizar el código en funciones reutilizables.

2. Escenario de trabajo

Vas a crear un pequeño sistema de análisis de productos de una tienda.

Cada producto tendrá esta información:

  • id: identificador numérico;
  • nombre: nombre del producto;
  • categoria: categoría del producto;
  • precio: precio en euros;
  • stock: unidades disponibles;
  • activo: indica si el producto está disponible para la venta.

Ejemplo de producto:

const producto = {
  id: 1,
  nombre: "Teclado",
  categoria: "periféricos",
  precio: 29.99,
  stock: 12,
  activo: true
};

Ejercicio 6. Crear productos con IVA

Crea un nuevo array llamado productosConIVA.

Cada elemento debe ser un objeto nuevo que mantenga los datos del producto original y añada una nueva propiedad:

precioConIVA

El IVA será del 21%.

Pista con spread:

const productosConIVA = productos.map((producto) => {
  return {
    ...producto,
    precioConIVA: producto.precio * 1.21
  };
});

console.log(productosConIVA);

Importante: para conservar precioConIVA como número, guarda el cálculo sin aplicar .toFixed(). Usa .toFixed(2) únicamente cuando muestres el precio por consola, ya que este método devuelve una cadena de texto.

13. Parte 9: función integradora

Ejercicio 25. Informe de productos

Crea una función llamada generarInformeProductos que reciba un array de productos y devuelva un objeto con esta información:

Con los datos iniciales de esta práctica, el resultado debe ser aproximadamente:

{
  totalProductos: 6,
  productosActivos: 5,
  productosSinStock: 1,
  valorInventario: 2549.48,
  precioMedio: 127.075
}

Al mostrar el precio medio por consola, puedes formatearlo con dos decimales para obtener 127.08. Dentro del objeto conviene conservar el valor como número.

Requisitos:

  • usa .length para el total;
  • usa filter para contar productos activos;
  • usa filter para contar productos sin stock;
  • usa reduce para calcular el valor del inventario;
  • usa reduce para calcular el precio medio;
  • devuelve un objeto con los resultados.

Estructura orientativa:

function generarInformeProductos(listaProductos) {
  const totalProductos = listaProductos.length;

  const productosActivos = listaProductos.filter((producto) => {
    return producto.activo;
  }).length;

  const productosSinStock = listaProductos.filter((producto) => {
    return producto.stock === 0;
  }).length;

  const valorInventario = listaProductos.reduce((acumulador, producto) => {
    return acumulador + producto.precio * producto.stock;
  }, 0);

  const sumaPrecios = listaProductos.reduce((acumulador, producto) => {
    return acumulador + producto.precio;
  }, 0);

  const precioMedio =
    totalProductos === 0
      ? null
      : sumaPrecios / totalProductos;

  return {
    totalProductos,
    productosActivos,
    productosSinStock,
    valorInventario,
    precioMedio
  };
}

const informe = generarInformeProductos(productos);
console.log(informe);

const informeVacio = generarInformeProductos([]);
console.log(informeVacio);

La comprobación con un array vacío evita dividir entre cero. En ese caso, precioMedio debe ser null.

19. Resumen final

En esta práctica has usado arrays de objetos para representar información más parecida a la que aparece en una aplicación real.

La idea importante es que, cuando los datos empiezan a tener varias propiedades, ya no basta con arrays simples de números o cadenas. En esos casos usamos objetos y, cuando tenemos muchos elementos del mismo tipo, usamos arrays de objetos.

Este patrón será fundamental en los siguientes bloques, especialmente cuando trabajemos con DOM, eventos, formularios y datos recibidos desde APIs.
