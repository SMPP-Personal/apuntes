Práctica global

UD2_B2. Práctica global: Panel dinámico de recursos con DOM

1. Finalidad de la práctica

En esta práctica construirás una página web llamada Panel de recursos de DWCC.

La página partirá de una estructura HTML básica y de un array de objetos. A partir de esos datos, JavaScript deberá generar y actualizar distintas zonas del documento mediante el DOM.

Esta es una práctica global de cierre de UD2_B2. Debes aplicar de forma autónoma los procedimientos trabajados en los apuntes, ejercicios y práctica guiada anteriores.

No se proporciona una solución paso a paso. El objetivo es que organices el código, selecciones las operaciones adecuadas y compruebes el resultado.

2. Qué debes demostrar

Al finalizar la práctica debes demostrar que sabes:

  • seleccionar elementos mediante querySelector o querySelectorAll;
  • modificar textos existentes con textContent;
  • crear elementos con createElement;
  • insertar nodos con append;
  • asignar clases con classList.add();
  • añadir atributos con setAttribute();
  • limpiar contenedores antes de volver a renderizar;
  • recorrer arrays de objetos;
  • filtrar datos con filter();
  • reutilizar funciones;
  • generar distintas vistas a partir de los mismos datos;
  • comprobar el DOM generado con las herramientas de desarrollo.

3. Contenidos que no debes utilizar

En esta práctica todavía no debes emplear:

  • eventos;
  • addEventListener;
  • botones interactivos;
  • formularios;
  • validación;
  • localStorage o sessionStorage;
  • fetch;
  • promesas;
  • async/await;
  • APIs externas.

Tampoco debes usar innerHTML para generar tarjetas, resúmenes o listas.

El contenido dinámico se construirá con:

createElement + textContent + append

4. Contexto

El panel mostrará diferentes recursos del módulo de DWCC:

  • apuntes;
  • ejercicios;
  • prácticas;
  • materiales de repaso;
  • recursos revisados;
  • recursos pendientes;
  • recursos destacados.

Cada recurso estará representado mediante un objeto dentro de un array.

La página deberá mostrar cuatro zonas dinámicas:

  1. un resumen general;
  2. el listado completo de recursos;
  3. los recursos destacados;
  4. los recursos pendientes de revisión.

6. HTML base

Puedes utilizar la siguiente estructura para index.html:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Panel de recursos de DWCC</title>
  <link rel="stylesheet" href="css/estilos.css">
</head>
<body>
  <header class="cabecera">
    <h1 id="titulo-principal">Panel de recursos de DWCC</h1>
    <p id="subtitulo">Contenido pendiente de generar.</p>
  </header>

  <main class="contenedor">
    <section class="bloque-resumen">
      <h2>Resumen del panel</h2>
      <div id="resumen" class="resumen"></div>
    </section>

    <section class="bloque-recursos">
      <h2>Listado de recursos</h2>
      <div id="lista-recursos" class="grid-recursos"></div>
    </section>

    <section class="bloque-destacados">
      <h2>Recursos destacados</h2>
      <div id="recursos-destacados" class="grid-recursos"></div>
    </section>

    <section class="bloque-pendientes">
      <h2>Recursos pendientes de revisión</h2>
      <ul id="lista-pendientes"></ul>
    </section>
  </main>

  <footer class="pie">
    <p id="fecha-generacion"></p>
  </footer>

  <script src="js/app.js"></script>
</body>
</html>

Puedes modificar los textos o añadir elementos auxiliares, pero debes conservar una estructura equivalente y todos los contenedores dinámicos.

Desarrollo de la práctica

9. Actividad 1. Seleccionar y modificar elementos existentes

Selecciona desde JavaScript:

  • #titulo-principal;
  • #subtitulo;
  • #fecha-generacion;
  • #resumen;
  • #lista-recursos;
  • #recursos-destacados;
  • #lista-pendientes.

Modifica el título y el subtítulo mediante textContent.

El resultado debe transmitir que el contenido ha sido generado dinámicamente mediante JavaScript.

Selecciona los elementos una sola vez y guarda las referencias en constantes con nombres claros.

10. Actividad 2. Mostrar la fecha de generación

Muestra en el pie de página una fecha con este formato:

Página generada el día/mes/año

Por ejemplo:

Página generada el 23/6/2026

La fecha debe calcularse al cargar la página. No debes escribirla manualmente.

Debes utilizar:

  • new Date();
  • getDate();
  • getMonth();
  • getFullYear().

Recuerda que el valor obtenido con getMonth() empieza en 0.

11. Actividad 3. Crear una tarjeta de recurso

Crea una función con esta firma:

function crearTarjetaRecurso(recurso) {
  // Tu código
}

La función debe crear y devolver un elemento article.

Cada tarjeta debe incluir:

  • título;
  • bloque;
  • tipo;
  • dificultad;
  • estado de revisión.

El estado no debe mostrarse como true o false. Utiliza textos comprensibles:

Estado: revisado

o:

Estado: pendiente de revisión

Además, la tarjeta debe:

  • incluir la clase base tarjeta-recurso;
  • tener un atributo data-bloque con el código del bloque;
  • recibir la clase recurso-revisado o recurso-pendiente;
  • recibir también la clase recurso-destacado cuando corresponda;
  • devolverse mediante return.

Comprobación

Al inspeccionar una tarjeta destacada y revisada, deberías encontrar una estructura equivalente a:

<article
  class="tarjeta-recurso recurso-revisado recurso-destacado"
  data-bloque="B03">
  ...
</article>

La estructura interna puede variar, pero las clases y el atributo deben aparecer.

12. Actividad 4. Renderizar el listado completo

Crea una función con esta firma:

function renderizarRecursos(lista) {
  // Tu código
}

La función debe:

  1. limpiar #lista-recursos;
  2. comprobar si la lista está vacía;
  3. mostrar un mensaje si no hay recursos;
  4. recorrer la lista si contiene elementos;
  5. crear una tarjeta por cada recurso;
  6. insertar las tarjetas en el contenedor.

Utiliza crearTarjetaRecurso() para evitar repetir código.

Resultado esperado

Con el array inicial deben aparecer 6 tarjetas.

13. Actividad 5. Generar el resumen

Crea una función con esta firma:

function renderizarResumen(lista) {
  // Tu código
}

La función debe calcular:

  • total de recursos;
  • recursos revisados;
  • recursos pendientes;
  • recursos destacados.

Genera el resumen mediante elementos creados con createElement.

No escribas todo el resumen como una única cadena y no uses innerHTML.

Total de recursos 6

Recursos pendientes 1
Recursos destacados 3

Si obtienes otros resultados, revisa los filtros y las condiciones.

14. Actividad 6. Renderizar los recursos destacados

Crea una función con esta firma:

function renderizarDestacados(lista) {
  // Tu código
}

La función debe:

  1. filtrar los recursos destacados;
  2. limpiar #recursos-destacados;
  3. gestionar el caso de que no haya resultados;
  4. reutilizar crearTarjetaRecurso();
  5. insertar las tarjetas correspondientes.

Resultado esperado

Con los datos iniciales deben mostrarse 3 recursos destacados.

Es normal que un recurso destacado aparezca tanto en el listado general como en esta sección. Se trata de dos vistas distintas construidas a partir de los mismos datos.

15. Actividad 7. Renderizar los recursos pendientes

Crea una función con esta firma:

function renderizarPendientes(lista) {
  // Tu código
}

La función debe:

  1. filtrar los recursos con revisado: false;
  2. limpiar #lista-pendientes;
  3. crear un li por cada recurso pendiente;
  4. mostrar en cada elemento el bloque y el título;
  5. mostrar un único li con un mensaje si no hay pendientes.

Formato recomendado:

B05C — Objetos predefinidos

Resultado esperado

Con los datos iniciales debe aparecer 1 recurso pendiente.

16. Actividad 8. Coordinar el renderizado

Crea una función que coordine la actualización completa del panel:

function renderizarPanel(lista) {
  // Llama aquí a las funciones de renderizado
}

Esta función debe llamar a:

  • renderizarRecursos(lista);
  • renderizarResumen(lista);
  • renderizarDestacados(lista);
  • renderizarPendientes(lista).

Al final de app.js, la página debe iniciarse con una única llamada:

renderizarPanel(recursos);

La fecha y los textos iniciales también deben quedar actualizados al cargar la página.

17. Actividad 9. Comprobar los estados vacíos

Comprueba temporalmente el funcionamiento con:

renderizarPanel([]);

La página no debe mostrar errores.

Cada zona debe presentar un mensaje comprensible, por ejemplo:

  • No hay recursos para mostrar.
  • No hay recursos destacados.
  • No hay recursos pendientes de revisión.

El resumen debe mostrar valores iguales a 0.

Después de realizar la comprobación, restaura:

renderizarPanel(recursos);

No entregues la página mostrando el array vacío.

20. Restricciones técnicas

La solución debe cumplir todas estas condiciones:

  • utilizar querySelector o querySelectorAll;
  • utilizar textContent;
  • utilizar createElement;
  • utilizar append;
  • utilizar classList.add();
  • utilizar setAttribute();
  • limpiar los contenedores antes de renderizar;
  • reutilizar crearTarjetaRecurso();

  • organizar el código mediante funciones;

24. Errores frecuentes

24.1. Confundir asignación y comparación

Esta instrucción modifica el dato y no sirve para filtrarlo:

recurso.revisado = true

Para comprobar el valor puedes utilizar directamente:

recurso.revisado

24.2. No devolver la tarjeta

Si crearTarjetaRecurso() no incluye:

return tarjeta;

las funciones de renderizado recibirán undefined.

24.3. Duplicar contenido

Si no limpias un contenedor antes de renderizar, las tarjetas o los elementos de lista pueden aparecer repetidos.

24.4. Mostrar booleanos directamente

Evita mostrar:

Revisado: true

Transforma el dato en un texto comprensible para el usuario.

24.5. Repetir la creación de tarjetas

No escribas una versión de la tarjeta para el listado general y otra para destacados.

Utiliza en ambos casos:

crearTarjetaRecurso(recurso)

24.6. Aplicar clases excluyentes de forma incorrecta

Una tarjeta revisada y destacada debe conservar ambas clases:

recurso-revisado recurso-destacado

La clase de destacado no sustituye a la clase de estado.

24.7. Olvidar gestionar listas vacías

Un array vacío no debe causar errores ni dejar zonas sin explicación.

24.8. Utilizar innerHTML

Aunque el resultado visual parezca correcto, una solución basada en innerHTML no cumple el objetivo de esta práctica.

  1. el título y el subtítulo se actualizan;
  2. se muestra la fecha de generación;
  3. aparecen seis tarjetas en el listado general;
  4. el resumen muestra los cuatro cálculos correctos;
  5. aparecen tres tarjetas en la sección de destacados;
  6. aparece un recurso en la lista de pendientes;
  7. las tarjetas tienen clases y atributos generados desde JavaScript;
  8. la consola no presenta errores;
  9. el inspector permite comprobar que el DOM se ha construido dinámicamente.
