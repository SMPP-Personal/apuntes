UD2_B2b. Ejercicios: BOM básico en JavaScript

1. Objetivo de la hoja

Esta hoja permite practicar el uso básico del BOM (Browser Object Model) desde JavaScript.

Trabajarás con:

  • window;
  • alert;
  • confirm;
  • prompt;
  • location;
  • history;
  • navigator;
  • screen;
  • setTimeout;
  • setInterval;
  • clearTimeout;
  • clearInterval;
  • integración básica entre BOM y DOM.

Al finalizar la hoja deberías ser capaz de:

  • consultar información básica de la ventana, la pantalla, la URL y el navegador;
  • mostrar esa información en la consola y en la página;
  • utilizar diálogos modales con control de cancelación y entradas vacías;
  • usar temporizadores de una sola ejecución y temporizadores repetidos;
  • cancelar temporizadores correctamente;
  • evitar que varios ejercicios interfieran entre sí;
  • organizar el código en funciones claras;
  • detectar y corregir errores frecuentes relacionados con el BOM.

Importante: estos ejercicios deben probarse en una página HTML abierta en el navegador. No deben ejecutarse solamente con Node.js, porque objetos como window, location, history, navigator o screen pertenecen al entorno del navegador.

2. Organización y preparación

2.1. Estructura del proyecto

Crea una carpeta llamada:

UD2_B2b_BOM_basico

Dentro crea estos archivos:

UD2_B2b_BOM_basico/
├── index.html
└── script.js

Puedes trabajar abriendo directamente index.html, pero es recomendable usar Live Server u otro servidor local.

Si abres el archivo directamente desde el disco:

  • location.protocol puede devolver file:;
  • location.hostname puede estar vacío;
  • la URL tendrá un formato diferente al de una página servida mediante http:// o https://.

Esto no significa necesariamente que el código esté mal.

2.3. Organización de script.js

Cada ejercicio debe estar identificado y organizado en una función propia.

Ejemplo:

function ejecutarEjercicio1() {
  // Código del ejercicio 1
}

function ejecutarEjercicio2() {
  // Código del ejercicio 2
}

No ejecutes automáticamente todos los ejercicios al cargar la página.

Los ejercicios que utilicen alert(), confirm(), prompt(), redirecciones o temporizadores deben iniciarse mediante botones para evitar:

  • una cadena de ventanas emergentes;
  • varias cuentas atrás funcionando al mismo tiempo;
  • resultados que se sobrescriben;
  • redirecciones inesperadas;
  • múltiples temporizadores activos.

2.4. Eventos usados como apoyo

Esta hoja se centra en el BOM. Si todavía no se ha trabajado el bloque de eventos, puedes usar esta estructura como apoyo:

const boton = document.querySelector("#mi-boton");

boton.addEventListener("click", () => {
  ejecutarEjercicio();
});

El objetivo principal es practicar el BOM, no evaluar todavía el dominio completo de los eventos.

2.5. Reglas de funcionamiento conjunto

Todos los ejercicios deben poder permanecer en el mismo proyecto.

Debes:

  • usar identificadores diferentes para los elementos de cada ejercicio;
  • evitar declarar varias veces variables globales con el mismo nombre;
  • guardar los identificadores de los temporizadores que deban cancelarse;
  • impedir que se inicien varias copias del mismo temporizador;
  • actualizar únicamente la zona de salida correspondiente;
  • mantener el código organizado y comentado.

Ejercicio 2. Diferencia entre ventana y pantalla

  1. Muestra por consola:

Parte 2. Diálogos básicos

Ejercicio 3. Mensaje con alert()

Añade al HTML:

<button type="button" id="boton-alert">Mostrar bienvenida</button>

En JavaScript:

  1. Selecciona el botón.
  2. Al pulsarlo, muestra un mensaje de bienvenida con alert().
  3. Después de cerrar el diálogo, muestra por consola:
El usuario cerró el mensaje de bienvenida
  1. Muestra también ese texto en #salida-parte-2.

No ejecutes el alert() automáticamente al cargar la página.

Ejercicio 4. Confirmar una acción

Añade al HTML:

<button type="button" id="boton-confirmar">Confirmar continuación</button>

Al pulsar el botón:

  1. Usa confirm() para preguntar:
¿Quieres continuar con la práctica?
  1. Si el usuario acepta, muestra en la página:
Has decidido continuar.
  1. Si cancela, muestra:
Has cancelado la práctica.
  1. Guarda el valor devuelto por confirm() en una variable.
  2. Añade un comentario indicando qué valor devuelve al aceptar y qué valor devuelve al cancelar.

Ejercicio 5. Pedir un nombre con prompt()

Añade al HTML:

<button type="button" id="boton-pedir-nombre">Pedir nombre</button>

Al pulsar el botón:

  1. Pide al usuario su nombre con prompt().
  2. Si el usuario escribe un nombre, muestra:
Hola, Ana
  1. Si el usuario cancela, muestra:
No has escrito ningún nombre.
  1. Si el usuario escribe solo espacios, usa trim() para tratarlo como nombre vacío.
  2. No llames a trim() antes de comprobar si el resultado es null.

Ejercicio 6. Pedir y validar una edad

Añade al HTML:

<button type="button" id="boton-pedir-edad">Comprobar edad</button>

Al pulsar el botón:

  1. Pide al usuario su edad con prompt().
  2. Si el usuario cancela, muestra:
Operación cancelada.
  1. Si escribe una cadena vacía o solo espacios, muestra:
No has introducido una edad.
  1. Convierte el valor a número con Number().
  2. Comprueba que:
    • no sea NaN;
    • sea un número entero;
    • sea mayor o igual que 0.
  3. Si no es una edad válida, muestra:
La edad no es válida.
  1. Si es menor de 18, muestra:
Eres menor de edad.
  1. Si es 18 o mayor, muestra:
Eres mayor de edad.

El orden de las comprobaciones debe ser:

  1. cancelación;
  2. entrada vacía;
  3. conversión;
  4. validación numérica;
  5. comparación de edad.

Ejercicio 8. Botón de recarga controlada

Añade al HTML:

Ejercicio 12. Información declarada por el navegador

Muestra por consola:

navigator.userAgent

Después responde en un comentario:

  1. ¿Qué tipo de información aparece?
  2. ¿Por qué no conviene basar toda la lógica de una aplicación en userAgent?

Añade una comprobación de funcionalidad sencilla:

Parte 6. Temporizadores

Ejercicio 13. Mensaje retardado con setTimeout()

Añade al HTML:

<button type="button" id="boton-mensaje-retardado">
  Iniciar mensaje retardado
</button>

<p id="mensaje-retardado"></p>

Al pulsar el botón:

  1. muestra:

El mensaje conserva la salida esperada del ejercicio original, pero recuerda que setTimeout() no garantiza una ejecución exacta: la función puede ejecutarse algo más tarde.

  1. evita iniciar varios temporizadores simultáneos al pulsar repetidamente el botón.

Parte 8. Corrige los errores

Ejercicio 18. Corrige este setTimeout()

Código incorrecto:

function saludar() {
  console.log("Hola");
}

setTimeout(saludar(), 2000);
  1. Corrige el código.
  2. Explica qué error había.
  3. Indica la diferencia entre:
    • saludar;
    • saludar().

Ejercicio 19. Corrige este prompt()

Código incorrecto:

const numero = prompt("Escribe un número");
console.log(numero + 5);

Corrige el código para que:

  1. sume numéricamente;
  2. controle si el usuario cancela;
  3. controle una cadena vacía;
  4. controle una entrada formada solo por espacios;
  5. controle un valor que no sea numérico;
  6. solo realice la suma cuando el dato sea válido.

Ejercicio 20. Corrige este intervalo

Código problemático:

setInterval(() => {
  console.log("Mensaje infinito");
}, 1000);
  1. Modifícalo para que se detenga después de 5 repeticiones.
  2. Guarda el identificador devuelto por setInterval().
  3. Usa clearInterval().
  4. Muestra el número de repetición.
  5. Muestra un mensaje final cuando se detenga.
  6. Explica por qué el código original podía seguir ejecutándose indefinidamente.

Indicadores de corrección

Uso correcto del BOM

Se valorará:

  • consulta adecuada de window, screen, location, history y navigator;
  • interpretación correcta de los valores;
  • ausencia de afirmaciones incorrectas sobre conexión o resolución de pantalla;
  • redirecciones y recargas controladas.

Temporizadores

Se valorará:

  • uso correcto de setTimeout() y setInterval();
  • almacenamiento de identificadores;
  • cancelación con clearTimeout() y clearInterval();
  • prevención de temporizadores duplicados;
  • finalización correcta de contadores y cuentas atrás.

Integración con DOM

Se valorará:

  • selección correcta de elementos;
  • uso de textContent;
  • zonas de salida independientes;
  • botones correctamente conectados;
  • resultados visibles y comprensibles.

Validación y casos especiales

Se valorará:

  • control de null;
  • control de cadenas vacías;
  • uso correcto de trim();
  • conversión numérica segura;
  • gestión de cancelaciones;
  • mensajes claros para el usuario.

Claridad, organización y comentarios

Se valorará:

  • una función por ejercicio;
  • nombres descriptivos;
  • ausencia de variables globales duplicadas;
  • ausencia de interferencias entre ejercicios;
  • funcionamiento de la mini-entrega sin comentar código anterior;
  • formato legible;
  • indentación coherente;
  • comentarios breves y útiles;
  • nombres de variables y funciones comprensibles;
  • ausencia de código innecesario o repetido.

La organización conjunta y la ausencia de interferencias forman parte de este criterio y también afectan a la corrección funcional de los apartados anteriores.

Comprobación final del funcionamiento

Antes de entregar, comprueba que:

  • la página carga sin errores en consola;
  • los botones ejecutan únicamente el ejercicio correspondiente;
  • los temporizadores pueden detenerse;
  • las zonas de salida no se sobrescriben entre sí;
  • la mini-entrega funciona sin comentar ni eliminar código anterior.
