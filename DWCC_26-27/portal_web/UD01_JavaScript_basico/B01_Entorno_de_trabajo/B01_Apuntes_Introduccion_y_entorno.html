B01 – Introducción y entorno de trabajo con JavaScript en DWCC

Qué papel juega JavaScript en el desarrollo web del lado cliente, cómo se ejecuta en el navegador y cómo preparar un entorno básico de trabajo.

Introducción

En el módulo Desarrollo Web en Entorno Cliente (DWCC) utilizamos JavaScript para dar comportamiento a las páginas web desde el navegador del usuario. Este lenguaje se combina con HTML (estructura) y CSS (presentación) para construir aplicaciones web completas del lado cliente.

En este primer bloque veremos qué papel juega JavaScript dentro de una aplicación web, por qué es tan importante hoy en día y cómo preparar un entorno de trabajo básico para empezar a programar. Todo ello sienta las bases para el resto del módulo y para poder avanzar, más adelante, hacia librerías y frameworks como React.

Nota: En algunos materiales previos puede aparecer la sigla DWEC. En esta versión se unifica la terminología a DWCC para mantener la coherencia con el módulo y con el portal de apuntes.

Objetivos

Al finalizar este bloque, el alumnado deberá ser capaz de:

  • Explicar el papel que juega JavaScript en una aplicación web del lado cliente dentro del módulo DWCC.
  • Distinguir el rol de HTML, CSS y JavaScript en una página web.
  • Entender dónde y cómo se ejecuta JavaScript en el navegador (motor JavaScript).
  • Utilizar la consola del navegador para ejecutar instrucciones simples y ver errores.
  • Crear una estructura mínima de trabajo con un archivo HTML y un archivo JavaScript enlazado mediante <script src="...">.
  • Organizar sus archivos en carpetas de práctica de forma clara y coherente.
  • Usar console.log() para mostrar mensajes y empezar a depurar su propio código.

1. JavaScript en el contexto de DWCC

JavaScript es el lenguaje de programación que utilizamos en el módulo Desarrollo Web en Entorno Cliente para dar comportamiento a las páginas web desde el navegador.

Lenguaje Responsabilidad Ejemplos
HTML Estructura del documento Etiquetas, secciones, formularios, encabezados
CSS Presentación visual Colores, tamaños, distribución, diseño responsive
JavaScript Comportamiento dinámico Eventos, validación, cambios de contenido, carga de datos

En DWCC nos centramos en el JavaScript del lado cliente, es decir, el que se ejecuta en el navegador del usuario, no en el servidor. Esto se corresponde con el currículo oficial del ciclo y con los resultados de aprendizaje relacionados con la sintaxis, las sentencias simples y la prueba y depuración básica del código JavaScript.

2. ¿Por qué JavaScript es tan importante?

Algunas razones clave por las que JavaScript es tan importante en desarrollo web:

  • Es el estándar de facto para programar en el lado cliente en la web.
  • Está presente en todos los navegadores modernos, sin necesidad de instalar nada adicional en el equipo del usuario.
  • Es la base sobre la que se apoyan muchas librerías y frameworks modernos, como React, Vue o Angular.

Aunque en este bloque nos centramos en JavaScript "puro" (JavaScript nativo del navegador), es importante tener en mente que más adelante estos conceptos servirán para entender mejor cómo funcionan los frameworks modernos.

3. Dónde y cómo se ejecuta JavaScript

JavaScript se ejecuta en el motor de JavaScript del navegador. Por ejemplo, Chrome y otros navegadores basados en Chromium utilizan V8, mientras que Firefox utiliza SpiderMonkey. Cada navegador implementa su propio motor, pero todos siguen el estándar de JavaScript, conocido como ECMAScript.

En esta primera unidad trabajaremos principalmente en dos escenarios:

  1. Consola del navegador.
  2. Ficheros .js enlazados a una página HTML sencilla.

Ambos enfoques son complementarios: la consola nos permite hacer pruebas rápidas y los ficheros .js nos acercan más a la estructura real de un proyecto.

4. Consola de desarrollo del navegador

Todos los navegadores modernos incluyen herramientas de desarrollo (DevTools) que nos permiten:

  • Ejecutar instrucciones de JavaScript de una en una.
  • Ver errores de ejecución.
  • Inspeccionar valores de variables y mensajes de depuración.

Para abrir la consola, los atajos más habituales son:

Chrome / Edge / Firefox: tecla F12 o combinación Ctrl + Shift + I, y después seleccionar la pestaña Console.

En la consola podemos escribir directamente instrucciones como estas:

2 + 2
"Hola" + " mundo"
console.log("Probando JavaScript en la consola")

Esto es muy útil en esta unidad para probar rápidamente expresiones, estructuras de control, funciones y arrays sin necesidad de montar todavía un proyecto completo.

Buena práctica: Utilizar la consola como espacio de prueba rápida antes de llevar el código a un archivo .js ayuda a detectar errores simples con más rapidez.

5. Ficheros JavaScript en un proyecto sencillo

Además de la consola, nos interesa trabajar desde el principio con ficheros .js que podamos reutilizar. Así nos acostumbramos a una estructura similar a la de proyectos reales.

Una estructura mínima de un proyecto de práctica podría ser:

  • Una carpeta de trabajo, por ejemplo, ud1-js-basico/.
  • Un archivo HTML muy simple, por ejemplo, index.html.
  • Un archivo JavaScript, por ejemplo, script.js.

Ejemplo de HTML mínimo para probar código JavaScript:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Pruebas JavaScript UD1</title>
</head>
<body>
  <h1>Pruebas de JavaScript básico</h1>

  <script src="script.js"></script>
</body>
</html>

En el archivo script.js escribiremos el código que queremos ejecutar cuando se cargue la página:

console.log("Hola desde script.js");

Al abrir index.html en el navegador y abrir la consola, deberíamos ver el mensaje anterior.

Advertencia: Si el archivo JavaScript no está correctamente enlazado con <script src="script.js"></script>, el navegador no ejecutará el código aunque el contenido del archivo sea correcto.

6. Entorno básico de trabajo

6.1. Editor de código o IDE

Para desarrollar JavaScript de forma cómoda, utilizaremos un editor de código o IDE adaptado al trabajo con HTML, CSS y JavaScript.

  • Un editor de código ofrece resaltado de sintaxis, sangrado, ayudas básicas y extensiones.
  • No es un procesador de textos: trabajamos siempre con texto plano, sin formato.

En DWCC, los archivos que manipularemos principalmente en la UD1 serán:

  • .html para la estructura básica de la página.
  • .js para el código JavaScript.

Más adelante, en unidades posteriores, añadiremos CSS y trabajaremos más en profundidad con el DOM y con conceptos que nos prepararán para usar librerías como React.

6.2. Organización mínima de archivos

Desde esta unidad es recomendable que el alumnado adquiera una estructura mínima de trabajo:

  • Cada práctica en una carpeta propia, por ejemplo, ud1-ejercicios/ejercicio01/.
  • Nombres de archivos significativos: index.html para el punto de entrada y main.js o script.js para el código JavaScript.

Buena práctica: Esta organización facilita la corrección y la continuidad del trabajo durante el módulo, y se parece a cómo se organizan los proyectos en entornos profesionales.

6.3. Uso básico de la consola para depurar

En JavaScript, una de las herramientas de depuración más habituales es el objeto console.

  • console.log(valor) muestra información en la consola.
  • Lo utilizaremos constantemente para ver el valor de variables, comprobar si una parte de código se está ejecutando y entender errores.
let a = 5;
let b = 3;
const resultado = a + b;

console.log("El resultado es:", resultado);

Además de los mensajes manuales, la consola muestra:

  • Errores de sintaxis, por ejemplo, llaves o paréntesis sin cerrar.
  • Errores de ejecución, por ejemplo, intentar usar una variable que no existe.

Nota: console.log() es una herramienta de apoyo para aprender y depurar. Más adelante se introducirán otras herramientas, pero en este bloque es suficiente y muy útil.

Ejemplos

Ejemplo 1: Probando expresiones en la consola

Contexto: Familiarizarse con la consola del navegador como herramienta rápida para comprobar operaciones sencillas.

2 + 2
"Hola" + " mundo"
console.log("Probando JavaScript en la consola");
  • 2 + 2 evalúa una operación aritmética simple.
  • "Hola" + " mundo" concatena dos cadenas de texto.
  • console.log(...) envía un mensaje a la consola para que podamos verlo.

Resultado esperado: En la consola veremos los resultados de las expresiones (4, "Hola mundo") y el mensaje "Probando JavaScript en la consola".

Relación con React: Más adelante, cuando trabajemos con React, usaremos también la consola del navegador para entender qué está pasando en nuestros componentes, especialmente al principio.

Ejemplo 2: HTML mínimo con script enlazado

Contexto: Crear la estructura mínima para ejecutar código JavaScript desde un archivo .js en lugar de escribirlo directamente en la consola.

Archivo index.html:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Pruebas JavaScript UD1</title>
</head>
<body>
  <h1>Pruebas de JavaScript básico</h1>

  <script src="script.js"></script>
</body>
</html>

Archivo script.js:

console.log("Hola desde script.js");

Explicación: El navegador carga index.html. Cuando llega a la etiqueta <script src="script.js"></script>, carga y ejecuta el contenido de script.js.

Resultado esperado: Al abrir index.html en el navegador y abrir la consola, veremos el mensaje "Hola desde script.js".

Relación con React: En React también tendremos archivos de JavaScript o TypeScript que se cargan como parte de la aplicación. Entender cómo se enlaza y ejecuta código JavaScript en una página es un paso previo necesario.

Ejemplo 3: Uso de console.log() para depurar

Contexto: Utilizar console.log() para entender qué valor tienen las variables y comprobar que el código funciona.

let a = 5;
let b = 3;
const resultado = a + b;

console.log("El resultado es:", resultado);

Explicación: Se definen dos variables (a y b) y se calcula su suma en resultado. Luego, console.log() muestra un mensaje junto con el valor de resultado.

Resultado esperado:

El resultado es: 8

Relación con React: En React también utilizaremos console.log() al principio para entender cómo cambian los estados y qué datos llegan a un componente.

Errores comunes

Error 1 — La página "no hace nada" pero hay un error en la consola

Pensar que si la página "no hace nada" no está pasando nada. Muchas veces el error se está mostrando en la consola, pero el alumnado no la mira.

Error 2 — Archivo .js no enlazado en el HTML

Escribir código JavaScript en un archivo .js, pero olvidarse de enlazarlo en el HTML con <script src="..."></script>.

Error 3 — Editar el archivo equivocado o no guardar antes de recargar

Guardar cambios en el archivo y recargar la página, pero no ver resultados porque se está editando un archivo distinto del que realmente carga el navegador o porque no se ha guardado antes de recargar.

Error 4 — Confundir editor de código con procesador de textos

Esto puede provocar problemas con comillas tipográficas u otros caracteres no válidos en el código.

Buenas prácticas

Acostumbrarse desde el primer día a mirar la consola del navegador cuando algo no funciona.

Usar console.log() para comprobar el valor de variables y el flujo de ejecución del código.

Organizar cada práctica en una carpeta propia (ud1-ejercicios/ejercicio01/, etc.) y utilizar nombres de archivo significativos (index.html, main.js, script.js).

Trabajar siempre con un editor de código como VS Code, y no con procesadores de texto, para evitar problemas de formato y aprovechar el resaltado de sintaxis.

Empezar desde el principio usando sintaxis moderna de JavaScript (let, const y funciones bien estructuradas), para facilitar más adelante el salto a frameworks como React.

Resumen

En este bloque hemos visto el papel de JavaScript en el desarrollo web del lado cliente dentro del módulo DWCC, en relación con HTML (estructura) y CSS (presentación). También hemos aprendido dónde y cómo se ejecuta JavaScript en el navegador y hemos utilizado la consola como herramienta para probar instrucciones simples y ver errores.

Además, hemos preparado un entorno básico de trabajo: un editor de código adecuado, una organización mínima de archivos y una estructura de proyecto sencilla con un archivo HTML y un archivo JavaScript enlazado. Por último, hemos insistido en el uso de console.log() y de la consola del navegador como herramientas fundamentales de depuración desde el primer día, algo que seguirá siendo clave cuando avancemos hacia aplicaciones más complejas y hacia el uso de librerías como React.
