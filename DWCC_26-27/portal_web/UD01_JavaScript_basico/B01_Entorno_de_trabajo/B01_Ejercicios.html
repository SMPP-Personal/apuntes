Actividades Bloque 1. Introducción y entorno de trabajo con JavaScript

Introducción

En este documento se recogen una serie de actividades prácticas para el Bloque 1 del módulo DWCC. Su objetivo es que el alumnado se familiarice con la consola del navegador, el uso de archivos JavaScript enlazados a HTML, la interpretación de errores básicos y la organización mínima de un pequeño proyecto.

Estas actividades están pensadas para reforzar el RA2 del currículo oficial: sentencias simples en JavaScript, sintaxis, prueba y depuración básica del código.

Objetivos

Al finalizar las actividades de este bloque, el alumnado debería ser capaz de:

  • Crear una estructura mínima de proyecto con HTML y un archivo script.js enlazado.
  • Usar la consola del navegador para ejecutar y comprobar código JavaScript.
  • Interpretar los mensajes de error básicos que aparecen en la consola.
  • Utilizar console.log de forma sistemática para depurar.
  • Trabajar con variables simples, concatenaciones, tipos de datos y operaciones básicas.
  • Responder a un cuestionario corto sobre conceptos de entorno y ejecución de JavaScript.
  • Realizar una pequeña práctica integradora que combine HTML, JavaScript y consola.

Nota: Puedes adaptar la dificultad y el ritmo de estas actividades según el grupo, haciendo algunas de ellas obligatorias y dejando otras como ampliación o refuerzo.

1. Actividad 1 – Primer contacto con la consola y script.js

Objetivo: Que el alumnado use la consola del navegador y un archivo .js enlazado a un HTML mínimo.

Enunciado orientativo:

  1. Crea una carpeta llamada ud1-bloque1-ej1.
  2. Dentro, crea un archivo index.html con un título “Pruebas JavaScript UD1”.
  3. Crea un archivo script.js y enlázalo desde el HTML con la etiqueta de script antes de </body>: 
    <script src="script.js"></script>
  4. En script.js, escribe el siguiente código y guarda el archivo: 
    console.log("Bienvenido a JavaScript en DWCC");
console.log("Este mensaje se muestra en la consola del navegador");
  5. Abre index.html en el navegador, abre las herramientas de desarrollador y la pestaña Consola. Comprueba que aparecen los mensajes.

Buena práctica: Coloca siempre la etiqueta <script src="..."></script> al final del <body> en estos ejemplos iniciales para asegurarte de que el HTML se ha cargado antes de ejecutar el JavaScript.

Variantes / rúbrica simple (para RA2):

  • 0: No funciona o no se ha enlazado el script.
  • 1: Script enlazado pero no se usa console.log correctamente.
  • 2: Script enlazado y muestra al menos dos mensajes bien formateados en la consola.

Nota: Puedes pedir al alumnado que modifique los mensajes para personalizarlos (por ejemplo, incluyendo su nombre o el ciclo que cursan).

2. Actividad 2 – Identificación de errores usando la consola

Objetivo: Empezar a usar la consola como herramienta de depuración.

Enunciado orientativo:

Dado este código en script.js:

console.log("Inicio del ejercicio");
let numero = 10
console.log("El número es: " numero);
console.log("Fin del ejercicio");
  1. Copia el código en tu proyecto y abre la consola.
  2. Observa qué error aparece en la consola e intenta interpretarlo.
  3. Corrige el código para que funcione correctamente.
  4. Añade un cuarto console.log que muestre el doble de numero.

Error común: Pensar que el problema es solo “falta un punto y coma”. En este caso, el error principal está en la concatenación de la cadena y la variable sin usar ningún operador.

Buena práctica: Lee siempre el mensaje de error completo (tipo de error, línea, columna) antes de empezar a tocar código al azar.

Puntos clave a trabajar:

  • Entender que la falta de punto y coma no siempre es el problema principal.
  • Detectar el error real en la concatenación: falta el operador +.
  • Interpretar el mensaje de error (línea, tipo de error) que da la consola.
  • Comprobar el resultado final una vez corregido el código.

3. Actividad 3 – Mini‑ficha de prácticas en consola

Objetivo: Practicar consola y primeras sentencias simples sin entrar aún en estructuras de control complejas.

Formato sugerido: Hoja de ejercicios cortos donde el alumnado debe ir ejecutando en consola y anotando resultados.

  1. Escribe en la consola: 
    console.log("Mi nombre es", "Ana");
    • ¿Qué se muestra exactamente en la consola?
  2. Declara una variable curso con el valor "DWCC" y muestra por consola el mensaje "Estoy cursando DWCC", usando la variable.
    • Pista: Puedes combinar texto fijo y la variable en el mismo console.log.
  3. Declara dos variables a y b con valores numéricos y muestra por consola:
    • La suma.
    • La resta.
    • El producto.
  4. Escribe el siguiente código en la consola: 
    let x = "10";
let y = 2;
console.log(x + y);
console.log(Number(x) + y);
    • Anota la diferencia entre ambos resultados.
  5. Usa typeof en al menos tres valores diferentes (42, "hola", true) y escribe qué devuelve la consola.

Nota: Esta ficha se puede hacer en parejas, pidiendo que comparen y comenten las respuestas antes de la puesta en común.

Buena práctica: Cuando no entiendas por qué JavaScript devuelve un resultado concreto, prueba a usar typeof para ver de qué tipo son los valores que estás usando.

4. Actividad 4 – Cuestionario corto / test de Bloque 1

Objetivo: Verificar comprensión básica de conceptos de entorno y ejecución.

Algunos ejemplos de preguntas tipo test (puedes adaptarlas o ampliarlas):

  1. ¿Dónde se ejecuta el JavaScript del módulo DWCC en esta unidad?

    • a) En el servidor web
    • b) En el navegador del usuario
    • c) En la base de datos
    • d) En el sistema operativo directamente

  2. ¿Qué herramienta del navegador usamos para ver mensajes de console.log y errores?

    • a) Inspector de red
    • b) Panel de almacenamiento
    • c) Consola de desarrollador
    • d) Administrador de tareas

  3. ¿Cuál de las siguientes opciones es la forma correcta de enlazar un archivo script.js al HTML?

    • a) <js href="script.js"></js>
    • b) <script src="script.js"></script>
    • c) <link rel="script" href="script.js">
    • d) <script link="script.js"></script>

  4. ¿Cuál de estas instrucciones mostrará un mensaje en la consola del navegador?

    • a) console.print("Hola");
    • b) log.console("Hola");
    • c) console.log("Hola");
    • d) alert.console("Hola");

  5. ¿Qué afirmación es correcta sobre console.log?

    • a) Sirve para modificar el contenido del HTML.
    • b) Sirve para mostrar mensajes en la consola, útil para depuración.
    • c) Sirve para guardar datos en una base de datos.
    • d) Sirve para cambiar el estilo CSS.

Nota: Puedes pasar este cuestionario en papel, en un formulario online o integrarlo en una plataforma tipo Moodle / Aula Virtual.

Buena práctica: Tras corregir el cuestionario, revisa con el grupo las preguntas que hayan fallado más y enlázalas con ejemplos en consola.

5. Actividad 5 – Pequeña práctica integradora del Bloque 1

Objetivo: Que el alumnado combine HTML mínimo + JavaScript + consola en una práctica sencilla.

Enunciado orientativo:

Crea una práctica llamada presentacion-js con:

  1. Un archivo index.html que muestre un título “Presentación con JavaScript”.
  2. Un archivo presentacion.js enlazado desde el HTML.
  3. En presentacion.js, escribe código que:
    • Declare variables con tu nombre, apellidos y el ciclo que cursas.
    • Use console.log para mostrar una presentación con varias líneas, por ejemplo:
    • "Hola, me llamo ..."
    • "Estoy estudiando ..."
    • Calcule tu edad aproximada a partir de tu año de nacimiento (variable fija) y la muestre en consola.
  4. Abre la página en el navegador y comprueba en la consola que toda la información se muestra correctamente.

Buena práctica: Empieza siempre calculando valores en variables (por ejemplo, edad) y luego muestra esas variables con console.log, en lugar de meter todos los cálculos directamente dentro del console.log.

Nota: Esta práctica se puede utilizar como mini‑presentación del alumnado en las primeras sesiones, combinando presentación personal y práctica de consola.

Resumen

En esta hoja de actividades del Bloque 1 se trabaja de manera progresiva:

  • El uso de la consola del navegador y el enlace de un archivo script.js a un HTML mínimo.
  • La interpretación de errores a través de ejemplos donde el código está mal a propósito.
  • La práctica de variables, concatenación y tipos de datos en la consola.
  • La comprobación de conceptos clave mediante un cuestionario corto.
  • Una práctica integradora que combina HTML, JavaScript y consola para crear una pequeña presentación personal.

Estas actividades refuerzan los contenidos teóricos del bloque y ayudan al alumnado a adquirir hábitos de trabajo que serán fundamentales en unidades posteriores y en el uso de librerías como React.
