Bloque 5B. Métodos avanzados de arrays en JavaScript

1. Qué vamos a trabajar en este bloque

En el bloque anterior estudiaste los arrays básicos:

  • crear arrays;
  • acceder a sus elementos mediante índices;
  • modificar posiciones;
  • usar length;
  • recorrer arrays con for;
  • usar métodos básicos como push(), pop(), shift(), unshift(), includes() e indexOf();
  • crear funciones que reciben arrays.

En este bloque vamos a dar un paso más: aprenderemos a usar métodos de array que reciben funciones como argumento.

Los métodos principales serán:

Método Uso principal Resultado
forEach() Recorrer un array y hacer algo con cada elemento No crea un nuevo array
map() Transformar todos los elementos Crea un nuevo array
filter() Seleccionar algunos elementos Crea un nuevo array
reduce() Acumular todos los elementos en un único resultado Devuelve un único valor

También veremos slice() como método complementario para copiar o extraer partes de un array.

Idea clave: en este bloque no vamos a sustituir completamente el bucle for. Vamos a aprender cuándo conviene usar métodos de array y qué aporta cada uno.

2. Recordatorio: funciones, callbacks y funciones flecha

En JavaScript una función puede pasarse como argumento a otra función.

Cuando una función se pasa a otra función para que esta la ejecute cuando corresponda, se suele llamar callback.

En los métodos de arrays de este bloque, el callback se ejecuta de forma síncrona, una vez por cada elemento del array.

Ejemplo sencillo:

function mostrarMensaje() {
  console.log("Hola");
}

function ejecutarFuncion(callback) {
  callback();
}

ejecutarFuncion(mostrarMensaje);

En los métodos avanzados de arrays ocurre algo parecido: el método recorre el array y ejecuta una función para cada elemento.

Por ejemplo:

const numeros = [1, 2, 3];

numeros.forEach((numero) => {
  console.log(numero);
});

Aquí:

  • forEach() es el método del array;
  • (numero) => { ... } es una función flecha;
  • esa función se ejecuta una vez por cada elemento;
  • numero representa el elemento actual del array en cada vuelta.

3. Del bucle for a los métodos de array

Hasta ahora, para recorrer un array usábamos un bucle for:

const nombres = ["Ana", "Luis", "María"];

for (let i = 0; i < nombres.length; i++) {
  console.log(nombres[i]);
}

Con forEach() podemos expresar la misma idea de otra forma:

const nombres = ["Ana", "Luis", "María"];

nombres.forEach((nombre) => {
  console.log(nombre);
});

Las dos versiones recorren el array.

La diferencia es que con for trabajamos manualmente con el índice i, mientras que con forEach() JavaScript se encarga de recorrer el array y nos entrega cada elemento.

¿Significa esto que for ya no se usa?

No.

El bucle for sigue siendo útil, especialmente cuando:

  • necesitamos controlar el índice manualmente;
  • queremos detener el bucle antes de llegar al final;
  • estamos empezando y queremos entender bien cómo se recorre un array.

Los métodos de array son útiles porque permiten escribir código más expresivo cuando ya tenemos claro qué queremos hacer con los datos.

Nota Con forEach() no se puede usar break para cortar el recorrido. Si necesitas detener un recorrido manualmente, al principio suele ser más claro usar un bucle for.

4. forEach(): recorrer un array

El método forEach() permite recorrer un array y ejecutar una acción por cada elemento.

4.1. Sintaxis básica

array.forEach((elemento) => {
  // instrucciones que se ejecutan por cada elemento
});

Ejemplo:

const dias = ["Lunes", "Martes", "Miércoles", "Jueves", "Viernes"];

dias.forEach((dia) => {
  console.log(dia);
});

Resultado:

Lunes
Martes
Miércoles
Jueves
Viernes

4.2. Usar también el índice

La función que recibe forEach() puede tener un segundo parámetro: el índice.

const dias = ["Lunes", "Martes", "Miércoles", "Jueves", "Viernes"];

dias.forEach((dia, indice) => {
  console.log("Día", indice + 1, ":", dia);
});

Resultado:

Día 1 : Lunes
Día 2 : Martes
Día 3 : Miércoles
Día 4 : Jueves
Día 5 : Viernes

Usamos indice + 1 porque los índices de los arrays empiezan en 0.

4.3. Importante: forEach() no crea un nuevo array

forEach() sirve para recorrer el array y ejecutar una acción, por ejemplo mostrar datos por consola.

No debe usarse cuando queremos obtener un nuevo array transformado.

Ejemplo:

const numeros = [1, 2, 3];

const resultado = numeros.forEach((numero) => {
  return numero * 2;
});

console.log(resultado); // undefined

Aunque dentro del forEach() haya un return, el método completo no devuelve un nuevo array.

Para transformar elementos y obtener un nuevo array debemos usar map().

5. map(): transformar todos los elementos

El método map() crea un nuevo array aplicando una transformación a cada elemento del array original.

5.1. Sintaxis básica

const nuevoArray = arrayOriginal.map((elemento) => {
  return nuevoValor;
});

Ejemplo:

const numeros = [1, 2, 3, 4];

const dobles = numeros.map((numero) => {
  return numero * 2;
});

console.log(dobles);  // [2, 4, 6, 8]
console.log(numeros); // [1, 2, 3, 4]

map() no modifica el array original. Devuelve un nuevo array con los valores transformados.

5.2. Versión compacta con función flecha

Cuando la función solo devuelve una expresión, podemos escribirla de forma más breve:

const numeros = [1, 2, 3, 4];

const dobles = numeros.map((numero) => numero * 2);

console.log(dobles); // [2, 4, 6, 8]

Esta forma es habitual, pero al principio puede resultar menos clara. Si tienes dudas, usa la versión con llaves y return.

5.3. Ejemplo: transformar notas numéricas en textos

const notas = [4, 7, 9, 3, 5];

const resultados = notas.map((nota) => {
  if (nota >= 5) {
    return "Aprobado";
  } else {
    return "Suspenso";
  }
});

console.log(resultados); // ["Suspenso", "Aprobado", "Aprobado", "Suspenso", "Aprobado"]

Aquí todas las notas se transforman en textos.

El array original sigue siendo el mismo:

console.log(notas); // [4, 7, 9, 3, 5]

6. filter(): seleccionar algunos elementos

El método filter() crea un nuevo array con los elementos que cumplen una condición.

6.1. Sintaxis básica

const nuevoArray = arrayOriginal.filter((elemento) => {
  return condicion;
});

La condición debe devolver true o false.

  • Si devuelve true, el elemento se queda en el nuevo array.
  • Si devuelve false, el elemento se descarta.

6.2. Ejemplo: obtener los aprobados

const notas = [4, 7, 9, 3, 5];

const aprobados = notas.filter((nota) => {
  return nota >= 5;
});

console.log(aprobados); // [7, 9, 5]

6.3. Versión compacta

const notas = [4, 7, 9, 3, 5];

const aprobados = notas.filter((nota) => nota >= 5);

console.log(aprobados); // [7, 9, 5]

6.4. Ejemplo: filtrar palabras largas

const palabras = ["sol", "javascript", "web", "programacion"];

const largas = palabras.filter((palabra) => {
  return palabra.length >= 5;
});

console.log(largas); // ["javascript", "programacion"]

En este ejemplo se usa palabra.length, que devuelve el número de caracteres de un texto.

7. reduce(): acumular en un único resultado

El método reduce() recorre un array y acumula sus elementos para obtener un único resultado final.

Se suele usar para calcular:

  • una suma;
  • un producto;
  • un máximo;
  • un mínimo;
  • un total acumulado.

Es el método más difícil de este bloque, así que conviene entenderlo paso a paso.

Recomendación En este curso escribiremos siempre un valor inicial en reduce(). Así el código es más claro y evitamos errores con arrays vacíos.

7.1. Ejemplo básico: sumar números

const numeros = [1, 2, 3, 4];

const suma = numeros.reduce((acumulador, numero) => {
  return acumulador + numero;
}, 0);

console.log(suma); // 10

Partes importantes:

numeros.reduce((acumulador, numero) => {
  return acumulador + numero;
}, 0);
  • acumulador guarda el resultado que se va construyendo.
  • numero es el elemento actual del array.
  • 0 es el valor inicial del acumulador.

7.2. Paso a paso del ejemplo anterior

Array original:

const numeros = [1, 2, 3, 4];

Valor inicial del acumulador: 0.

Vuelta acumulador antes numero actual acumulador después
1 0 1 1
2 1 2 3
3 3 3 6
4 6 4 10

Resultado final: 10.

7.3. Ejemplo: calcular el total de precios

const precios = [10, 25, 8, 12];

const total = precios.reduce((acumulador, precio) => {
  return acumulador + precio;
}, 0);

console.log("Total:", total); // Total: 55

7.4. Versión compacta

const precios = [10, 25, 8, 12];

const total = precios.reduce((acumulador, precio) => acumulador + precio, 0);

console.log(total); // 55

Al principio es recomendable usar la versión con llaves y return, porque permite ver mejor qué está ocurriendo.

8. Comparación entre forEach(), map(), filter() y reduce()

Método Pregunta que responde Devuelve Ejemplo de uso
forEach() ¿Qué quiero hacer con cada elemento? undefined Mostrar cada elemento por consola
map() ¿Cómo quiero transformar cada elemento? Un nuevo array Duplicar todos los números
filter() ¿Con qué elementos me quiero quedar? Un nuevo array Quedarse con las notas aprobadas
reduce() ¿Cómo reduzco todos los elementos a un único resultado? Un valor final Sumar todos los números

9. Comparación práctica: for, forEach() y map()

Supongamos que queremos duplicar todos los números de un array.

9.1. Con for

const numeros = [1, 2, 3];
const dobles = [];

for (let i = 0; i < numeros.length; i++) {
  dobles.push(numeros[i] * 2);
}

console.log(dobles); // [2, 4, 6]

9.2. Con forEach()

const numeros = [1, 2, 3];
const dobles = [];

numeros.forEach((numero) => {
  dobles.push(numero * 2);
});

console.log(dobles); // [2, 4, 6]

Funciona, pero estamos creando el array nuevo manualmente.

9.3. Con map()

const numeros = [1, 2, 3];

const dobles = numeros.map((numero) => {
  return numero * 2;
});

console.log(dobles); // [2, 4, 6]

Esta es la opción más adecuada cuando queremos transformar todos los elementos y obtener un nuevo array.

10. slice(): copiar y extraer partes de un array

El método slice() permite crear una copia de un array o extraer una parte de él.

slice() no modifica el array original.

10.1. Copiar un array

const originales = [1, 2, 3];
const copia = originales.slice();

copia[0] = 99;

console.log(originales); // [1, 2, 3]
console.log(copia);      // [99, 2, 3]

En este ejemplo, copia es otro array.

10.2. Extraer una parte del array

const letras = ["a", "b", "c", "d", "e"];

const parte1 = letras.slice(1, 3);
const parte2 = letras.slice(2);

console.log(parte1); // ["b", "c"]
console.log(parte2); // ["c", "d", "e"]

En slice(inicio, fin):

  • inicio es la posición desde la que empieza la copia;
  • fin es la posición donde se detiene;
  • el elemento de inicio se incluye;
  • el elemento de fin no se incluye.

Por eso letras.slice(1, 3) devuelve los elementos de las posiciones 1 y 2.

11. Métodos que modifican y métodos que no modifican el array original

No todos los métodos de array se comportan igual.

Método ¿Modifica el array original? Qué hace
push() Sí Añade al final
pop() Sí Elimina el último
shift() Sí Elimina el primero
unshift() Sí Añade al principio
forEach() No por sí mismo Recorre el array
map() No Devuelve un nuevo array transformado
filter() No Devuelve un nuevo array filtrado
reduce() No Devuelve un valor acumulado
slice() No Devuelve una copia o una parte

Ojo: forEach() no modifica el array por sí mismo, pero dentro de su función podrías escribir código que sí modificase elementos. En este bloque evitaremos hacerlo salvo que sea necesario y esté claro.

12. Errores frecuentes

12.1. Usar forEach() esperando un nuevo array

Incorrecto:

const numeros = [1, 2, 3];

const dobles = numeros.forEach((numero) => {
  return numero * 2;
});

console.log(dobles); // undefined

Correcto:

const numeros = [1, 2, 3];

const dobles = numeros.map((numero) => {
  return numero * 2;
});

console.log(dobles); // [2, 4, 6]

12.2. Olvidar el return en map()

Incorrecto:

const numeros = [1, 2, 3];

const dobles = numeros.map((numero) => {
  numero * 2;
});

console.log(dobles); // [undefined, undefined, undefined]

Correcto:

const numeros = [1, 2, 3];

const dobles = numeros.map((numero) => {
  return numero * 2;
});

console.log(dobles); // [2, 4, 6]

También sería correcto usar la versión compacta:

const numeros = [1, 2, 3];

const dobles = numeros.map((numero) => numero * 2);

console.log(dobles); // [2, 4, 6]

12.3. Confundir map() y filter()

map() transforma elementos.

const numeros = [1, 2, 3];
const dobles = numeros.map((numero) => numero * 2);

console.log(dobles); // [2, 4, 6]

filter() selecciona elementos.

const numeros = [1, 2, 3, 4, 5];
const mayoresQueTres = numeros.filter((numero) => numero > 3);

console.log(mayoresQueTres); // [4, 5]

12.4. Usar reduce() sin entender el valor inicial

Es recomendable indicar siempre un valor inicial en reduce().

const numeros = [1, 2, 3];

const suma = numeros.reduce((acumulador, numero) => {
  return acumulador + numero;
}, 0);

console.log(suma); // 6

En este caso, el acumulador empieza en 0.

12.5. Crear callbacks demasiado complejos

Si la función que pasas a map(), filter() o reduce() se vuelve muy larga, puede ser más claro escribir una función aparte.

function esAprobado(nota) {
  return nota >= 5;
}

const notas = [4, 7, 9, 3, 5];
const aprobados = notas.filter(esAprobado);

console.log(aprobados); // [7, 9, 5]

13. Ejemplos guiados combinando métodos

13.1. Obtener las notas aprobadas y subirles un punto

const notas = [4, 7, 9, 3, 5];

const aprobadosSubidos = notas
  .filter((nota) => nota >= 5)
  .map((nota) => nota + 1);

console.log(aprobadosSubidos); // [8, 10, 6]

Explicación:

  1. filter() se queda con las notas aprobadas: [7, 9, 5].
  2. map() suma 1 a cada una: [8, 10, 6].

13.2. Calcular el total de precios con IVA

const precios = [10, 20, 15];

const preciosConIVA = precios.map((precio) => {
  return precio * 1.21;
});

const total = preciosConIVA.reduce((acumulador, precio) => {
  return acumulador + precio;
}, 0);

console.log(preciosConIVA); // [12.1, 24.2, 18.15]
console.log(total);         // 54.45

En este ejemplo:

  1. map() crea un nuevo array con los precios con IVA.
  2. reduce() suma todos los precios con IVA.

15. Resumen final

En este bloque has aprendido que JavaScript ofrece métodos de array que permiten trabajar con listas de datos de forma más expresiva.

Ideas principales:

  • forEach() recorre un array y ejecuta una acción por cada elemento.
  • map() transforma todos los elementos y devuelve un nuevo array.
  • filter() selecciona los elementos que cumplen una condición y devuelve un nuevo array.
  • reduce() acumula todos los elementos en un único resultado.
  • slice() permite copiar un array o extraer una parte sin modificar el original.
  • No todos los métodos devuelven lo mismo.
  • No todos los métodos modifican el array original.
  • Es importante elegir el método adecuado según el problema.

Guía rápida

  • Si solo quiero hacer algo con cada elemento: forEach().
  • Si quiero obtener otro array con todos los elementos transformados: map().
  • Si quiero quedarme solo con algunos elementos: filter().
  • Si quiero obtener un único resultado final: reduce().

La pregunta más útil antes de elegir un método es:

¿Quiero recorrer, transformar, filtrar o acumular?

Según la respuesta, elegiré forEach(), map(), filter() o reduce().
