Bloque 3. Operadores, condicionales y bucles en JavaScript

1. Qué vas a aprender en este bloque

En el bloque anterior trabajamos la sintaxis básica de JavaScript, las variables, las constantes, los tipos de datos y algunas conversiones entre tipos. En este bloque vamos a utilizar esos conocimientos para controlar el flujo de ejecución de un programa.

Al finalizar este bloque deberías ser capaz de:

  • construir expresiones booleanas con operadores de comparación;
  • combinar condiciones mediante operadores lógicos;
  • entender cómo JavaScript interpreta valores como verdaderos o falsos;
  • utilizar if, else if, else y switch para tomar decisiones;
  • utilizar el operador ternario en casos sencillos;
  • repetir instrucciones mediante bucles while, do...while y for;
  • detectar errores frecuentes en condiciones y bucles.

Idea clave
Un programa no siempre se ejecuta línea a línea sin desviarse. Las estructuras de control permiten decidir qué instrucciones se ejecutan y cuántas veces se repiten.

3. Control de flujo

En JavaScript, si no indicamos lo contrario, las instrucciones se ejecutan de arriba abajo, una detrás de otra. A esto se le llama ejecución secuencial. 

console.log("Primera línea");
console.log("Segunda línea");
console.log("Tercera línea");

La salida será:

Primera línea
Segunda línea
Tercera línea

Las estructuras de control permiten cambiar ese orden natural:

  • Condicionales: ejecutan un bloque solo si se cumple una condición.
  • Bucles: repiten un bloque varias veces mientras se cumpla una condición.

Ejemplo conceptual:

const edad = 17;

if (edad >= 18) {
  console.log("Puedes acceder");
} else {
  console.log("No puedes acceder");
}

Aquí no se ejecutan todas las instrucciones posibles. El programa toma una decisión según el valor de edad.

4. Expresiones booleanas y operadores de comparación

Una expresión booleana es una expresión cuyo resultado es true o false.

Ejemplos:

console.log(5 > 3);       // true
console.log(10 < 4);      // false
console.log(7 === 7);     // true
console.log("7" === 7);   // false

4.1. Operadores de comparación más usados

Operador Significado Ejemplo Resultado
> Mayor que 8 > 3 true
< Menor que 8 < 3 false
>= Mayor o igual que 5 >= 5 true
<= Menor o igual que 4 <= 2 false
=== Igualdad estricta 3 === 3 true
!== Diferencia estricta 3 !== "3" true

4.2. Igualdad estricta frente a igualdad débil

En JavaScript existen dos formas habituales de comparar igualdad: 

console.log(0 == "0");    // true
console.log(0 === "0");   // false
  • == compara haciendo conversiones automáticas de tipo.
  • === compara valor y tipo, sin conversión automática.

En código nuevo se recomienda usar === y !== como norma general. 

const edad = "18";

console.log(edad == 18);   // true, porque convierte tipos
console.log(edad === 18);  // false, porque string y number no son el mismo tipo

Regla práctica
Usa === y !==. Si necesitas convertir un dato, hazlo explícitamente con Number(), String() o Boolean().

5. Operadores lógicos

Los operadores lógicos sirven para combinar o modificar condiciones.

Los tres operadores principales son:

Operador Nombre Significado
&& AND / Y lógico Todas las condiciones deben ser verdaderas.
|| OR / O lógico Basta con que una condición sea verdadera.
! NOT / NO lógico Invierte el valor lógico.

5.1. AND lógico: &&

&& devuelve true solo cuando todas las condiciones son verdaderas. 

const edad = 20;
const tieneCarnet = true;

const puedeConducir = edad >= 18 && tieneCarnet === true;

console.log(puedeConducir); // true

Tabla básica:

Expresión Resultado
true && true true
true && false false
false && true false
false && false false

Ejemplo con un resultado falso:

const edad = 20;
const tieneCarnet = false;

console.log(edad >= 18 && tieneCarnet === true); // false

La edad cumple la condición, pero tieneCarnet === true no se cumple.

5.2. OR lógico: ||

|| devuelve true cuando al menos una condición es verdadera. 

const esAdmin = false;
const esEditor = true;

const puedeEditar = esAdmin === true || esEditor === true;

console.log(puedeEditar); // true

Tabla básica:

Expresión Resultado
true || true true
true || false true
false || true true
false || false false

5.3. NOT lógico: !

! invierte el valor lógico.

const logueado = false;

console.log(!logueado); // true

Ejemplos:

console.log(!true);   // false
console.log(!false);  // true

También puede usarse para expresar condiciones como “si no está logueado”: 

const logueado = false;

if (!logueado) {
  console.log("Debes iniciar sesión");
}

5.4. Condiciones compuestas

Podemos combinar varias condiciones:

const edad = 22;
const tieneEntrada = true;
const expulsado = false;

if (edad >= 18 && tieneEntrada && !expulsado) {
  console.log("Puede acceder al evento");
} else {
  console.log("No puede acceder al evento");
}

Lectura de la condición:

Si la edad es mayor o igual que 18, tiene entrada y no está expulsado, puede acceder.

Cuando una condición sea larga, conviene dividirla en variables intermedias: 

const esMayorDeEdad = edad >= 18;
const puedeEntrar = tieneEntrada && !expulsado;

if (esMayorDeEdad && puedeEntrar) {
  console.log("Puede acceder al evento");
}

6. Valores truthy y falsy

En JavaScript, una condición no tiene que ser necesariamente true o false. Cuando un valor se usa dentro de un if, JavaScript lo interpreta como verdadero o falso.

A los valores que se interpretan como false se les llama falsy.

A los valores que se interpretan como true se les llama truthy.

6.1. Valores falsy más importantes

Estos valores se comportan como false en una condición: 

false
0
""
null
undefined
NaN

Ejemplos:

if (0) {
  console.log("Esto no se ejecuta");
}

if ("") {
  console.log("Esto tampoco se ejecuta");
}

if (null) {
  console.log("Esto tampoco se ejecuta");
}

6.2. Valores truthy

Casi todos los demás valores se comportan como true.

Ejemplos importantes:

if ("Hola") {
  console.log("Una cadena no vacía es truthy");
}

if ("0") {
  console.log("La cadena '0' es truthy");
}

if ("false") {
  console.log("La cadena 'false' también es truthy");
}

Esto puede sorprender al principio:

console.log(Boolean("false")); // true
console.log(Boolean("0"));     // true
console.log(Boolean(""));      // false

Cuidado
La cadena "false" no es el valor booleano false. Es texto, y como no está vacía, es truthy.

6.3. Uso habitual en validaciones sencillas

const nombre = "Ana";

if (nombre) {
  console.log("Hay un nombre guardado");
} else {
  console.log("No hay nombre");
}

Si nombre es una cadena vacía (""), la condición será falsa.

7. Cortocircuito y valores por defecto

Los operadores && y || no siempre evalúan toda la expresión. Evalúan de izquierda a derecha y se detienen cuando ya saben el resultado. A esto se le llama cortocircuito.

7.1. Cortocircuito con &&

En una expresión con &&, si la primera parte es falsa, JavaScript ya sabe que el resultado completo será falso. Por tanto, no evalúa lo que viene después. 

function saludar() {
  console.log("Hola desde la función");
  return true;
}

false && saludar(); // saludar() no se ejecuta

7.2. Cortocircuito con ||

En una expresión con ||, si la primera parte es verdadera, JavaScript ya sabe que el resultado completo será verdadero. Por tanto, no evalúa lo que viene después. 

function saludar() {
  console.log("Hola desde la función");
  return true;
}

true || saludar(); // saludar() no se ejecuta

7.3. Uso de || como valor por defecto

Un uso habitual de || es proporcionar un valor por defecto cuando el primer valor es falsy. 

let nombreUsuario = null;

const nombreMostrado = nombreUsuario || "Invitado";

console.log(nombreMostrado); // "Invitado"

Si nombreUsuario tiene un valor truthy, se usa ese valor: 

let nombreUsuario = "Ana";

const nombreMostrado = nombreUsuario || "Invitado";

console.log(nombreMostrado); // "Ana"

7.4. Advertencia sobre 0 y ""

El uso de || como valor por defecto puede dar problemas cuando 0 o "" son valores válidos. 

const puntos = 0;
const puntosMostrados = puntos || 100;

console.log(puntosMostrados); // 100

El resultado es 100 porque 0 es falsy. Pero quizá 0 era un valor válido y no queríamos sustituirlo.

En JavaScript moderno existe el operador ??, que solo usa el valor por defecto cuando el primer valor es null o undefined: 

const puntos = 0;
const puntosMostrados = puntos ?? 100;

console.log(puntosMostrados); // 0

En este bloque basta con entender bien ||. El operador ?? queda como ampliación.

8. Condicional if

La estructura if permite ejecutar un bloque de código solo si se cumple una condición.

8.1. Sintaxis

if (condicion) {
  // Código que se ejecuta si la condición es true
}

La condición va entre paréntesis. El bloque de código va entre llaves.

8.2. Ejemplo básico

const edad = 20;

if (edad >= 18) {
  console.log("Puedes entrar");
}

Si edad >= 18 es true, se ejecuta el console.log().

Si la condición es false, el bloque no se ejecuta.

8.3. if con valores truthy y falsy

const nombre = "Ana";

if (nombre) {
  console.log("Nombre indicado");
}

La variable nombre contiene una cadena no vacía, por tanto se interpreta como true.

Otro ejemplo:

const nombre = "";

if (nombre) {
  console.log("Nombre indicado");
}

En este caso no se muestra nada, porque la cadena vacía es falsy.

9. Condicional if / else

Con else podemos indicar qué debe ocurrir cuando la condición del if no se cumple.

9.1. Sintaxis

if (condicion) {
  // Código si la condición es true
} else {
  // Código si la condición es false
}

Solo se ejecuta uno de los dos bloques.

9.2. Ejemplo

const edad = 16;

if (edad >= 18) {
  console.log("Eres mayor de edad");
} else {
  console.log("Eres menor de edad");
}

Como edad >= 18 es false, se ejecuta el bloque del else.

9.3. Ejemplo con usuario logueado

const logueado = false;

if (logueado) {
  console.log("Mostrar panel de usuario");
} else {
  console.log("Mostrar botón de iniciar sesión");
}

10. Condicional if / else if / else

Cuando hay más de dos posibilidades, podemos encadenar condiciones con else if.

10.1. Sintaxis

if (condicion1) {
  // Código si condicion1 es true
} else if (condicion2) {
  // Código si condicion2 es true
} else if (condicion3) {
  // Código si condicion3 es true
} else {
  // Código si ninguna condición anterior es true
}

JavaScript evalúa las condiciones en orden. Cuando una condición es true, ejecuta ese bloque y salta el resto.

10.2. Ejemplo: nota numérica

const nota = 7;

if (nota >= 9) {
  console.log("Sobresaliente");
} else if (nota >= 7) {
  console.log("Notable");
} else if (nota >= 5) {
  console.log("Aprobado");
} else {
  console.log("Suspenso");
}

El orden importa. Si nota es 7, se cumple nota >= 7, se muestra "Notable" y ya no se revisa el resto.

10.3. Validación de rangos

El ejemplo anterior clasifica notas, pero no comprueba si la nota está dentro del rango correcto. 

const nota = 25;

if (nota < 0 || nota > 10) {
  console.log("Nota no válida");
} else if (nota >= 9) {
  console.log("Sobresaliente");
} else if (nota >= 7) {
  console.log("Notable");
} else if (nota >= 5) {
  console.log("Aprobado");
} else {
  console.log("Suspenso");
}

Aquí primero se comprueba si la nota es válida. Solo después se clasifica.

11. Operador ternario

El operador ternario es una forma abreviada de escribir un if / else sencillo cuando queremos elegir entre dos valores.

11.1. Sintaxis

condicion ? valorSiTrue : valorSiFalse

La estructura tiene tres partes:

  1. condición;
  2. valor si la condición es verdadera;
  3. valor si la condición es falsa.

11.2. Ejemplo básico

const edad = 18;

const mensaje = edad >= 18 ? "Mayor de edad" : "Menor de edad";

console.log(mensaje); // "Mayor de edad"

Equivale a esto:

const edad = 18;
let mensaje;

if (edad >= 18) {
  mensaje = "Mayor de edad";
} else {
  mensaje = "Menor de edad";
}

console.log(mensaje);

11.3. Ejemplo con login

const logueado = false;

const textoBoton = logueado ? "Cerrar sesión" : "Iniciar sesión";

console.log(textoBoton); // "Iniciar sesión"

11.4. Cuándo usar ternario

Usa ternario cuando:

  • hay una condición sencilla;
  • hay solo dos valores posibles;
  • el resultado se guarda en una variable o se muestra directamente.

Evítalo cuando:

  • la condición es larga;
  • hay varias decisiones encadenadas;
  • el código pierde legibilidad.

Ejemplo poco recomendable para empezar:

const mensaje = nota >= 9 ? "Sobresaliente" : nota >= 7 ? "Notable" : nota >= 5 ? "Aprobado" : "Suspenso";

Aunque funciona, es difícil de leer. En este caso es mejor usar if / else if / else.

12. switch

switch permite elegir entre varios casos posibles comparando una misma expresión con distintos valores.

Es útil cuando tenemos muchos valores concretos posibles.

12.1. Sintaxis

switch (expresion) {
  case valor1:
    // Código si expresion === valor1
    break;

  case valor2:
    // Código si expresion === valor2
    break;

  default:
    // Código si ningún case coincide
    break;
}

Puntos clave:

  • La expresión se evalúa una vez.
  • Cada case se compara usando igualdad estricta.
  • break evita que se sigan ejecutando los casos siguientes.
  • default es opcional, pero suele ser recomendable.

12.2. Ejemplo: día de la semana

const dia = 3;
let nombreDia;

switch (dia) {
  case 1:
    nombreDia = "Lunes";
    break;
  case 2:
    nombreDia = "Martes";
    break;
  case 3:
    nombreDia = "Miércoles";
    break;
  case 4:
    nombreDia = "Jueves";
    break;
  case 5:
    nombreDia = "Viernes";
    break;
  case 6:
    nombreDia = "Sábado";
    break;
  case 7:
    nombreDia = "Domingo";
    break;
  default:
    nombreDia = "Día no válido";
    break;
}

console.log(nombreDia); // "Miércoles"

12.3. Fall-through

Si no ponemos break, JavaScript continúa ejecutando los casos siguientes. Esto se llama fall-through.

A veces es un error:

const opcion = 1;

switch (opcion) {
  case 1:
    console.log("Opción 1");
  case 2:
    console.log("Opción 2");
  default:
    console.log("Otra opción");
}

Si opcion vale 1, se imprimirán varios mensajes porque faltan los break.

Pero también puede usarse de forma intencionada:

const dia = 6;

switch (dia) {
  case 1:
  case 2:
  case 3:
  case 4:
  case 5:
    console.log("Día laborable");
    break;
  case 6:
  case 7:
    console.log("Fin de semana");
    break;
  default:
    console.log("Día no válido");
    break;
}

12.4. Cuándo usar if y cuándo usar switch

Estructura Cuándo usarla
if Una condición simple.
if / else Dos caminos posibles.
if / else if / else Varias condiciones, rangos o expresiones diferentes.
switch Una misma expresión comparada con varios valores concretos.

Ejemplos:

  • Para clasificar una nota por rangos, suele ser mejor if / else if / else.
  • Para elegir una acción según un texto como "crear", "editar" o "borrar", puede ser mejor switch.

13. Bucles

Un bucle es una estructura que permite repetir un bloque de código varias veces.

En JavaScript usaremos principalmente:

  • while;
  • do...while;
  • for.

Los bucles son imprescindibles para automatizar tareas repetitivas: contar, acumular valores, recorrer intervalos, repetir validaciones o procesar listas de datos.

13.1. Bucle while

El bucle while repite un bloque mientras una condición sea verdadera.

Sintaxis

while (condicion) {
  // Código que se repite mientras la condición sea true
}

Ejemplo: contar de 1 a 5

let numero = 1;

while (numero <= 5) {
  console.log(numero);
  numero = numero + 1;
}

Funcionamiento:

  1. numero empieza valiendo 1.
  2. Se comprueba si numero <= 5.
  3. Si la condición es verdadera, se ejecuta el bloque.
  4. Se incrementa numero.
  5. Cuando numero vale 6, la condición es falsa y el bucle termina.

También podríamos escribir el incremento así:

numero++;

numero++ equivale a:

numero = numero + 1;

Cuidado con los bucles infinitos

Un bucle infinito ocurre cuando la condición nunca se vuelve falsa. 

let x = 0;

while (x < 10) {
  console.log(x);
  // Falta actualizar x
}

Este bucle no termina porque x siempre vale 0.

Regla práctica:

Dentro del bucle debe cambiar alguna variable que afecte a la condición.

13.2. Bucle do...while

El bucle do...while se parece a while, pero garantiza que el bloque se ejecuta al menos una vez.

Sintaxis

do {
  // Código que se ejecuta al menos una vez
} while (condicion);

Primero se ejecuta el bloque. Después se comprueba la condición.

Ejemplo básico

let numero = 10;

do {
  console.log(numero);
  numero++;
} while (numero <= 5);

Aunque numero <= 5 es falso desde el principio, el bloque se ejecuta una vez.

Ejemplo: pedir un número mayor que 0

let numero;

do {
  numero = Number(prompt("Introduce un número mayor que 0"));
} while (numero <= 0 || Number.isNaN(numero));

console.log("Número correcto:", numero);

prompt() se utiliza aquí como recurso sencillo para practicar. Más adelante, en formularios, se sustituirá por entradas HTML.

13.3. Bucle for

El bucle for se usa mucho cuando controlamos un contador o sabemos el número de repeticiones.

Sintaxis

for (inicializacion; condicion; actualizacion) {
  // Código que se repite mientras la condición sea true
}

Partes:

  • inicialización: se ejecuta una vez al principio;
  • condición: se comprueba antes de cada vuelta;
  • actualización: se ejecuta al final de cada vuelta.

Ejemplo: contar de 1 a 5

for (let i = 1; i <= 5; i++) {
  console.log(i);
}

Equivale a este while:

let i = 1;

while (i <= 5) {
  console.log(i);
  i++;
}

13.4. Cuándo usar cada bucle

Bucle Cuándo usarlo
while Cuando no sabemos cuántas veces se repetirá y dependemos de una condición.
do...while Cuando queremos ejecutar el bloque al menos una vez.
for Cuando usamos un contador o conocemos el rango de repetición.

Ejemplos típicos:

  • while: seguir pidiendo datos mientras no sean válidos.
  • do...while: mostrar un menú al menos una vez.
  • for: contar de 1 a 10 o recorrer posiciones de una lista.

13.5. Contadores y acumuladores

Un contador aumenta o disminuye para contar repeticiones. 

for (let i = 1; i <= 5; i++) {
  console.log("Vuelta", i);
}

Un acumulador guarda un resultado que se va construyendo vuelta a vuelta. 

let suma = 0;

for (let i = 1; i <= 10; i++) {
  suma = suma + i;
}

console.log("La suma es:", suma); // 55

También se puede escribir:

suma += i;

que equivale a:

suma = suma + i;

13.6. Ejemplos integradores

Tabla de multiplicar

const numero = 5;

for (let i = 1; i <= 10; i++) {
  console.log(numero + " x " + i + " = " + numero * i);
}

Validar una entrada numérica

let entrada;
let numero;

do {
  entrada = prompt("Introduce un número:");
  numero = Number(entrada);
} while (Number.isNaN(numero));

console.log("Has introducido el número:", numero);

Este ejemplo comprueba que la entrada pueda convertirse a número. Si queremos exigir que sea un entero, habría que añadir Number.isInteger(numero): 

let entrada;
let numero;

do {
  entrada = prompt("Introduce un número entero:");
  numero = Number(entrada);
} while (Number.isNaN(numero) || !Number.isInteger(numero));

console.log("Has introducido el entero:", numero);

14. Errores frecuentes

14.1. Usar = en lugar de ===

Incorrecto:

let edad = 18;

if (edad = 18) {
  console.log("Edad igual a 18");
}

= es asignación, no comparación.

Correcto:

const edad = 18;

if (edad === 18) {
  console.log("Edad igual a 18");
}

14.2. Olvidar las llaves

JavaScript permite escribir un if sin llaves cuando solo hay una instrucción, pero para aprender es mejor usarlas siempre.

Poco recomendable:

if (edad >= 18)
  console.log("Mayor de edad");

Recomendado:

if (edad >= 18) {
  console.log("Mayor de edad");
}

14.3. Construir condiciones demasiado largas

Difícil de leer:

if (edad >= 18 && tieneEntrada && !expulsado && saldo >= precio && eventoAbierto) {
  console.log("Puede entrar");
}

Más claro:

const esMayorDeEdad = edad >= 18;
const entradaValida = tieneEntrada && !expulsado;
const puedePagar = saldo >= precio;

if (esMayorDeEdad && entradaValida && puedePagar && eventoAbierto) {
  console.log("Puede entrar");
}

14.4. Bucles infinitos

Incorrecto:

let i = 1;

while (i <= 10) {
  console.log(i);
}

Correcto:

let i = 1;

while (i <= 10) {
  console.log(i);
  i++;
}

14.5. Condiciones mal ordenadas

Incorrecto:

const nota = 8;

if (nota >= 5) {
  console.log("Aprobado");
} else if (nota >= 7) {
  console.log("Notable");
} else if (nota >= 9) {
  console.log("Sobresaliente");
}

Si nota vale 8, se muestra "Aprobado", porque nota >= 5 ya se cumple y las demás condiciones no se revisan.

Correcto:

const nota = 8;

if (nota >= 9) {
  console.log("Sobresaliente");
} else if (nota >= 7) {
  console.log("Notable");
} else if (nota >= 5) {
  console.log("Aprobado");
} else {
  console.log("Suspenso");
}

14.6. Confundir || como valor por defecto en todos los casos 

const puntos = 0;
const resultado = puntos || "Sin puntos";

console.log(resultado); // "Sin puntos"

Si 0 es un valor válido, esta solución no sirve. Hay que comprobar el caso concreto o usar otra estrategia.

17. Resumen final

En este bloque hemos estudiado cómo controlar el flujo de ejecución de un programa JavaScript.

Ideas principales:

  • Las comparaciones devuelven valores booleanos: true o false.
  • En JavaScript se recomienda usar === y !== en lugar de == y !=.
  • Los operadores lógicos permiten combinar condiciones:
    • &&: todas las condiciones deben cumplirse;
    • ||: basta con que se cumpla una;
    • !: invierte el valor lógico.
  • Los valores falsy más importantes son false, 0, "", null, undefined y NaN.
  • if, else if y else permiten tomar decisiones según condiciones.
  • El operador ternario permite elegir entre dos valores cuando la condición es sencilla.
  • switch es útil cuando se compara una misma expresión con varios valores concretos.
  • Los bucles permiten repetir instrucciones:
    • while: repite mientras se cumpla una condición;
    • do...while: se ejecuta al menos una vez;
    • for: suele usarse con contadores o rangos.
  • Todo bucle debe tener una condición que pueda llegar a ser falsa.
