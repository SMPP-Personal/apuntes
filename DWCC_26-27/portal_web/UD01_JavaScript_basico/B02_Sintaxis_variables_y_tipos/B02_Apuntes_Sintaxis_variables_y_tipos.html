1. Qué vas a aprender en este bloque

En este bloque se trabajan los fundamentos mínimos para empezar a escribir código JavaScript de forma correcta:

  • cómo se escriben instrucciones, bloques y comentarios;
  • cómo se nombran variables, constantes y otros identificadores;
  • cómo declarar variables con let y constantes con const;
  • por qué no se recomienda usar var en código nuevo;
  • qué significa que JavaScript tenga tipado dinámico;
  • qué tipos de datos básicos existen;
  • qué son undefined, null, NaN e Infinity;
  • cómo usar typeof para inspeccionar valores;
  • cómo convertir valores entre tipos;
  • por qué es preferible usar comparaciones estrictas con === y !==.

Este bloque es la base para poder estudiar después operadores, condicionales, bucles, funciones, arrays, objetos, DOM y eventos.

3. Cómo ejecutar JavaScript en este bloque

Durante este bloque se recomienda trabajar de dos formas.

3.1. Consola del navegador

La consola permite probar instrucciones rápidas sin crear un proyecto completo.

Ejemplo:

let mensaje = "Hola desde la consola";
console.log(mensaje);

La consola es útil para:

  • probar expresiones;
  • ver el resultado de typeof;
  • comprobar errores de sintaxis;
  • entender conversiones entre tipos.

3.2. Fichero .js enlazado desde HTML

Para trabajar de forma más ordenada, se usará un fichero .js enlazado desde una página HTML.

Estructura recomendada:

mi-proyecto/
├── index.html
└── js/
    └── bloque02.js

Ejemplo de index.html:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Bloque 2 - JavaScript</title>
</head>
<body>
  <h1>Pruebas de JavaScript</h1>

  <script src="js/bloque02.js"></script>
</body>
</html>

Ejemplo de js/bloque02.js:

const nombre = "Ana";

Idea clave: en este bloque no necesitamos modificar todavía el DOM. Lo importante es saber escribir código básico y comprobar resultados en la consola.

4. Sintaxis básica de JavaScript

La sintaxis es el conjunto de reglas que indican cómo debe escribirse el código para que el navegador pueda interpretarlo correctamente.

4.1. Sentencias o instrucciones

Una sentencia es una orden completa.

let saludo = "Hola";
console.log(saludo);

JavaScript puede inferir algunos puntos y coma automáticamente, pero en este nivel es recomendable escribir ; al final de las instrucciones simples para evitar ambiguëdades.

4.2. Bloques de código

Un bloque agrupa instrucciones entre llaves { }.

{
  let x = 3;
  console.log(x);
}

Los bloques se usan en funciones, condicionales, bucles y otros elementos del lenguaje. En este bloque inicial nos interesan sobre todo porque afectan al ámbito de las variables declaradas con let y const.

4.3. Comentarios

Los comentarios sirven para explicar el código. El navegador los ignora al ejecutar el programa.

// Comentario de una línea
let precio = 25;

/*
  Comentario de varias líneas.
  Puede ocupar más de una línea.
*/
const IVA = 0.21;

Un buen comentario no repite lo evidente. Debe ayudar a entender la intención del código.

// Mal comentario: no aporta información útil
let edad = 18; // edad vale 18

// Mejor comentario: explica el criterio usado
let edadMinima = 18; // mayoría de edad legal para este ejemplo

5. Identificadores y reglas de nombres

Un identificador es el nombre que usamos para una variable, constante, función, parámetro u otro elemento del programa.

Ejemplos de identificadores:

let nombreUsuario = "Ana";
const MAX_INTENTOS = 3;
let precioFinal = 19.99;

5.1. Reglas obligatorias

En JavaScript, un identificador:

  • puede empezar por una letra, _ o $;
  • no puede empezar por un número;
  • puede contener letras, números, _ y $ después del primer carácter;
  • no puede ser una palabra reservada del lenguaje;
  • distingue entre mayúsculas y minúsculas.

Ejemplos válidos:

let nombre = "Ana";
let _contador = 0;
let $elemento = "boton";
let precio2 = 15;

Ejemplos no válidos:

let 2precio = 15;     // Error: empieza por número
let let = "texto";    // Error: let es palabra reservada
let nombre usuario;   // Error: contiene un espacio

5.2. Convenciones recomendadas

ElementoConvención habitualEjemplo

Constantes de configuraciónSNAKE_UPPER_CASEMAX_INTENTOS, TIEMPO_ESPERA_MS

Recomendación de aula: usar nombres claros aunque sean algo más largos. precioFinal es mejor que pf.

6. Variables y constantes

Una variable es un nombre que referencia un valor. En JavaScript no se declara el tipo de la variable: se declara el nombre y se le asigna un valor.

let edad = 20;
let nombre = "Lucía";

El tipo pertenece al valor, no al nombre de la variable.

6.1. Declarar e inicializar

Declarar una variable significa crearla.

let edad;

Inicializar una variable significa darle un valor por primera vez.

edad = 20;

También se puede declarar e inicializar en una sola línea:

let edad = 20;

Si una variable declarada con let no recibe valor inicial, su valor será undefined.

let dato;
console.log(dato);        // undefined
console.log(typeof dato); // "undefined"

6.2. let

let se usa para declarar variables cuyo valor puede cambiar.

let contador = 0;
contador = 1;
contador = contador + 1;

console.log(contador); // 2

Características principales:

  • permite reasignar valores;
  • no permite redeclarar el mismo nombre dentro del mismo bloque;
  • tiene ámbito de bloque.
let edad = 20;
edad = 21; // Correcto

// let edad = 22; // Error si está en el mismo bloque

6.3. const

const se usa para declarar constantes. Una constante debe inicializarse en el momento de declararse y no puede reasignarse después.

const PI = 3.14159;
console.log(PI);

// PI = 3; // Error: no se puede reasignar una constante

Esto es incorrecto:

// const x; // Error: una constante debe inicializarse

const no significa inmutable

Si una constante guarda un objeto o un array, no se puede cambiar la referencia, pero sí pueden modificarse sus propiedades o elementos.

const config = {
  apiUrl: "https://ejemplo.com",
  intentosMaximos: 3
};

config.intentosMaximos = 5; // Permitido: cambia una propiedad

// config = {}; // Error: intenta reasignar la constante

const impide reasignar la variable. const no convierte automáticamente un objeto en inmutable.

6.4. ¿Cuándo usar let y cuándo usar const?

SituaciónForma recomendadaEjemplo
El valor no debe reasignarseconstconst IVA = 0.21;
El valor cambiará durante la ejecuciónletlet contador = 0;
Código antiguo que ya existevar puede aparecervar nombre = "Ana";
Código nuevoevitar varusar let o const

Regla práctica: usar const por defecto y cambiar a let solo cuando realmente sea necesario reasignar.

7. Ámbito de las variables

El ámbito o scope es la zona del código donde una variable existe y puede usarse.

7.1. Ámbito de bloque con let y const

Las variables declaradas con let y const solo existen dentro del bloque { } donde se han declarado.

if (true) {
  let mensaje = "Dentro del bloque";
  console.log(mensaje); // Correcto
}

// console.log(mensaje); // Error: mensaje no existe fuera del bloque

Aunque todavía no hayamos estudiado if, este ejemplo permite ver que las llaves { } delimitan un bloque.

7.2. var y ámbito de función

var es la forma clásica de declarar variables en JavaScript. No se recomienda usarla en código nuevo porque tiene un comportamiento menos intuitivo.

if (true) {
  let a = 1;
  var b = 2;
}

// console.log(a); // Error: a no existe fuera del bloque
console.log(b);    // 2: b sí queda accesible fuera del bloque

var tiene ámbito de función, no de bloque, y además está afectada por el hoisting. Para este nivel basta con recordar:

En código nuevo, usa let y const. Debes reconocer var porque aparecerá en ejemplos antiguos.

8. Tipado dinámico y tipado débil

JavaScript es un lenguaje con tipado dinámico y se suele describir como débilmente tipado.

8.1. Tipado dinámico

Significa que no se indica el tipo al declarar la variable. El tipo se determina en tiempo de ejecución según el valor que tenga en ese momento.

let dato = 10;       // number
dato = "diez";      // string
dato = false;       // boolean

console.log(typeof dato); // "boolean"

Idea clave: en JavaScript, el tipo pertenece al valor, no a la variable.

8.2. Tipado débil y coerción

JavaScript permite operar con valores de tipos diferentes y, en algunas operaciones, intenta convertirlos automáticamente. A esto se le llama coerción de tipos.

console.log("3" * 2); // 6: convierte "3" a número
console.log("3" + 2); // "32": concatena como texto
console.log(true + 1); // 2: true se convierte a 1

Este comportamiento puede ser cómodo, pero también puede producir errores difíciles de detectar. Por eso es importante aprender a convertir datos de forma explícita y a comparar con igualdad estricta.

10. number: números, NaN e Infinity

En JavaScript, el tipo number se usa tanto para enteros como para decimales.

let unidades = 5;
let precio = 19.99;
let temperatura = -3;

10.1. NaN

NaN significa Not a Number. Aparece cuando el resultado de una operación numérica no es un número válido.

console.log(Number("hola")); // NaN
console.log(0 / 0);          // NaN
console.log(typeof NaN);     // "number"

Aunque resulte extraño, typeof NaN devuelve "number" porque NaN es un valor especial dentro del tipo numérico.

Si se opera con NaN, normalmente el resultado sigue siendo NaN.

console.log(NaN + 5);  // NaN
console.log(NaN * 10); // NaN

Para comprobar si un valor es NaN, es preferible usar Number.isNaN().

console.log(Number.isNaN(NaN));       // true
console.log(Number.isNaN("hola"));    // false
console.log(Number.isNaN(Number("hola"))); // true

10.2. Infinity y -Infinity

Infinity y -Infinity representan valores numéricos infinitos.

console.log(10 / 0);   // Infinity
console.log(-10 / 0);  // -Infinity

En código real, la aparición de Infinity suele indicar que conviene revisar la operación, por ejemplo una división entre cero.

11. string: cadenas de texto

Un string es una cadena de texto. Puede escribirse con comillas dobles, comillas simples o acentos graves (backticks).

let nombre = "Ana";
let ciudad = 'Ourense';
let mensaje = `Hola, Ana`;

11.1. Concatenación

El operador + permite unir textos.

let nombre = "Ana";
let saludo = "Hola, " + nombre;

console.log(saludo); // "Hola, Ana"

Cuando se usa + con un texto y un número, JavaScript puede convertir el número a texto.

let x = "5";
let y = 2;

console.log(x + y); // "52"

Para sumar numéricamente, hay que convertir primero.

let x = "5";
let y = 2;

console.log(Number(x) + y); // 7

12. boolean: verdadero o falso

Un valor booleano solo puede ser true o false.

let esMayorEdad = true;
let usuarioActivo = false;

Los booleanos se usarán mucho en condiciones, validaciones, formularios y control del flujo del programa.

En este bloque solo se introducen como tipo de dato. Los operadores lógicos y las estructuras condicionales se trabajarán con más detalle en un bloque posterior.

13. undefined y null

undefined y null representan ausencia de valor, pero no significan exactamente lo mismo.

13.1. undefined

undefined suele indicar que una variable existe, pero todavía no tiene un valor asignado.

let x;

console.log(x);        // undefined
console.log(typeof x); // "undefined"

También puede aparecer cuando se accede a una propiedad inexistente de un objeto o cuando una función no devuelve nada explícitamente.

const persona = {
  nombre: "Ana"
};

console.log(persona.edad); // undefined

Aunque una persona podría asignar undefined manualmente, no es lo recomendable en código inicial. Normalmente se deja que JavaScript lo use para indicar ausencia de valor asignado.

13.2. null

null representa una ausencia intencionada de valor.

let usuarioActual = null;
console.log(usuarioActual); // null

Se usa cuando queremos indicar de forma explícita que en una variable no hay ningún valor útil en ese momento.

let seleccion = null; // de momento no se ha seleccionado nada

seleccion = "producto-123";
console.log(seleccion);

13.3. Comparación rápida

ValorIdea básicaEjemplo
undefinedNo hay valor asignado todavía.let x;
nullSe ha decidido guardar “ningún valor”.let usuario = null;

undefined suele significar “aún no hay valor”; null significa “he puesto explícitamente que no hay valor”.

13.4. El caso especial de typeof null

Hay un detalle histórico muy importante:

console.log(typeof null); // "object"

Esto no significa que null sea un objeto normal. Es un comportamiento histórico del lenguaje que se mantiene por compatibilidad.

Para comprobar si un valor es exactamente null, usa comparación estricta:

let usuario = null;

console.log(usuario === null); // true

14. Objetos y arrays: tipos de referencia

Todo lo que no es un tipo primitivo se trata como objeto o está relacionado con objetos.

Ejemplos:

const persona = {
  nombre: "Ana",
  edad: 20
};

const numeros = [1, 2, 3];

En este bloque no se estudian todavía objetos y arrays en profundidad, pero conviene entender una idea básica: cuando asignamos un objeto o un array a otra variable, ambas variables pueden apuntar al mismo contenido.

const persona1 = {
  nombre: "Ana"
};

const persona2 = persona1;
persona2.nombre = "Luis";

console.log(persona1.nombre); // "Luis"

No se ha copiado una persona independiente. Se ha copiado la referencia al mismo objeto.

15. typeof: comprobar el tipo de un valor

El operador typeof devuelve una cadena de texto con el tipo de un valor.

console.log(typeof 5);        // "number"
console.log(typeof "hola");   // "string"
console.log(typeof true);     // "boolean"
console.log(typeof undefined);// "undefined"
console.log(typeof NaN);      // "number"
console.log(typeof null);     // "object"

Tabla de referencia:

ExpresiónResultado

typeof "hola""string"
typeof true"boolean"
typeof undefined"undefined"
typeof NaN"number"
typeof null"object"
typeof {}"object"
typeof []"object"
typeof function() {}"function"

Cuidado: typeof [] devuelve "object", no "array". Los arrays se estudiarán más adelante.

16. Conversión de tipos

La conversión de tipos consiste en transformar un valor de un tipo a otro.

Hay dos tipos de conversión:

  • conversión explícita: la hace la persona que programa;
  • conversión implícita o coerción: la hace JavaScript automáticamente.

16.1. Conversión explícita con Number()

Number() intenta convertir un valor a número.

console.log(Number("3"));     // 3
console.log(Number("3.14"));  // 3.14
console.log(Number("hola"));  // NaN
console.log(Number(true));    // 1
console.log(Number(false));   // 0
console.log(Number(null));    // 0

Ejemplo habitual en DWCC:

let edadTexto = "20";
let edadNumero = Number(edadTexto);

console.log(edadNumero + 1); // 21

16.2. Conversión explícita con String()

String() convierte un valor a texto.

console.log(String(42));      // "42"
console.log(String(true));    // "true"
console.log(String(null));    // "null"
console.log(String(undefined)); // "undefined"

Ejemplo:

let edad = 20;
let mensaje = "Edad: " + String(edad);

console.log(mensaje); // "Edad: 20"

16.3. Conversión explícita con Boolean()

Boolean() convierte un valor a true o false.

console.log(Boolean(1));         // true
console.log(Boolean(0));         // false
console.log(Boolean("hola"));    // true
console.log(Boolean(""));        // false
console.log(Boolean(null));      // false
console.log(Boolean(undefined)); // false
console.log(Boolean(NaN));       // false

Los valores que se comportan como false en contextos booleanos se llaman falsy. Los demás se llaman truthy.

Valores falsy principales:

false, 0, "", null, undefined, NaN

En este bloque se introduce la idea. Se retomará con más detalle al estudiar condiciones y operadores lógicos.

16.4. Conversión implícita o coerción

La conversión implícita aparece cuando JavaScript convierte tipos automáticamente para intentar resolver una operación.

console.log("3" * 2); // 6
console.log("3" + 2); // "32"
console.log(true + 1); // 2

El caso más peligroso al inicio suele ser el operador +, porque puede sumar números o concatenar textos.

console.log(5 + 2);     // 7
console.log("5" + 2);   // "52"
console.log(5 + "2");   // "52"

Cuando un dato venga como texto y se necesite hacer cálculos, conviene convertirlo explícitamente con Number() antes de operar.

17. Asignaciones y operadores básicos vinculados a variables

Este bloque no desarrolla todavía todos los operadores del lenguaje. Solo se incluyen los imprescindibles para trabajar con variables, tipos y conversiones.

17.1. Asignación

El operador = asigna un valor a una variable.

let edad = 20;
edad = 21;

console.log(edad); // 21

No debe confundirse asignación con comparación.

let edad = 20;

// edad = 18 asigna un valor
// edad === 18 compara un valor

let x = 5;

console.log(x); // 21

18. Comparación: == frente a ===

JavaScript tiene dos formas habituales de comparar igualdad.

18.1. Igualdad no estricta: ==

== compara después de permitir conversiones implícitas.

console.log(0 == "0"); // true
console.log(false == 0); // true

Puede producir resultados poco intuitivos.

18.2. Igualdad estricta: ===

=== compara valor y tipo, sin hacer conversión implícita.

console.log(0 === "0"); // false
console.log(0 === 0);   // true

También existe la desigualdad estricta:

console.log(0 !== "0"); // true

Usar === y !== por defecto. Evitar == y != salvo que se explique expresamente por qué se usan.

19. Errores frecuentes en este bloque

Error 1. Copiar mal el bloque de código

Incorrecto:

js let edad = 20;

Correcto:

let edad = 20;

La palabra js no forma parte del código. Solo se usa en Markdown para indicar que el bloque contiene JavaScript.

Error 2. Usar una variable antes de declararla correctamente

console.log(nombre);
let nombre = "Ana";

Este ejemplo produce error porque nombre se usa antes de inicializarse.

Error 3. Confundir = con ===

let edad = 18;

console.log(edad = 20);   // Asigna 20
console.log(edad === 20); // Compara con 20

Error 4. Pensar que const hace inmutable un objeto

const usuario = {
  nombre: "Ana"
};

usuario.nombre = "Luis"; // Permitido

const impide reasignar usuario, pero no bloquea sus propiedades.

Error 5. No tener en cuenta la conversión implícita

let cantidad = "5";

console.log(cantidad + 1);         // "51"
console.log(Number(cantidad) + 1); // 6

Error 6. Comparar NaN directamente

console.log(NaN === NaN); // false

Para comprobar NaN, usa:

console.log(Number.isNaN(NaN)); // true

Error 7. Olvidar el caso especial de typeof null

console.log(typeof null); // "object"

Es un comportamiento histórico. Para comprobar null, usa:

let dato = null;
console.log(dato === null); // true

20. Ejemplos guiados para clase

20.1. Variables y tipos mezclados

let dato = 5;
console.log(dato, typeof dato);

dato = "cinco";
console.log(dato, typeof dato);

dato = true;
console.log(dato, typeof dato);

Qué se debe observar:

  • la variable se llama igual todo el tiempo;
  • el valor cambia;
  • el tipo cambia porque depende del valor actual.

20.2. undefined, null y typeof

let a;
const b = null;

console.log(a);        // undefined
console.log(b);        // null
console.log(typeof a); // "undefined"
console.log(typeof b); // "object"

Qué se debe observar:

  • a existe, pero no tiene valor asignado;
  • b contiene intencionadamente null;
  • typeof null devuelve "object".

20.3. Conversión de texto a número

let precioTexto = "19.99";
let unidadesTexto = "3";

let precio = Number(precioTexto);
let unidades = Number(unidadesTexto);
let total = precio * unidades;

console.log(total); // 59.97

Qué se debe observar:

  • los datos pueden llegar como texto;
  • si necesitamos operar matemáticamente, conviene convertirlos antes.

20.4. Conversión incorrecta

let valor = "hola";
let numero = Number(valor);

console.log(numero);              // NaN
console.log(typeof numero);       // "number"
console.log(Number.isNaN(numero)); // true

Qué se debe observar:

  • Number("hola") no puede producir un número válido;
  • el resultado es NaN;
  • NaN pertenece al tipo number.

20.5. Igualdad estricta

console.log(0 == "0");   // true
console.log(0 === "0");  // false

console.log(false == 0);  // true
console.log(false === 0); // false

Qué se debe observar:

  • == permite conversiones implícitas;
  • === compara valor y tipo;
  • para evitar sorpresas, en este módulo usaremos ===.

24. Resumen final

Para repasar este bloque, debes dominar estas ideas:

  1. JavaScript se puede ejecutar en la consola del navegador o desde un fichero .js enlazado con HTML.
  2. Las instrucciones se escriben siguiendo reglas de sintaxis.
  3. Los comentarios ayudan a documentar el código.
  4. Los identificadores deben tener nombres válidos y claros.
  5. En código moderno se usan let y const; var se reserva para entender código antiguo.
  6. let permite reasignar; const no permite reasignar.
  7. const no hace inmutables los objetos.
  8. let y const tienen ámbito de bloque.
  9. JavaScript tiene tipado dinámico: el tipo pertenece al valor.
  10. La coerción de tipos puede producir resultados inesperados.
  11. undefined suele indicar falta de valor asignado.
  12. null indica ausencia intencionada de valor.
  13. typeof null devuelve "object", aunque null no sea un objeto normal.
  14. NaN es un valor especial del tipo number.
  15. Para comparar, se recomienda usar === y !==.
  16. Para convertir tipos de forma clara, se usan Number(), String() y Boolean().

25. Para bloques posteriores

Este documento no desarrolla en profundidad algunos contenidos porque encajan mejor en el bloque siguiente de operadores y estructuras de control.

Dejar para el bloque posterior:

  • operadores de comparación completos: <, >, <=, >=, !=, !==;
  • operadores lógicos: &&, ||, !;
  • tablas de verdad;
  • valores truthy y falsy aplicados a condiciones reales;
  • cortocircuito en && y ||;
  • estructuras condicionales: if, else if, else;
  • operador ternario: condicion ? valorSiTrue : valorSiFalse;
  • uso del ternario solo en casos sencillos y legibles;
  • bucles: for, while, do...while;
  • patrones habituales: contadores, acumuladores y recorridos;
  • errores frecuentes en condiciones y bucles.

Nota docente: el operador ternario queda expresamente reservado para el bloque de operadores y estructuras de control, no para este bloque de sintaxis, variables y tipos.
