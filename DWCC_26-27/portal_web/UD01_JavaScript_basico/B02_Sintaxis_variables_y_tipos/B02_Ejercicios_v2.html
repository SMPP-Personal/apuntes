Bloque 2. Hoja de ejercicios: sintaxis, variables y tipos en JavaScript

Módulo: DWCC — Desenvolvemento web en contorno cliente

Esta hoja de ejercicios acompaña al bloque de apuntes sobre sintaxis básica, variables, constantes, tipos de datos y conversión de tipos en JavaScript.

El objetivo no es memorizar resultados, sino aprender a predecir, ejecutar, comprobar y explicar el comportamiento básico de JavaScript en el navegador.

1. Objetivos de la hoja

Al terminar estos ejercicios deberías ser capaz de:

  • escribir identificadores válidos en JavaScript;
  • declarar variables y constantes con let y const;
  • distinguir declaración, inicialización y reasignación;
  • reconocer tipos básicos de datos;
  • usar typeof para comprobar tipos;
  • diferenciar undefined, null, NaN e Infinity;
  • realizar conversiones explícitas con Number(), String() y Boolean();
  • detectar resultados inesperados por conversión implícita;
  • usar correctamente la igualdad estricta === y la desigualdad estricta !==;
  • probar fragmentos de código en la consola del navegador o en un fichero .js.

2. Relación con la programación del módulo

Esta hoja trabaja principalmente contenidos de la UD1, relacionados con:

  • RA2: escribe sentencias simples, aplicando la sintaxis del lenguaje y verificando su ejecución sobre navegadores web.
  • CA2.2: utilización de variables y operadores disponibles en el lenguaje.
  • CA2.3: identificación de ámbitos de utilización de las variables.
  • CA2.4: reconocimiento de peculiaridades del lenguaje respecto a conversiones entre tipos de datos.
  • CA2.7: incorporación de comentarios al código.
  • CA2.8: uso de herramientas y entornos para facilitar la programación, prueba y documentación del código.

3. Instrucciones de trabajo

Para cada ejercicio:

  1. Intenta responder sin ejecutar primero el código.
  2. Comprueba después el resultado en la consola del navegador o en un fichero .js enlazado desde HTML.
  3. Corrige tu respuesta si te equivocaste.
  4. Escribe una explicación breve, no solo el resultado.

Cuando se pida código, debe estar bien indentado y con nombres de variables claros.
