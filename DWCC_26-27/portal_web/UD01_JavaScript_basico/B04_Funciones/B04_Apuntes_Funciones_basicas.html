Bloque 4. Funciones básicas en JavaScript

1. Qué vas a aprender en este bloque

En este bloque aprenderás a organizar el código JavaScript usando funciones.

Una función permite agrupar instrucciones bajo un nombre para poder reutilizarlas varias veces. Esto evita repetir código, mejora la claridad del programa y facilita la comprobación de errores.

Al terminar este bloque deberías ser capaz de:

  • declarar funciones usando la sintaxis clásica function;
  • llamar a una función desde el programa principal;
  • diferenciar entre parámetros y argumentos;
  • usar return para devolver un resultado;
  • distinguir entre mostrar un valor con console.log() y devolverlo con return;
  • comprender el ámbito local de las variables declaradas dentro de una función;
  • escribir funciones sencillas que usen condicionales o bucles;
  • aplicar buenas prácticas básicas al nombrar y diseñar funciones.

2. Relación con lo visto hasta ahora

Antes de estudiar funciones ya hemos trabajado:

  • variables y constantes;
  • tipos de datos;
  • operadores;
  • expresiones booleanas;
  • condicionales;
  • bucles.

Las funciones permiten encapsular todo eso en bloques reutilizables.

Por ejemplo, en lugar de escribir varias veces el mismo cálculo, podemos escribirlo una sola vez dentro de una función y llamarla cuando la necesitemos. 

function calcularDoble(numero) {
  return numero * 2;
}

console.log(calcularDoble(5));
console.log(calcularDoble(12));

3. Qué es una función y por qué se usa

Una función es un bloque de código con un nombre, pensado para realizar una tarea concreta.

Podemos imaginar una función como una receta:

  • tiene un nombre;
  • puede recibir datos de entrada;
  • ejecuta una serie de instrucciones;
  • puede devolver un resultado.

Ejemplo sencillo:

function saludar() {
  console.log("Hola desde DWCC");
}

En este ejemplo, saludar es el nombre de la función. Dentro de las llaves { ... } están las instrucciones que forman la función.

¿Por qué usamos funciones?

Usamos funciones porque ayudan a:

  1. Evitar repetir código. Si una operación se repite varias veces, podemos ponerla en una función y llamarla cuando haga falta.

  2. Ordenar el programa. Cada función puede encargarse de una tarea concreta.

  3. Mejorar la legibilidad. Un nombre bien elegido ayuda a entender qué hace una parte del programa.

  4. Facilitar las pruebas. Podemos comprobar una función con distintos valores de entrada.

  5. Reutilizar código. Una función bien diseñada puede usarse en diferentes partes del programa.

Ejemplo:

function precioConIVA(precioSinIVA, tipoIVA) {
  const iva = precioSinIVA * (tipoIVA / 100);
  const total = precioSinIVA + iva;
  return total;
}

const totalLibro = precioConIVA(20, 4);
console.log("Libro con IVA:", totalLibro);

const totalProducto = precioConIVA(50, 21);
console.log("Producto con IVA:", totalProducto);

La lógica del cálculo aparece una sola vez, dentro de precioConIVA, pero se reutiliza con datos diferentes.

4. Declarar una función

Para crear una función usamos la palabra reservada function, seguida del nombre de la función, paréntesis y un bloque de código entre llaves.

Sintaxis básica:

function nombreDeLaFuncion() {
  // instrucciones
}

Ejemplo:

function mostrarMensaje() {
  console.log("Estoy aprendiendo funciones");
}

En este punto la función está declarada, pero todavía no se ha ejecutado.

Declarar una función significa decirle a JavaScript:

“Existe una función con este nombre y estas instrucciones.”

5. Llamar a una función

Para ejecutar una función usamos su nombre seguido de paréntesis. 

function mostrarMensaje() {
  console.log("Estoy aprendiendo funciones");
}

mostrarMensaje();

La línea:

mostrarMensaje();

es una llamada a la función.

Una función puede llamarse varias veces:

function saludar() {
  console.log("Hola");
}

saludar();
saludar();
saludar();

La función se escribe una vez, pero se ejecuta tantas veces como la llamemos.

6. Parámetros y argumentos

Las funciones pueden recibir datos de entrada.

A esos datos, dentro de la declaración de la función, los llamamos parámetros.

Cuando llamamos a la función y le pasamos valores concretos, esos valores se llaman argumentos.

6.1. Función con un parámetro

function saludar(nombre) {
  console.log("Hola " + nombre);
}

saludar("Ana");
saludar("Marcos");

En este ejemplo:

  • nombre es el parámetro;
  • "Ana" y "Marcos" son argumentos;
  • en cada llamada, el parámetro nombre toma el valor que se le pasa.

La salida sería:

Hola Ana
Hola Marcos

6.2. Función con varios parámetros

function mostrarSuma(a, b) {
  console.log(a + b);
}

mostrarSuma(3, 4);
mostrarSuma(10, 5);

En este caso:

  • a y b son parámetros;
  • 3 y 4 son argumentos en la primera llamada;
  • 10 y 5 son argumentos en la segunda llamada.

6.3. Si faltan argumentos

En JavaScript, si llamamos a una función sin pasar todos los argumentos, los parámetros que falten tendrán el valor undefined. 

function sumar(a, b) {
  return a + b;
}

console.log(sumar(3));

En este ejemplo, a vale 3, pero b vale undefined. El resultado será NaN, porque JavaScript intenta calcular: 

3 + undefined

Por eso es importante llamar a las funciones con los argumentos que necesitan.

7. Funciones que muestran datos y funciones que devuelven datos

Este punto es importante.

No es lo mismo mostrar un dato que devolver un dato.

7.1. Mostrar con console.log()

console.log() muestra información en la consola, pero no devuelve un resultado útil para seguir calculando. 

function mostrarSuma(a, b) {
  console.log(a + b);
}

mostrarSuma(3, 4);

Esta función muestra 7 en consola, pero no permite guardar ese resultado directamente en una variable para reutilizarlo.

7.2. Devolver con return

return devuelve un valor al lugar desde donde se llamó la función. 

function devolverSuma(a, b) {
  return a + b;
}

const resultado = devolverSuma(3, 4);
console.log("Resultado:", resultado);

Aquí la función devuelve 7, y ese valor se guarda en la variable resultado.

7.3. Comparación directa

function sumarYMostrar(a, b) {
  console.log(a + b);
}

function sumarYDevolver(a, b) {
  return a + b;
}

sumarYMostrar(3, 4);

const total = sumarYDevolver(3, 4);
console.log(total * 2);

La segunda función es más reutilizable porque el resultado puede guardarse, combinarse con otros cálculos o pasarse a otra función.

8. return: devolver un valor y terminar la función

La instrucción return tiene dos efectos:

  1. Devuelve un valor.
  2. Termina inmediatamente la ejecución de la función.

Ejemplo:

function sumar(a, b) {
  const resultado = a + b;
  return resultado;
}

const total = sumar(3, 4);
console.log("Total:", total);

También podemos escribirlo de forma más compacta:

function sumar(a, b) {
  return a + b;
}

8.1. Código después de return

Cualquier instrucción escrita después de un return ejecutado no se ejecutará. 

function saludar(nombre) {
  return "Hola " + nombre;
  console.log("Esta línea no se ejecuta");
}

console.log(saludar("Ana"));

La función termina en el return. La línea posterior queda inaccesible.

8.2. Función sin return

Si una función no tiene un return explícito, devuelve undefined. 

function mostrarMensaje() {
  console.log("Hola");
}

const resultado = mostrarMensaje();
console.log(resultado); // undefined

La función muestra "Hola", pero no devuelve ningún valor útil.

9. Variables locales y ámbito de función

Las variables declaradas dentro de una función solo existen dentro de esa función. 

function ejemplo() {
  let x = 10;
  console.log(x);
}

ejemplo();

// console.log(x); // Error: x is not defined

La variable x se ha creado dentro de ejemplo(), por tanto no se puede usar desde fuera.

Esto se llama ámbito local.

9.1. Parámetros como variables locales

Los parámetros también se comportan como variables locales de la función. 

function saludar(nombre) {
  console.log("Hola " + nombre);
}

saludar("Ana");

// console.log(nombre); // Error: nombre is not defined

El parámetro nombre solo existe dentro de la función saludar.

9.2. Evitar depender de variables globales

Una función es más clara y reutilizable si recibe los datos que necesita mediante parámetros.

Menos recomendable:

let precio = 100;

function calcularIVA() {
  return precio * 0.21;
}

console.log(calcularIVA());

Más recomendable:

function calcularIVA(precio) {
  return precio * 0.21;
}

console.log(calcularIVA(100));

La segunda versión es mejor porque la función no depende de una variable externa llamada precio.

10. Funciones con condicionales

Una función puede contener condicionales. 

function esMayorDeEdad(edad) {
  if (edad >= 18) {
    return true;
  } else {
    return false;
  }
}

console.log(esMayorDeEdad(20));
console.log(esMayorDeEdad(15));

La función anterior puede simplificarse porque la condición edad >= 18 ya produce un valor booleano: 

function esMayorDeEdad(edad) {
  return edad >= 18;
}

Otro ejemplo con texto:

function obtenerResultadoNota(nota) {
  if (nota >= 5) {
    return "Aprobado";
  }

  return "Suspenso";
}

console.log(obtenerResultadoNota(7));
console.log(obtenerResultadoNota(3));

En este ejemplo, si se cumple nota >= 5, la función devuelve "Aprobado" y termina. Si no se cumple, continúa y devuelve "Suspenso".

11. Funciones con bucles

Una función también puede contener bucles. 

function imprimirHasta(n) {
  for (let i = 1; i <= n; i++) {
    console.log(i);
  }
}

imprimirHasta(5);

La función imprimirHasta recibe un número y muestra todos los números desde 1 hasta ese número.

Otro ejemplo:

function sumarHasta(n) {
  let suma = 0;

  for (let i = 1; i <= n; i++) {
    suma = suma + i;
  }

  return suma;
}

const resultado = sumarHasta(10);
console.log("Suma:", resultado);

Aquí el bucle se usa para calcular un resultado, y la función lo devuelve con return.

12. Buenas prácticas básicas

12.1. Usar nombres claros

El nombre de una función debe indicar qué hace.

Malos nombres:

function cosa() {
  // ...
}

function funcion1() {
  // ...
}

Mejores nombres:

function calcularTotal() {
  // ...
}

function esMayorDeEdad(edad) {
  // ...
}

function mostrarMensajeError() {
  // ...
}

En JavaScript se recomienda usar camelCase para nombres de funciones: 

calcularPrecioFinal
mostrarResultado
obtenerNombreUsuario

12.2. Usar normalmente verbos

Una función suele representar una acción. Por eso muchos nombres empiezan por verbos:

  • calcularTotal
  • mostrarMensaje
  • obtenerDescuento
  • validarEdad
  • convertirMinutosASegundos

12.3. Una función, una tarea principal

Una función debe intentar hacer una tarea clara.

Menos recomendable:

function hacerTodo() {
  // pide datos
  // calcula
  // valida
  // muestra resultados
  // modifica muchas variables
}

Más recomendable:

function calcularPrecioConIVA(precio, iva) {
  return precio + precio * (iva / 100);
}

12.4. Preferir return cuando el resultado se va a reutilizar

Si solo queremos ver algo durante una prueba, console.log() está bien.

Pero si queremos usar el resultado más adelante, conviene devolverlo. 

function calcularCuadrado(numero) {
  return numero * numero;
}

const cuadrado = calcularCuadrado(5);
console.log(cuadrado + 10);

13. Errores frecuentes

13.1. Declarar la función pero no llamarla

function saludar() {
  console.log("Hola");
}

Este código no muestra nada porque la función está declarada, pero no llamada.

Correcto:

function saludar() {
  console.log("Hola");
}

saludar();

13.2. Confundir parámetro y argumento

function saludar(nombre) {
  console.log("Hola " + nombre);
}

saludar("Ana");
  • nombre es el parámetro.
  • "Ana" es el argumento.

13.3. Esperar que console.log() devuelva un valor 

function sumar(a, b) {
  console.log(a + b);
}

const resultado = sumar(3, 4);
console.log(resultado); // undefined

La función muestra 7, pero no devuelve 7.

Correcto:

function sumar(a, b) {
  return a + b;
}

const resultado = sumar(3, 4);
console.log(resultado);

13.4. Escribir código después de return pensando que se ejecutará 

function ejemplo() {
  return "Fin";
  console.log("Esto no se ejecuta");
}

La línea posterior al return no se ejecuta.

13.5. Usar variables locales fuera de la función

function crearMensaje() {
  const mensaje = "Hola";
}

// console.log(mensaje); // Error

mensaje solo existe dentro de la función.

13.6. No pasar todos los argumentos necesarios

function multiplicar(a, b) {
  return a * b;
}

console.log(multiplicar(5)); // NaN

Falta el segundo argumento, por lo que b vale undefined.

14. Qué no vamos a tratar todavía

En este bloque vamos a trabajar solo con la forma clásica: 

function nombreFuncion(parametro1, parametro2) {
  // instrucciones
}

De momento no vamos a estudiar en profundidad:

  • funciones flecha;
  • funciones anónimas;
  • callbacks;
  • funciones como valores;
  • closures;
  • recursividad;
  • métodos de arrays.

Estos contenidos aparecerán más adelante cuando ya tengamos una base más sólida.

Nota sobre hoisting

Las declaraciones de función como esta:

function saludar() {
  console.log("Hola");
}

son conocidas por JavaScript antes de ejecutar el código. Por eso, técnicamente, una función declarada con function nombre() { ... } puede llamarse antes de aparecer escrita en el archivo.

Sin embargo, en este bloque usaremos siempre este estilo:

function saludar() {
  console.log("Hola");
}

saludar();

Primero declaramos la función y después la llamamos. Es más claro para aprender y evita confusiones.

15. Resumen final

Una función es un bloque de código con nombre que podemos reutilizar.

Ideas clave:

  • Se declara con function.
  • Se ejecuta llamándola por su nombre seguido de paréntesis.
  • Puede recibir datos mediante parámetros.
  • Los valores concretos que se pasan al llamar son argumentos.
  • Puede devolver un resultado con return.
  • return termina la función.
  • Una función sin return explícito devuelve undefined.
  • Las variables declaradas dentro de una función tienen ámbito local.
  • Una función puede contener condicionales y bucles.
  • Es mejor usar nombres claros y funciones con una tarea concreta.

Ejemplo final:

function calcularPrecioFinal(precio, descuento) {
  const rebaja = precio * (descuento / 100);
  return precio - rebaja;
}

const total = calcularPrecioFinal(100, 15);
console.log("Precio final:", total);

En este ejemplo:

  • precio y descuento son parámetros;
  • 100 y 15 son argumentos;
  • la función calcula un resultado;
  • return devuelve ese resultado;
  • el resultado se guarda en total y después se muestra en consola.
