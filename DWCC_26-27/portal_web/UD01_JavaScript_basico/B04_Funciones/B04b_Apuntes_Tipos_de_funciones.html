Bloque 4b. Tipos de funciones en JavaScript

1. Relación con el bloque anterior

En el bloque anterior vimos la forma básica de declarar y usar funciones en JavaScript: 

function sumar(a, b) {
  return a + b;
}

const resultado = sumar(3, 4);
console.log(resultado); // 7

Esta forma sigue siendo correcta y la usaremos muchas veces. En este bloque veremos otras formas habituales de escribir y utilizar funciones en JavaScript moderno:

  • funciones anónimas;
  • funciones flecha;
  • callbacks, es decir, funciones que se pasan como argumento a otras funciones.

La idea principal es comprender que en JavaScript una función no solo se puede declarar y llamar: también se puede guardar en una variable o constante, y también se puede pasar como dato a otra función.

2. Formas de escribir funciones en JavaScript

En JavaScript podemos escribir funciones de varias formas. En este bloque nos centraremos en tres:

Forma Ejemplo Uso básico
Declaración de función function saludar() { ... } Forma clásica y clara para empezar.
Función anónima asignada const saludar = function () { ... }; Función sin nombre propio guardada en una constante.
Función flecha const saludar = () => { ... }; Forma moderna y compacta de escribir funciones.

Ejemplo comparativo:

// Declaración de función
function sumar(a, b) {
  return a + b;
}

// Función anónima asignada a una constante
const sumarAnonima = function (a, b) {
  return a + b;
};

// Función flecha asignada a una constante
const sumarFlecha = (a, b) => {
  return a + b;
};

console.log(sumar(3, 4));        // 7
console.log(sumarAnonima(3, 4)); // 7
console.log(sumarFlecha(3, 4));  // 7

Cuidado
Las funciones guardadas en const, tanto anónimas como flecha, deben estar creadas antes de llamarlas. Para empezar, escribe siempre primero la función y después la llamada.

Las tres funciones hacen lo mismo. La diferencia está en la forma de escribirlas y en algunos detalles avanzados que veremos más adelante.

3. Funciones anónimas

Una función anónima es una función que no tiene nombre propio después de la palabra function.

Normalmente se guarda en una variable o constante para poder usarla más tarde. 

const saludar = function () {
  console.log("Hola desde una función anónima");
};

saludar();

En este ejemplo:

  • la función no tiene nombre propio;
  • la constante saludar permite referirse a esa función;
  • para ejecutarla usamos saludar().

La constante no guarda un número ni un texto: guarda una función. 

const multiplicar = function (a, b) {
  return a * b;
};

const resultado = multiplicar(4, 5);
console.log(resultado); // 20

3.1. Función anónima no significa función imposible de llamar

Una función puede ser anónima y aun así podemos llamarla si está asignada a una variable o constante. 

const mostrarMensaje = function () {
  console.log("Mensaje de prueba");
};

mostrarMensaje();

Lo que no tiene nombre es la función en sí, pero la constante mostrarMensaje actúa como referencia.

4. Funciones flecha

Una función flecha es una forma más corta de escribir funciones. En lugar de usar la palabra function, se usa el símbolo =>. 

const sumar = (a, b) => {
  return a + b;
};

console.log(sumar(3, 4)); // 7

La función anterior equivale, para este uso básico, a esta función clásica: 

function sumar(a, b) {
  return a + b;
}

En este bloque usaremos funciones flecha solo en casos simples. Las diferencias avanzadas entre funciones tradicionales y funciones flecha, especialmente el comportamiento de this, se verán más adelante.

5. Funciones flecha con retorno implícito

Cuando una función flecha solo tiene una instrucción y esa instrucción devuelve un valor, podemos escribirla de forma más compacta.

Versión con llaves y return explícito:

const doble = (n) => {
  return n * 2;
};

console.log(doble(4)); // 8

Versión compacta con retorno implícito:

const doble = n => n * 2;

console.log(doble(4)); // 8

En la forma compacta:

  • no se escriben llaves {};
  • no se escribe return;
  • el resultado de la expresión se devuelve automáticamente.

5.1. Un parámetro

Si la función flecha tiene un solo parámetro, los paréntesis son opcionales. 

const cuadrado = n => n * n;

console.log(cuadrado(5)); // 25

También se puede escribir con paréntesis:

const cuadrado = (n) => n * n;

Las dos formas son válidas.

5.2. Varios parámetros

Si hay varios parámetros, los paréntesis son obligatorios. 

const sumar = (a, b) => a + b;

console.log(sumar(3, 7)); // 10

5.3. Ningún parámetro

Si no hay parámetros, hay que escribir paréntesis vacíos. 

const saludar = () => "Hola DWCC";

console.log(saludar()); // Hola DWCC

6. Callbacks: funciones como argumentos

Un callback es una función que se pasa como argumento a otra función.

Hasta ahora habíamos pasado a las funciones números, textos o valores booleanos: 

function saludar(nombre) {
  console.log("Hola " + nombre);
}

saludar("Ana");

Pero en JavaScript también podemos pasar una función como argumento.

6.1. Ejemplo síncrono básico

function sumar(a, b) {
  return a + b;
}

function multiplicar(a, b) {
  return a * b;
}

function calcular(a, b, operacion) {
  return operacion(a, b);
}

console.log(calcular(3, 4, sumar));       // 7
console.log(calcular(3, 4, multiplicar)); // 12

En este ejemplo:

  • sumar es una función;
  • multiplicar es una función;
  • calcular recibe una función en el parámetro operacion;
  • dentro de calcular, se ejecuta operacion(a, b).

Cuando llamamos a calcular(3, 4, sumar), estamos diciendo:

Usa los valores 3 y 4, y aplica la función sumar.

Cuando llamamos a calcular(3, 4, multiplicar), estamos diciendo:

Usa los valores 3 y 4, y aplica la función multiplicar.

Este es el concepto básico de callback: una función que se entrega a otra función para que esta la use.

7. Pasar una función no es lo mismo que ejecutarla

Este punto es muy importante.

No es lo mismo escribir el nombre de una función que escribir una llamada a la función. 

function mostrarMensaje() {
  console.log("Hola");
}

mostrarMensaje;   // referencia a la función, no se ejecuta
mostrarMensaje(); // llamada a la función, sí se ejecuta

Cuando pasamos una función como callback, normalmente pasamos la función sin paréntesis. 

function mostrarMensaje() {
  console.log("Han pasado 2 segundos");
}

setTimeout(mostrarMensaje, 2000);

Aquí no escribimos mostrarMensaje() porque no queremos ejecutarla inmediatamente. Queremos que setTimeout la ejecute después de 2000 milisegundos.

7.1. Error frecuente

function mostrarMensaje() {
  console.log("Han pasado 2 segundos");
}

setTimeout(mostrarMensaje(), 2000); // error de enfoque

En este caso, mostrarMensaje() se ejecuta inmediatamente, antes de que setTimeout pueda esperar los 2 segundos.

La forma correcta es:

setTimeout(mostrarMensaje, 2000);

Nota
En este bloque usamos setTimeout solo para entender cómo se pasa una función como callback. La asincronía se estudiará con más detalle en un bloque posterior.

8. Callback con función flecha

También podemos escribir el callback directamente como función flecha. 

setTimeout(() => {
  console.log("Han pasado 2 segundos");
}, 2000);

En este ejemplo:

  • no declaramos antes una función llamada mostrarMensaje;
  • escribimos directamente una función flecha anónima dentro de setTimeout;
  • setTimeout ejecutará esa función después de 2000 milisegundos.

Versión equivalente usando una función declarada:

function mostrarMensaje() {
  console.log("Han pasado 2 segundos");
}

setTimeout(mostrarMensaje, 2000);

Las dos versiones son válidas. En niveles iniciales suele ser más clara la segunda; en JavaScript moderno verás mucho la primera.

9. Qué no vamos a tratar todavía

En este bloque solo queremos entender las formas básicas de escribir y pasar funciones.

Dejamos para documentos posteriores:

  • el comportamiento de this en funciones tradicionales y funciones flecha;
  • funciones flecha dentro de objetos;
  • métodos de array que reciben callbacks, como forEach, map, filter y reduce;
  • callbacks aplicados a eventos del DOM;
  • asincronía real;
  • promesas;
  • async / await;
  • closures;
  • recursividad.

No se eliminan del curso: simplemente se verán cuando el alumnado tenga base suficiente.

10. Errores frecuentes

10.1. Olvidar que una función flecha asignada a const necesita nombre de constante

Incorrecto:

(a, b) => a + b;

Eso crea una función, pero no la estamos guardando ni usando.

Correcto:

const sumar = (a, b) => a + b;

console.log(sumar(2, 3));

10.2. Usar llaves y olvidar return

Incorrecto:

const doble = n => {
  n * 2;
};

console.log(doble(4)); // undefined

Correcto:

const doble = n => {
  return n * 2;
};

También correcto en forma compacta:

const doble = n => n * 2;

10.3. Confundir pasar una función con ejecutarla

Incorrecto en un callback como setTimeout:

setTimeout(mostrarMensaje(), 2000);

Correcto:

setTimeout(mostrarMensaje, 2000);

10.4. Pensar que una función anónima no se puede usar

Una función anónima puede usarse si está guardada en una variable o constante. 

const saludar = function () {
  console.log("Hola");
};

saludar();

11. Ejercicios de transformación

Ejercicio 1. Función clásica a función anónima

Código de partida:

function saludar() {
  console.log("Hola DWCC");
}

Tarea:

  • Reescribe la función como función anónima asignada a una constante llamada saludar.
  • Llama a la función para comprobar que sigue funcionando igual.

Ejercicio 2. Función clásica a función flecha con llaves

Código de partida:

function cuadrado(n) {
  return n * n;
}

Tarea:

  • Reescribe la función como función flecha guardada en una constante cuadrado.
  • Usa la versión con llaves y return explícito.
  • Prueba con console.log(cuadrado(5));.

Ejercicio 3. Función clásica a función flecha compacta

Código de partida:

function doble(n) {
  return n * 2;
}

Tarea:

  • Reescribe la función como función flecha en una sola línea.
  • No uses llaves.
  • No escribas return.
  • Prueba con console.log(doble(4));.

Ejercicio 4. Función con dos parámetros a función flecha

Código de partida:

function sumar(a, b) {
  return a + b;
}

Tarea:

  • Reescribe la función como función flecha guardada en const sumar.
  • Prueba con console.log(sumar(3, 7));.

Ejercicio 5. Callback con función clásica a función flecha inline

Código de partida:

setTimeout(function () {
  console.log("Mensaje después de 2 segundos");
}, 2000);

Tarea:

  • Reescribe el código usando una función flecha anónima en lugar de function () { ... }.
  • No cambies el mensaje ni el tiempo.

Ejercicio 6. Identificar tipos de funciones

Indica qué tipo de función aparece en cada caso:

function restar(a, b) {
  return a - b;
}

const dividir = function (a, b) {
  return a / b;
};

const triplicar = n => n * 3;

Opciones:

  • declaración de función;
  • función anónima asignada a constante;
  • función flecha.

Ejercicio 7. Corregir un callback

Código de partida:

function mostrarMensaje() {
  console.log("Hola después de 1 segundo");
}

setTimeout(mostrarMensaje(), 1000);

Tarea:

  • Corrige el código para que mostrarMensaje se ejecute después de 1 segundo, no inmediatamente.
  • Explica en una frase cuál era el error.

Ejercicio 8. Callback síncrono básico

Completa el código para que calcular pueda usar una función recibida como tercer argumento. 

function sumar(a, b) {
  return a + b;
}

function calcular(a, b, operacion) {
  // completar
}

console.log(calcular(5, 2, sumar)); // 7

12. Resumen final

  • Una función anónima es una función sin nombre propio.
  • Una función anónima puede guardarse en una constante.
  • Una función flecha usa => y permite escribir funciones de forma más compacta.
  • Si una función flecha tiene una sola expresión, puede usar retorno implícito.
  • Un callback es una función que se pasa como argumento a otra función.
  • Pasar una función no es lo mismo que ejecutarla.
  • mostrarMensaje referencia la función.
  • mostrarMensaje() ejecuta la función.
  • setTimeout(funcion, tiempo) recibe una función y un tiempo en milisegundos.
