UD2_B4b. Apuntes: localStorage y persistencia básica en JavaScript

1. Qué vamos a aprender en este documento

En los documentos anteriores hemos trabajado datos en variables, arrays, objetos y JSON. El problema es que, cuando se recarga la página, esos datos se pierden.

En este documento vamos a estudiar almacenamiento local básico con localStorage.

Al finalizar, deberías ser capaz de:

  • entender qué significa persistir datos en el navegador;
  • guardar datos con localStorage.setItem();
  • recuperar datos con localStorage.getItem();
  • borrar datos con removeItem() y clear();
  • guardar arrays y objetos usando JSON.stringify();
  • recuperar arrays y objetos usando JSON.parse();
  • aplicar localStorage a preferencias, formularios sencillos o pequeñas listas;
  • identificar errores frecuentes y riesgos básicos.

2. Datos temporales y datos persistentes

Una variable guarda información mientras la página está cargada.

let nombre = "Ana";
console.log(nombre);

Pero si recargamos la página, esa variable vuelve a crearse desde cero.

En cambio, localStorage permite guardar información en el navegador para que siga disponible aunque:

  • se recargue la página;
  • se cierre y se vuelva a abrir el navegador;
  • el usuario vuelva más tarde a la misma aplicación.

Ejemplos de uso:

  • guardar el nombre de usuario en una práctica;
  • recordar un tema claro/oscuro;
  • guardar una lista de tareas;
  • conservar el borrador de un formulario;
  • guardar preferencias de visualización.

3. Qué es localStorage

localStorage es una API del navegador que permite almacenar pares clave-valor.

La clave es un nombre.

El valor se guarda como texto.

Ejemplo:

localStorage.setItem("usuario", "Ana");

Aquí:

  • "usuario" es la clave;
  • "Ana" es el valor.

Para recuperar el dato:

const usuario = localStorage.getItem("usuario");
console.log(usuario); // "Ana"

4. Guardar datos con setItem

La sintaxis básica es:

localStorage.setItem(clave, valor);

Ejemplo:

localStorage.setItem("curso", "DWCC");
localStorage.setItem("idioma", "es");

Podemos comprobarlo desde las herramientas de desarrollo del navegador, normalmente en:

Application / Storage / Local Storage

El nombre exacto puede variar según el navegador.

5. Recuperar datos con getItem

La sintaxis básica es:

localStorage.getItem(clave);

Ejemplo:

const curso = localStorage.getItem("curso");
console.log(curso);

Si la clave no existe, devuelve null.

const valor = localStorage.getItem("noExiste");
console.log(valor); // null

Por eso es habitual comprobar si el dato existe antes de usarlo:

const nombre = localStorage.getItem("nombre");

if (nombre !== null) {
  console.log("Hola, " + nombre);
} else {
  console.log("No hay nombre guardado");
}

6. Borrar datos con removeItem

Para borrar una clave concreta:

localStorage.removeItem("usuario");

Ejemplo:

localStorage.setItem("usuario", "Ana");
localStorage.removeItem("usuario");

console.log(localStorage.getItem("usuario")); // null

7. Borrar todo con clear

clear() borra todo el almacenamiento local del sitio actual.

localStorage.clear();

Hay que tener cuidado: elimina todas las claves de localStorage asociadas a esa página o dominio.

En prácticas de aula es mejor borrar claves concretas con removeItem().

8. localStorage siempre guarda texto

Aunque guardemos un número, localStorage lo almacena como texto.

localStorage.setItem("edad", 20);

const edad = localStorage.getItem("edad");

console.log(edad);        // "20"
console.log(typeof edad); // "string"

Si queremos usarlo como número:

const edadNumero = Number(localStorage.getItem("edad"));
console.log(edadNumero + 1);

9. Guardar booleanos

Los booleanos también se guardan como texto.

localStorage.setItem("acepta", true);

const acepta = localStorage.getItem("acepta");
console.log(acepta);        // "true"
console.log(typeof acepta); // "string"

Para convertirlo de nuevo:

const aceptaBooleano = localStorage.getItem("acepta") === "true";

10. Guardar objetos con JSON

No podemos guardar directamente un objeto como objeto real en localStorage.

Incorrecto:

const usuario = {
  nombre: "Ana",
  edad: 20
};

localStorage.setItem("usuario", usuario);

Eso guardaría algo poco útil como:

[object Object]

La forma correcta es convertirlo a JSON:

const usuario = {
  nombre: "Ana",
  edad: 20
};

const usuarioJSON = JSON.stringify(usuario);

localStorage.setItem("usuario", usuarioJSON);

Para recuperarlo:

const textoGuardado = localStorage.getItem("usuario");
const usuarioRecuperado = JSON.parse(textoGuardado);

console.log(usuarioRecuperado.nombre);
console.log(usuarioRecuperado.edad);

11. Guardar arrays con JSON

También podemos guardar arrays.

const tareas = ["estudiar", "practicar JS", "repasar DOM"];

localStorage.setItem("tareas", JSON.stringify(tareas));

Para recuperarlo:

const tareasGuardadas = localStorage.getItem("tareas");
const tareas = JSON.parse(tareasGuardadas);

console.log(tareas);

12. Recuperar arrays de forma segura

Si la clave no existe, getItem() devuelve null.
Si intentamos hacer JSON.parse(null), puede generar resultados no deseados o dificultar la lógica.

Una forma sencilla:

const tareasGuardadas = localStorage.getItem("tareas");

let tareas;

if (tareasGuardadas !== null) {
  tareas = JSON.parse(tareasGuardadas);
} else {
  tareas = [];
}

Forma abreviada habitual:

const tareas = JSON.parse(localStorage.getItem("tareas")) || [];

Esta segunda forma es práctica, pero conviene entender primero la versión larga.

13. Ejemplo completo: guardar preferencia de tema

HTML:

<button id="tema-claro">Tema claro</button>
<button id="tema-oscuro">Tema oscuro</button>

JavaScript:

const botonClaro = document.querySelector("#tema-claro");
const botonOscuro = document.querySelector("#tema-oscuro");

const temaGuardado = localStorage.getItem("tema");

if (temaGuardado !== null) {
  document.body.className = temaGuardado;
}

botonClaro.addEventListener("click", () => {
  document.body.className = "claro";
  localStorage.setItem("tema", "claro");
});

botonOscuro.addEventListener("click", () => {
  document.body.className = "oscuro";
  localStorage.setItem("tema", "oscuro");
});

La próxima vez que se cargue la página, se recuperará el tema guardado.

14. Ejemplo completo: lista de tareas persistente

let tareas = JSON.parse(localStorage.getItem("tareas")) || [];

function guardarTareas() {
  localStorage.setItem("tareas", JSON.stringify(tareas));
}

function agregarTarea(texto) {
  tareas.push(texto);
  guardarTareas();
}

agregarTarea("Estudiar localStorage");
agregarTarea("Practicar JSON");

console.log(tareas);

Cada vez que se añade una tarea, se actualiza el almacenamiento local.

15. Diferencia entre localStorage y sessionStorage

Existe otra API parecida: sessionStorage.

APIDuración aproximada
localStoragePersiste aunque se cierre el navegador
sessionStoragePersiste solo durante la sesión/pestaña

En este documento trabajaremos principalmente con localStorage.

16. Qué NO debemos guardar en localStorage

No se deben guardar datos sensibles, como:

  • contraseñas;
  • tokens secretos;
  • datos personales delicados;
  • información privada que no debería quedar en el navegador.

localStorage es útil para prácticas, preferencias y datos sencillos, pero no es una base de datos segura.

17. Errores frecuentes

Error 1. Olvidar que todo se guarda como texto

localStorage.setItem("contador", 5);

const contador = localStorage.getItem("contador");
console.log(contador + 1); // "51"

Corrección:

const contador = Number(localStorage.getItem("contador"));
console.log(contador + 1); // 6

Error 2. Guardar objetos sin JSON.stringify

Incorrecto:

localStorage.setItem("usuario", { nombre: "Ana" });

Correcto:

localStorage.setItem("usuario", JSON.stringify({ nombre: "Ana" }));

Error 3. Recuperar JSON sin comprobar si existe

const usuario = JSON.parse(localStorage.getItem("usuario"));
console.log(usuario.nombre);

Si no hay usuario guardado, puede provocar errores en la lógica posterior.

Mejor:

const usuarioGuardado = localStorage.getItem("usuario");

if (usuarioGuardado !== null) {
  const usuario = JSON.parse(usuarioGuardado);
  console.log(usuario.nombre);
}

Error 4. Usar clear() sin pensar

localStorage.clear();

Puede borrar más datos de los que queríamos. Es más seguro borrar claves concretas:

localStorage.removeItem("tareas");

19. Resumen final

localStorage permite guardar información sencilla en el navegador.

Las operaciones básicas son:

localStorage.setItem("clave", "valor");
localStorage.getItem("clave");
localStorage.removeItem("clave");
localStorage.clear();

Para guardar arrays y objetos usamos JSON:

localStorage.setItem("datos", JSON.stringify(datos));
const datos = JSON.parse(localStorage.getItem("datos"));

Es una herramienta muy útil para pequeñas aplicaciones interactivas, especialmente cuando queremos conservar datos entre recargas.
