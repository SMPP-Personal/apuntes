Actividades Bloque 1. Introducción y entorno de trabajo con JavaScript

Introducción Actividad 1 Actividad 2 Actividad 3 Actividad 4 Actividad 5

Introducción y Objetivos

En este documento se recogen una serie de actividades prácticas para el Bloque 1 del módulo DWCC. Su objetivo es que el alumnado se familiarice con la consola del navegador, el uso de archivos JavaScript enlazados a HTML, la interpretación de errores básicos y la organización mínima de un pequeño proyecto.

Estas actividades están pensadas para reforzar el RA2 del currículo oficial: sentencias simples en JavaScript, sintaxis, prueba y depuración básica del código.

Objetivos de Aprendizaje

  • Crear una estructura mínima de proyecto con HTML y un archivo script.js enlazado.
  • Usar la consola del navegador para ejecutar y comprobar código JavaScript.
  • Interpretar los mensajes de error básicos que aparecen en la consola.
  • Utilizar console.log de forma sistemática para depurar.
  • Trabajar con variables simples, concatenaciones, tipos de datos y operaciones básicas.
  • Responder a un cuestionario corto sobre conceptos de entorno y ejecución de JavaScript.
  • Realizar una pequeña práctica integradora que combine HTML, JavaScript y consola.
Nota pedagógica: Puedes adaptar la dificultad y el ritmo de estas actividades según el grupo, haciendo algunas de ellas obligatorias y dejando otras como ampliación o refuerzo.

1. Actividad 1 – Primer contacto con la consola y script.js

Objetivo: Que el alumnado use la consola del navegador y un archivo .js enlazado a un HTML mínimo.

Enunciado orientativo:

  1. Crea una carpeta llamada ud1-bloque1-ej1.
  2. Dentro, crea un archivo index.html con un título “Pruebas JavaScript UD1”.
  3. Crea un archivo script.js y enlázalo desde el HTML con la etiqueta de script antes de la etiqueta de cierre </body>: 
    <script src="script.js"></script>
  4. En script.js, escribe el siguiente código y guarda el archivo: 
    console.log("Bienvenido a JavaScript en DWCC");
console.log("Este mensaje se muestra en la consola del navegador");
  5. Abre index.html en el navegador, abre las herramientas de desarrollador y la pestaña Consola. Comprueba que aparecen los mensajes.
Buena práctica: Coloca siempre la etiqueta <script src="..."></script> al final del <body> en estos ejemplos iniciales para asegurarte de que el HTML se ha cargado por completo antes de ejecutar el JavaScript.

Variantes / rúbrica simple (para RA2):

  • 0: No funciona o no se ha enlazado el script de forma correcta.
  • 1: Script enlazado pero no se usa console.log correctamente.
  • 2: Script enlazado perfectamente y muestra al menos dos mensajes bien formateados en la consola.
Sugerencia: Puedes pedir al alumnado que modifique los mensajes para personalizarlos (por ejemplo, incluyendo su nombre o el ciclo que cursan).

2. Actividad 2 – Identificación de errores usando la consola

Objetivo: Empezar a usar la consola como herramienta activa de depuración.

Enunciado orientativo:

Dado este código erróneo en script.js:

console.log("Inicio del ejercicio");
let numero = 10
console.log("El número es: " numero);
console.log("Fin del ejercicio");
  1. Copia el código en tu proyecto y abre la consola del navegador.
  2. Observa qué error aparece reflejado en la consola e intenta interpretarlo.
  3. Corrige el código para que funcione de manera impecable.
  4. Añade un cuarto console.log que muestre el doble del valor de la variable numero.
Error común: Pensar que el problema es exclusivamente "la falta de un punto y coma". En este caso, el error principal de compilación/ejecución está en la concatenación de la cadena y la variable sin usar ningún tipo de operador.
Buena práctica: Lee siempre el mensaje de error completo (tipo de error, archivo, línea, columna) antes de empezar a modificar código de manera aleatoria.

Puntos clave a trabajar de cara a la evaluación:

  • Entender que la falta de punto y coma no siempre es el problema principal en JavaScript.
  • Detectar el error real en la concatenación: falta el operador +.
  • Interpretar de forma fluida el mensaje de error que reporta el navegador.

3. Actividad 3 – Mini-ficha de prácticas en consola

Objetivo: Practicar de forma directa en consola las primeras sentencias simples sin introducir estructuras de control complejas.

Formato sugerido:

Hoja de ejercicios cortos de ejecución rápida donde el alumnado debe ir anotando los resultados arrojados por el intérprete.

  1. Escribe directamente en la consola: 
    console.log("Mi nombre es", "Ana");
    ¿Qué se muestra exactamente en el panel de salida?
  2. Declara una variable curso con el valor "DWCC" y muestra por consola el mensaje "Estoy cursando DWCC", haciendo uso de dicha variable.
    Pista: Puedes combinar texto fijo y la variable dentro del mismo console.log.
  3. Declara dos variables numéricas, a y b, y genera salidas en consola para:
    • La suma de ambas.
    • La resta.
    • El producto resultante.
  4. Escribe e interpreta el siguiente bloque de sentencias: 
    let x = "10";
let y = 2;
console.log(x + y);
console.log(Number(x) + y);
    Anota razonadamente la diferencia entre ambos resultados.
  5. Utiliza el operador unario typeof en al menos tres tipos de valores diferentes (42, "hola", true) y documenta qué devuelve la consola.
Metodología: Esta ficha es ideal para trabajarse en parejas de código (Peer Programming), pidiendo que comparen las conclusiones antes de realizar la puesta en común general.
Buena práctica: Cuando no comprendas por qué JavaScript devuelve un resultado inesperado, acostúmbrate a usar typeof para verificar dinámicamente el tipo de datos con el que estás operando.

4. Actividad 4 – Cuestionario corto / test de Bloque 1

Objetivo: Verificar la comprensión teórica y conceptual de la ejecución de JavaScript en el ecosistema cliente.

Preguntas Tipo Test de Evaluación

  1. ¿Dónde se ejecuta el JavaScript del módulo DWCC en esta unidad?
    a) En el servidor web
    b) En el navegador del usuario (cliente)
    c) En la base de datos relacional
    d) Directamente sobre el kernel del Sistema Operativo
  2. ¿Qué herramienta integrada usamos para visualizar salidas de console.log?
    a) Inspector de red
    b) Panel de Almacenamiento Local
    c) Consola de Desarrollador (DevTools)
    d) Administrador de tareas de Chromium
  3. ¿Cuál es la sintaxis semántica y correcta para enlazar un archivo externo?
    a) <js href="script.js"></js>
    b) <script src="script.js"></script>
    c) <link rel="script" href="script.js">
    d) <script link="script.js"></script>
  4. ¿Cuál de estas instrucciones mostrará de forma efectiva un mensaje de depuración?
    a) console.print("Hola");
    b) log.console("Hola");
    c) console.log("Hola");
    d) alert.console("Hola");
  5. ¿Qué afirmación define con precisión la utilidad de console.log?
    a) Sirve para alterar de forma directa la estructura del DOM HTML.
    b) Muestra mensajes internos en el panel del navegador, ideal para tareas de depuración.
    c) Almacena datos persistentes en el disco local.
    d) Se encarga de alternar las clases CSS de presentación visual.
Sugerencia de despliegue: Puedes exportar o mapear este cuestionario en un formulario digital o integrarlo de manera directa dentro de tu aula virtual (Moodle).
Buena práctica: Al corregir los cuestionarios con el grupo, retoma las preguntas con mayor índice de error y monta un ejemplo rápido directo en la consola para consolidar el concepto.

5. Actividad 5 – Pequeña práctica integradora del Bloque 1

Objetivo: Combinar de forma sólida la maquetación HTML mínima, el enlazado lógico de scripts y el flujo de salida por consola.

Enunciado orientativo:

Crea una mini-aplicación de prácticas denominada presentacion-js que conste de:

  1. Un archivo estructurado index.html cuyo encabezado visible sea "Presentación con JavaScript".
  2. Un archivo script complementario enlazado llamado presentacion.js.
  3. Dentro de presentacion.js, implementa código estructurado que realice lo siguiente:
    • Declare variables explícitas que contengan tu nombre, apellidos y el ciclo formativo actual.
    • Haga uso de console.log para pintar en pantalla una presentación multilínea bien formateada.
    • Calcule de forma automatizada tu edad aproximada restando el año en curso a tu año de nacimiento (guardado en una variable de asignación fija) e imprima el resultado final combinado.
Buena práctica: Separa siempre las fases del desarrollo: primero inicializa y procesa los cálculos en variables de memoria lógica (ej: edad) y, posteriormente, pinta dichas variables, evitando mezclar operaciones complejas inline dentro del propio console.log.
Consejo docente: Esta práctica integradora rompe el hielo de forma fantástica durante las primeras sesiones del curso, sirviendo como presentación formal del alumnado y diagnóstico de su soltura técnica.

Resumen del Bloque de Actividades

Esta propuesta didáctica del Bloque 1 guía al estudiante de forma progresiva: desde la toma de contacto guiada con el motor del cliente, la detección quirúrgica de errores semánticos, el testeo conceptual y, por último, el desarrollo autónomo de un script integrador.

Adquirir estos hábitos limpios de inspección, control de tipos modernos (let y const) y depuración sistemática de errores sentará los cimientos indispensables antes de abordar manipulaciones complejas del DOM o dar el salto definitivo hacia frameworks reactivos.
