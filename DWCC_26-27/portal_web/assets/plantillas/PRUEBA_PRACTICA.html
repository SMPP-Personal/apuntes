Práctica global de cierre

UD2_B5. Práctica global: aplicación web interactiva

1. Finalidad de la práctica

Esta práctica sirve como cierre de la UD2: JavaScript avanzado en entorno cliente.

El objetivo es construir una pequeña aplicación web completa que integre los contenidos trabajados en los bloques anteriores:

  • objetos y arrays de objetos;
  • métodos avanzados de arrays;
  • DOM;
  • eventos;
  • formularios;
  • validación;
  • JSON;
  • asincronía;
  • fetch;
  • async / await;
  • gestión de errores;
  • renderizado dinámico de información.

No se trata de copiar código de los apuntes, sino de demostrar que se sabe organizar una aplicación sencilla, probarla, depurarla y explicar las decisiones tomadas.

2. Contexto de la práctica

Vas a crear una aplicación llamada:

Panel interactivo de recursos de aprendizaje

La aplicación mostrará una colección de recursos relacionados con el módulo de DWCC: apuntes, ejercicios, prácticas, vídeos, enlaces o actividades.

Los datos estarán guardados en un archivo JSON y se cargarán desde JavaScript usando fetch.

La aplicación permitirá:

  • cargar recursos desde un archivo JSON;
  • mostrarlos dinámicamente en la página;
  • buscar recursos por texto;
  • filtrar por tipo o dificultad;
  • ordenar resultados;
  • mostrar estadísticas;
  • validar un formulario para proponer un nuevo recurso;
  • añadir temporalmente ese recurso a la lista mostrada;
  • gestionar mensajes de carga, error y resultado vacío.

El nuevo recurso añadido mediante el formulario no tiene que guardarse de forma permanente. Basta con añadirlo al array en memoria y volver a renderizar la página.

7. Fase 1: cargar datos con fetch

En script.js, crea una variable global para guardar los recursos:

let recursos = [];

Después crea una función asíncrona para cargar los datos:

async function cargarRecursos() {
  try {
    mostrarMensaje("Cargando recursos...");

    const respuesta = await fetch("datos/recursos.json");

    if (!respuesta.ok) {
      throw new Error("No se pudieron cargar los recursos");
    }

    recursos = await respuesta.json();

    mostrarMensaje("");
    actualizarVista();
  } catch (error) {
    mostrarMensaje("Error al cargar los datos: " + error.message);
  }
}

Esta función debe ejecutarse al cargar la página.

9. Fase 3: buscar, filtrar y ordenar

Crea una función principal para recalcular lo que se muestra:

function obtenerRecursosVisibles() {
  // código
}

Debe aplicar:

  1. Búsqueda por texto.
  2. Filtro por tipo.
  3. Filtro por dificultad.
  4. Ordenación.

Puedes usar filter, sort y otros métodos de array.

Ejemplo orientativo:

function obtenerRecursosVisibles() {
  const texto = inputBusqueda.value.trim().toLowerCase();
  const tipo = selectTipo.value;
  const dificultad = selectDificultad.value;
  const orden = selectOrdenacion.value;

  let resultado = recursos.filter((recurso) => {
    const coincideTexto = recurso.titulo.toLowerCase().includes(texto);
    const coincideTipo = tipo === "todos" || recurso.tipo === tipo;
    const coincideDificultad = dificultad === "todas" || recurso.dificultad === dificultad;

    return coincideTexto && coincideTipo && coincideDificultad;
  });

  resultado = [...resultado].sort((a, b) => {
    if (orden === "titulo") {
      return a.titulo.localeCompare(b.titulo);
    }

    if (orden === "duracionAsc") {
      return a.duracion - b.duracion;
    }

    if (orden === "duracionDesc") {
      return b.duracion - a.duracion;
    }

    return 0;
  });

  return resultado;
}

  • número de recursos pendientes;

const duracionTotal = lista.reduce((total, recurso) => total + recurso.duracion, 0);

Si no hay recursos visibles, la duración media debe evitar divisiones entre cero.

11. Fase 5: eventos de búsqueda, filtro y ordenación

Debes asociar eventos a los controles:

inputBusqueda.addEventListener("input", actualizarVista);
selectTipo.addEventListener("change", actualizarVista);
selectDificultad.addEventListener("change", actualizarVista);
selectOrdenacion.addEventListener("change", actualizarVista);

La función actualizarVista() debe:

  1. Obtener los recursos visibles.
  2. Renderizar las tarjetas.
  3. Renderizar las estadísticas.
function actualizarVista() {
  const visibles = obtenerRecursosVisibles();
  renderizarRecursos(visibles);
  renderizarEstadisticas(visibles);
}

13. Fase 7: añadir un nuevo recurso

Al enviar el formulario:

  1. Usa preventDefault().
  2. Valida los datos.
  3. Si hay errores, no añadas nada.
  4. Si todo es correcto, crea un nuevo objeto recurso.
  5. Añádelo al array recursos.
  6. Actualiza la vista.
  7. Limpia el formulario.
  8. Muestra un mensaje de confirmación.

Ejemplo de estructura del nuevo objeto:

const nuevoRecurso = {
  id: Date.now(),
  titulo: titulo.value.trim(),
  tipo: tipo.value,
  dificultad: dificultad.value,
  duracion: Number(duracion.value),
  etiquetas: etiquetas.value.split(",").map((etiqueta) => etiqueta.trim()).filter((etiqueta) => etiqueta !== ""),
  publicado: false
};

Puedes usar Date.now() para generar un identificador temporal.

14. Fase 8: usar JSON en la aplicación

Además de cargar el archivo JSON con fetch, debes demostrar que entiendes JSON.stringify() y JSON.parse().

Añade una sección de depuración o exportación que permita mostrar en consola una copia de los recursos actuales en formato JSON:

function exportarRecursosAJSON() {
  const textoJSON = JSON.stringify(recursos, null, 2);
  console.log(textoJSON);

  const copia = JSON.parse(textoJSON);
  console.log("Copia recuperada:", copia);
}

Puedes llamarla desde el código o asociarla a un botón si quieres ampliar la práctica.

17. Restricciones

En esta práctica no se debe usar:

  • frameworks;
  • librerías externas;
  • jQuery;
  • Bootstrap obligatorio;
  • código copiado sin entender;
  • almacenamiento persistente obligatorio;
  • APIs con clave o autenticación;
  • backend propio.

La aplicación debe funcionar con HTML, CSS y JavaScript puro.

18. Pruebas obligatorias

Antes de entregar, comprueba:

  • La página carga sin errores en consola.
  • Los datos se cargan desde recursos.json.
  • Si cambias mal la ruta del JSON, aparece un mensaje de error.
  • La búsqueda filtra correctamente.
  • Los filtros por tipo y dificultad funcionan.
  • La ordenación funciona.
  • Las estadísticas se actualizan al filtrar.
  • El formulario no permite enviar datos vacíos.
  • El formulario añade un recurso válido.
  • El nuevo recurso aparece en el listado.
  • Las etiquetas del nuevo recurso se convierten correctamente en array.
  • La aplicación no se rompe si no hay resultados visibles.

En esta práctica debes demostrar que sabes construir una aplicación web interactiva completa usando JavaScript en el navegador.

La aplicación debe cargar datos, mostrarlos en el DOM, permitir interacción del usuario, validar formularios, procesar arrays de objetos y gestionar errores de forma clara.

Si la práctica funciona correctamente, tendrás una evidencia global de los contenidos principales de la UD2.
