Comandos básicos: Windows e Linux

Apuntes do módulo IMSO · CIFP A Carballeira

Bloque UD1 – Manexo básico de consola en (CMD e PowerShell)Windows
1. Concepto xeral

Nun contorno profesional Windows (tanto escritorio como Windows Server), un administrador debe dominar CMD e, especialmente, PowerShell para:

  • Xestionar ficheiros e cartafoles.
  • Consultar e filtrar información en ficheiros (por exemplo, logs).
  • Controlar e supervisar procesos e servizos.

Este bloque céntrase en comandos básicos pero moi útiles para o día a día.

2. Rutas e navegación en Windows

2.1. Rutas absolutas e relativas en Windows

  • Ruta absoluta:
    • Comeza pola unidade, por exemplo C:\.
    • Exemplos:
      • C:\Usuarios\alumno\Documentos\informe.docx
      • D:\Datos\Logs\hoxe.log
    • Ruta relativa:
      • Defínese respecto ao directorio actual.
      • Se estamos en C:\Usuarios\alumno,
        • Documentos\informe.docx

          é unha ruta relativa.

Símbolos especiais:

.

→ directorio actual.

..

→ directorio pai.

2.2. Navegación en CMD

Comandos básicos en CMD:

cd
  • Mostra ou cambia o directorio actual.
  • cd

    (sen argumentos) mostra a ruta actual.

  • cd ..

    sobe un nivel.

dir
  • Lista o contido do directorio actual (ficheiros e cartafoles).
mkdir nome

/ md nome

  • Crea un novo cartafol.
rmdir nome

/ rd nome

  • Elimina un cartafol baleiro.

2.3. Navegación en PowerShell

En PowerShell os conceptos son os mesmos, aínda que os cmdlets teñen outros nomes:

Get-Location
  • Mostra o directorio actual.
  • Alias: pwd.
Set-Location ruta
  • Cambia de directorio.
  • Alias: cd, chdir.
Get-ChildItem
  • Lista o contido do directorio actual.
  • Alias: ls, dir.
New-Item -ItemType Directory -Name nome
  • Crea un directorio novo.
Remove-Item nome
  • Elimina un ficheiro ou un directorio baleiro.
  • Para eliminar un cartafol con contido: Remove-Item nome -Recurse (usar con moito coidado).
3. Xestión básica de ficheiros en Windows

3.1. En CMD

copy orixe destino
  • Copia un ficheiro.
  • Exemplo: copy informe.txt copia_informe.txt.
move orixe destino
  • Move ou renomea ficheiros.
del ficheiro
  • Elimina ficheiros.
  • Exemplo: del antigo.log.
type ficheiro
  • Mostra o contido dun ficheiro de texto pola pantalla.

3.2. En PowerShell

Copy-Item orixe destino
  • Copia ficheiros ou directorios.
Move-Item orixe destino
  • Move ou renomea.
Remove-Item ficheiro_ou_carpeta
  • Elimina ficheiros ou cartafoles baleiros.
  • Con contido dentro: Remove-Item cartafol -Recurse (usar con moito coidado).
New-Item nome -ItemType File
  • Crea un ficheiro baleiro.
Get-Content ficheiro

(alias gc, type)

  • Mostra o contido dun ficheiro de texto.
4. Redireccións e tuberías en Windows

4.1. Redirección de saída

En CMD e en PowerShell:

>
  • Redirixe a saída dun comando a un ficheiro, sobrescribindo o contido anterior.
  • Exemplo:
    • dir > lista.txt
>>
  • Engade a saída dun comando ao final dun ficheiro.
  • Exemplo:
    • dir C:\Windows >> lista.txt

4.2. Pipes (tuberías)

O símbolo | permite encadear comandos:

  • En CMD:
    • dir | find "txt"
      • Mostra só as liñas de dir que conteñen “txt”.
    • En PowerShell:
      • Get-ChildItem *log*
        • Lista ficheiros e cartafoles cuxo nome contén “log”.
        • Get-ChildItem | Where-Object { $_.Name -like '*log*' }
          • Mesmo filtro; útil cando se encadea con outros cmdlets.
5. Localización de información en ficheiros (logs) en Windows

Revisar logs e ficheiros de texto grandes é unha tarefa habitual de administración.

5.1. En CMD

type ficheiro.log
  • Mostra o contido dun ficheiro de log (útil só se non é moi grande).
find "texto" ficheiro.log
  • Mostra as liñas de ficheiro.log que conteñen “texto”.
findstr "texto" ficheiro.log
  • Versión máis potente; permite buscar cadeas e patróns.
  • Opción /I ignora maiúsculas/minúsculas.
  • Sen comillas, varias palabras actúan como OR (calquera coincidencia).
  • Exemplos:
    • findstr /I error warning sistema.log

      → liñas con “error” ou “warning”.

    • findstr /I "error critico" sistema.log

      → liñas coa frase exacta “error critico”.

5.2. En PowerShell

Get-Content ficheiro.log
  • Le o contido completo do ficheiro.
Get-Content ficheiro.log -Tail 20
  • Mostra só as últimas 20 liñas.
Get-Content ficheiro.log -Wait -Tail 10
  • Semellante a tail -f en Linux: segue mostrando novas liñas segundo se escriben no log.
Select-String -Path ficheiro.log -Pattern "ERROR"
  • Busca liñas que conteñen “ERROR”.
  • Combinación típica:
    • Get-Content sistema.log | Select-String -Pattern "failed"
6. Xestión básica de procesos en Windows

Un administrador debe saber ver que procesos están activos e, se é necesario, rematar procesos problemáticos.

6.1. En CMD

tasklist
  • Lista os procesos en execución co seu nome, PID (identificador de proceso) e consumo básico.
taskkill /PID número /F
  • Termina o proceso co PID indicado.
  • /F

    forza a finalización.

taskkill /IM nome.exe /F
  • Termina procesos polo nome de imaxe (por exemplo, taskkill /IM notepad.exe /F).

6.2. En PowerShell

Get-Process
  • Lista procesos en execución (semellante a tasklist).
Get-Process nome
  • Filtra por nome, por exemplo Get-Process notepad.
Stop-Process -Name nome
  • Detén procesos polo nome.
Stop-Process -Id PID
  • Detén procesos polo identificador.

Estes comandos poden combinarse con Where-Object e outros filtros máis adiante (por exemplo en UD3), pero na UD1 abonda con que entendan:

  • Como ver a lista de procesos.
  • Como localizar un proceso concreto.
  • Como detelo se está bloqueado ou consome demasiados recursos.
Bloque UD1 – Manexo básico de consola enLinux
1. Concepto xeral

En servidores Linux, contornos na nube e moitos sistemas de produción, non hai escritorio gráfico ou está desactivado.

Todo o traballo de instalación, mantemento, análise de logs e resolución de incidencias faise en terminal.

Este bloque introduce:

  • Navegación polo sistema de ficheiros.
  • Xestión básica de ficheiros e directorios.
  • Redireccións e tuberías.
  • Busca e filtrado de información en ficheiros (logs simples).
  • Unha primeira ollada á xestión de procesos.
2. Rutas e navegación en Linux

2.1. Rutas absolutas e relativas en Linux

  • Ruta absoluta:
    • Comeza sempre por /.
    • Exemplos:
      • /home/alumno/documentos/informe.txt
      • /var/log/syslog
    • Ruta relativa:
      • Defínese respecto ao directorio actual.
      • Se estamos en /home/alumno,
        • documentos/informe.txt

          é unha ruta relativa.

Símbolos especiais:

.

→ directorio actual.

..

→ directorio pai.

2.2. Navegación básica

Comandos principais:

pwd
  • Mostra a ruta absoluta do directorio actual.
ls
  • Lista o contido do directorio actual.
  • ls -l

    : información detallada (permisos, propietario, tamaño…).

cd ruta
  • Cambia de directorio.
  • Exemplos:
    • cd /home/alumno
    • cd documentos
    • cd ..
mkdir nome
  • Crea un novo directorio.
rmdir nome
  • Elimina un directorio baleiro.
3. Xestión básica de ficheiros en Linux

3.1. Creación, copia, movemento, borrado

touch ficheiro
  • Crea un ficheiro baleiro ou actualiza a súa data de modificación.
cp orixe destino
  • Copia ficheiros ou directorios (con -r para directorios).
mv orixe destino
  • Move ou renomea ficheiros/directorios.
rm ficheiro
  • Elimina ficheiros.
  • Coidado: non vai á papeleira.
rm -r directorio
  • Elimina un directorio e o seu contido.
  • Usar con moita precaución, especialmente se se combina con sudo.

3.2. Visualización de contido

cat ficheiro
  • Mostra todo o contido dun ficheiro de texto.
less ficheiro
  • Mostra o ficheiro paginado (útil para ficheiros longos).
  • Sair con q.
head ficheiro
  • Mostra as primeiras liñas (por defecto 10).
tail ficheiro
  • Mostra as últimas liñas.
tail -f ficheiro
  • Mostra as últimas liñas e segue actualizando en tempo real (ideal para logs).
4. Redireccións e tuberías en Linux

4.1. Redirección de saída

>
  • Escribe a saída dun comando nun ficheiro, sobrescribindo o contido anterior.
  • Exemplo: ls > lista.txt
>>
  • Engade a saída dun comando ao final dun ficheiro.
  • Exemplo: ls /etc >> lista.txt

4.2. Tuberías (pipes)

comando1 | comando2
  • Envía a saída de comando1 como entrada de comando2.

Exemplos:

ls /etc | less
  • Lista /etc e pásao a less para ver máis comodamente.
dmesg | grep -i usb
  • Filtra as mensaxes do kernel para ver só liñas que conteñen “usb”.
5. Localización de información en ficheiros (logs) en Linux

En Ubuntu 24, moitos rexistros do sistema están no journal de systemd (journalctl), ademais dos ficheiros en /var/log/. Para buscar texto dentro dun ficheiro de log, grep segue sendo a ferramenta máis directa; para consultar os rexistros xerais do sistema traballarédes máis adiante con journalctl.

5.1. grep – busca de cadeas en texto

grep é unha das ferramentas máis importantes para traballar con logs e ficheiros de texto.

Formato básico:

grep texto ficheiro
  • Mostra as liñas de ficheiro que conteñen “texto”.

Opcións útiles:

-i

→ ignora maiúsculas/minúsculas.

-n

→ mostra o número de liña.

Exemplos:

grep ERROR sistema.log
grep -in fallo /var/log/syslog

(ficheiro de log clásico; en Ubuntu 24 pode estar baleiro se o sistema usa sobre todo o journal).

journalctl --no-pager | grep -i fallo

(rexistros do sistema vía systemd).

Uso combinado con pipes:

dmesg | grep -i error
journalctl --no-pager | grep -i network

5.2. cut – extracción sinxela de columnas

cut permite extraer campos dunha liña, se están separados por un delimitador.

cut -d':' -f1 ficheiro
  • Co delimitador :, extrae o primeiro campo de cada liña.

Exemplos prácticos:

  • Dado un ficheiro tipo usuario:data:acción,
    • cut -d':' -f1 rexistro.txt

      → lista só os usuarios.

6. Xestión básica de procesos en Linux

Para supervisar que está a pasar no sistema e detectar procesos problemáticos:

ps
  • Mostra procesos en execución asociados á sesión actual.
ps aux
  • Lista detallada de procesos de todo o sistema.
top
  • Interface en tempo real con procesos, uso de CPU e memoria.
  • Permite ordenar, buscar, matar procesos desde unha interface interactiva.
htop
  • Versión mellorada de top (se está instalada).
kill PID
  • Envía un sinal a un proceso (por defecto, terminar).
kill -9 PID
  • Forza a finalización do proceso co PID indicado (sinal SIGKILL).

Conceptos clave:

  • PID: identificador numérico dun proceso.
  • Saber localizar un proceso que consome moitos recursos e detelo se é necesario é unha habilidade básica dun administrador.
