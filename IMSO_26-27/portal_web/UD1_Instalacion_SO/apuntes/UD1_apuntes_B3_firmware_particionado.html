1. Firmware: BIOS e UEFI
1.1. Que é o firmware de arranque

O firmware é o software básico que se executa ao acender o equipo e que se encarga de inicializar o hardware e buscar desde onde arranca o sistema operativo.

Na práctica, cando falamos de BIOS ou UEFI estamos a falar deste firmware de arranque.

1.2. BIOS clásica

A BIOS é o firmware tradicional:

  • execútase ao acender o ordenador,
  • comproba memoria, discos e dispositivos básicos,
  • busca un dispositivo de arranque (disco, USB, CD/DVD),
  • ten limitacións en discos grandes e en sistemas modernos.

Hoxe en día considérase unha tecnoloxía antiga e, en equipos actuais, está a ser substituída por UEFI.

1.3. UEFI

UEFI é o firmware moderno que substitúe á BIOS na maioría de equipos actuais.

Vantaxes principais de UEFI:

  • mellor compatibilidade con discos grandes,
  • arranque máis flexible,
  • soporte para partición EFI,
  • compatibilidade con Secure Boot,
  • mellor integración con sistemas operativos modernos.

En equipos modernos recoméndase traballar con UEFI + GPT.

1.4. Secure Boot

Secure Boot é unha función de UEFI que comproba que o software de arranque é de confianza.

  • Reduce o risco de malware que se cargue antes do sistema operativo.
  • Pode causar problemas se se queren instalar sistemas ou controladores non asinados.

Windows 11 require UEFI con Secure Boot activado para unha instalación compatible nun equipo moderno.

1.5. TPM 2.0

O TPM é un módulo de seguridade que almacena chaves criptográficas e axuda a protexer o sistema.

  • Windows 11 require TPM 2.0 para unha instalación compatible nun equipo moderno.
  • En máquinas virtuais actuais (por exemplo, VirtualBox 7), pode configurarse un TPM virtual.

Na UD1 interesa que o alumnado saiba que é TPM 2.0 e por que é relevante para Windows 11, aínda que a configuración detallada quede máis ligada a prácticas de laboratorio.

1.6. Virtualización por hardware

Para usar máquinas virtuais con bo rendemento, o procesador debe soportar virtualización por hardware:

Fabricante Nome da tecnoloxía
Intel Intel VT‑x
AMD AMD‑V

Debe estar activada no firmware BIOS/UEFI do equipo físico.

2. Particionado: MBR, GPT e partición EFI
2.1. Que é o particionado

O particionado consiste en dividir un disco en partes lóxicas chamadas particións.

Cada partición pode ter unha finalidade distinta:

  • sistema operativo,
  • datos,
  • arranque,
  • recuperación,
  • memoria de intercambio (swap),
  • etc.
2.2. MBR e GPT

O material resume así os dous esquemas principais:

Sistema Características
MBR Modelo antigo, con máis limitacións, usado con BIOS clásico.
GPT Modelo moderno, usado habitualmente con UEFI; permite máis particións e discos grandes.

En equipos modernos recoméndase UEFI + GPT.

2.3. Partición EFI

A partición EFI é unha partición especial usada por sistemas con UEFI.

  • Contén ficheiros de arranque como Windows Boot Manager, GRUB e outros cargadores.
  • Adoita estar formatada en FAT32.

É un concepto clave para entender por que un erro ao borrar esta partición pode deixar un equipo sen arranque, aínda que o sistema operativo siga instalado no disco.

2.4. Sistemas de ficheiros básicos

O cadro de sistemas de ficheiros que se traballa na UD1 é:

Sistema de ficheiros Uso habitual
NTFS Windows
FAT32 USB, partición EFI, compatibilidade
exFAT USB ou discos externos grandes
ext4 Linux
Btrfs Linux avanzado (predeterminado en openSUSE e Fedora)
XFS Servidores Linux
APFS macOS

A parte máis avanzada (Btrfs, XFS, APFS) enlaza coa futura UD3 sobre sistemas de ficheiros e tolerancia a fallos, pero xa se poden ir mencionando aquí como contexto.

2.5. Particións habituais en Windows

Nunha instalación moderna de Windows poden aparecer varias particións:

Partición Función
EFI Arranque UEFI
MSR Reservada por Microsoft
Principal Onde se instala Windows
Recuperación Ferramentas de recuperación

Saber recoñecelas nun esquema de disco é un obxectivo explícito da sesión sobre particionado e lóxica de arranque en Windows.

2.6. Particións habituais en Linux (Ubuntu)

Nunha instalación típica de Linux (por exemplo, Ubuntu 24.04) poden aparecer:

Partición Punto de montaxe Función
raíz / Sistema principal
home /home Datos dos usuarios
swap swap Espazo auxiliar de memoria
EFI /boot/efi Arranque UEFI

Para unha máquina virtual de Ubuntu, o material propone como exemplo:

Partición Tamaño orientativo Uso
EFI 512 MB Arranque
raíz 25–40 GB Sistema completo
swap 2–4 GB Memoria de intercambio

Nunha primeira práctica é válido deixar que o instalador cree as particións automaticamente, sempre que o alumnado entenda que está a facer o instalador.

Nota: En máquinas virtuais adóitase omitir /home como partición separada, xa que non se espera que o alumnado garde datos persistentes entre sesións. Nun equipo físico de uso real, separar /home é unha boa práctica que facilita reinstalacións sen perder os datos do usuario.

3. Proceso xeral de instalación dun sistema operativo

O proceso xeral de instalación que recollen os apuntes segue normalmente unha serie de pasos comúns:

3.1. Pasos xerais
Paso Descrición
1 Preparar o equipo ou máquina virtual.
2 Descargar a ISO ou imaxe de instalación.
3 Crear ou montar o medio de instalación (USB, ISO, etc.).
4 Arrancar desde USB, DVD ou ISO.
5 Escoller idioma, teclado e rexión.
6 Aceptar licenza, se procede.
7 Escoller tipo de instalación (limpa, actualización, etc.).
8 Configurar particións.
9 Crear usuario e contrasinal.
10 Configurar rede, se procede.
11 Copiar ficheiros e instalar o sistema.
12 Reiniciar.
13 Actualizar o sistema.
14 Instalar controladores ou ferramentas adicionais.
15 Documentar o proceso.

Este esquema úsase despois para comparar o fluxo de instalación de Windows 11 e Ubuntu 24.04.

Paso Windows 11 Ubuntu 24.04 LTS

4. Virtualización con VirtualBox: recursos e modos de rede
4.1. Recursos recomendados para Windows 11 en VM

Segundo o material de referencia, para unha VM de Windows 11 en VirtualBox recoméndase:

  • 4 GB de RAM como mínimo, 6–8 GB para aula.
  • 2 núcleos de CPU.
  • 80 GB de disco.
  • Firmware UEFI.
  • TPM 2.0 virtual.
  • Secure Boot activado se se quere cumprir requisitos.

Non se recomenda configurar Windows 11 con 2 GB de RAM e 40 GB de disco, porque queda por debaixo dos requisitos mínimos dunha instalación estándar compatible.

4.2. Recursos recomendados para Ubuntu 24.04 en VM

Para Ubuntu 24.04 en VirtualBox, o cadro indica:

  • 2 GB de RAM en virtualización como mínimo; 4 GB recomendados para aula.
  • 2 núcleos de CPU.
  • Disco de 25 GB como mínimo; 30–40 GB recomendados.
  • 128–256 MB de VRAM; recoméndase activar aceleración 3D para escritorio GNOME.

Estas cifras serven tamén para exercicios de “configurar a VM e xustificar decisións”.

4.3. Modos de rede en VirtualBox

Os modos de rede máis habituais en VirtualBox son:

Modo de rede Descrición Uso recomendado
NAT A VM sae a Internet usando a conexión do host Uso básico e seguro
Adaptador ponte A VM aparece como outro equipo na rede real Prácticas de rede e servidores
Rede interna Só se comunican máquinas virtuais entre si Laboratorios illados
Só anfitrión Comunicación entre host e VM Probas locais sen saír á rede real
Non conectado A tarxeta existe pero sen conexión Probas sen rede

Para instalar Ubuntu e actualizar paquetes chega con NAT; para practicar servidores, Samba ou dominio, adoita ser mellor usar adaptador ponte ou rede interna, segundo o deseño da práctica.

4.4. Guest Additions e snapshots
  • As Guest Additions melloran a integración entre a VM e o equipo anfitrión (resolución, portapapeis, carpetas compartidas, etc.).
  • Un snapshot ou instantánea é unha “foto” do estado da VM nun momento concreto e permite volver atrás se algo sae mal.

Un snapshot non substitúe unha copia de seguridade completa, pero é moi útil antes de modificar particións ou facer prácticas complexas.

5. Dual boot: idea xeral e riscos
5.1. Que é o dual boot

O dual boot consiste en instalar dous sistemas operativos no mesmo equipo e escoller cal iniciar ao acender.

Exemplo habitual: Windows 11 + Ubuntu 24.04 LTS.

5.2. Recomendacións antes dun dual boot

O material sinala estes pasos previos:

  1. Facer copia de seguridade.
  2. Comprobar se o equipo usa UEFI ou BIOS.
  3. Revisar se o disco usa GPT ou MBR.
  4. Liberar espazo no disco.
  5. Instalar normalmente primeiro Windows.
  6. Instalar despois Linux.
  7. Revisar o xestor de arranque GRUB.
  8. Non borrar particións descoñecidas.
5.3. Riscos frecuentes
Risco Explicación
Borrar datos Un erro no particionado pode eliminar ficheiros.
Eliminar o arranque Pode danarse GRUB ou Windows Boot Manager.
Problemas con Secure Boot Certos controladores ou sistemas poden non arrancar.
Confusión de particións O alumnado pode seleccionar o disco ou partición erróneos.

Por estes riscos, no contexto de FP adoita priorizarse practicar dual boot en máquinas virtuais ou en equipos específicos de laboratorio, non en portátiles persoais sen supervisión.
