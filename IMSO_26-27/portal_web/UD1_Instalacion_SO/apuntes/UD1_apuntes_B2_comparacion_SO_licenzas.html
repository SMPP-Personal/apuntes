1. Tipos de sistemas operativos.

En el bloque 1 se introdujo la idea general de sistema operativo; aquí se profundiza en su clasificación y se preparan las comparaciones que luego aparecerán en las prácticas y en las pruebas.

1.1. Según la finalidad principal
Tipo Finalidad Ejemplos
Escritorio Uso personal, ofimática, navegación, programas comunes Windows 11, Ubuntu Desktop, macOS
Servidor Ofrecer servicios a otros equipos Windows Server, Ubuntu Server, Debian
Móvil Teléfonos y tabletas Android, iOS
Tiempo real Respuesta muy rápida y previsible QNX, VxWorks, FreeRTOS
Embebido Integrado en un dispositivo concreto Routers, smart TV, sistemas industriales
Distribuido Varios equipos que cooperan como un sistema Clústeres y sistemas de computación distribuida

Esta clasificación ayuda a justificar por qué no tiene sentido instalar un sistema móvil en un servidor, o usar un sistema de tiempo real para tareas de ofimática general.

1.2. Según tipo de licencia
  • Software libre: permite estudiar, modificar y redistribuir el código fuente según las condiciones de la licencia (por ejemplo, GPL, MIT o BSD).
  • Software propietario: el código fuente no suele estar disponible y la copia, modificación o redistribución se limitan según el contrato de licencia.
  • OEM: licencia asociada a un equipo concreto (por ejemplo, un Windows preinstalado en un portátil).
  • Licencia por suscripción: se paga un importe periódico por el derecho de uso (por ejemplo, servicios tipo Microsoft 365).
Modelos de distribución y negocio asociados

No son tipos de licencia en sentido estricto, pero aparecen con frecuencia en el mercado:

  • Freemium: ofrece una versión básica gratuita y funciones avanzadas de pago. La licencia subyacente puede ser propietaria o libre.
  • Freeware: software gratuito pero generalmente propietario; no permite modificar ni redistribuir el código fuente.

Idea clave: software libre no significa automáticamente gratuito, y un programa gratuito puede seguir siendo propietario.

1.3. Software libre frente a software propietario
Característica Software libre Software propietario
Código fuente Accesible según licencia Normalmente cerrado
Modificación Permitida con condiciones Normalmente no permitida
Redistribución Permitida con condiciones Restringida
Coste Puede ser gratis o de pago Puede ser de pago, OEM o subscripción
Ejemplos Ubuntu, Debian, Firefox, GIMP Windows, macOS, Microsoft Office

Esta tabla te servirá también como plantilla para el alumnado en actividades de comparación.

2. Comparación básica: Windows 11 y Ubuntu 24.04

En UD1 se trabajará especialmente con Windows 11 y Ubuntu 24.04 LTS como referentes de sistema propietario y libre para escritorio y laboratorio.

2.1. Ficha resumen de Windows 11
Aspecto Descripción
Familia Windows NT
Núcleo Híbrido
Licencia Propietaria
Interfaz Gráfica (GUI), con consola CMD y PowerShell (CLI)
Uso habitual Escritorio, empresa, juegos, software comercial

Requisitos mínimos generales (instalación estándar compatibles):

  • CPU de 1 GHz o más, con 2 núcleos, 64 bits.
  • 4 GB de RAM.
  • 64 GB de espacio en disco.
  • Firmware UEFI con Secure Boot.
  • TPM 2.0.
  • Gráfica compatible con DirectX 12 o posterior con driver WDDM 2.0, y pantalla de más de 9 pulgadas en diagonal a resolución mínima de 720p.
2.2. Ficha resumen de Ubuntu 24.04 LTS
Aspecto Descripción
Familia GNU/Linux (basada en Debian)
Núcleo Linux (monolítico modular)
Licencia Mayoritariamente software libre
Interfaz Gráfica (GNOME) y terminal (CLI)
Uso habitual Escritorio, servidor, desarrollo, docencia

Requisitos orientativos (Ubuntu Desktop 24.04):

  • CPU de doble núcleo a 2 GHz (64 bits).
  • En equipo físico, 4 GB de RAM; en VM, 2 GB mínimos, 4 GB recomendables.
  • 25 GB de disco como mínimo; en prácticas, suele ser mejor 30‑40 GB.
  • Aceleración 3D recomendable y pantalla de 1024×768 o superior.
2.3. Tabla comparativa síntesis
Sistema Licencia Uso principal Requisitos clave
Windows 11 Propietaria Escritorio, empresa, juegos UEFI, Secure Boot, TPM 2.0, 4 GB RAM, 64 GB disco
Ubuntu 24.04 LTS Mayoritariamente libre Escritorio, servidor, desarrollo 2–4 GB RAM, 25–40 GB disco, sin exigir TPM

Esta comparativa es una base directa para ejercicios escritos del tipo “Tabla Windows/Ubuntu” y para justificar por qué se recomienda empezar prácticas intensivas en VM con Ubuntu.

3. Requisitos y consideraciones previas a la instalación

Antes de instalar un sistema operativo, BC1 indica que hay que revisar requisitos, versiones, licencias y consideraciones previas, tanto para sistemas libres como propietarios.

3.1. Comprobar los requisitos de hardware

Cada sistema operativo define unos requisitos mínimos, que suelen incluir:

  • CPU compatible y número de núcleos.
  • Cantidad de memoria RAM necesaria.
  • Espacio mínimo en disco.
  • Soporte para UEFI, Secure Boot y TPM en el caso de Windows 11.
  • Tarjeta gráfica y resolución de pantalla compatibles.

Windows 11 resulta más exigente que muchas distribuciones Linux debido, sobre todo, a los requisitos de TPM 2.0, Secure Boot y UEFI.

3.2. Elegir el tipo de instalación

Los apuntes resumen los tipos de instalación habituales así:

Tipo de instalación Descripción
Instalación limpia Se borra o sustituye el sistema anterior
Actualización Se pasa de una versión a otra manteniendo datos y programas
Dual boot Dos sistemas operativos en el mismo equipo, eligiendo en el arranque
Instalación en máquina virtual El sistema se instala dentro de un equipo virtual
Instalación en la nube Se crea una instancia en Azure, AWS, Google Cloud u otro proveedor
Instalación desatendida El proceso se automatiza mediante ficheros de respuesta

En la UD1 grande se trabajará principalmente con instalación limpia en VM, usando dual boot solo a nivel conceptual y dejando nube y desatendida para la UD00 de ampliación.

3.3. Copia de seguridad previa

Antes de instalar un sistema operativo en un equipo real, es obligatorio realizar una copia de seguridad de los datos importantes:

  • documentos personales,
  • imágenes y proyectos,
  • configuraciones y claves,
  • datos específicos de aplicaciones.

El motivo es que un error en el particionado puede provocar la pérdida total de los datos del disco.

3.4. Descargar la imagen ISO desde fuente oficial

Una imagen ISO es un archivo que contiene el instalador de un sistema operativo y que se puede usar para crear un medio de instalación (USB, DVD o VM).

Siempre debe descargarse desde fuentes oficiales o de confianza, utilizando nombres de archivo que correspondan a versiones reales y actualizadas, como por ejemplo:

Win11_24H2_Spanish_x64.iso

(ejemplo de nomenclatura para Windows 11).

ubuntu-24.04.2-desktop-amd64.iso

.

3.5. Verificar la integridad de la ISO (hash)

En entornos profesionales es recomendable verificar que la imagen ISO no está corrupta ni ha sido modificada maliciosamente, mediante un hash o suma de comprobación.

  • Algoritmos habituales: SHA256, SHA512; MD5 ya no se recomienda para seguridad.
  • La verificación consiste en comparar el hash publicado por el proveedor con el hash calculado sobre el archivo descargado.
3.6. Crear el medio de instalación

El sistema operativo puede instalarse desde:

  • un USB arrancable,
  • un DVD,
  • una ISO montada en una máquina virtual,
  • un arranque por red,
  • una imagen en la nube.

Herramientas habituales para crear USB de instalación:

Herramienta Uso típico
Rufus Crear USB arrancables en Windows
Ventoy Crear USB con varias ISOs en el mismo dispositivo
Balena Etcher Grabar imágenes en USB de forma sencilla
Media Creation Tool Crear un medio oficial de Windows
Creador de discos de Ubuntu Crear USB de Ubuntu desde Linux

En el contexto de la UD1 en FP, muchas veces basta con montar directamente la ISO en VirtualBox y trabajar en máquinas virtuales.

4. Instalación en máquinas virtuales y en nube.
4.1. Instalación en máquina virtual

La virtualización permite instalar un sistema operativo dentro de otro sin modificar el equipo físico y es especialmente útil en docencia.

Ventajas principales:

  • probar sistemas sin riesgo,
  • repetir instalaciones,
  • crear snapshots,
  • restaurar estados anteriores,
  • practicar con redes virtuales,
  • trabajar con varios sistemas a la vez.
4.2. Recursos recomendados en VirtualBox

Para una VM de Windows 11 en VirtualBox se recomienda configurar:

  • 6 GB de RAM,
  • 2 núcleos de CPU,
  • 80 GB de disco,
  • firmware UEFI,
  • TPM 2.0 virtual,
  • red NAT.

Para Ubuntu 24.04 en VirtualBox, los recursos recomendados son:

  • 2 GB de RAM como mínimo en VM (4 GB en equipo físico), 4 GB para trabajar con comodidad,
  • 2 núcleos de CPU,
  • 30–40 GB de disco,
  • 256 MB de VRAM y, si es posible, aceleración 3D.

La recomendación explícita es evitar configuraciones muy por debajo de los requisitos, como 2 GB de RAM y 40 GB de disco para Windows 11, porque no cumplen los mínimos de una instalación estándar compatible.

4.3. Instalación en la nube y automatización
  • es posible desplegar sistemas operativos en la nube mediante instancias en plataformas como Azure, AWS o Google Cloud (C1.10),
  • la instalación puede automatizarse usando ficheros de respuesta, como unattend.xml en Windows o configuraciones de autoinstalación en Linux (C1.11, C1.12).

5. Checklist previa a la instalación
  1. Confirmar qué sistema se va a instalar y con qué objetivo.
  2. Revisar requisitos mínimos y recomendados del sistema operativo.
  3. Comprobar firmware (BIOS/UEFI), Secure Boot, TPM y soporte de virtualización si procede.
  4. Elegir el tipo de instalación adecuado (limpia, actualización, dual boot, VM, nube).
  5. Asegurar copia de seguridad si se va a tocar un equipo real.
  6. Descargar la ISO desde fuente oficial.
  7. Verificar integridad mediante hash si es necesario.
  8. Preparar el medio de instalación o montar la ISO en una VM.
  9. Definir un esquema de particionado básico (se trabajará más en el bloque de particionado).
  10. Tener lista una plantilla de documentación para registrar la instalación.
