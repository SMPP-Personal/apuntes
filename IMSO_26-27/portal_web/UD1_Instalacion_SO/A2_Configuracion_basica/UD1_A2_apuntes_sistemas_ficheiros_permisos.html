4. Sistemas de ficheiros, particións e permisos básicos

Conceptos fundamentais para entender como o sistema operativo organiza o disco e controla o acceso á información.

4.1 Sistema de ficheiros: a «forma de organizar» o disco

Un sistema de ficheiros é o método que usa o sistema operativo para organizar os datos nun disco ou partición: cartafoles, ficheiros, permisos, atributos, etc. Cada SO ten os seus sistemas preferidos, aínda que normalmente pode ler ou escribir en varios.

Sistema de ficheiros Uso habitual
NTFSWindows (partición principal)
FAT32USB, partición EFI, compatibilidade
exFATUSB e discos externos de gran capacidade
ext4Distribucións Linux modernas

APFSmacOS
Nunha VM de laboratorio, o máis habitual é usar NTFS en Windows e ext4 en Ubuntu.

4.2 Particións e volumes: dividir o disco

O particionado consiste en dividir un disco en partes lóxicas chamadas particións. Cada partición pode ter o seu propio sistema de ficheiros e unha función distinta: sistema, datos, arranque, recuperación, etc.

Esquemas de particionado

  • MBR — modelo antigo; con limitacións de tamaño máximo de partición (2 TB) e número de particións primarias (máx. 4).
  • GPT — modelo moderno, recomendado en equipos actuais con UEFI; elimina as limitacións de MBR.
Nos equipos modernos recoméndase UEFI + GPT, tanto para Windows como para Linux.

Nunha primeira práctica en VM é aceptable deixar que o instalador cree as particións automaticamente, pero é importante comprender que está a facer.

4.3 Permisos de ficheiros: idea xeral

Os permisos de ficheiros controlan quen pode ler, modificar ou executar un ficheiro ou cartafol. Serven para protexer información e impedir cambios non autorizados no sistema.

Conceptos comúns

  • Propietario — usuario dono do ficheiro.
  • Grupo — conxunto de usuarios con permisos comúns.
  • Outros — o resto de usuarios do sistema.
  • Accións — lectura, escritura, execución.
A implementación é distinta en Windows e Linux, pero a idea de fondo é a mesma: non todo o mundo debe poder facer de todo en calquera ficheiro.

4.4 Permisos en Linux (visión básica para esta UD)

En Linux, cada ficheiro ten tres bloques de permisos: un para o usuario propietario, outro para o grupo e outro para o resto de usuarios (outros). En cada bloque pódense establecer tres tipos de permiso:

Tipos de permiso en Linux
  • r read – lectura: ver o contido do ficheiro ou listar un cartafol.
  • w write – escritura: modificar ou eliminar o ficheiro.
  • x execute – execución nun ficheiro; acceso (cd) nun cartafol.

Nesta UD abórdanse os permisos de forma introdutoria, aplicados a cartafoles persoais e ficheiros de práctica. A parte avanzada (permisos numéricos, chmod, chown, bits especiais) verase en UDs posteriores.

4.5 Permisos NTFS en Windows (visión básica)

En Windows, sobre sistemas de ficheiros NTFS, os permisos xestiónanse dende as propiedades dun ficheiro ou cartafol, na pestana Seguridade. Pódense asignar a usuarios individuais ou a grupos.

Permisos básicos de NTFS
Permiso Permite…
LerVer o contido do ficheiro ou cartafol
Ler e executarVer e abrir programas
EscribirCrear e modificar ficheiros
ModificarLer + escribir + borrar
Control totalTodos os permisos anteriores + cambiar permisos

Ideas clave para esta UD

  • Non é o mesmo entrar como administrador que como usuario estándar.
  • Pódense limitar os permisos dunha carpeta (por exemplo, compartida nunha aula).
  • As decisións de permisos afectan directamente á seguridade dos datos.
A parte avanzada de permisos en Windows (herdanzas, listas de control de acceso ACL, permisos de recurso compartido) traballarase con máis detalle na UD2.

4.6 Relación entre particións, sistemas de ficheiros e permisos

Estes tres conceptos forman unha cadea lóxica: primeiro créase a estrutura no disco, logo asígnaselle un formato e finalmente contrólase quen pode acceder ao seu contido.

1. Partición / Volume
Dividir o disco
2. Sistema de ficheiros
Formatar (NTFS, ext4…)
3. Permisos
Controlar o acceso

Exemplo práctico: Ubuntu en VM con VirtualBox

  1. O disco virtual créase en VirtualBox (ficheiro .vdi de ~40 GB).
  2. Durante a instalación de Ubuntu, créase partición EFI (FAT32) e partición raíz / con ext4.
  3. No sistema instalado, os cartafoles en /home/usuario teñen permisos que permiten ao dono traballar libremente, pero impiden que outro usuario local modifique os seus ficheiros.
Comprender esta cadea será clave máis adiante, cando entremos en RAID, cotas de disco e copias de seguridade (Actividade 3).
