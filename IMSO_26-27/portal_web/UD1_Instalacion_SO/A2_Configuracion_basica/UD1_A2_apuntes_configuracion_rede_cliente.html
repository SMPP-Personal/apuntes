5. Configuración básica de rede en sistemas operativos cliente

Conceptos fundamentais de TCP/IP, DHCP, permisos de rede e diagnóstico básico de conectividade en Windows 11 e Ubuntu 24.04.

5.1 Conceptos fundamentais de TCP/IP

A maioría das redes actuais usan o protocolo TCP/IP. Cada equipo conéctase á rede mediante unha ou varias interfaces (tarxeta de rede física ou virtual) ás que se lles asigna unha dirección IP.

Elementos básicos

Enderezo IP — rangos privados habituais
Rango Notación CIDR Uso típico
10.0.0.0 – 10.255.255.255/8Redes grandes, corporativas
172.16.0.0 – 172.31.255.255/12Redes medianas
192.168.0.0 – 192.168.255.255/16Redes domésticas e de aula
  • Máscara de rede — indica que parte do enderezo corresponde á rede e cal ao host. Exemplo: 255.255.255.0 é equivalente a /24.
  • Porta de enlace (gateway) — equipo que permite saír cara a outras redes, normalmente o router.
  • DNS — servidores que traducen nomes (www.exemplo.com) a enderezos IP.
Nunha aula con VirtualBox, o máis habitual é usar DHCP (IP automática) cando a VM está en modo NAT ou adaptador ponte.

5.2 Configuración automática vs manual (DHCP vs IP fixa)

Automática (DHCP)
  • O equipo solicita unha IP a un servidor DHCP.
  • Máis cómodo en redes grandes e aulas.
  • Cambiar de rede require pouca ou ningunha intervención.
Manual (IP fixa)
  • IP, máscara, gateway e DNS especifícanse a man.
  • Útil para servidores, impresoras de rede ou prácticas concretas.

O que debes saber nesta UD

  • Ver se unha interface ten IP.
  • Diferenciar cando está en DHCP e cando en IP fixa.
  • Corrixir unha configuración moi básica (IP ou DNS mal posta) durante unha incidencia.

5.3 Configuración básica de rede en Windows 11 (visión básica)

Ruta de acceso en Windows 11
  1. Configuración → Rede e Internet
  2. Seleccionar a conexión activa (Ethernet ou Wi‑Fi) e premer en Ver propiedades.
  3. Para ver ou modificar IPv4: ir ao apartado Asignación de IP e activar manual se se precisa.
En Windows 11 o acceso ás propiedades do adaptador cambiou respecto a versións anteriores. Xa non está na icona de rede do panel de control clásico de forma directa; o camiño principal é Configuración → Rede e Internet.

Puntos clave

  • Saber localizar: IP actual, porta de enlace e servidores DNS.
  • Sen IP válida, o equipo non se comunica coa rede.
  • Sen DNS correcto, pode haber IP pero non se resolve nomes — parece «non hai Internet».
Os comandos de consola (ipconfig, ping, etc.) traballaranse de forma máis intensa en UD2 e redes máis avanzadas.

5.4 Configuración básica de rede en Ubuntu 24.04

En Ubuntu 24.04 pódese configurar a rede a través da interface gráfica (icono de rede → Configuración de rede) ou mediante terminal.

Comandos de consulta en Ubuntu
ip a consulta 
$ ip a
  • Mostra todas as interfaces e as IPs asignadas.
  • Permite ver se a interface da VM ten unha IP de tipo 10.x, 172.16–31.x ou 192.168.x.x.
O nome da interface varía segundo a VM e o controlador de rede (exemplos: enp0s3, enp0s8, eth0…). Comproba co comando ip a cal é o nome real na túa VM antes de traballar con ela.
ping 8.8.8.8 -c 4 conectividade IP 
$ ping 8.8.8.8 -c 4

Proba conectividade IP básica con Internet. Se isto falla, o problema é de rede (IP, gateway ou modo VirtualBox).

ping www.google.com -c 4 resolución DNS 
$ ping www.google.com -c 4

Proba resolución DNS. Se a IP funciona pero isto falla, o problema adoita ser de DNS.

O que chega con saber nesta UD

  • Saber ler a información de ip a.
  • Entender a diferenza entre non ter IP, ter IP pero non gateway, e ter problema de DNS.
A configuración completa de IP fixa mediante ficheiros (/etc/netplan, etc.) verase con máis detalle nas UDs seguintes.

5.5 Modos de rede en VirtualBox

Coñecer os modos de rede de VirtualBox é imprescindible para entender por que unha VM ten ou non ten conectividade.

Modo de rede Descrición breve Uso recomendado
NAT A VM sae a Internet usando a conexión do host Uso básico e seguro (instalación, actualizacións)
Adaptador ponte A VM aparece como outro equipo da rede real Prácticas de servidores e dominios
Rede interna Só se comunican entre si as VMs nesta rede Laboratorios illados
Só anfitrión Comunicación entre host e VM, sen saír á rede real Probas locais sen saír á rede
Non conectado A tarxeta existe pero non ten conexión Probas sen rede / incidencias simuladas
Lembra:
  • Para instalar Ubuntu e actualizar paquetes, NAT chega de sobra.
  • Se unha VM non ten Internet, hai que revisar o modo de rede en VirtualBox e despois a IP dentro do sistema.

5.6 Relación co diagnóstico de incidencias de rede

Esta sección conecta directamente coa parte de incidencias xa traballada. As causas máis habituais de que unha VM non teña Internet son:

  • Modo de rede «Non conectado» en VirtualBox.
  • IP mal configurada ou sen IP asignada.
  • DNS erróneo ou non configurado.

Procedemento de diagnóstico — orde recomendada

1
Revisar o modo de rede en VirtualBox
Se está en «Non conectado», cámbialo antes de continuar. Non ten sentido revisar a IP se o cable virtual está desconectado.
2
Ver IP e estado da interface
ip a en Ubuntu · propiedades do adaptador en Windows.
3
Probar ping a unha IP externa 
ping 8.8.8.8 -c 4
4
Probar ping a un nome 
ping www.google.com -c 4
Se o paso 3 funciona pero o paso 4 falla, o problema é de DNS, non de conectividade IP.
Na UD2 iremos moito máis lonxe en protocolos, DNS, ficheiros de configuración e resolución de problemas de conectividade.
