Perfís móbiles e cartafoles persoais

Configuración de recursos compartidos ocultos, perfil móbil e unidade persoal H: para alumnado do dominio.

Nota de laboratorio: nesta práctica usamos dcbuho tamén como servidor de ficheiros para simplificar. Nun contorno profesional é recomendable separar o controlador de dominio e o servidor de ficheiros.

1. Título

Actividade práctica guiada en dominio Windows.

Actividade 4 – Configuración de perfís móbiles e cartafoles persoais para alumnado en Windows Server 2022

2. Obxectivos da actividade

Ao rematar a actividade, deberías ser capaz de:

Crear un recurso compartido para perfís móbiles.
Crear un recurso compartido para cartafoles persoais.
Configurar o Profile path dun usuario de dominio.
Configurar un home folder como unidade de rede H:.
Comprobar o funcionamento desde un cliente unido ao dominio.
Diferenciar perfil móbil, cartafol persoal e redirección de cartafoles.

3. Punto de partida

Estado mínimo antes de iniciar a práctica.

  • O dominio cifpsauce.local está instalado e funcionando.
  • O servidor dcbuho.cifpsauce.local funciona como controlador de dominio e servidor DNS.
  • Existe OU=Alumnado → OU=Curso1 cos usuarios alu.curso1.001 e alu.curso1.002.
  • Existe o grupo global de seguridade GG_Alumnado_Curso1, cos usuarios anteriores como membros.
  • Existe un equipo cliente Windows 10/11 unido ao dominio, por exemplo PC-AULA01.
  • Traballarás en dcbuho cunha conta administrativa do dominio, por exemplo CIFPSAUCE\Administrador.

4. Antes de empezar: conceptos que non debes mesturar

ElementoQue garda ou para que serveOnde se configura
Perfil móbilParte do contorno do usuario: configuracións, escritorio, datos do perfil, etc.Pestana Profile, campo Profile path
Cartafol persoalEspazo persoal de traballo, normalmente montado como unidade de rede H:.Pestana Profile, sección Home folder
Redirección de cartafolesFai que carpetas como Documentos ou Escritorio apunten a unha ruta de servidor.Normalmente mediante GPO
Aviso importante: un perfil móbil non é o mesmo que un cartafol persoal. O perfil móbil afecta ao contorno de sesión; o cartafol persoal é unha localización de almacenamento.
Rendemento: os perfís móbiles poden facer máis lento o inicio e peche de sesión se conteñen moitos ficheiros.

5. Parte A – Preparar os recursos compartidos no servidor dcbuho

A.1

Crear a estrutura de carpetas no servidor

Directorios base para perfís e cartafoles persoais.

No servidor dcbuho, crea a seguinte estrutura:

C:\DatosDominio\Perfis
C:\DatosDominio\Usuarios
  • Cartafol C:\DatosDominio\Perfis creado.
  • Cartafol C:\DatosDominio\Usuarios creado.
Se no teu laboratorio xa se está usando outra unidade ou ruta estándar, adapta a ruta segundo indique o profesorado.
A.2

Crear recursos compartidos ocultos

Usaremos $ para que non aparezan facilmente nos listados de rede.

Cartafol localRecurso compartidoRuta UNCUso
C:\DatosDominio\Perfisperfis$\\dcbuho\perfis$Perfís móbiles
C:\DatosDominio\Usuariosusuarios$\\dcbuho\usuarios$Cartafoles persoais

Crear o recurso perfis$

  1. Clic dereito sobre C:\DatosDominio\Perfis.
  2. Entra en Propiedades.
  3. Vai á pestana Compartir.
  4. Preme en Uso compartido avanzado / Advanced Sharing.
  5. Marca Compartir esta carpeta.
  6. Nome do recurso compartido: perfis$.
  7. Permisos de compartición para laboratorio: GG_Alumnado_Curso1 con Control total.
  8. Acepta os cambios.
  • Recurso compartido \\dcbuho\perfis$ creado.

Crear o recurso usuarios$

  1. Clic dereito sobre C:\DatosDominio\Usuarios.
  2. Propiedades → Compartir → Uso compartido avanzado.
  3. Nome do recurso compartido: usuarios$.
  4. Permisos de compartición para laboratorio: GG_Alumnado_Curso1 con Control total.
  5. Acepta os cambios.
  • Recurso compartido \\dcbuho\usuarios$ creado.
Non confundas permisos de compartición con permisos NTFS. No acceso por rede actúan os dous.
Evita usar Everyone como solución por defecto. Usa grupos do dominio e afina con NTFS.
A.3

Axustar permisos NTFS básicos

O usuario debe poder crear e usar o seu propio espazo.

A configuración exacta de NTFS pode variar segundo o enfoque do profesorado. Como criterio didáctico:

  • Evita dar permisos directamente usuario por usuario se podes facelo por grupos.
  • Evita abrir os permisos a Everyone.
  • Comproba sempre o acceso cun usuario real de dominio.
CartafolPermiso orientativo
C:\DatosDominio\PerfisGG_Alumnado_Curso1 con permisos suficientes para crear/escribir o seu perfil.
C:\DatosDominio\UsuariosGG_Alumnado_Curso1 con permisos suficientes para crear/escribir o seu cartafol persoal.
AmbosAdministradores e/ou administradores do dominio con control administrativo.
  • Permisos NTFS revisados en C:\DatosDominio\Perfis.
  • Permisos NTFS revisados en C:\DatosDominio\Usuarios.

6. Parte B – Configurar o perfil móbil dun usuario

B

Perfil móbil de alu.curso1.001

Configuración do campo Profile path.

B.1. Abrir as propiedades do usuario

  1. En dcbuho, abre Server Manager.
  2. Vai a Tools → Active Directory Users and Computers.
  3. Navega a cifpsauce.local → Alumnado → Curso1.
  4. Clic dereito sobre alu.curso1.001.
  5. Preme en Properties.
  • Propiedades do usuario alu.curso1.001 abertas.

B.2. Definir a ruta do perfil móbil

  1. Na xanela de propiedades do usuario, vai á pestana Profile.
  2. No campo Profile path, escribe:
\\dcbuho\perfis$\alu.curso1.001
  1. Acepta os cambios.
  • Ruta de perfil móbil configurada como \\dcbuho\perfis$\alu.curso1.001.
O cartafol físico C:\DatosDominio\Perfis\alu.curso1.001 pode crearse automaticamente no primeiro inicio de sesión se a ruta, DNS, conectividade e permisos son correctos.

7. Parte C – Configurar o cartafol persoal (home folder)

C

Unidade persoal H: para alu.curso1.001

Configuración da sección Home folder.

  1. Na mesma xanela de propiedades do usuario, pestana Profile, localiza a sección Home folder.
  2. Selecciona Connect.
  3. Escolle a letra de unidade H:.
  4. No campo de ruta escribe:
\\dcbuho\usuarios$\alu.curso1.001
  1. Acepta os cambios.
  • Cartafol persoal configurado como unidade H:.
  • Ruta configurada como \\dcbuho\usuarios$\alu.curso1.001.
Se o cartafol físico non aparece tras o primeiro inicio de sesión, revisa ruta UNC, DNS, permisos de compartición, permisos NTFS e eventos do cliente.

8. Parte D – Comprobación previa desde o equipo cliente

Comprobar DNS e conectividade antes do inicio de sesión

Se isto falla, non diagnostiques aínda perfís nin permisos.

No equipo cliente unido ao dominio, abre CMD ou PowerShell e executa:

ping dcbuho.cifpsauce.local
nslookup dcbuho.cifpsauce.local
net view \\dcbuho
  • ping dcbuho.cifpsauce.local responde correctamente.
  • nslookup dcbuho.cifpsauce.local resolve a IP correcta do servidor.
  • net view \\dcbuho non falla por resolución de nome.
Se estas comprobacións fallan, revisa DNS, conectividade e unión do cliente ao dominio antes de continuar.

9. Parte E – Proba desde un equipo cliente

E.1

Primeiro inicio de sesión con perfil móbil

Inicio de sesión como usuario de dominio.

  1. No equipo cliente, por exemplo PC-AULA01, chega á pantalla de inicio de sesión.
  2. Inicia sesión como usuario de dominio:
CIFPSAUCE\alu.curso1.001

Ou usando o UPN:

alu.curso1.001@cifpsauce.local
  1. Agarda a que cargue a sesión.
  • Inicio de sesión correcto de alu.curso1.001 no cliente.
O primeiro inicio de sesión pode tardar máis porque se crea e sincroniza a estrutura inicial do perfil.
E.2

Comprobar a unidade H:

Validación do cartafol persoal no cliente.

No cliente, abre o Explorador de ficheiros e comproba:

  • aparece unha unidade de rede H:;
  • apunta a \\dcbuho\usuarios$\alu.curso1.001;
  • podes crear un ficheiro de proba, por exemplo proba.txt.
  • Unidade H: visible no equipo cliente.
  • Unidade H: apunta á ruta correcta.
  • Ficheiro proba.txt creado en H:.

Tamén podes comprobar desde CMD:

net use
  • net use mostra a unidade H: conectada.
E.3

Pechar sesión para sincronizar o perfil

A sincronización debe comprobarse tras peche de sesión correcto.

  1. Crea unha modificación visible, por exemplo un ficheiro no escritorio chamado perfil-proba.txt.
  2. Pecha sesión correctamente.
  3. Non apagues a VM de golpe.
  • Ficheiro de proba creado no escritorio.
  • Sesión pechada correctamente para permitir sincronización do perfil.
Os cambios do perfil móbil sincronízanse especialmente no peche de sesión. Se apagas ou reinicias de forma brusca, a comprobación pode non ser fiable.

10. Parte F – Verificación no servidor

Comprobar cartafoles creados e contido

Validación desde dcbuho.

  1. Abre C:\DatosDominio\Perfis e comproba se existe C:\DatosDominio\Perfis\alu.curso1.001.
  2. Abre C:\DatosDominio\Usuarios e comproba se existe C:\DatosDominio\Usuarios\alu.curso1.001.
  3. Comproba que dentro do cartafol persoal aparece o ficheiro proba.txt creado desde o cliente.
  • Cartafol de perfil C:\DatosDominio\Perfis\alu.curso1.001 existe.
  • Cartafol persoal C:\DatosDominio\Usuarios\alu.curso1.001 existe.
  • O ficheiro proba.txt aparece no cartafol persoal.

11. Parte G – Segunda proba noutro equipo cliente (opcional)

Comprobar comportamento noutro equipo unido ao dominio.

  1. Pecha sesión no primeiro equipo.
  2. Inicia sesión como alu.curso1.001 no segundo equipo.
  3. Comproba se aparece a unidade H:, se o ficheiro proba.txt segue dispoñible e se parte do contorno do usuario se conserva.
Inicio de sesión correcto no segundo equipo.
Unidade H: visible no segundo equipo.
Contido do cartafol persoal dispoñible.
Comportamento do perfil móbil observado.

12. Capturas obrigatorias

  1. Carpetas C:\DatosDominio\Perfis e C:\DatosDominio\Usuarios creadas no servidor.
  2. Propiedades de compartición de perfis$.
  3. Propiedades de compartición de usuarios$.
  4. Propiedades do usuario alu.curso1.001, pestana Profile, co campo Profile path visible.
  5. Propiedades do usuario alu.curso1.001, pestana Profile, coa sección Home folder e a unidade H: visible.
  6. Comprobación desde cliente con ping, nslookup ou net view.
  7. Inicio de sesión no cliente como alu.curso1.001.
  8. Unidade H: visible no cliente.
  9. Ficheiro proba.txt creado en H:.
  10. Cartafol de perfil e cartafol persoal visibles no servidor.
  11. Se fas a parte opcional, proba no segundo equipo cliente.

13. Resultado esperado

Estado final mínimo da actividade.

Recurso \\dcbuho\perfis$.
Recurso \\dcbuho\usuarios$.
Profile path configurado.
Cartafol persoal como unidade H:.
Inicio de sesión correcto no cliente.
Unidade H: visible e funcional.

Rutas finais esperadas

Perfil móbil:
\\dcbuho\perfis$\alu.curso1.001

Cartafol persoal H:
\\dcbuho\usuarios$\alu.curso1.001

Ademais, deberías comprender a diferenza básica entre perfil móbil e cartafol persoal.

14. Erros frecuentes

ErroConsecuencia
Escribir mal a ruta UNCO perfil ou a unidade H: non cargan.

Permisos de compartición demasiado restritivosO usuario non pode crear ou acceder ao seu cartafol.
Permisos NTFS demasiado abertosOutros usuarios poderían acceder a datos alleos.
Usar Everyone como solución por defectoMala práctica de seguridade.
Cliente que non resolve dcbuhoO problema será de DNS/conectividade, non do perfil.
Iniciar sesión cun usuario localNon se aplican propiedades do usuario de dominio.
Non pechar sesión correctamenteO perfil pode non sincronizarse ben.
Gardar ficheiros grandes no escritorioO perfil móbil pode tardar moito en cargar e gardar.

15. Pequenas cuestións de reflexión

Responde brevemente, en 2 ou 3 liñas por pregunta.

  1. Que diferenzas hai entre perfil móbil e cartafol persoal?
  2. Que datos ou configuracións poderían perderse se falla o perfil móbil?
  3. Que datos poderían perderse se falla o cartafol persoal?
  4. Que vantaxes ten para un centro de formación que o alumnado garde traballos nun cartafol persoal de servidor en vez de no disco local do equipo da aula?
  5. Que problemas poden aparecer se moitos usuarios teñen perfís móbiles moi grandes?
  6. Por que é importante comprobar DNS e conectividade antes de diagnosticar un erro de perfil?

16. Ideas clave da actividade

O perfil móbil permite que parte do contorno do usuario viaxe entre equipos.
O cartafol persoal é un espazo de traballo propio, normalmente como unidade de rede.
As rutas deben escribirse como UNC, por exemplo \\dcbuho\usuarios$\alu.curso1.001.
Os recursos ocultos con $ son útiles para carpetas de infraestrutura.
Os permisos de compartición e NTFS deben estar correctamente configurados.
Everyone non debería usarse como solución por defecto.
Antes de culpar ao perfil, comproba DNS, conectividade e inicio de sesión no dominio.
Os perfís móbiles grandes poden xerar inicios e peches de sesión lentos.
