UD2 · B1 Actividade práctica guiada GUI + PowerShell

Actividade 2 – Instalación e posta en marcha do dominio cifpsauce.local

Práctica guiada para configurar un servidor Windows Server 2022 como primeiro controlador de dominio dun novo bosque, usando interface gráfica e comandos PowerShell.

Windows Server 2022 AD DS + DNS PowerShell 3 capturas mínimas

2. Obxectivo da práctica

Ao rematar a práctica, o alumnado deberá ser capaz de:

Configurar correctamente a rede e o nome do servidor que será o primeiro controlador de dominio.
Instalar os roles AD DS e DNS Server tanto con interface gráfica como con PowerShell.
Crear un novo bosque cun dominio raíz chamado cifpsauce.local.
Promover o servidor dcbuho como primeiro controlador de dominio dese novo bosque.
Verificar o funcionamento básico do dominio, do servizo DNS e das ferramentas administrativas de Active Directory.
Comparar o proceso realizado mediante interface gráfica co proceso equivalente realizado mediante comandos PowerShell.

3. Escenario inicial para o alumnado

Partimos dunha máquina virtual con Windows Server 2022 recén instalada.

A máquina aínda non ten:

  • □o nome definitivo do servidor;
  • □unha IP fixa configurada;
  • □os roles AD DS nin DNS instalados;
  • □ningún dominio creado.

O obxectivo é convertela no primeiro controlador de dominio do dominio interno de laboratorio:

Dominiocifpsauce.local
Primeiro DCdcbuho.cifpsauce.local
IP do servidor192.168.50.10
Rede de laboratorio192.168.50.0/24
Nota sobre o nome do dominio: nesta práctica úsase .local como dominio interno de laboratorio. Nun contorno real, a elección do nome DNS do dominio debe planificarse con máis coidado.

4. Instrucións xerais para o alumnado

Nesta práctica farás o mesmo proceso de dúas formas diferentes:

Parte A – Instalación usando GUI
Usarás a interface gráfica de Windows Server: Server Manager, asistentes de instalación e consolas administrativas.
Parte B – Instalación usando PowerShell
Repetirás o proceso mediante comandos, partindo dunha máquina limpa ou dun snapshot previo.
Moi importante: non podes crear dúas veces o mesmo dominio na mesma instalación xa promovida a controlador de dominio. Para facer a Parte B debes usar unha VM limpa, unha copia anterior da VM ou un snapshot previo á promoción do servidor a controlador de dominio.

Ao final da práctica entregarás a ficha ou checklist cuberto, as capturas de pantalla indicadas e, opcionalmente, un ficheiro de texto cos comandos PowerShell empregados.

Parte B – Checklist PowerShell: instalación do dominio con comandos

Segunda execución do proceso empregando PowerShell. Debe facerse nunha máquina limpa ou nun snapshot anterior á promoción.

Recordatorio obrigatorio: esta parte debe facerse nunha VM limpa, nunha copia anterior ou nun snapshot anterior á promoción. Non executes Install-ADDSForest nun servidor que xa é controlador de dominio.

B.1. Comprobación inicial da rede

Abre PowerShell como administrador. Antes de modificar a configuración, revisa o estado actual da rede:

Get-NetAdapter
Get-NetIPConfiguration

Nome do adaptador: _______________________________________________

Importante: nos comandos seguintes debes substituír NOME_ADAPTADOR polo nome real do adaptador. Por exemplo, pode ser Ethernet, pero debes comprobar o teu caso concreto.

B.2. Configuración de IP estática con PowerShell

Configura a IP estática 192.168.50.10/24 e a gateway 192.168.50.1.

New-NetIPAddress -InterfaceAlias "NOME_ADAPTADOR" -IPAddress "192.168.50.10" -PrefixLength 24 -DefaultGateway "192.168.50.1"
  • □IP configurada correctamente.
  • □Gateway configurada correctamente.
O prefixo /24 equivale á máscara 255.255.255.0. Se o adaptador xa tiña unha configuración IP previa, pode ser necesario partir dunha VM limpa ou eliminar a configuración anterior.

B.3. Configuración do DNS do adaptador

Configura o servidor DNS do adaptador para que apunte á propia IP do servidor:

Set-DnsClientServerAddress -InterfaceAlias "NOME_ADAPTADOR" -ServerAddresses "192.168.50.10"
  • □DNS configurado correctamente.
Pode parecer estraño que o servidor apunte a si mesmo como DNS antes de rematar toda a instalación. O importante é que, ao finalizar a promoción a controlador de dominio, dcbuho use o seu propio servizo DNS para resolver cifpsauce.local.

B.4. Comprobación da configuración de rede

Comproba a configuración aplicada:

Get-NetIPAddress -InterfaceAlias "NOME_ADAPTADOR"
Get-DnsClientServerAddress -InterfaceAlias "NOME_ADAPTADOR"
ipconfig /all
  • □A IP é 192.168.50.10.
  • □A máscara corresponde a /24 ou 255.255.255.0.
  • □A gateway é 192.168.50.1 ou a indicada polo profesorado.
  • □O DNS preferido é 192.168.50.10.

B.5. Cambio de nome do equipo con PowerShell

Cambia o nome do equipo a dcbuho e forza o reinicio:

Rename-Computer -NewName "dcbuho" -Restart

Despois do reinicio, comproba o nome do equipo:

hostname
  • □O equipo reiniciou correctamente.
  • □O novo nome do equipo é dcbuho.

B.6. Instalación dos roles AD DS e DNS con PowerShell

Abre de novo PowerShell como administrador. Instala os roles AD DS e DNS, incluíndo as ferramentas de administración:

Install-WindowsFeature AD-Domain-Services, DNS -IncludeManagementTools
  • □Instalación de AD DS completada.
  • □Instalación de DNS completada.
  • □Ferramentas de administración instaladas.

Podes comprobar o resultado con:

Get-WindowsFeature AD-Domain-Services, DNS
Durante a promoción con Install-ADDSForest tamén se pode instalar ou configurar DNS. Nesta práctica instalámolo explicitamente antes para que o proceso sexa máis claro.

B.7. Creación do novo bosque e dominio con PowerShell

Importa o módulo de despregamento de AD DS:

Import-Module ADDSDeployment

Executa o seguinte comando para crear o novo bosque co dominio raíz cifpsauce.local:

Install-ADDSForest `
  -DomainName "cifpsauce.local" `
  -DomainNetbiosName "CIFPSAUCE" `
  -SafeModeAdministratorPassword (Read-Host "Introduce o contrasinal DSRM" -AsSecureString) `
  -InstallDNS `
  -Force

Durante a execución:

  • introduce o contrasinal DSRM cando o solicite;
  • revisa as mensaxes de advertencia;
  • agarda ao reinicio automático do servidor.
  • □O comando executouse sen erros críticos.
  • □O contrasinal DSRM foi definido.
  • □O servidor reiniciou automaticamente ao final.
Recorda: o contrasinal DSRM non debe aparecer nunha captura nin nun documento público de entrega.

B.8. Verificación con PowerShell

1

Verificar dominio e bosque

Despois do reinicio, inicia sesión como usuario do dominio. Usuario orientativo:

CIFPSAUCE\Administrador

ou equivalente.

Get-ADDomain
Get-ADForest
  • □Get-ADDomain devolve datos de cifpsauce.local.
  • □Get-ADForest indica un bosque único co dominio raíz cifpsauce.local.
2

Verificar o controlador de dominio

Get-ADDomainController
  • □Aparece dcbuho como controlador de dominio.
  • □O dominio asociado é cifpsauce.local.
3

Verificar resolución DNS

Resolve-DnsName dcbuho.cifpsauce.local
Resolve-DnsName cifpsauce.local
nslookup dcbuho.cifpsauce.local
  • □dcbuho.cifpsauce.local resolve correctamente.
  • □A resolución devolve a IP 192.168.50.10.
  • □nslookup devolve unha resposta coherente.
4

Verificación gráfica final

Abre as seguintes consolas administrativas:

Tools > Active Directory Users and Computers
Tools > DNS

Comproba que a situación é equivalente á instalación feita mediante GUI:

  • □Existe o dominio cifpsauce.local.
  • □Existe a zona DNS cifpsauce.local.
  • □O servidor dcbuho aparece como controlador de dominio.

Non é necesario repetir todas as capturas se xa se fixeron na Parte A, agás que o profesorado o indique.

B.9. Proba opcional avanzada: diagnóstico DNS do controlador de dominio

Esta proba é opcional. Serve para obter un diagnóstico máis completo do funcionamento DNS asociado ao controlador de dominio.

Executa nunha consola con permisos de administrador:

dcdiag /test:dns

Resultado esperado:

  • non deberían aparecer erros críticos relacionados coa resolución DNS do dominio;
  • poden aparecer avisos que haberá que interpretar segundo a configuración da rede de laboratorio.

7. Ideas clave da práctica

Un dominio de Active Directory necesita unha configuración DNS correcta.
O primeiro controlador de dominio crea o novo bosque e o dominio raíz.
dcbuho non é un controlador “principal”; é o primeiro controlador de dominio da práctica.
O servidor debe usar como DNS preferido o seu propio servizo DNS.
O proceso pode facerse mediante asistentes gráficos ou mediante PowerShell.
A Parte GUI e a Parte PowerShell deben facerse en máquinas limpas ou snapshots separados.
En Windows Server 2022 é normal que o nivel funcional máis recente dispoñible sexa Windows Server 2016.
