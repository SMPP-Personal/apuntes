Instalación e posta en marcha do dominio

Preparación teórica para crear o dominio cifpsauce.local en Windows Server 2022: requisitos, AD DS, DNS, novo bosque, boas prácticas e erros frecuentes.

Primeiro DC: dcbuho Dominio: cifpsauce.local Rede: 192.168.50.0/24
Dominio internocifpsauce.local
Primeiro DCdcbuho.cifpsauce.local
IP fixa do servidor192.168.50.10
Rede de laboratorio192.168.50.0/24
Nota sobre .local: nesta UD usamos cifpsauce.local como dominio interno de laboratorio. Nun contorno real, a elección do nome DNS do dominio debe planificarse con coidado, preferiblemente a partir dun dominio rexistrado pola organización, para evitar conflitos de nomes e problemas de resolución.

1. Que imos facer neste bloque

Neste bloque non imos crear aínda usuarios, grupos, OU nin políticas de grupo avanzadas. Iso farase en bloques posteriores.

O obxectivo agora é preparar a infraestrutura básica do dominio:

  1. Preparar a máquina virtual con Windows Server 2022.
  2. Configurar o nome do servidor e a rede.
  3. Instalar os roles necesarios.
  4. Crear un novo bosque cuxo dominio raíz será cifpsauce.local.
  5. Converter o servidor dcbuho no primeiro controlador de dominio.
  6. Comprobar que AD DS e DNS funcionan correctamente.

3. Requisitos para montar un dominio en laboratorio

Antes de instalar Active Directory Domain Services (AD DS) nun servidor, é necesario comprobar requisitos de hardware, software e rede.

Hardware e virtualización

  • CPU: 2 vCPU ou máis.
  • RAM: 4 GB poden servir para unha práctica moi básica; 6–8 GB é máis recomendable.
  • Disco: 60–80 GB nun laboratorio.
  • Plataforma: VirtualBox, VMware, Hyper-V ou equivalente.

Software

  • Windows Server 2022.
  • Rol AD DS.
  • Rol DNS Server.
  • Ferramentas: Server Manager, ADUC, DNS Manager, PowerShell e módulos de AD.

Rede

  • Enderezo IP estático.
  • Nome definitivo antes da promoción.
  • DNS correctamente configurado.
  • Conectividade cos clientes que se unirán ao dominio.
Idea clave: nun laboratorio non buscamos dimensionar un servidor real de produción, senón crear unha contorna estable para practicar AD DS, DNS, unión de clientes ao dominio e administración básica.

4. Rol Active Directory Domain Services en Windows Server 2022

4.1. Que é AD DS

O compoñente que permite crear e administrar dominios de Active Directory.

AD DS permite:

  • Almacenar información sobre usuarios, equipos, grupos, OU, políticas e outros obxectos.
  • Centralizar a autenticación dos usuarios do dominio.
  • Aplicar autorizacións e permisos a recursos de rede.
  • Organizar os obxectos do dominio de forma xerárquica.
  • Administrar configuracións de usuarios e equipos mediante directivas de grupo.
  • Integrarse con DNS para que os clientes localicen os servizos do dominio.
Instalar o rol AD DS é o primeiro paso para converter un servidor nun controlador de dominio, pero despois hai que executar a promoción a controlador de dominio.

4.2. AD DS, LDAP, Kerberos e DNS

Active Directory non debe entenderse como “só LDAP”.

LDAPPermite consultar e modificar información do directorio.
KerberosProtocolo principal de autenticación no dominio.
DNSPermite localizar controladores de dominio e servizos de AD.
SMB/RPCUtilizados en operacións administrativas e de acceso a recursos.
Group PolicyPermite aplicar configuracións centralizadas a usuarios e equipos.

Moitas ferramentas gráficas, como Active Directory Users and Computers, son interfaces de administración que traballan contra estes servizos de forma transparente para a persoa usuaria.

5. Integración con DNS

DNS non é un complemento opcional en AD DS. Nun dominio Active Directory, DNS é unha peza central para localizar controladores de dominio e servizos.

5.1. Por que DNS é imprescindible nun dominio

Os clientes consultan DNS para atopar os servizos do dominio.

Nun dominio como cifpsauce.local, os equipos necesitan resolver nomes e rexistros especiais asociados a Kerberos, LDAP e catálogo global.

AD DS crea rexistros especiais en DNS, especialmente rexistros SRV, para que os clientes poidan localizar:

controladores de dominio dispoñibles;
servizos LDAP;
servizos Kerberos;
catálogo global, se procede;

Se DNS falla, poden fallar tamén a unión de clientes ao dominio, novos inicios de sesión, aplicación de GPO, localización de DC e acceso a recursos que dependen da autenticación de dominio.

6. Creación do primeiro dominio nun novo bosque

Na UD2 imos crear un novo bosque cuxo dominio raíz será cifpsauce.local. Non imos engadir un controlador de dominio a un bosque xa existente nin crear un dominio adicional dentro dunha infraestrutura previa.

Novo bosque

Úsase cando non existe ningunha infraestrutura AD previa. É a opción da UD2.

Novo dominio

Úsase cando xa existe un bosque e se quere engadir outro dominio.

DC adicional

Úsase cando xa existe o dominio e se quere engadir outro controlador de dominio.

Nivel funcional: en Windows Server 2022 non existe un nivel funcional específico chamado “Windows Server 2022”. O nivel funcional máis recente dispoñible segue sendo normalmente Windows Server 2016. Na práctica deixaremos os valores predeterminados que ofrece o asistente.

6.3. Contrasinal DSRM

Directory Services Restore Mode.

Durante a promoción, o asistente pedirá un contrasinal para o Directory Services Restore Mode (DSRM).

Este contrasinal non é o contrasinal normal dunha conta de usuario do dominio. Utilízase en situacións de recuperación ou mantemento avanzado de Active Directory.

  • Definir un contrasinal seguro.
  • Non deixalo en branco.
  • Non escribilo nunha folla de entrega do alumnado.
  • Gardalo no lugar que indique o profesorado.

7. Consideracións sobre seguridade e boas prácticas iniciais

7.1. Contas e privilexios

A conta Administrador non debe converterse nun hábito de traballo diario.

Nun laboratorio pode usarse a conta Administrador para simplificar a instalación inicial, pero non debe presentarse como hábito profesional.

Boas prácticas:
  • Usar unha conta normal para tarefas cotiás.
  • Reservar contas administrativas para tarefas de administración.
  • Non navegar nin descargar ficheiros cunha conta con privilexios elevados.
Exemplos de contas e grupos:
  • alu.curso1.001, alu.curso2.001
  • prof.001, prof.002
  • GG_Alumnado_Curso1, GG_Profesorado

Idea clave: os permisos non deberían asignarse usuario por usuario. O habitual é organizar as contas en grupos e aplicar os permisos aos grupos.

8. Erros frecuentes ao crear un dominio

Antes de facer a práctica, convén coñecer algúns erros típicos.

Erro frecuenteConsecuencia posible
Deixar o servidor con IP por DHCPO DC pode cambiar de IP e os clientes non localizarán o dominio.
Poñer como DNS principal un DNS externoOs clientes non atoparán os rexistros internos de AD DS.
Cambiar o nome do servidor despois de promoveloPode provocar problemas complexos no dominio.
Non documentar o contrasinal DSRMComplica tarefas de recuperación.
Confundir instalar AD DS con crear o dominioInstalar o rol non equivale a promover o servidor.
Non comprobar a zona DNSO dominio pode crearse pero non funcionar correctamente para os clientes.
Non usar snapshot antes da prácticaSe algo falla, hai que reinstalar ou desfacer manualmente.
Usar sempre a conta Administrador do dominioMala práctica de seguridade.

9. Esquema resumo do proceso de posta en marcha

1

Planificación previa

Definir dominio, servidor, FQDN, IP fixa e estrutura inicial de OU.

2

Preparación do servidor

Instalar Windows Server 2022, configurar nome dcbuho, IP estática, DNS propio e comprobar rede.

3

Instalación dos roles

Instalar AD DS, DNS Server, ferramentas de administración e comprobar que non hai erros.

4

Promoción a controlador de dominio

Crear novo bosque, indicar cifpsauce.local, activar DNS, manter catálogo global e definir DSRM.

5

Verificación básica

Comprobar inicio de sesión no dominio, existencia da zona DNS, resolución de dcbuho.cifpsauce.local e documentación final.

10. Ideas clave do bloque

Instalar AD DS non é o mesmo que crear o dominio.
O servidor debe ter IP fixa antes de ser promovido a controlador de dominio.
O DNS é imprescindible en Active Directory.
Nun laboratorio cun único DC, o propio servidor actuará como DC e DNS.
En Windows Server 2022 o nivel funcional máis recente dispoñible segue sendo normalmente Windows Server 2016.
O primeiro dominio creado será o dominio raíz dun novo bosque.
O contrasinal DSRM debe tratarse como información sensible.
A documentación inicial evita moitos problemas nas prácticas posteriores.
