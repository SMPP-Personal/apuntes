Guión práctico 2 – Instalación e creación do dominio usando só PowerShell

Creación do dominio cifpsauce.local co primeiro controlador de dominio dcbuho, empregando comandos PowerShell para preparar rede, instalar roles e promover o servidor.

Método: PowerShell Dominio: cifpsauce.local DC: dcbuho IP: 192.168.50.10/24 8 evidencias

0. Punto de partida

Situación inicial da máquina virtual.

Nesta práctica vas crear o dominio cifpsauce.local empregando só PowerShell para preparar o servidor, instalar os roles necesarios e promover o equipo a controlador de dominio.

O resultado final debe ser equivalente ao obtido co guión práctico GUI:

Dominiocifpsauce.local
NetBIOSCIFPSAUCE
Primeiro DCdcbuho.cifpsauce.local
IP192.168.50.10/24
DNS preferido192.168.50.10
RolesAD DS e DNS Server

Moi importante

Este guión debe facerse nunha VM limpa ou nun snapshot anterior á instalación de AD DS. Non intentes crear de novo o bosque cifpsauce.local nun servidor que xa foi promovido a controlador de dominio.

1. Antes de empezar

Requisitos previos e snapshot da máquina.

1.1. Requisitos previos

  • Windows Server 2022 instalado.
  • Nome provisional de equipo, por exemplo SERVER2022.
  • Sesión iniciada cunha conta con permisos de administración local.
  • Acceso a PowerShell como administrador.
  • Adaptador conectado á rede de laboratorio correspondente.

1.2. Snapshot da máquina

  1. Apaga ou deixa estable a VM, segundo indique o profesorado.
  2. Crea un snapshot ou unha copia da máquina.
  3. Ponlle un nome claro.
Antes_ADDS_PowerShell

Este snapshot permitirá repetir a práctica se algo falla ou se necesitas volver ao estado inicial.

2. Comprobación inicial da rede e do equipo

Antes de modificar a configuración, revisa o estado actual.

Abre Windows PowerShell como administrador e executa:

Get-NetAdapter
Get-NetIPConfiguration
hostname

Debes anotar:

  • nome real do adaptador de rede;
  • IP actual, se a ten;
  • gateway, se existe;
  • servidor DNS configurado actualmente;
  • nome actual do equipo.

Nome do adaptador

Nos exemplos usarase o adaptador Ethernet. Se no teu servidor aparece outro nome, debes substituír Ethernet polo nome real do teu adaptador.

3. Configurar IP estática con PowerShell

Configuración de rede necesaria para o primeiro DC.

3.1. Identificar o adaptador de rede

Executa:

Get-NetAdapter

Localiza o adaptador principal. Normalmente chamarase Ethernet, pero pode ter outro nome dependendo da VM.

Nome do adaptador: ______________________________

3.2. Configurar IP, máscara e gateway

Se o adaptador se chama Ethernet, executa:

New-NetIPAddress `
  -InterfaceAlias "Ethernet" `
  -IPAddress "192.168.50.10" `
  -PrefixLength 24 `
  -DefaultGateway "192.168.50.1"
ParámetroSignificado
-InterfaceAliasNome do adaptador de rede.
-IPAddressIP fixa que terá o servidor.
-PrefixLength 24Equivale á máscara 255.255.255.0.
-DefaultGatewayPorta de enlace da rede de laboratorio.

Posible erro

Se o adaptador xa ten unha IP estática ou unha gateway configurada, este comando pode devolver erro. Nesta práctica recoméndase partir dunha VM limpa ou dun snapshot anterior. Se aparece un erro, consulta co profesorado antes de eliminar configuracións de rede.

3.3. Configurar o DNS do servidor

O servidor debe apuntar como DNS preferido á súa propia IP:

Set-DnsClientServerAddress `
  -InterfaceAlias "Ethernet" `
  -ServerAddresses "192.168.50.10"

Por que se apunta a si mesmo?

Ao finalizar a promoción, este servidor será controlador de dominio e tamén servidor DNS do dominio. Debe resolver a zona interna cifpsauce.local. Non deixes como DNS principal un DNS externo como 8.8.8.8 nin o router da aula.

3.4. Comprobar a configuración de rede

Get-NetIPAddress -InterfaceAlias "Ethernet"
Get-DnsClientServerAddress -InterfaceAlias "Ethernet"
ipconfig /all
ElementoValor esperado
IP192.168.50.10
Máscara255.255.255.0 ou prefixo /24
Gateway192.168.50.1, se existe na rede de laboratorio
DNS preferido192.168.50.10

Se hai gateway real na rede de laboratorio, podes probar:

ping 192.168.50.1

4. Cambiar o nome do equipo a dcbuho

O nome debe quedar aplicado antes da promoción.

Executa PowerShell como administrador:

Rename-Computer -NewName "dcbuho" -Restart

O servidor reiniciarase automaticamente. Despois do reinicio, inicia sesión de novo como administrador local e comproba o nome:

hostname

O resultado debe ser:

dcbuho

Erro frecuente

Non continúes coa instalación de AD DS se o servidor aínda non se chama dcbuho. Cambiar o nome despois de promover o servidor a controlador de dominio non é unha boa práctica nun laboratorio inicial.

5. Instalar os roles AD DS e DNS con PowerShell

Instalación dos compoñentes necesarios antes da promoción.

Abre PowerShell como administrador e instala os roles Active Directory Domain Services e DNS Server, incluíndo as ferramentas de administración:

Install-WindowsFeature AD-Domain-Services, DNS -IncludeManagementTools

Ao rematar, revisa o resultado do comando. Debes comprobar que a instalación se completou correctamente.

Tamén podes verificar os roles instalados con:

Get-WindowsFeature AD-Domain-Services, DNS

Non confundas rol con dominio

Instalar o rol AD DS non crea aínda o dominio. Só prepara o servidor para poder ser promovido posteriormente a controlador de dominio.

6. Promover o servidor a controlador de dominio dun novo bosque

Creación de cifpsauce.local mediante Install-ADDSForest.

6.1. Importar o módulo de despregamento de AD DS

Import-Module ADDSDeployment

6.2. Crear o novo bosque cifpsauce.local

Executa o seguinte comando:

Install-ADDSForest `
  -DomainName "cifpsauce.local" `
  -DomainNetbiosName "CIFPSAUCE" `
  -SafeModeAdministratorPassword (Read-Host "Introduce o contrasinal DSRM" -AsSecureString) `
  -InstallDNS `
  -Force

Durante a execución pedirase o contrasinal DSRM.

Contrasinal DSRM

É o contrasinal do modo de restauración de servizos de directorio. Debe conservarse nun lugar seguro indicado polo profesorado. Non debe escribirse nunha ficha pública nin nun documento compartido sen protección.

6.3. Explicación dos parámetros principais

ParámetroSignificado
-DomainName "cifpsauce.local"Nome DNS completo do dominio raíz do novo bosque.
-DomainNetbiosName "CIFPSAUCE"Nome NetBIOS curto do dominio.
-SafeModeAdministratorPasswordDefine o contrasinal DSRM.
-InstallDNSConfigura DNS para o novo dominio.
-ForceReduce preguntas interactivas adicionais.

DNS

Nesta práctica instalamos explicitamente o rol DNS antes da promoción e ademais usamos -InstallDNS durante Install-ADDSForest. Isto garante que o novo bosque queda configurado co DNS necesario para cifpsauce.local.

Nivel funcional

En Windows Server 2022, o nivel funcional máis recente dispoñible segue sendo normalmente Windows Server 2016. Ao non indicar parámetros específicos de nivel funcional, deixamos que o cmdlet empregue os valores predeterminados axeitados.

6.4. Reinicio automático

O cmdlet comprobará os requisitos, instalará e configurará AD DS, creará o novo bosque e reiniciará o servidor automaticamente ao rematar.

7. Inicio de sesión despois da promoción

Primeiro acceso despois de converter o servidor en DC.

Tras o reinicio, inicia sesión xa no dominio. Podes empregar:

CIFPSAUCE\Administrador

Tamén podes seleccionar o dominio correspondente na pantalla de inicio de sesión. Comproba que o equipo xa pertence ao dominio cifpsauce.local.

8. Verificación básica con PowerShell

Comprobación de dominio, bosque, DC, IP e DNS.

8.1. Verificar dominio e bosque

Get-ADDomain
Get-ADForest

Debes comprobar que:

  • o dominio é cifpsauce.local;
  • o bosque ten como dominio raíz cifpsauce.local;
  • o nome NetBIOS é CIFPSAUCE.

8.2. Verificar que dcbuho é controlador de dominio

Get-ADDomainController

Debe aparecer dcbuho como controlador de dominio do dominio cifpsauce.local.

8.3. Verificar IP e DNS do propio servidor

Get-NetIPConfiguration
Get-DnsClientServerAddress -InterfaceAlias "Ethernet"

Comproba que o DNS segue apuntando a:

192.168.50.10

9. Verificación DNS

Comprobación de nomes e rexistros necesarios para AD DS.

9.1. Resolver o nome completo do controlador de dominio

Resolve-DnsName dcbuho.cifpsauce.local
nslookup dcbuho.cifpsauce.local

O resultado debe apuntar á IP:

192.168.50.10

9.2. Resolver o dominio

Resolve-DnsName cifpsauce.local
nslookup cifpsauce.local

9.3. Comprobación dos rexistros SRV

nslookup -type=SRV _ldap._tcp.dc._msdcs.cifpsauce.local

Esta comprobación verifica a existencia de rexistros SRV empregados polos clientes para localizar controladores de dominio.

Interpretación inicial

Se esta comprobación resulta difícil de interpretar, non te preocupes: o importante nesta práctica inicial é entender que Active Directory usa DNS non só para nomes simples, senón tamén para localizar servizos internos do dominio.

10. Verificación gráfica opcional con MMC

Comprobación visual aínda que o guión sexa de PowerShell.

Desde Server Manager → Tools, podes abrir:

  • Active Directory Users and Computers;
  • DNS.

Comproba:

  • que existe o dominio cifpsauce.local;
  • que aparecen os contedores básicos de Active Directory;
  • que existe a zona DNS cifpsauce.local;
  • que hai un rexistro A para dcbuho;
  • que existen contedores como _msdcs e rexistros relacionados con AD DS.

11. Comprobación avanzada opcional

Proba adicional de DNS no controlador de dominio.

Se o profesorado o solicita, executa:

dcdiag /test:dns

Aviso sobre dcdiag

dcdiag pode mostrar avisos ou información técnica avanzada. Non todos os avisos implican que a práctica estea mal. O profesorado indicará que resultados deben considerarse correctos neste nivel.

14. Erros frecuentes

Problemas típicos ao facer a instalación por PowerShell.

ErroConsecuencia
Executar PowerShell sen permisos de administradorOs comandos de rede, roles ou promoción poden fallar.
Copiar powershell como parte do comandoO comando non funciona. A palabra powershell indica só a linguaxe do bloque de código.
Non substituír o nome real do adaptadorNew-NetIPAddress ou Set-DnsClientServerAddress fallan.
Executar o guión nun servidor xa promovidoNon se pode crear de novo o mesmo bosque nun DC existente.
Esquecer reiniciar tras Rename-ComputerO cambio de nome non queda aplicado antes da promoción.
Deixar como DNS principal un DNS externoO servidor pode non resolver correctamente cifpsauce.local.
Escribir mal cifpsauce.localCréase un dominio incorrecto ou falla a promoción.
Publicar ou perder o contrasinal DSRMSupón un problema de seguridade e recuperación.
Non revisar os rexistros DNSPoden quedar problemas de localización do DC sen detectar.

15. Ideas clave

Resumo conceptual da práctica.

Instalar o rol AD DS non é o mesmo que crear un dominio.
Install-ADDSForest crea un novo bosque e promove o servidor como controlador de dominio.
O primeiro DC debe ter IP fixa e DNS correctamente configurado.
Active Directory depende de DNS para localizar controladores de dominio e servizos internos.
PowerShell permite repetir, documentar e automatizar a instalación, pero require escribir os comandos con precisión.
