Guión práctico 1 – Instalación e creación do dominio con interface gráfica

Creación do dominio cifpsauce.local co primeiro controlador de dominio dcbuho, empregando Server Manager e o asistente gráfico de AD DS.

Dominio: cifpsauce.local DC: dcbuho IP: 192.168.50.10 7 evidencias

0. Punto de partida

Situación inicial antes de modificar a máquina virtual.

Nesta práctica vas crear o dominio cifpsauce.local empregando a interface gráfica de Windows Server 2022.

Partimos dunha máquina virtual con:

  • Windows Server 2022 instalado.
  • Nome provisional do equipo, por exemplo SERVER2022.
  • Sesión iniciada cunha conta con permisos de administración local.
  • Rede configurada na mesma rede de laboratorio que se empregará nas prácticas da UD2.

O obxectivo é converter este servidor no primeiro controlador de dominio do dominio cifpsauce.local.

Antes de empezar

Crea un snapshot ou unha copia da máquina virtual antes de realizar esta práctica.

A promoción a controlador de dominio cambia profundamente o estado do servidor. Non é unha operación cómoda de desfacer se despois queres repetir a práctica dende cero.

Nota sobre o dominio .local

Nesta unidade usamos cifpsauce.local como nome interno de laboratorio. Nun contorno real, o nome DNS dun dominio debe planificarse con máis coidado, normalmente partindo dun dominio rexistrado pola organización.

2. Configurar a IP estática do servidor

Un controlador de dominio non debe depender dunha IP asignada por DHCP.

O primeiro controlador de dominio debe ter IP fixa. Os clientes do dominio dependerán del para resolver nomes internos, localizar servizos e acceder a recursos.

2.1. Abrir configuración de rede

  1. Abre o Panel de control ou fai clic dereito na icona de rede.
  2. Entra en Configuración de rede e Internet.
  3. Accede a Cambiar as opcións do adaptador.
  4. Fai clic dereito sobre o adaptador de rede da VM.
  5. Entra en Propiedades.
  6. Selecciona Protocolo de Internet versión 4 (TCP/IPv4) e preme Propiedades.

2.2. Configurar IPv4

CampoValor
Enderezo IP192.168.50.10
Máscara255.255.255.0
Gateway192.168.50.1 ou indicado
DNS preferido192.168.50.10

Importante sobre DNS

O servidor DNS preferido debe ser a propia IP do servidor: 192.168.50.10.

Non deixes como DNS principal un servidor externo como 8.8.8.8, nin o router da aula, porque Active Directory necesita resolver os rexistros internos do dominio.

2.3. Comprobación da configuración de rede

Abre unha consola de comandos e executa:

ipconfig /all
hostname
ping 192.168.50.10

Se existe gateway real na rede de laboratorio, comproba tamén:

ping 192.168.50.1
  • a IP debe ser 192.168.50.10;
  • a máscara debe ser 255.255.255.0;
  • o DNS preferido debe ser 192.168.50.10;
  • o nome do equipo aínda será provisional ou o que tivese a máquina antes de cambialo.

3. Cambiar o nome do equipo a dcbuho

O nome debe quedar definido antes da promoción a controlador de dominio.

3.1. Cambio desde a interface gráfica

  1. Fai clic dereito en Inicio.
  2. Entra en Sistema.
  3. Abre a sección Información.
  4. Preme en Cambiar o nome deste PC.

Tamén podes facelo pola ruta clásica: Panel de control → Sistema → Configuración avanzada do sistema → Nome de equipo → Cambiar...

Novo nome

dcbuho

Acepta os cambios e reinicia o servidor cando o sistema o solicite.

3.2. Comprobar o novo nome

Despois do reinicio, inicia sesión de novo e comproba o nome do equipo:

hostname

O resultado debe ser:

dcbuho

4. Engadir os roles AD DS e DNS

Instalar roles non é o mesmo que crear o dominio.

Neste paso imos instalar os roles necesarios, pero aínda non estaremos creando o dominio. Instalar AD DS só prepara o servidor para poder ser promovido a controlador de dominio.

4.1. Abrir o asistente de roles

  1. Abre Server Manager / Xestor do servidor.
  2. No panel principal, fai clic en Add roles and features / Engadir roles e características.
  3. Before you begin / Antes de comezar → Next.
  4. Installation type / Tipo de instalación: selecciona Role-based or feature-based installation.
  5. Server Selection / Selección do servidor: selecciona o servidor local, que debería aparecer como dcbuho.

4.2. Seleccionar os roles

  1. Na páxina Server Roles / Roles de servidor, marca Active Directory Domain Services.
  2. Cando o asistente pregunte se queres engadir características necesarias, preme Add Features.
  3. Marca tamén DNS Server se non aparece xa marcado.
  4. Se o asistente volve pedir características adicionais, acepta con Add Features.
Captura obrigatoria 3 · roles AD DS e DNS seleccionados

4.3. Completar a instalación

  1. Na páxina Features / Características, deixa os valores por defecto.
  2. Le as páxinas informativas sobre AD DS e DNS.
  3. Na páxina Confirmation / Confirmación, preme Install.
  4. Agarda a que remate a instalación.

Lembra

Neste punto só instalaches os roles. O dominio cifpsauce.local aínda non existe. Para crealo realmente hai que promover o servidor a controlador de dominio.

5. Promover o servidor a controlador de dominio dun novo bosque

Creación do bosque novo co dominio raíz cifpsauce.local.

Agora imos crear un novo bosque cuxo dominio raíz será cifpsauce.local. Neste laboratorio partimos de cero.

5.1. Iniciar promoción

Ao finalizar a instalación dos roles, en Server Manager aparecerá unha advertencia na parte superior. Preme na ligazón:

Promote this server to a domain controller

En galego/castelán pode aparecer como Promover este servidor a controlador de dominio.

5.2. Configuración da implementación

Na páxina Deployment Configuration / Configuración da implementación, selecciona:

Add a new forest

No campo Root domain name / Nome do dominio raíz, escribe:

cifpsauce.local
Captura obrigatoria 4

5.3. Opcións do controlador de dominio

Nivel funcional

Deixa os valores predeterminados que ofrece o asistente. En Windows Server 2022, o nivel funcional máis recente dispoñible normalmente aparecerá como Windows Server 2016.

  • Mantén marcada a opción Domain Name System (DNS) server.
  • Mantén marcada a opción Global Catalog (GC).
  • Define o contrasinal de Directory Services Restore Mode (DSRM).

Contrasinal DSRM

Empregarase en situacións de recuperación de Active Directory. Debe conservarse nun lugar seguro indicado polo profesorado e non debe escribirse nunha ficha pública nin nun documento compartido sen protección.

5.4. Opcións DNS

Na páxina DNS Options / Opcións DNS pode aparecer un aviso relacionado coa delegación DNS:

A delegation for this DNS server cannot be created...

Nun laboratorio coma este, ese aviso normalmente non é un erro bloqueante. Ocorre porque non existe unha zona DNS superior que poida delegar automaticamente cifpsauce.local.

5.5–5.7. Opcións finais, rutas e instalación

  1. En Additional Options / Opcións adicionais, comproba que o NetBIOS domain name é CIFPSAUCE.
  2. En Paths / Rutas, deixa as rutas por defecto para base de datos, logs e SYSVOL.
  3. En Review Options / Revisar opcións, comproba que se vai crear un novo bosque co dominio raíz cifpsauce.local.
  4. En Prerequisites Check / Comprobación de requisitos previos, se só aparecen avisos informativos e permite continuar, preme Install.
  5. O servidor configurará AD DS, converterase en controlador de dominio e reiniciarase automaticamente.

6. Iniciar sesión no dominio

Despois do reinicio, o servidor xa debería ser DC do dominio.

  1. Na pantalla de inicio de sesión, comproba que aparece o dominio CIFPSAUCE ou cifpsauce.
  2. Inicia sesión como:
CIFPSAUCE\Administrador

A conta local de Administrador pasa a comportarse como conta de administrador do dominio despois da promoción do primeiro DC.

7. Verificación gráfica da instalación

Comprobación con Server Manager, ADUC e DNS.

7.1. Server Manager

  • O servidor aparece como dcbuho.
  • Figuran os roles AD DS e DNS.
  • Non hai erros críticos pendentes.

7.2. ADUC

Abre Tools / Ferramentas → Active Directory Users and Computers e comproba que aparece:

cifpsauce.local

Observa contedores iniciais como Builtin, Computers, Domain Controllers e Users.

8. Verificación con comandos básicos

A práctica é GUI, pero estas comprobacións detectan erros típicos.

Abre Símbolo do sistema ou PowerShell e executa:

hostname
ipconfig /all
ping dcbuho.cifpsauce.local
nslookup dcbuho.cifpsauce.local
  • o nome do equipo debe ser dcbuho;
  • a IP debe ser 192.168.50.10;
  • o DNS preferido debe ser 192.168.50.10;
  • dcbuho.cifpsauce.local debe resolver á IP 192.168.50.10.
Captura obrigatoria 7 · nslookup dcbuho.cifpsauce.local

8.1. Comprobación DNS recomendada

nslookup cifpsauce.local
nslookup -type=SRV _ldap._tcp.dc._msdcs.cifpsauce.local

A segunda orde comproba rexistros SRV empregados polos clientes para localizar controladores de dominio.

8.2. Comprobación opcional con PowerShell

Get-ADDomain
Get-ADDomainController

Estas ordes permiten comprobar que o dominio cifpsauce.local existe e que dcbuho figura como controlador de dominio.

11. Erros frecuentes

Problemas típicos que convén evitar antes de continuar.

Erro frecuenteConsecuencia
Deixar o servidor con IP por DHCPA IP pode cambiar e os clientes poden deixar de atopar o DC ou o DNS.
Poñer como DNS principal 8.8.8.8 ou o routerO servidor non resolverá correctamente os rexistros internos do dominio.
Esquecer reiniciar despois de cambiar o nomeO servidor pode promocionarse cun nome incorrecto.
Instalar AD DS pero non promover o servidorO dominio aínda non existe.
Escribir mal cifpsauce.localCrearase un dominio con nome incorrecto ou fallará a configuración.
Alarmarse polo aviso de delegación DNSNun laboratorio illado adoita ser só un aviso informativo.
Non conservar o contrasinal DSRM nun lugar seguroPode dificultar operacións de recuperación.
Non verificar DNS despois da instalaciónPoden quedar erros ocultos que afectarán á unión de clientes ao dominio.

12. Ideas clave da práctica

Resumo conceptual antes de pasar a usuarios, grupos e OU.

Instalar o rol AD DS non é o mesmo que crear un dominio.
O servidor debe ter IP fixa antes de converterse en DC.
O primeiro DC adoita actuar tamén como servidor DNS do dominio.
Active Directory depende de DNS para localizar controladores de dominio e servizos.
O dominio créase ao seleccionar Add a new forest.
En Windows Server 2022, o nivel funcional máis recente dispoñible normalmente segue sendo Windows Server 2016.
As comprobacións finais con ADUC, DNS, ping e nslookup son imprescindibles antes de continuar con usuarios, grupos e OU.
