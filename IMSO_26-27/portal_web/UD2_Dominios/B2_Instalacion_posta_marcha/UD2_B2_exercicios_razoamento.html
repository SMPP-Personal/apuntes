Cuestións de razoamento

Instalación e posta en marcha do dominio cifpsauce.local.

Estas cuestións serven para comprobar se comprendiches os conceptos principais do Bloque 2: requisitos previos, instalación de AD DS, papel de DNS, creación dun novo bosque, uso de GUI e PowerShell, e documentación da infraestrutura.

Dominio: cifpsauce.local DC: dcbuho.cifpsauce.local IP: 192.168.50.10

Instrución xeral

Non se trata de copiar definicións dos apuntes. En cada resposta debes explicar a idea coas túas propias palabras e aplicala ao escenario da UD2.

Dominio cifpsauce.local
Primeiro DC dcbuho.cifpsauce.local
IP de exemplo 192.168.50.10
1

IP estática e controlador de dominio

Rede DC DNS dos clientes

Nun laboratorio de dominios, decides deixar o servidor que vai ser controlador de dominio con IP por DHCP.

Responde de forma razoada:

  1. Explica por que isto é unha mala práctica nun contorno de dominio.
  2. Indica polo menos dúas consecuencias prácticas que se poden producir se a IP do controlador de dominio cambia.
  3. Explica que problemas poden aparecer se os clientes teñen configurado un servidor DNS incorrecto ou seguen apuntando a unha IP antiga.

Para orientar a resposta

  • localización do controlador de dominio;
  • resolución da zona cifpsauce.local;
  • inicio de sesión dos usuarios;
  • acceso a recursos compartidos;
  • aplicación de políticas de grupo.

2

Papel de DNS en Active Directory

DNS Localización Rexistros SRV

Active Directory depende de DNS para que os clientes poidan localizar os servizos do dominio.

Responde de forma razoada:

  1. Explica por que un cliente de dominio necesita consultar o DNS do dominio para localizar un controlador de dominio.
  2. Aclara por que DNS non autentica o usuario directamente, senón que axuda a atopar os servizos necesarios para iniciar sesión no dominio.
  3. Describe que tipo de información especial se garda na zona DNS cifpsauce.local, especialmente os rexistros SRV.
  4. Explica para que serven eses rexistros no funcionamento dun dominio Active Directory.

Para orientar a resposta

  • resolución de nomes como dcbuho.cifpsauce.local;
  • localización de controladores de dominio;
  • servizos como Kerberos, LDAP ou catálogo global;
  • consecuencias dun DNS mal configurado.

3

Diferenza entre instalar o rol AD DS e crear o dominio

Rol AD DS Promoción Verificación

Nunha práctica, unha persoa instala o rol Active Directory Domain Services, pero non completa a promoción do servidor a controlador de dominio.

Responde de forma razoada:

  1. Explica a diferenza entre:
    • ter instalado o rol AD DS;
    • ter promovido o servidor a controlador de dominio;
    • ter creado realmente o dominio cifpsauce.local.
  2. Indica que faltaría facer se só se instalou o rol, pero non se executou a promoción.
  3. Explica como poderías comprobar se o dominio foi creado correctamente ou se só está instalado o rol.

Ferramentas ou comprobacións posibles

  • Server Manager;
  • Active Directory Users and Computers;
  • consola de DNS;
  • Get-ADDomain;
  • Get-ADForest;
  • Get-ADDomainController;
  • nslookup dcbuho.cifpsauce.local.

Novo bosque, dominio existente e controlador adicional

Bosque Dominio DC adicional

No asistente de promoción de AD DS, para crear cifpsauce.local escollemos a opción Create a new forest, porque non existe unha infraestrutura previa de Active Directory.

Responde de forma razoada:

  1. Explica por que neste laboratorio escollemos Create a new forest.
  2. Indica en que caso se escollería Add a domain controller to an existing domain.
  3. Indica en que caso se escollería Add a new domain to an existing forest.
  4. Pon un exemplo sinxelo, a nivel teórico, no que un centro educativo podería necesitar engadir un novo dominio dentro dun bosque xa existente en lugar de crear un bosque novo.

Para orientar a resposta

  • ampliar unha estrutura xa existente con outro dominio.

Comparando GUI e PowerShell

GUI PowerShell Automatización

Na UD2 traballamos a instalación e creación do dominio cifpsauce.local tanto con interface gráfica como con PowerShell.

Responde de forma razoada:

  1. Indica polo menos dúas vantaxes de usar PowerShell para instalar AD DS e crear o dominio, en comparación coa interface gráfica.
  2. Non abonda con dicir que PowerShell é “máis rápido”: explica por que pode ser máis reproducible, documentable ou automatizable.
  3. Indica polo menos unha vantaxe de usar a interface gráfica, especialmente pensando en alumnado que está aprendendo por primeira vez.
  4. Indica tamén un posible risco de usar PowerShell se se copian comandos sen entender os parámetros.

Para orientar a resposta

  • repetición da mesma instalación en varias máquinas;
  • gardar os comandos usados;
  • reducir erros manuais;
  • comprender visualmente os pasos do asistente;
  • risco de executar comandos nunha VM incorrecta ou cun parámetro mal escrito.

6

Documentación da infraestrutura recén creada

Documentación Comprobacións DSRM

Despois de crear un dominio, é imprescindible documentar a infraestrutura creada.

Responde de forma razoada:

  1. Elabora unha lista dos datos mínimos que deberías recoller na ficha de documentación do dominio despois de crear cifpsauce.local.
  2. Explica por que esa documentación é útil para a administración diaria.
  3. Explica por que tamén será útil para futuras intervencións, como engadir un segundo controlador de dominio, resolver incidencias ou preparar unha práctica/exame.

A ficha debería incluír, como mínimo

  • nome DNS do dominio;
  • indicación de que se trata dun bosque novo co dominio raíz cifpsauce.local;
  • nome NetBIOS do dominio;
  • nome do primeiro controlador de dominio;
  • FQDN do controlador de dominio;
  • IP, máscara, gateway e DNS configurados;
  • versión do sistema operativo;
  • roles instalados;
  • nivel funcional do bosque e do dominio;
  • existencia da zona DNS cifpsauce.local;
  • comprobacións realizadas despois da instalación;
  • data da instalación;
  • persoa ou equipo que realizou a práctica;
  • indicación de onde se garda de forma segura o contrasinal DSRM, sen escribilo directamente nun documento entregable.

Peche da actividade

Ao rematar estas cuestións deberías ser capaz de explicar, sen copiar definicións, por que a creación dun dominio non consiste só en “instalar un rol”, senón en preparar correctamente a rede, DNS, AD DS, o primeiro controlador de dominio e a documentación mínima da infraestrutura.
